In Modal, these are two different levels of granularity:

**App ID** identifies an entire deployment/run of your Modal app — essentially one invocation of `modal run ...`. In your screenshot, each "cue-eval-harness" row is a separate app. The app ID looks something like `ap-Qw6PhcydR9BDAHF7bSZL5a` (you can see one in your stack trace URL).

**Input ID** identifies a single function call *within* that app. If your app's `run` function gets called once, that's one input. If you had a `.map()` over 100 items, each item would be a separate input. It's the most granular unit of work Modal tracks — each input has its own logs, status, retry history, etc.

So the hierarchy is roughly: **App → Function → Input**. One app contains one or more functions, and each function processes one or more inputs.
