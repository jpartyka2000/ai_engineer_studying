Yes, that's a reasonable way to describe it. `__dict__` is a built-in attribute that Python automatically creates on most objects—it's the dictionary where the object's instance attributes are stored:

```python
class Thing:
    def __init__(self):
        self.foo = 1
        self.bar = 2

t = Thing()
print(t.__dict__)  # {'foo': 1, 'bar': 2}
```

When you do `t.foo`, Python looks in `t.__dict__` (among other places) to find it.

You might also call it a "special attribute" or "magic attribute"—the double underscores indicate it's part of Python's internal machinery rather than something you defined yourself.
