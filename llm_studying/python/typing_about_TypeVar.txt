## **Understanding TypeVar in Python**

### **What is TypeVar?**
`TypeVar` is a special construct in Python's typing system that creates **generic type variables**. It's a placeholder that represents "some type" while maintaining type consistency across a function or class.

### **Basic Usage**

```python
from typing import TypeVar

T = TypeVar('T')

def first_element(items: list[T]) -> T:
    return items[0]

# Type checker knows:
result1: int = first_element([1, 2, 3])      # T = int
result2: str = first_element(["a", "b"])     # T = str
```

### **Why Use TypeVar?**

**Without TypeVar (loses type information):**
```python
from typing import Any

def first_element(items: list[Any]) -> Any:
    return items[0]

# Type checker can't help us here:
result = first_element([1, 2, 3])  # result is Any, not int!
```

**With TypeVar (preserves type information):**
```python
T = TypeVar('T')

def first_element(items: list[T]) -> T:
    return items[0]

result = first_element([1, 2, 3])  # result is int ✅
```

### **Common Patterns**

**1. Functions with consistent types:**
```python
T = TypeVar('T')

def swap(a: T, b: T) -> tuple[T, T]:
    return (b, a)

# Both parameters and return must be same type
x, y = swap(1, 2)          # T = int
a, b = swap("hi", "bye")   # T = str
```

**2. Multiple TypeVars:**
```python
T = TypeVar('T')
U = TypeVar('U')

def pair(first: T, second: U) -> tuple[T, U]:
    return (first, second)

result = pair(42, "hello")  # tuple[int, str]
```

**3. Constrained TypeVars (limited types):**
```python
T = TypeVar('T', int, float)  # T can only be int or float

def add(a: T, b: T) -> T:
    return a + b

add(1, 2)      # ✅ int
add(1.5, 2.5)  # ✅ float
add("a", "b")  # ❌ Type error!
```

**4. Bounded TypeVars (must be subclass):**
```python
from typing import TypeVar

T = TypeVar('T', bound=str)  # T must be str or a subclass of str

def repeat(value: T, times: int) -> list[T]:
    return [value] * times

repeat("hello", 3)  # ✅ Works
repeat(42, 3)       # ❌ Type error - int is not a subclass of str
```

### **TypeVar in Classes (Generics)**

```python
from typing import TypeVar, Generic

T = TypeVar('T')

class Box(Generic[T]):
    def __init__(self, item: T) -> None:
        self.item = item
    
    def get(self) -> T:
        return self.item

int_box = Box(42)         # Box[int]
str_box = Box("hello")    # Box[str]

x: int = int_box.get()    # Type checker knows this is int
```

### **Common Convention**

**Naming:**
- Single letter: `T`, `U`, `V` for general types
- Descriptive: `ValueT`, `KeyT` for specific purposes

```python
T = TypeVar('T')                    # Generic type
KT = TypeVar('KT')                  # Key type
VT = TypeVar('VT')                  # Value type
NumberT = TypeVar('NumberT', int, float)  # Number type
```

### **Key Takeaways**

✅ **Use TypeVar when:** You want to maintain type relationships across parameters and returns

✅ **TypeVar creates:** A placeholder that represents a consistent type throughout usage

✅ **Must define:** `T = TypeVar('T')` before using `T` in type hints

✅ **Enables:** Generic programming with full type safety
