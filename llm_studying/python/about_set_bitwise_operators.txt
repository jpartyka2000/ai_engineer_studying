In Python, the built-in `set` type overloads several “bitwise” operators to perform the usual set‐theoretic operations. Under the hood these are just method calls, but the operators make your code more concise:

| Operator | Method                            | Meaning                              |                            |
| :------: | :-------------------------------- | :----------------------------------- | -------------------------- |
|    `&`   | `set1.intersection(set2)`         | Elements common to both sets         |                            |
|    \`    | \`                                | `set1.union(set2)`                   | All elements in either set |
|    `^`   | `set1.symmetric_difference(set2)` | Elements in exactly one set          |                            |
|    `-`   | `set1.difference(set2)`           | Elements in first set but not second |                            |

---

## 1. Intersection (`&`)

Returns the overlap of two sets:

```python
>>> a = {1, 2, 3, 4}
>>> b = {3, 4, 5, 6}
>>> a & b
{3, 4}

# equivalent to
>>> a.intersection(b)
{3, 4}
```

This runs in roughly *O(min(len(a), len(b)))* time, since it iterates the smaller set and checks membership in the larger.

---

## 2. Union (`|`)

Combines all unique elements from both sets:

```python
>>> a | b
{1, 2, 3, 4, 5, 6}

# equivalent to
>>> a.union(b)
{1, 2, 3, 4, 5, 6}
```

---

## 3. Symmetric Difference (`^`)

Keeps elements that appear in exactly one of the sets:

```python
>>> a ^ b
{1, 2, 5, 6}

# equivalent to
>>> a.symmetric_difference(b)
{1, 2, 5, 6}
```

---

## 4. Difference (`-`)

Subtracts elements of the right-hand set from the left:

```python
>>> a - b
{1, 2}

# equivalent to
>>> a.difference(b)
{1, 2}
```

---

## 5. In-place Variants

You can also modify a set in place:

```python
a &= b   # same as a.intersection_update(b)
a |= b   # same as a.update(b)
a ^= b   # same as a.symmetric_difference_update(b)
a -= b   # same as a.difference_update(b)
```

---

## 6. Practical Tips

* **Commutativity**
  Union and intersection are commutative:

  ```python
  a | b == b | a
  a & b == b & a
  ```
* **Associativity**
  You can chain them without parentheses:

  ```python
  (a | b) | c  ==  a | (b | c)
  (a & b) & c  ==  a & (b & c)
  ```
* **Operator Precedence**
  Bitwise operators on sets follow the same precedence as for integers:

  ```python
  a & b | c   # interpreted as (a & b) | c
  ```
* **Use in Comprehensions**
  You might also combine set comprehensions with these for more complex filters:

  ```python
  common_evens = {x for x in a & b if x % 2 == 0}
  ```

---

### Summary

The “bitwise” operators on Python’s `set` type (`&`, `|`, `^`, `-`) map directly to the familiar set‐theoretic operations intersection, union, symmetric difference, and difference. They offer a concise, readable way to combine and compare sets.
