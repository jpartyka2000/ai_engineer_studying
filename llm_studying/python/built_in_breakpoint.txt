In Python, a built-in breakpoint is a function that allows you to pause the execution of your program at a specific point and enter an interactive debugging session. This is incredibly useful for inspecting the current state of the program (like variables, function calls, etc.) when you're debugging.

### How it works

The built-in function is `breakpoint()`, and it works by triggering the debugger.

#### Python 3.7 and later
In Python 3.7+, the `breakpoint()` function is built-in and triggers the default debugger (which is usually `pdb`—Python's built-in debugger) unless you configure it to use another debugger.

#### Example usage

```python
def add(a, b):
    result = a + b
    breakpoint()  # This will pause the execution and open a debugger
    return result

add(2, 3)
```

When the program execution reaches `breakpoint()`, it will pause and open a debugging prompt where you can interact with the program:

```plaintext
> /path/to/script.py(5)add()
-> return result
(Pdb)
```

At the `(Pdb)` prompt, you can use various debugging commands like:
- `n` to execute the next line.
- `c` to continue execution.
- `p <variable>` to print the value of a variable.
- `q` to quit the debugger.

#### Customizing the debugger
You can customize the debugger that `breakpoint()` triggers by setting the `PYTHONBREAKPOINT` environment variable. For example:

```bash
export PYTHONBREAKPOINT=ipdb.set_trace
```

This would trigger `ipdb` (an enhanced version of `pdb`) instead of the default `pdb` debugger.

#### How `breakpoint()` differs from `pdb.set_trace()`
- `breakpoint()` is a built-in function in Python 3.7 and later.
- `pdb.set_trace()` is a function from the `pdb` module, which you could call explicitly in earlier versions of Python.

### Key points
- `breakpoint()` pauses execution and starts a debugger session.
- In Python 3.7+, it's the recommended way to insert breakpoints in your code.
- You can customize which debugger is used by modifying the `PYTHONBREAKPOINT` environment variable.
