Yes, when you create a **normal Python class**, Python does provide **default implementations** of `__init__`, `__eq__`, and `__repr__`, but these defaults are often **not as useful** as the ones provided by a `dataclass`.

### **Comparison: Default Class vs. `dataclass`**
Let’s compare a **normal class** and a **dataclass** with the same attributes.

#### **1. Regular Python Class (Without `dataclass`)**
```python
class Person:
    def __init__(self, name, age, city):
        self.name = name
        self.age = age
        self.city = city

p1 = Person("Alice", 30, "New York")
p2 = Person("Alice", 30, "New York")

print(p1)          # <__main__.Person object at 0x7f8e4a3c6d90>
print(p1 == p2)    # False
```
- The default `__repr__` only prints a generic object reference (`<__main__.Person object at ...>`).
- The default `__eq__` (from `object`) **compares memory addresses** instead of actual attributes, so `p1 == p2` is `False`, even though their data is identical.

---

#### **2. Using `dataclass`**
```python
from dataclasses import dataclass

@dataclass
class Person:
    name: str
    age: int
    city: str

p1 = Person("Alice", 30, "New York")
p2 = Person("Alice", 30, "New York")

print(p1)          # Person(name='Alice', age=30, city='New York')
print(p1 == p2)    # True
```
- `__repr__` is automatically generated to print meaningful information.
- `__eq__` is overridden to compare **attribute values** instead of memory addresses, so `p1 == p2` is `True`.

---

### **Key Differences**
| Feature        | Regular Class | `dataclass` |
|---------------|--------------|-------------|
| `__init__`    | Must be manually defined | Auto-generated |
| `__repr__`    | Default is not useful | Auto-generated with readable output |
| `__eq__`      | Compares memory addresses | Compares attribute values |
| Boilerplate Code | More code required | Less code, more readable |

### **When the Default Class Implementations Are Enough**
If you **don’t need meaningful string representations or attribute-based comparisons**, a normal class might be fine. But in most cases where you’re working with structured data, `dataclass` makes your life easier.
