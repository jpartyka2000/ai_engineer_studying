Sure — `dict.fromkeys()` is one of those “quietly powerful” Python built-ins that most people don’t use to its full potential.

---

## **What it is**

`dict.fromkeys(iterable, value=None)`
is a **class method** that creates a new dictionary with:

* Keys from the given **iterable**
* All values set to the same value (default is `None`)

---

### **Basic example**

```python
d = dict.fromkeys(["a", "b", "c"])
print(d)
# {'a': None, 'b': None, 'c': None}

d = dict.fromkeys(["a", "b", "c"], 0)
print(d)
# {'a': 0, 'b': 0, 'c': 0}
```

---

## **Why it preserves order now**

Since **Python 3.7** (officially guaranteed from 3.7, but true in CPython 3.6), dictionaries **preserve insertion order** for keys.

That means if you do:

```python
list(dict.fromkeys(["b", "a", "c", "a"]))
# ['b', 'a', 'c']
```

The first time `"a"` is seen, it’s kept; the second `"a"` is ignored, preserving the order of first occurrences.

---

## **Why it’s great for removing duplicates**

The “order preservation” + “unique keys” properties of dictionaries make `dict.fromkeys()` a very clean deduplication trick:

```python
items = ["b", "a", "c", "a", "b"]
deduped = list(dict.fromkeys(items))
print(deduped)
# ['b', 'a', 'c']
```

It’s:

* **Short** — one line.
* **Fast** — O(n) complexity.
* **Pythonic** — uses built-in semantics.

---

## **Caveats**

1. **All values are the same**
   If you pass `value=some_list`, every key will share the same list object:

   ```python
   d = dict.fromkeys("abc", [])
   d['a'].append(1)
   print(d)  # {'a': [1], 'b': [1], 'c': [1]}
   ```

   This is because the **same mutable object** is reused — so be careful.

2. **Order preservation only guaranteed in Python ≥ 3.7**
   In Python 3.5 or earlier, dicts are unordered; you’d use `collections.OrderedDict.fromkeys()` for the same effect.

---

## **Summary**

* `dict.fromkeys()` quickly makes a dict with default values.
* In Python 3.7+, it’s a super-clean way to remove duplicates **while preserving the order of first occurrences**.
* Avoid passing the same mutable default value unless that’s intentional.
