When you use bracket syntax on an object, Python automatically calls that object's `__getitem__()` method behind the scenes:

```python
output['final_answer']
# Python translates this to:
output.__getitem__('final_answer')
```

This is how Python makes syntax like `my_list[0]` or `my_dict['key']` work—lists, dicts, and other built-in types all have `__getitem__` defined internally.

By defining `__getitem__` on your own class, you're hooking into this same mechanism. You're telling Python "when someone uses brackets on my object, here's what to do."

`get()`, by contrast, is just a normal method. Python doesn't treat it specially—it's simply a convention (from dicts) for "lookup with a fallback default." You call it explicitly like any other method:

```python
output.get('final_answer', 'N/A')  # just a regular method call
```
