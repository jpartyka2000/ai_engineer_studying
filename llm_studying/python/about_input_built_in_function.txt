Yes—Python has a built-in function called `input()`. Here’s what you need to know:

1. **Purpose**
   It reads a line from standard input (usually the keyboard), strips the trailing newline, and returns it as a **string**.

2. **Basic Usage**

   ```python
   name = input("What’s your name? ")
   print(f"Hello, {name}!")
   ```

3. **Python 2 vs. Python 3**

   * **Python 3:**

     * `input()` → always returns a `str`.
     * There is no `raw_input()`—`input()` replaced it.
   * **Python 2:**

     * `raw_input()` → reads a line and returns a `str`.
     * `input()` → reads a line and then **evaluates** it as a Python expression (dangerous if you don’t control the input).

4. **Converting to Other Types**
   Since `input()` returns text, you often need to cast:

   ```python
   age_str = input("How old are you? ")
   age = int(age_str)      # convert to integer
   height = float(input("Height in meters? "))
   ```

5. **Handling End-of-File / Interrupts**
   If the user sends an EOF (Ctrl+D on Unix, Ctrl+Z on Windows) or interrupts (Ctrl+C), `input()` will raise an exception (`EOFError` or `KeyboardInterrupt`), so you may want to wrap it:

   ```python
   try:
       answer = input("Continue? (y/n) ")
   except EOFError:
       answer = 'n'
   ```

Beyond `input()`, there’s no separate “command” for reading console input—`input()` is the standard way in modern Python to prompt the user and capture what they type.
