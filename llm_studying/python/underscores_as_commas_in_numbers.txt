Yes, in Python, it is legal to use underscores (`_`) in numeric literals to improve readability, and it serves exactly the purpose you're describing: to separate groups of digits, much like commas in written numbers in English.

### Example:
```python
# Readable large numbers with underscores
million = 1_000_000
billion = 1_000_000_000
```

The underscores have no effect on the value itself; they are purely for readability. Python will interpret `1_000_000` as `1000000`.

### Notes:
1. You can use underscores in integers, floating-point numbers, and even hexadecimal or binary literals:
   ```python
   hex_value = 0xFF_FF_FF
   binary_value = 0b1010_1101_0001
   float_value = 1_000.000_001
   ```

2. You cannot:
   - Start or end a number with an underscore.
   - Place an underscore directly next to a decimal point or other non-digit characters (e.g., in `1_.0` or `0._1`).

### Examples of Invalid Usage:
```python
# These will raise SyntaxError:
invalid = _1000      # Starts with an underscore
invalid = 1000_      # Ends with an underscore
invalid = 1_.0       # Underscore right before the decimal point
invalid = 0._1       # Underscore right after the decimal point
```

Using underscores for grouping numbers is especially helpful when working with large constants in code!
