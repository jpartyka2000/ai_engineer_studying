### **What Is a `pyenv` Shim?**
A **shim** in `pyenv` is a lightweight **proxy script** that sits between your system and the actual Python binaries installed by `pyenv`.  

When you run `python`, `pip`, or any other Python-related command, `pyenv` **intercepts** the request through its **shim system** and redirects it to the appropriate Python version.

---

### **How Does It Work?**
1. **`pyenv` Adds a "Shims" Directory to Your `$PATH`**
   When you install `pyenv`, it modifies your shell configuration (e.g., `~/.bashrc`, `~/.zshrc`) to **prepend** its `shims` directory to your `$PATH`:
   ```bash
   export PATH="$HOME/.pyenv/shims:$PATH"
   ```

2. **When You Run `python`, It First Checks the Shim**
   - Instead of directly calling `/usr/bin/python`, your system executes `~/.pyenv/shims/python`.
   - The `python` shim then determines **which Python version** should be used, based on:
     - `.python-version` (local project setting)
     - `PYENV_VERSION` (environment variable)
     - `pyenv global` (default setting)
   - Finally, it **forwards the command** to the correct Python binary.

---

### **Example: How `pyenv` Decides Which Python to Use**
Assume you have the following setup:

```bash
pyenv versions
# system (default)
# 3.9.10
# 3.10.6 (set by /Users/jpartyka/Desktop/project/.python-version)
```

- If you run `python` inside `/Users/jpartyka/Desktop/project/`, `pyenv` will execute **Python 3.10.6**.
- If you're in a different directory, it may use the **system Python** (or another global setting).

You can check which actual Python binary is being used via:

```bash
which python
# Output: /Users/jpartyka/.pyenv/shims/python
```

And see which real Python version is being used:

```bash
python --version
# Output: Python 3.10.6 (if inside the project directory)
```

---

### **Why Are Shims Useful?**
✅ **No need to modify system paths manually**  
✅ **Allows per-project Python versions**  
✅ **Avoids conflicts with system-wide Python installations**
