When you import a specific function, such as `call_openai` from `call_openai.py`, Python **only makes that function directly accessible** in your namespace. The supporting functions in the same file are not automatically imported into your namespace unless you explicitly import them.

However, **the other functions will still be "loaded" into memory** as part of the module import process, because Python executes the file when it's imported. This means the supporting functions are present in the module's scope, but they will not clutter your `main.py` namespace.

### Example to Clarify:

#### `call_openai.py`
```python
def helper1():
    print("This is helper1")

def helper2():
    print("This is helper2")

def call_openai():
    print("Calling OpenAI...")
    helper1()
    helper2()
```

---

#### `main.py`
```python
from call_openai import call_openai

call_openai()
```

When you run `main.py`, the following happens:

1. The **`call_openai` function** is imported into `main.py`'s namespace, but **not `helper1` or `helper2`**.
2. When you call `call_openai()`, it will still **access the `helper1` and `helper2` functions internally** because they exist in the module scope of `call_openai.py`.

---

### Key Takeaways:
- **Namespace in `main.py`**: Only `call_openai` is accessible.
- **Module Execution**: All functions in `call_openai.py` are loaded into memory because the module is executed when imported.
- **Supporting Functions**: The supporting functions are not imported into `main.py` but are accessible to `call_openai` because they exist in the `call_openai.py` module scope.

If you tried to call `helper1()` directly in `main.py`, it would raise a `NameError` because `helper1` is not in `main.py`'s namespace.

---

### To Summarize:
The supporting functions will **not** be directly imported into `main.py`. Only `call_openai` will be accessible, but the supporting functions will be available to `call_openai` because they exist in the same module's scope.
