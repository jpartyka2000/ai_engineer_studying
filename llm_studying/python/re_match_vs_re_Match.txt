Great question! The reason the `M` in `Match` is capitalized in `Optional[re.Match]` is because it refers to the **type** (or class) `re.Match` from Python's `re` module, not the method `re.match`.

---

### Key Difference Between `re.Match` and `re.match`

1. **`re.Match` (Type/Class)**:
   - This is a type that represents the result of a regex operation, such as `re.search()` or `re.match()`.
   - It is used to specify the **type** of the object returned by these methods when a match is found.
   - Example:
     ```python
     from re import Match
     result: Match = re.search(r"\d+", "123abc")
     ```

2. **`re.match` (Function)**:
   - This is a **method** in the `re` module that performs regex matching at the start of a string.
   - Its name is lowercase because it’s a regular Python function, following the naming convention.

---

### Why Capitalized `Match` is Used in Typing
In Python, **types** or **classes** are conventionally capitalized (e.g., `str`, `int`, `List`, `Match`). Since `re.search()` and `re.match()` return objects of type `re.Match` (if a match is found), you use the class `re.Match` to annotate those results in typing.

---

### Example:
```python
import re
from typing import Optional

# Annotating the return type of re.search
def find_match(pattern: str, string: str) -> Optional[re.Match]:
    return re.search(pattern, string)

# Example usage
match = find_match(r"\d+", "abc123")
if match:
    print(f"Found match: {match.group()}")
```

---

### Summary
- **`re.Match`** is a class/type that represents the object returned by regex operations.
- **`re.match`** is a function used to perform regex matching at the beginning of a string.
- Typing uses `re.Match` (capitalized) because it refers to the type, adhering to Python's convention for type names.
