If your **fixtures** are defined in a separate file, you need to ensure that pytest can discover and use them in your test files. Here's how to do it:

---

## **📌 Step 1: Create a Fixture File (`conftest.py`)**
Pytest automatically discovers fixtures defined in a file named **`conftest.py`**, so you should define your fixtures there.

### **Example: `tests/conftest.py`**
```python
import pytest

@pytest.fixture
def sample_data():
    return {"a": 10, "b": 20}
```

---

## **📌 Step 2: Use the Fixture in a Test File**
You can now **use the fixture in any test file** without needing to import it!

### **Example: `tests/test_math_operations.py`**
```python
def test_addition(sample_data):
    result = sample_data["a"] + sample_data["b"]
    assert result == 30
```
✅ **No import statement needed!** Pytest will automatically find `sample_data` in `conftest.py`.

---

## **📌 Alternative: Store Fixtures in a Separate Module (Non-`conftest.py`)**
If you don’t want to use `conftest.py`, you can define your fixtures in a separate file (e.g., `fixtures.py`) and **import them manually**.

### **Example: `tests/fixtures.py`**
```python
import pytest

@pytest.fixture
def sample_data():
    return {"a": 10, "b": 20}
```

### **Example: `tests/test_math_operations.py`**
```python
import pytest
from tests.fixtures import sample_data  # Explicitly import the fixture

def test_addition(sample_data):
    result = sample_data["a"] + sample_data["b"]
    assert result == 30
```

---

## **📌 Summary**
| Fixture Location  | Do You Need to Import? | Recommended? |
|-------------------|----------------------|--------------|
| `conftest.py`    | ❌ No                 | ✅ Yes (Best practice) |
| `fixtures.py` (or another file) | ✅ Yes (Must import) | ⚠️ Only if necessary |
