That used to be true in older versions of Python, but not anymore in modern Python (3.7 and later).

Here’s the breakdown:

Old behavior (Python < 3.6):

Dictionaries (dict) and sets had non-deterministic iteration order.

This means the order in which keys/items appeared during iteration could change each time the program ran—even if you inserted the items in the same order.


Transition period (Python 3.6):

Python 3.6 (CPython implementation) preserved insertion order for dict, but this was considered an implementation detail, not a language guarantee.


Current behavior (Python 3.7+):

Dicts officially preserve insertion order. That is now part of the Python language specification.

Sets, however, do not guarantee insertion order. Their iteration order may still appear random or change between runs.


Why this matters:

If you're iterating over a set, don’t rely on the order unless you convert it to a list and explicitly sort or order it.

dict order is now reliable if you're using Python 3.7 or newer.


Quick example:

# Python 3.7+
my_dict = {'a': 1, 'b': 2, 'c': 3}
print(list(my_dict))  # Always ['a', 'b', 'c']

my_set = {'x', 'y', 'z'}
print(list(my_set))   # Could be ['x', 'z', 'y'], or any other order
