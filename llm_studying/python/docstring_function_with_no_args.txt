When writing a docstring for a function that doesn't accept any arguments, you can simply omit the `Args` section in the docstring. It's common to describe the function's behavior and any relevant information about its purpose or side effects. The `Returns` section should also describe what the function returns, even if it returns `None`.

### Example of a Function with No Arguments:

```python
def greet_user():
    """
    Prints a greeting message to the user.
    
    This function does not accept any arguments and simply prints
    a friendly message to the console.
    
    Returns:
        None
    """
    print("Hello, user!")
```

### PEP 257 Guidelines for Docstrings
- **No Arguments**: If there are no arguments, the `Args` section is omitted entirely.
- **Describing Behavior**: Focus on describing what the function does. You can mention that it has no parameters, as shown in the docstring above.

### When to Use `Args` or `Parameters`
If your function has no parameters, it's typically enough to focus on the function's purpose and output. If you were to include an `Args` section despite having no arguments, it would be unnecessary and redundant.
