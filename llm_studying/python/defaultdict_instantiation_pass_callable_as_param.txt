### Understanding `defaultdict` and Callables

When you create a `defaultdict` in Python, you pass it a **callable** (like a function, class, or `lambda`) as its first argument. This callable is used to **generate a default value** whenever you access a key that doesn’t exist yet.

For example:

```python
from collections import defaultdict

# Each new key will start with 0 as its value
counts = defaultdict(int)

counts["apples"] += 1
print(counts)  
# {'apples': 1}
```

Here, `int` is the callable. Each time a missing key is referenced, `int()` is called to produce `0`.

---

### Why it must be a callable

If you were to pass in a literal value like `0` or `{}`, every missing key would share that exact same object. Instead, by passing in a callable, `defaultdict` can **call it fresh** each time, creating a new object for each key.

Example with a `lambda`:

```python
# Each new key gets its own dictionary
stats = defaultdict(lambda: {"total": 0, "correct": 0})

stats["easy"]["total"] += 1
stats["hard"]["total"] += 2
print(stats)
# {'easy': {'total': 1, 'correct': 0}, 'hard': {'total': 2, 'correct': 0}}
```

Each key gets a new independent dictionary because the `lambda` is executed separately for each missing key.

---

✅ **Takeaway:**
When instantiating a `defaultdict`, you pass in a **callable factory**. Each time you access a missing key, that callable is invoked to generate the default value. This ensures every key gets its own fresh value, rather than sharing one object across all keys.
