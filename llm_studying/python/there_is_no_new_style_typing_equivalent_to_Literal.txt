You're right that `Any` doesn't have a new-style equivalent — you still need to import it from `typing` (or `typing_extensions`). There's no built-in syntax that replaces it.

`Literal` is the same situation — there's no new-style syntax for it. You still need:

```python
from typing import Literal

def set_mode(mode: Literal["read", "write"]) -> None: ...
```

In general, the new-style syntax (PEP 604's `X | Y`, built-in `list[str]`, `dict[str, int]`, `tuple[int, ...]`, etc.) only replaces things that map to generic aliases of built-in types or `Union`/`Optional`. Anything that's a special form — `Any`, `Literal`, `Final`, `ClassVar`, `TypeGuard`, `TypeVar`, `Protocol`, `Annotated`, etc. — still requires an import from `typing`.
