**`list[str]` vs `List[str]`**

`list[str]` is the modern syntax (Python 3.9+) using the built-in `list` type directly. `List[str]` is the older style requiring `from typing import List`. The built-in lowercase version is now preferred — `typing.List` is considered legacy and may eventually be deprecated.

**`list[str] | str` vs `Union[list[str], str]`**

Same story — the `|` union syntax was introduced in Python 3.10+. `Union` requires `from typing import Union`. The `|` syntax is preferred for its readability. Note it's `Union[...]` with square brackets, not `Union(...)` with parentheses.

**Summary:**

| Modern (preferred) | Legacy | Min Python |
|---|---|---|
| `list[str]` | `List[str]` | 3.9 |
| `dict[str, int]` | `Dict[str, int]` | 3.9 |
| `tuple[int, ...]` | `Tuple[int, ...]` | 3.9 |
| `str \| None` | `Optional[str]` | 3.10 |
| `list[str] \| str` | `Union[list[str], str]` | 3.10 |

If you need to support older Python versions but want the modern syntax, you can add this at the top of your file:

```python
from __future__ import annotations
```

This makes all annotations strings (lazily evaluated), so the modern syntax works even on Python 3.7+.
