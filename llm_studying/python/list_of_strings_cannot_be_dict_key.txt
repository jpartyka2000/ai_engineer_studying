Python dictionaries require their keys to be **hashable** (i.e. immutable) objects. A plain `list` is **not** hashable, so you **cannot** use a list as a dict key:

```python
>>> { ['a','b'] : 123 }
TypeError: unhashable type: 'list'
```

If you want to use a sequence of strings as a single key, you have a few options:

1. **Use a tuple** (an immutable sequence):

   ```python
   key = tuple(['a', 'b', 'c'])   # ('a','b','c')
   d = { key: "some value" }
   print(d[('a','b','c')])        # -> "some value"
   ```

2. **Use a `frozenset`** (an immutable unordered collection):

   ```python
   key = frozenset(['a', 'b', 'c'])
   d = { key: "some value" }
   # since frozenset is unordered, order of items in key doesn’t matter:
   print(d[frozenset(['c','b','a'])])  # -> "some value"
   ```

3. **Serialize the list to a string** (e.g. JSON):

   ```python
   import json
   key = json.dumps(['a','b','c'])
   d = { key: "some value" }
   print(d[json.dumps(['a','b','c'])])  # -> "some value"
   ```

Choose whichever approach best matches your needs. Most commonly, wrapping your list of strings in a `tuple` is the simplest way to get a hashable, ordered key.
