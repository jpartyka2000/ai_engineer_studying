Good question — and it’s one that often gets overlooked.

According to **PEP 8**, the official Python style guide, **comments should start with a capital letter** *if they are complete sentences*.

Here’s the relevant guidance:

---

### 🧠 From PEP 8 (Comments section)

> “Comments should be complete sentences. The first word should be capitalized, unless it is an identifier that begins with a lower case letter (never alter the case of identifiers!).”

---

### ✅ So:

* If your comment is a **full sentence**, start it with a **capital letter**:

  ```python
  # Load data from the external API.
  # This function handles retries and logging.
  ```

* If your comment is a **short phrase** or **inline note**, it can start lowercase:

  ```python
  value = get_rate()  # temporary fix for missing data
  ```

* If your comment starts with a **variable name, function name, or keyword**, keep its original case:

  ```python
  # df is a pandas DataFrame
  # uses np.log() to transform values
  ```

---

### 🧩 Summary

| Type of Comment      | Starts With             | Example                      |
| -------------------- | ----------------------- | ---------------------------- |
| Full sentence        | Capital                 | `# Parse user input safely.` |
| Phrase / fragment    | Lowercase               | `# quick check`              |
| Refers to identifier | Same case as identifier | `# df contains raw results`  |
