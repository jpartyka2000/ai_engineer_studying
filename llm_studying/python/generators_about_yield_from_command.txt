QUESTION:
What is the output of the following code?

    def inner():
        yield 1
        yield 2

    def outer():
        yield "start"
        yield from inner()
        yield "end"

    result = list(outer())
    print(result)

A) ['start', 'end']
B) ['start', [1, 2], 'end']
C) ['start', 1, 2, 'end']
D) ['start', <generator object>, 'end']

------------------------------------------------------------
YOUR ANSWER: D

CORRECT ANSWER: C) ['start', 1, 2, 'end']

RESULT: ✗ Incorrect

EXPLANATION:
- `yield from` delegates to another generator/iterable
- It yields EACH VALUE from the inner generator one by one
- This "flattens" the values into the outer generator's stream

KEY DISTINCTION:
- `yield inner()`      → yields the generator object itself
- `yield from inner()` → yields each value FROM the generator

`yield from` is equivalent to:
    for item in inner():
        yield item

ADDITIONAL BENEFITS OF yield from:
- Properly propagates send() and throw() to inner generator
- Returns the inner generator's return value
- Cleaner syntax for generator delegation/chaining
