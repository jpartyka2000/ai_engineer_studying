If you don't want the `deque` to automatically remove elements when it exceeds a certain size, you can simply **not** set the `maxlen` parameter when initializing the `deque`. This way, the `deque` will not enforce any size limit on its own.

You can then manage the removal of messages manually based on your specific needs. Here's how you can modify your class to implement this:

1. Use a `deque` without a `maxlen` limit.
2. Implement a custom method to remove the oldest messages, such as removing the oldest 2 messages when necessary.

### Example:

```python
from collections import deque

class LLMMessageList:
    def __init__(self, max_history_length=10):
        self.max_history_length = max_history_length
        self.message_list = deque()

    def add_message(self, role, content):
        # Add the new message to the deque
        self.message_list.append({"role": role, "content": content})
        
        # If the deque exceeds the max history length, remove the oldest 2 messages
        if len(self.message_list) > self.max_history_length:
            print("History length exceeded, removing 2 oldest messages...")
            # Remove the oldest 2 messages manually
            self.message_list.popleft()  # Remove the oldest message
            self.message_list.popleft()  # Remove the second oldest message

    def get_messages(self):
        return list(self.message_list)  # Convert deque to list if needed
```

### Key Points:
- **Manual Removal:** Instead of relying on the `maxlen` property of the `deque`, you are manually removing the oldest 2 messages inside the `add_message` method.
- **Flexibility:** This allows you to tailor the behavior for message removal based on your specific criteria, such as removing exactly 2 messages or any other number.
