Yes 🙂 — when you use `csv.DictWriter.writerow()`, it **does add a newline at the end of each row by default**. That’s the normal behavior of the `csv` module: each call to `.writerow()` writes one line of CSV, ending with a line terminator (usually `\r\n` on Windows or `\n` on Unix-like systems).

---

### Why you sometimes see “blank lines”

On **Windows**, if you open the file in text mode without the `newline=""` argument, the `csv` module will insert **extra blank lines** between rows.
That’s why the official docs recommend always opening CSV files like this:

```python
with open("file.csv", "w", newline="") as f:
    writer = csv.DictWriter(f, fieldnames=headers)
```

The `newline=""` tells Python not to “helpfully” translate `\n` into `\r\n\r\n`, so the `csv` module can handle line breaks consistently itself.

---

✅ **Summary:**

* Yes, each `writerow()` call ends with a newline.
* Always open CSV files with `newline=""` to avoid doubled blank lines (especially on Windows).
