### Understanding `csv.DictWriter.writerows()`

When you’re working with Python’s built-in `csv` module, the `DictWriter` class provides two main ways to write data:

* `writerow()` → writes **one row** at a time
* `writerows()` → writes **many rows** at once

This post explains the latter: **`writerows()`**.

---

#### 🔹 What it does

`writerows()` takes an **iterable of dictionaries** (usually a list of dicts) and writes each one as a row to the CSV file, in the order given.

Each dictionary should have keys matching the `fieldnames` you specified when creating the `DictWriter`.

---

#### 🔹 Example

```python
import csv

headers = ["name", "age", "city"]

people = [
    {"name": "Alice", "age": 30, "city": "Toronto"},
    {"name": "Bob", "age": 25, "city": "Vancouver"},
    {"name": "Charlie", "age": 35, "city": "Montreal"},
]

with open("people.csv", "w", newline="") as f:
    writer = csv.DictWriter(f, fieldnames=headers)
    writer.writeheader()
    writer.writerows(people)
```

**Resulting CSV file:**

```
name,age,city
Alice,30,Toronto
Bob,25,Vancouver
Charlie,35,Montreal
```

---

#### 🔹 Handling missing/extra keys

* Missing keys → blank cells in that row.
* Extra keys → by default, raise `ValueError`. You can allow ignoring them with:

  ```python
  writer = csv.DictWriter(f, fieldnames=headers, extrasaction="ignore")
  ```

---

#### 🔹 Key points

* Accepts **any iterable** (list, tuple, generator) of dicts.
* Equivalent to calling `writerow()` in a loop, but more convenient.
* Still adds newlines automatically — use `newline=""` when opening files to avoid blank lines on Windows.

---

✅ **Takeaway:**
Use `csv.DictWriter.writerows()` when you want to write multiple dictionaries to a CSV file in one call, instead of looping and calling `writerow()` repeatedly. It’s a batch-writing convenience method.
