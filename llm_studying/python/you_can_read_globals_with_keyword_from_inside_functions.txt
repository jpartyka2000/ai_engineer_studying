Exactly—if you only **read** a global variable inside a function (i.e. you never assign to that name), Python will automatically look it up in the module’s (or notebook’s) global scope:

```python
my_var = 10

def print_var():
    # No global declaration needed, because we're only reading
    print(my_var)

print_var()   # → 10
```

But the moment you do an assignment to `my_var` inside the function, Python treats it as a new **local** variable unless you declare it:

```python
my_var = 10

def try_to_set():
    my_var = 20    # this creates a *local* my_var, leaving the global untouched

try_to_set()
print(my_var)    # → still 10
```

To rebind the **global** `my_var` inside your function, you must use:

```python
my_var = 10

def set_global():
    global my_var
    my_var = 20    # now this changes the module‐level variable

set_global()
print(my_var)    # → 20
```

So yes—you can freely **read** globals without any keyword, but you need `global` if you want to **write** (assign) to them.
