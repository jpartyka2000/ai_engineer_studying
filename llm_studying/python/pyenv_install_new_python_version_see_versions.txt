If you set a Python version for `pyenv` in a specific local directory using:

```sh
pyenv local 3.11.6  # Example version
```

but **that version is not installed**, then when you try to run `python`, you'll likely see an error message like:

```
pyenv: version `3.11.6` not installed
```

### **How to Fix It**
If you see this error, you need to install the specified Python version:

```sh
pyenv install 3.11.6
```

Once the installation is complete, `pyenv` will automatically use the specified version in that directory.

### **How to Check Which Versions Are Installed**
To see which Python versions are currently installed on your system via `pyenv`, run:

```sh
pyenv versions
```

If the version you set locally is missing from the list, you need to install it before using it.
