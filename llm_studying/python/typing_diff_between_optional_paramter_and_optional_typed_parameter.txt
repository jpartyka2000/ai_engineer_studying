## **Optional Type Hints vs Optional Parameters**

### **Optional Parameters** (Function Design)
These are parameters with **default values**. They're "optional" because the caller doesn't have to provide them.

```python
def greet(name: str, greeting: str = "Hello") -> str:
    return f"{greeting}, {name}!"

# Both calls are valid:
greet("Alice")                    # Uses default
greet("Alice", "Hi")              # Provides value
```

### **Optional Type Hints** (Type System)
`Optional[T]` is a type hint meaning the value can be `T` **or `None`**. It's shorthand for `T | None` or `Union[T, None]`.

```python
from typing import Optional

def find_user(user_id: int) -> Optional[str]:
    # Can return a string OR None
    if user_id == 1:
        return "Alice"
    return None
```

### **Common Mistake: Mixing Them Up**

```python
# ❌ WRONG - Says the parameter can be bool OR None, but default is False
def process(flag: Optional[bool] = False) -> None:
    pass

# This would allow: process(None) - probably not what you want!

# ✅ CORRECT - Parameter has default, but type is just bool
def process(flag: bool = False) -> None:
    pass
```

### **When to Use Optional Type Hints**

```python
# ✅ When None is explicitly allowed and meaningful
def find_user(user_id: int) -> Optional[str]:
    return None  # "user not found"

# ✅ When a parameter can truly be None
def format_name(first: str, middle: Optional[str] = None, last: str = "") -> str:
    if middle is None:
        return f"{first} {last}"
    return f"{first} {middle} {last}"
```

### **Quick Rule:**
- **Default value → optional parameter** (caller doesn't have to provide it)
- **`Optional[T]` → type can be None** (value can literally be `None`)
- These are **independent concepts** - you can have one, both, or neither!
