def receiver():
    received = yield "Ready"
    yield f"You sent: {received}"
```

The magic here is that **`yield` is an expression**, not just a statement. It can do two things:

1. **Send a value OUT** (to the caller)
2. **Receive a value IN** (from `send()`)

Here's the flow:

| Step | Code | What Happens |
|------|------|--------------|
| 1 | `gen = receiver()` | Creates generator, no code runs yet |
| 2 | `next(gen)` | Runs until `yield "Ready"`, outputs `"Ready"`, pauses |
| 3 | `gen.send("Hello")` | Resumes, assigns `"Hello"` to `received`, continues to next `yield`, outputs `"You sent: Hello"` |

The key insight: `received = yield "Ready"` is a single line that **yields first, then receives when resumed**.
```
