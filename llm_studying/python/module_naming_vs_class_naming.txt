You should use **`class_name.py`** (snake_case) for naming Python files (modules).  

### **Why?**
✅ **PEP 8 – Python's Style Guide** recommends using **lowercase with underscores** for filenames (modules).  
✅ **Consistency with standard libraries** – Most built-in Python modules follow this convention (e.g., `os.py`, `json.py`).  
✅ **Avoids conflicts** – Using `ClassName.py` (PascalCase) can confuse Python, as it looks like a class instead of a module.  

### **Best Practice**
If you have a class **`CustomerData`**, name the file **`customer_data.py`**, and then you can import it like:
```python
from customer_data import CustomerData
```
