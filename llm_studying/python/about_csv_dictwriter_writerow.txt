### Understanding `csv.DictWriter.writerow()`

When you’re using Python’s built-in `csv` module, the `DictWriter` class lets you write CSV rows from dictionaries. The key method for adding data is **`writerow()`**.

---

#### 🔹 What it does

`writerow()` takes a **dictionary** and writes one row to the CSV file.

* Each key in the dictionary should match one of the field names you passed when creating the `DictWriter`.
* The corresponding values become the cell values in that row.
* If a key is missing, its cell will be blank.
* If there are extra keys not in `fieldnames`, it raises an error (unless you set `extrasaction="ignore"`).

---

#### 🔹 Example

```python
import csv

headers = ["name", "age", "city"]

with open("people.csv", "w", newline="") as f:
    writer = csv.DictWriter(f, fieldnames=headers)
    writer.writeheader()

    writer.writerow({"name": "Alice", "age": 30, "city": "Toronto"})
    writer.writerow({"name": "Bob", "age": 25, "city": "Vancouver"})
```

**Resulting CSV file:**

```
name,age,city
Alice,30,Toronto
Bob,25,Vancouver
```

---

#### 🔹 Handling missing/extra keys

```python
writer.writerow({"name": "Eve"})  
# age and city will be blank

writer = csv.DictWriter(f, fieldnames=headers, extrasaction="ignore")
writer.writerow({"name": "Eve", "age": 28, "city": "Calgary", "country": "Canada"})
# "country" is ignored
```

---

#### 🔹 Key points

* Always pass a **dict**, not a list or string.
* One call = one row written.
* For multiple rows, use `writerows([dict1, dict2, ...])`.
* Newlines are added automatically; use `newline=""` when opening the file to avoid blank lines (especially on Windows).

---

✅ **Takeaway:**
`csv.DictWriter.writerow()` writes a single dictionary as a row in your CSV. It’s the core method for adding data, mapping your dict’s keys to the CSV’s columns.
