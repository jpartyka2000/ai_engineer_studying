# The Single-Element Tuple Gotcha in Python

One of Python's subtle syntax quirks trips up even experienced developers: creating a tuple with just one element.

## The Problem

Parentheses alone don't make a tuple — the **comma** does.

```python
not_a_tuple = (42)
actually_a_tuple = (42,)

print(type(not_a_tuple))    # <class 'int'>
print(type(actually_a_tuple))  # <class 'tuple'>
```

In the first case, Python interprets `(42)` as just an integer wrapped in parentheses for grouping, the same way you'd write `(2 + 3) * 4`. The parentheses are ignored.

## Why This Matters

This becomes a real issue when building lists of tuples programmatically:

```python
# Broken — first two elements are integers, not tuples
ranges = [(0), (1), (2, 5), (10, None)]

# Fixed — all elements are tuples
ranges = [(0,), (1,), (2, 5), (10, None)]
```

If you iterate over the broken list expecting tuples:

```python
for item in ranges:
    print(len(item))

# Broken version throws: TypeError: object of type 'int' has no len()
# Fixed version prints: 1, 1, 2, 2
```

## The Rule

A tuple is defined by commas, not parentheses. Parentheses are optional except when needed for clarity:

```python
a = 1, 2, 3      # Valid tuple
b = (1, 2, 3)    # Also valid tuple
c = 1,           # Valid single-element tuple
d = (1,)         # Also valid single-element tuple
e = (1)          # NOT a tuple — just the integer 1
```

## Quick Check

When in doubt, verify with `type()`:

```python
>>> type((42))
<class 'int'>

>>> type((42,))
<class 'tuple'>
```

The trailing comma may look odd, but it's essential for single-element tuples.
