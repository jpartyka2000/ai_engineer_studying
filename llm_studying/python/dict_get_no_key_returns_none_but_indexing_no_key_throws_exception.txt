When you use the **“indexing” syntax** (`my_dict['my_key']`):

* If the key exists → you get the value.
* If the key does **not** exist → Python raises a `KeyError`.

Example:

```python
my_dict = {"a": 1}

print(my_dict["a"])   # ✅ 1
print(my_dict["b"])   # ❌ KeyError: 'b'
```

---

In contrast, with `.get()` you avoid the exception:

```python
print(my_dict.get("a"))        # ✅ 1
print(my_dict.get("b"))        # ✅ None (safe default)
print(my_dict.get("b", 42))    # ✅ 42 (custom default)
```

---

✅ **Answer:**
Yes — `my_dict['my_key']` throws a `KeyError` if the key is missing, whereas `my_dict.get('my_key')` safely returns `None` (or a default you specify).
