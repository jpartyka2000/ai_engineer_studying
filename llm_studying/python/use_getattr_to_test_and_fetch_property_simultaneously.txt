Using `getattr()` you can both check for an attribute’s existence and retrieve its value in one go—optionally supplying a default if the attribute isn’t present. This keeps your code concise and avoids a separate `hasattr()` check.

---

## Basic pattern

```python
value = getattr(obj, "attr_name", default_value)
```

* **`obj`**: the object you’re inspecting
* **`"attr_name"`**: the name of the attribute as a string
* **`default_value`**: what to return if `obj` has no `attr_name`

If the attribute exists, `value` is set to `obj.attr_name`. Otherwise, `value` becomes `default_value`—and no exception is raised.

---

## Examples

1. **Fetch with `None` default**

   ```python
   # returns the name if it exists, else None
   name = getattr(event, "name", None)
   if name is not None:
       print("Event name:", name)
   else:
       print("No name on this event.")
   ```

2. **Fetch with a fallback string**

   ```python
   # returns the name or a placeholder
   name = getattr(event, "name", "<unnamed event>")
   print("Processing:", name)
   ```

3. **Using in a single expression**

   ```python
   # increment a counter only if the event has a 'count' attribute
   total += getattr(event, "count", 0)
   ```

---

### Why use `getattr()` this way?

* **Atomicity**: one function call handles both existence check and retrieval.
* **Clarity**: your intent (“get this attribute or fall back to X”) is explicit.
* **Safety**: avoids `AttributeError` without cluttering your code with `try`/`except` or separate `hasattr()` checks.
