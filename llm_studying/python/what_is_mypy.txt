**mypy** is a **static type checker for Python**. You add type hints to your code (PEP 484 style), and mypy analyzes it *without running it* to catch type errors early—great for large codebases and refactors.

### What it does

* Checks functions, classes, and modules against your type hints.
* Supports **gradual typing**: you can annotate a little, then tighten over time.
* Understands `Optional`, `Union`/`|`, `TypedDict`, `Protocol`, generics, dataclasses, and more.
* Uses type stubs from **typeshed** (and packages like `pandas-stubs`) for library types.

### Quick start

```bash
pip install mypy
mypy your_project/
```

```python
# example.py
from typing import List

def mean(xs: List[float]) -> float:
    return sum(xs) / len(xs)

print(mean([1, 2, 3]))     # ok
print(mean(["a", "b"]))    # mypy: error: List[str] incompatible with List[float]
```

### Config (recommended)

Create `mypy.ini` or `pyproject.toml`:

```ini
# mypy.ini
[mypy]
python_version = 3.11
disallow_untyped_defs = True
warn_redundant_casts = True
warn_unused_ignores = True
strict_optional = True
```

Run with `mypy .` (add `--strict` for an even tighter set).

### Handy tricks

* Suppress a specific check: `# type: ignore[assignment]`
* Inspect inferred types while developing: `reveal_type(expr)` (reported in mypy output)
* CI/pre-commit: add `mypy` to your pipeline to keep types from regressing.
