You absolutely **can** define methods in a `@dataclass` — that’s a common misconception 👍

---

## ✅ What `@dataclass` Does

When you decorate a class with `@dataclass`:

* Python automatically generates some boilerplate methods for you (`__init__`, `__repr__`, `__eq__`, etc.) based on the fields.
* But it doesn’t restrict you from adding **your own methods**.

In fact, most real-world dataclasses include helper methods alongside the auto-generated ones.

---

## Example

```python
from dataclasses import dataclass, field
from typing import Dict, Any

@dataclass
class Store:
    items: Dict[str, Any] = field(default_factory=dict)

    def put(self, key: str, value: Any) -> None:
        self.items[key] = value

    def get(self, key: str) -> Any:
        return self.items.get(key)
```

Usage:

```python
s = Store()
s.put("x", 42)
print(s.get("x"))  # 42
```

Here:

* `@dataclass` took care of `__init__` so you didn’t have to write it.
* But you still defined `put` and `get` as custom methods.

---

## ✅ Key Takeaway

* `@dataclass` **generates boilerplate for you**, but it doesn’t limit what else you put in the class.
* You can freely define methods, properties, and even override generated methods (like writing your own `__init__` if needed).
