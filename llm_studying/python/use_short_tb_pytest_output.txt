Yes, you can turn off all output when using `pytest`'s `--collect-only` option by combining it with the `-q` (quiet) switch and the `--tb=short` option.

### **Command:**
```bash
pytest --collect-only -q --tb=short
```

### **Explanation of the Switches:**

1. **`--collect-only`:** This option tells pytest to collect and show all the tests it would run without actually running them. This is useful for discovering tests without executing them.
   
2. **`-q` (Quiet):** This switch suppresses unnecessary output, making the output more concise. It will reduce the verbosity of the collection process.

3. **`--tb=short`:** This switch configures the traceback output to be shorter. While traceback usually shows long error details for failed tests, using `--tb=short` limits the traceback output to just the essential lines.

### **Expected Output:**
With these switches, the output will show only the names of the tests discovered by pytest, without additional verbosity or tracebacks.

#### Example of Output:
```plaintext
tests/test_example.py::test_case_1
tests/test_example.py::test_case_2
```

This combination effectively minimizes the output, making it easier to parse or read the list of collected tests.
