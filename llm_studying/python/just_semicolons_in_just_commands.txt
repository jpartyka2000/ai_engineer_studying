Yes, the semicolons are necessary here. Here's why:

## The backslashes (`\`) escape the newlines

This makes the shell see everything as **one logical line**:

```bash
if [ -n "{{run_name}}" ]; then uv run ... ; else uv run ... ; fi
```

## Semicolons separate shell statements

In shell syntax, you need either a **newline** or a **semicolon** between statements. Compare:

**Multi-line (no semicolons needed):**
```bash
if [ -n "{{run_name}}" ]
then
    uv run ...
else
    uv run ...
fi
```

**Single-line (semicolons required):**
```bash
if [ -n "{{run_name}}" ]; then uv run ...; else uv run ...; fi
```

## Without semicolons, you'd get a syntax error:

```bash
# This is invalid:
if [ -n "foo" ] then echo "yes" else echo "no" fi
# bash: syntax error near unexpected token `else'
```

## Summary

| Location | Purpose |
|----------|---------|
| `]; then` | Ends the test command, separates from `then` |
| `...; else` | Ends the then-block command, separates from `else` |
| `...; fi` | Ends the else-block command, separates from `fi` |
