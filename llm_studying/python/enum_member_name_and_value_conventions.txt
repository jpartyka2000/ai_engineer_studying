There's no strict standard requiring enum member names and values to be identical in Python. Often developers choose to make them the same for convenience and clarity, especially when the enum is used primarily as a set of constant identifiers. However, sometimes you may want the member names to be concise (e.g., `CLV`) while the values carry more descriptive detail (e.g., `"customer_lifetime_value"`). 

In essence, it depends on your use case:

- **Same Names and Values:**  
  This can be convenient when you simply need an enumeration of constant symbols. It makes the code more concise:
  ```python
  class StatsGoal(Enum):
      CLV = "CLV"
      MONTE_CARLO = "MONTE_CARLO"
      CHURN = "CHURN"
  ```

- **Different Names and Values:**  
  This is useful when the value needs to convey a more descriptive or differently formatted piece of information than the name:
  ```python
  class StatsGoal(Enum):
      CLV = "customer_lifetime_value"
      MONTE_CARLO = "monte_carlo"
      CHURN = "churn"
      RETENTION = "retention"
      ARPU = "average_revenue_per_user"
  ```
  Here, the member names are short identifiers, and the values are more descriptive strings that you could, for example, display in user interfaces or logs.

Ultimately, the choice comes down to your specific requirements and readability preferences. There's no enforced convention in Python, so both approaches are acceptable.
