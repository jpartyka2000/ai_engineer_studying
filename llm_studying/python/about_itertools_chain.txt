The function `itertools.chain` takes multiple iterables and returns a single iterator that yields all their elements in sequence, one iterable after the other. In other words, it “chains” them together without creating an intermediate list.

---

### Basic usage

```python
from itertools import chain

a = [1, 2, 3]
b = ('x', 'y')
c = {10, 20}

for item in chain(a, b, c):
    print(item)
# Output (order of c’s set elements may vary):
# 1
# 2
# 3
# x
# y
# 10
# 20
```

Here, `chain(a, b, c)` yields
1 → 2 → 3 → 'x' → 'y' → 10 → 20.

---

### `chain.from_iterable`

If you have *one* iterable whose elements are themselves iterables, you can flatten it one level with `chain.from_iterable`:

```python
from itertools import chain

nested = [[1,2], [3,4], [5]]
flat = chain.from_iterable(nested)
print(list(flat))
# [1, 2, 3, 4, 5]
```

---

### Why use it?

* **Memory‑efficient**: doesn’t build a new list of all elements at once.
* **Lazy**: pulls items only as you iterate.
* **Convenient**: avoids needing nested loops or list concatenations when you just want to iterate over everything in order.

Under the hood, `chain` just keeps an internal pointer to which sub‑iterator it’s on and advances through each until exhaustion before moving to the next.
