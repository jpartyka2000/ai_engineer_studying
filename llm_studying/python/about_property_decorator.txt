The **`@property` decorator** in Python is a way to turn a method into an **attribute-like property** of a class.

It lets you **define dynamic, computed attributes** while keeping the syntax clean — you access them like regular attributes, not like method calls.

---

## Basic Example

```python
class Rectangle:
    def __init__(self, width, height):
        self.width = width
        self.height = height

    @property
    def area(self):
        return self.width * self.height
```

Usage:

```python
r = Rectangle(4, 5)
print(r.area)    # 20   (no parentheses!)
```

Here, `area` looks like a normal attribute, but it’s actually calculated each time you access it.

---

## Why Use `@property`?

1. **Encapsulation**

   * Hide internal details but present a clean API.

2. **Computed values**

   * Calculate something on demand instead of storing it.

3. **Control over attribute access**

   * You can add validation, caching, or lazy evaluation later without changing the external API.

---

## Extending with Setters and Deleters

The `@property` decorator can be paired with `.setter` and `.deleter` to manage assignment and deletion:

```python
class Person:
    def __init__(self, name):
        self._name = name   # convention: leading underscore for "private" var

    @property
    def name(self):
        return self._name

    @name.setter
    def name(self, value):
        if not value:
            raise ValueError("Name cannot be empty")
        self._name = value

    @name.deleter
    def name(self):
        print("Deleting name...")
        del self._name
```

Usage:

```python
p = Person("Alice")
print(p.name)     # Alice
p.name = "Bob"    # calls setter
del p.name        # calls deleter
```

---

## ✅ Summary

* `@property` makes a method behave like an attribute.
* Useful for **computed attributes**, **validation**, and **encapsulation**.
* Can be extended with `@<prop>.setter` and `@<prop>.deleter`.
