Great question! There are several ways Python knows to start from the `src` directory for imports. Here are the main mechanisms:

## 1. **Package Installation**
The most common and robust approach is that the project has a `setup.py` or `pyproject.toml` file that defines `src` as a package. When installed (even in "editable" mode), Python knows where to find it.

Look for files like:
- `setup.py` with something like: `packages=find_packages(where="src")`
- `pyproject.toml` with a `[tool.setuptools]` section specifying package directories

## 2. **PYTHONPATH Environment Variable**
The `src` directory might be added to the `PYTHONPATH`:
```bash
export PYTHONPATH="${PYTHONPATH}:/path/to/repo/src"
```

## 3. **sys.path Manipulation**
The code or a startup script might explicitly add `src` to Python's path:
```python
import sys
from pathlib import Path
sys.path.insert(0, str(Path(__file__).parent / "src"))
```

## 4. **IDE/Tool Configuration**
- IDEs like PyCharm or VS Code can mark `src` as a "Sources Root"
- Tools like pytest can be configured to add `src` to the path

## How to Verify

To check which mechanism is being used in this repo, look for:

1. **`setup.py` or `pyproject.toml`** in the repo root
2. **`.env` files** or shell scripts that set PYTHONPATH
3. **pytest.ini or setup.cfg** with path configurations
4. **README or documentation** explaining installation steps
