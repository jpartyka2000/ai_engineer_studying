The function `random.randrange()` is part of Python’s built-in `random` module. It returns a randomly selected element from the specified range, and its signature is essentially:

```python
random.randrange(start, stop=None, step=1)
```

### Parameters

* **start**

  * If you supply only one argument, it’s treated as `stop` and `start` defaults to 0.
  * Otherwise, `start` is the beginning of the range (inclusive).

* **stop** (optional)

  * The end of the range (exclusive).
  * If omitted, the single argument is used as the `stop` value.

* **step** (optional, default=1)

  * The step size between successive values in the range.

### Behavior

* Picks a random element from the arithmetic progression defined by `range(start, stop, step)`.
* Equivalent to:

  ```python
  choice(range(start, stop, step))
  ```
* Raises `ValueError` if the resulting range is empty (e.g. `start >= stop` or a step that skips all values).

### How It Differs from `randint`

* `random.randint(a, b)` returns a random integer **inclusive** of both `a` and `b`.
* `random.randrange(a, b)` returns from `a` up to but **not including** `b`.

### Examples

```python
import random

# Equivalent to choice(range(0, 10))
print(random.randrange(10))        # e.g. 3

# From 5 up to (but not including) 15
print(random.randrange(5, 15))     # e.g. 12

# From 0 to 100 stepping by 10: {0,10,20,...,90}
print(random.randrange(0, 100, 10))  # e.g. 70

# Rolling a six-sided die (1–6 inclusive):
die_roll = random.randrange(1, 7)
print(die_roll)                     # e.g. 4
```

### Common Pitfalls

* Passing only one argument `n` gives you `0 ≤ result < n`.
* If `step` doesn’t evenly partition the interval, you’ll still get only the values that fit (it doesn’t error unless there are none).
* For truly uniform floats, use `random.uniform()` instead.

That’s the gist of `random.randrange()`—a handy way to pick a random integer from an arithmetic sequence without having to build the sequence yourself.
