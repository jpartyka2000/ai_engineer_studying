Here’s a classic Python demo of non-associativity with IEEE 754 floats:

```python
a = 1e20
b = -1e20
c = 3.14

# Grouping 1: (a + b) + c
res1 = (a + b) + c
print("(a + b) + c =", res1)   # → 3.14

# Grouping 2: a + (b + c)
res2 = a + (b + c)
print("a + (b + c) =", res2)   # → 0.0
```

**What’s happening?**  
1. **`a + b`** is `1e20 + (−1e20)` → exactly `0.0`.  
2. **`(a + b) + c`** → `0.0 + 3.14` → `3.14`.  
3. **`b + c`** is `−1e20 + 3.14` → rounded to `−1e20` (the `3.14` is too small to shift the mantissa).  
4. **`a + (b + c)`** → `1e20 + (−1e20)` → `0.0`.  

Because floating-point addition isn’t associative—thanks to rounding and finite mantissa length—the two ways of parenthesizing give **different** answers.
