Great question! The reason taking the exponential of \(-300\) (e.g., \(e^{-300}\)) doesn’t result in underflow has less to do with Python's handling of large numbers **in this case** and more to do with the limits of floating-point precision.

Here’s the breakdown:

---

### **Why \(e^{-300}\) Works in Python**

1. **Floating-Point Precision in Python**:
   - Python uses **double-precision floating-point numbers** (64-bit IEEE 754 standard) for most numerical computations.
   - This allows representation of values as small as approximately \(2.2 \times 10^{-308}\) (the smallest positive normalized double-precision number).
   - \(e^{-300} \approx 2.06 \times 10^{-131}\), which is much larger than the underflow threshold. Therefore, this value can be represented without any issues.

2. **Exponential Growth and Floating-Point Numbers**:
   - The exponential function \(e^x\) is numerically stable as long as the result lies within the range of double-precision floats:
     - Smallest positive number: ~\(10^{-308}\)
     - Largest positive number: ~\(10^{308}\)
   - Since \(-300\) is well within the acceptable range of inputs, Python computes \(e^{-300}\) without underflow.

3. **Python’s Arbitrary Precision Is Not Used Here**:
   - Python’s `float` type (which underlies `math.exp`) is based on hardware-supported double-precision floats. These are **not arbitrary precision**.
   - However, Python does support arbitrary precision using libraries like `decimal` or `mpmath` for calculations involving extreme numbers beyond the floating-point range.

---

### **When Underflow Happens**

If you were to compute \(e^{-1000}\), for example:
- \(e^{-1000} \approx 1.97 \times 10^{-435}\), which is smaller than the smallest representable positive number in double-precision floating-point (\(~10^{-308}\)).
- This would result in **underflow**, and Python would return \(0.0\) as the result.

Example:
```python
import math
print(math.exp(-1000))  # Outputs: 0.0 (underflow)
```

---

### **How Python Handles Very Small Numbers**
1. **Double-Precision Floating-Point**:
   - Regular `float` computations are constrained by the IEEE 754 standard.
   - Extremely small values (below ~\(10^{-308}\)) are treated as \(0.0\) to avoid representing denormals (subnormal numbers) or further computational instability.

2. **Arbitrary Precision Libraries**:
   - To work with numbers smaller than \(10^{-308}\), you need to use libraries like `decimal` or `mpmath`:
     ```python
     from decimal import Decimal
     print(Decimal('-1000').exp())  # Works for very small numbers
     ```

---

### **Summary**

- \(e^{-300}\) doesn’t underflow because its result (\(\approx 2.06 \times 10^{-131}\)) is well above the minimum threshold for double-precision floats.
- Python's `float` does not use arbitrary precision, but libraries like `decimal` and `mpmath` allow you to work with arbitrarily small or large numbers if needed.
- Underflow happens when the result is too small to represent in the floating-point system, which Python handles by returning \(0.0\).
