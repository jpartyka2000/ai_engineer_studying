Yes, running:  

```sh
pyenv local 3.11.6
```
  
sets Python **3.11.6** **only for the current directory and its subdirectories**.  

### **What Happens?**
- A `.python-version` file is created (or updated) in the current directory.
- When you run `python` in this directory (or any subdirectory), it will use Python 3.11.6.
- Other directories remain unaffected—they will use their own local version (if set) or fall back to the global version.

### **How to Check?**
You can verify the active Python version with:

```sh
pyenv version
```

To see all version settings and their precedence, run:

```sh
pyenv which python
```
