Python’s f‑strings let you embed expressions inside string literals, and you can control *exactly* how those expressions are formatted by adding a *format specifier* after a colon. The general syntax is:

```python
f"{expression:format_spec}"
```

Here’s a breakdown of the most common pieces you can include in `format_spec` (in order):

| Component                                                                           | Meaning                                                                                   | Example  | Result         |
| ----------------------------------------------------------------------------------- | ----------------------------------------------------------------------------------------- | -------- | -------------- |
| **Fill & Align**                                                                    | `[fill][align]` — pad with `fill` character to reach the field width, and align text.     | `{:*<8}` | `"foo*****"`   |
|                                                                                     | `align` can be: `<` (left), `>` (right), `^` (center), `=` (pad after sign, for numbers). | `{:^8}`  | `"  foo   "`   |
| **Sign**                                                                            | `[+- ]` — how to show signs for numbers.                                                  | `{:+}`   | `+42` or `-42` |
| **# Alternate**                                                                     | `#` — with integers/floats, include e.g. `0x` prefix for hex, forced decimal point, etc.  | `{:#x}`  | `0xff`         |
| **0 Zero‑pad**                                                                      | `0` — shorthand for `fill='0'` and `align='='` (pad after sign).                          | `{:08}`  | `"00000042"`   |
| **Width**                                                                           | `<width>` — minimum total field width (pads if content is smaller).                       | `{:5}`   | `"   hi"`      |
| **Grouping**                                                                        | `,_` — thousands separator (\`\`,`for comma,`\_\` for underscore).                        | `{:,}`   | `"1,234,567"`  |
| **Precision**                                                                       | `.<precision>` — for floats, number of digits after decimal; for strings, max length.     | `{:.2f}` | `"3.14"`       |
| **Type**                                                                            | One of                                                                                    |          |                |
|   - **Integers**: `b` (binary), `d` (decimal), `o` (octal), `x`/`X` (hex)           |                                                                                           |          |                |
|   - **Floats**: `f` (fixed), `e`/`E` (scientific), `g`/`G` (general), `%` (percent) |                                                                                           |          |                |
|   - **Strings**: `s` (string)                                                       |                                                                                           |          |                |
|   - **Others**: `c` (character), `r` (repr), etc.                                   |                                                                                           |          |                |
|                                                                                     |                                                                                           | `{:.2%}` | `"314.16%"`    |

Putting those together, a full spec looks like:

```
[fill][align][sign][#][0][width][,][_][.precision][type]
```

---

## Examples

1. **Right‑aligned in 8‑char field, two decimals**

   ```python
   val = 3.14159
   f"{val:>8.2f}"   # width=8, align right, precision=2, fixed‑point
   # → "    3.14"
   ```

2. **Zero‑padded integer with sign**

   ```python
   n = -42
   f"{n:+05d}"      # sign, zero‑pad, width=5, decimal
   # → "-0042"
   ```

3. **Center‑aligned text with custom fill**

   ```python
   name = "Hi"
   f"{name:*^10s}"  # fill='*', center, width=10
   # → "***Hi*****"
   ```

4. **Thousands separator and percent**

   ```python
   frac = 0.314159
   f"{frac:,.1%}"   # comma grouping, 1 decimal, percent
   # → "31.4%"
   ```

5. **Hex with alternate prefix**

   ```python
   x = 255
   f"{x:#06x}"      # '#': include "0x", width=6, zero‑pad
   # → "0x00ff"
   ```

---

### Quick Reference

* **Alignment**: `<`, `>`, `^`, `=`
* **Sign**: `+`, `-`, or space
* **Width**: integer
* **Precision**: `.<digits>`
* **Type**: `b`, `d`, `f`, `e`, `%`, `s`, etc.
* **Grouping**: `,` or `_`
* **Alternate form**: `#`
* **Zero‑pad**: `0`

Using these options you can produce neatly aligned tables, control numeric precision, insert separators, or even build complex reports entirely in formatted strings.
