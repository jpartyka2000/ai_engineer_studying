Test coverage is usually reported as a percentage of code that is executed when running tests. It is typically measured using tools like **coverage.py** in Python. The key metrics in a test coverage report include:

1. **Coverage Percentage** – The proportion of code lines executed during tests, e.g., **85% coverage**.
2. **Missing Lines** – The specific lines that were not executed.
3. **Branch Coverage** – Measures whether both **true** and **false** branches of conditionals were tested.
4. **Statement Coverage** – The percentage of executed lines of code.
5. **Function/Class Coverage** – The percentage of functions or classes that were executed at least once.

### Example of Coverage Report from `coverage.py`:
After running:
```bash
coverage run -m pytest
coverage report -m
```
You might see:
```
Name                        Stmts   Miss  Cover   Missing
---------------------------------------------------------
project/module1.py           120     10    91%    45-50, 75-78
project/module2.py            80     20    75%    22-25, 60-65
---------------------------------------------------------
TOTAL                        200     30    85%
```
This shows how much of each module is covered, where missing lines are, and the total coverage.

For a more visual report, you can use:
```bash
coverage html
```
This generates an **HTML report** with color-coded lines showing which were executed and which were missed.
