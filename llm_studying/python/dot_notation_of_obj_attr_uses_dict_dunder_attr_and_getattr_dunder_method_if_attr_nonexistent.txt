It depends on whether the attribute exists.

For normal attribute access, Python looks up the attribute through the object's `__dict__` and its class hierarchy. No dunder method is called if the attribute is found normally.

But if the attribute **doesn't exist**, Python calls `__getattr__()` as a fallback:

```python
class Thing:
    def __init__(self):
        self.real = 'I exist'
    
    def __getattr__(self, name):
        return f'{name} not found, but __getattr__ caught it'

t = Thing()
t.real      # 'I exist' — found normally, no dunder called
t.fake      # '__getattr__ caught it' — not found, so __getattr__ runs
```

There's also `__getattribute__()` which is called for **every** attribute access (even ones that exist), but it's rarely used since it's easy to cause infinite recursion.

So the short answer: normal attribute lookup doesn't call a dunder method—it just finds the attribute. Dunders only get involved as a fallback or for interception.
