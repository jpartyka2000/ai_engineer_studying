**Question:**
Complete the following class so that `print(dog)` outputs `Dog(name='Buddy', age=3)`:

```python
class Dog:
    def __init__(self, name, age):
        self.name = name
        self.age = age
    
    def _____________(self):
        return f"Dog(name='{self.name}', age={self.age})"

dog = Dog("Buddy", 3)
print(dog)
```

**Your Answer:** `__str__`

**Correct Answer:** `__repr__`

**Result:** ⚠️ Partially Correct

---

**Explanation:**

Both `__str__` and `__repr__` would technically work here with `print()`, but `__repr__` is the better answer for two reasons:

1. **Convention matters:** The output format `Dog(name='Buddy', age=3)` follows the `__repr__` convention — it looks like a valid Python expression that could recreate the object. `__repr__` is meant to be "unambiguous" and developer-focused.

2. **`__str__` is for readability:** `__str__` should return a "nice" human-readable string. For a Dog, that might be something like `"Buddy (3 years old)"`.

3. **Fallback behavior:** If only `__repr__` is defined, `print()` will use it. If only `__str__` is defined, the REPL and debugging tools won't benefit from it.

**Rule of thumb:**
- `__repr__` → for developers, ideally looks like `ClassName(args)`
- `__str__` → for end users, friendly and readable
