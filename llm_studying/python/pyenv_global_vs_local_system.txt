The difference between `pyenv global system` and `pyenv local system` lies in how they affect Python version selection.

### **1. `pyenv global system`**
- This sets the **global Python version** to the system's default Python.
- It affects **all directories** unless overridden by `pyenv local` or `pyenv shell`.
- Example:
  ```sh
  pyenv global system
  ```
- After running this, `python` will use the system-installed Python (e.g., `/usr/bin/python3` on macOS).

### **2. `pyenv local system`**
- This sets the **Python version for a specific directory** to the system's default Python.
- It creates (or updates) a `.python-version` file in the current directory.
- Example:
  ```sh
  pyenv local system
  ```
- When inside this directory, `python` will use the system Python, but other directories may still use a different `pyenv` version.

### **Key Differences**
| Command | Scope | Affects All Directories? | Stores Setting? |
|---------|------|-----------------|----------------|
| `pyenv global system` | Global | Yes | Changes `~/.pyenv/version` |
| `pyenv local system` | Local | No (only current dir & subdirs) | Creates/updates `.python-version` |
