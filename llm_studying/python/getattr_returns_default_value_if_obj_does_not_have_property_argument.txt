Yes, exactly. That's what the third argument to `getattr()` does — it's the default value returned when the attribute doesn't exist.

```python
getattr(agent, 'is_legacy', False)
```

| Scenario | Result |
|----------|--------|
| `agent` has `is_legacy = True` | Returns `True` |
| `agent` has `is_legacy = False` | Returns `False` |
| `agent` has no `is_legacy` attribute | Returns `False` (the default) |

**Without the default, it would raise an error:**

```python
getattr(agent, 'is_legacy')  # Raises AttributeError if not found
```

So the pattern `getattr(obj, 'attr', default)` is a safe way to check for optional attributes without try/except.
