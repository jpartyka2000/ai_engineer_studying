Mypy is a static type checker for Python. It allows you to add type hints to your Python code and then checks the code for type-related errors without actually running it. This helps catch potential bugs and improves code quality and readability.

### Key Features of Mypy:
1. **Static Type Checking**: Mypy analyzes your code and verifies that variables, functions, and return types match the specified type hints.
2. **Supports Optional Typing**: Since Python is a dynamically typed language, type hints are optional. Mypy integrates seamlessly with Python, allowing you to gradually add type hints.
3. **Error Detection**: It catches common type-related issues, such as assigning the wrong type to a variable or calling a function with incorrect arguments.
4. **Integration with IDEs**: Many modern IDEs and editors (like PyCharm, VS Code) support Mypy, highlighting errors directly in the editor.

### How to Install Mypy:
You can install Mypy using `pip`:
```bash
pip install mypy
```

### Example Usage:
Here’s an example to demonstrate Mypy in action:

#### Code with Type Hints:
```python
def add_numbers(a: int, b: int) -> int:
    return a + b

result = add_numbers(5, "10")  # This will cause a type error
```

#### Running Mypy:
Run the following command in the terminal:
```bash
mypy script.py
```

#### Output:
```text
script.py:5: error: Argument 2 to "add_numbers" has incompatible type "str"; expected "int"
```

Mypy identifies that the second argument passed to `add_numbers` is a `str`, but the function expects an `int`.

### Benefits of Using Mypy:
- **Bug Detection**: Catch bugs earlier in the development process.
- **Improved Code Quality**: Enforces a consistent coding style and makes code easier to understand.
- **Documentation**: Type hints serve as documentation, making it clear what types are expected for each function or variable.
- **Enhanced IDE Support**: IDEs can provide better autocomplete and error checking with type hints.

### When to Use Mypy:
- In larger projects where maintaining type consistency is crucial.
- When working in teams, as type hints improve collaboration and code readability.
- For libraries or APIs to clarify how the functions are intended to be used.

No, Mypy does not automatically add type hints to a `.py` file that lacks them. Instead, it analyzes the type hints that already exist in your code and checks for type correctness. If your code lacks type hints, Mypy will not modify your file to include them—it only performs static type checking based on the hints provided.

### How to Add Type Hints to a File Without Them
If you want to add type hints to a `.py` file that doesn’t have any, you need to manually annotate your functions and variables. Here's an example:

#### Original Code Without Type Hints:
```python
def greet(name):
    return "Hello, " + name
```

#### Manually Added Type Hints:
```python
def greet(name: str) -> str:
    return "Hello, " + name
```

### Tools to Assist with Adding Type Hints
If you want to automate the process of suggesting type hints for your code, you can use specialized tools or plugins. Here are some options:

1. **MonkeyType**:
   - A dynamic type inference tool that collects type information at runtime.
   - It observes how your code is executed and generates type annotations based on actual usage.
   - Example:
     ```bash
     pip install monkeytype
     monkeytype run script.py
     monkeytype stub script
     ```

2. **Pyright or Pylance**:
   - Tools integrated into editors like VS Code that provide real-time type inference suggestions based on your code.

3. **Pyre**:
   - Another type checker similar to Mypy that includes some tools for inferring types.

### Summary
- Mypy does not add type hints to your code—it only checks the type hints that you provide.
- You can use tools like **MonkeyType** to generate type hints dynamically based on your code's runtime behavior if you want automated suggestions.
