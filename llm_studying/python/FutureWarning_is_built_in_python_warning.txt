## Explanation

### `FutureWarning` — Works Without Import

`FutureWarning` is a **built-in Python exception class**. It's part of Python's standard warning categories and is always available in the global namespace without any import, just like `Exception`, `TypeError`, `ValueError`, etc.

```python
# These are all built-in, no import needed:
FutureWarning
DeprecationWarning
UserWarning
RuntimeWarning
```

### `ConstantInputWarning` — Requires Import

`ConstantInputWarning` is **not a built-in**. It's a specific warning class defined in `scipy.stats`. Your editor shows a red line because the name is undefined in the current namespace.

To fix it:

```python
from scipy.stats import ConstantInputWarning

warnings.filterwarnings('ignore', category=ConstantInputWarning)
```

### Why Your Code Might Still Run

If you have `ConstantInputWarning` in your code without importing it, a few things could be happening:

1. **The line hasn't been executed yet** — If it's in a conditional branch or function that hasn't been called, Python won't raise the `NameError` until it tries to execute that specific line.

2. **You're running different code** — The version you're running may differ from what's in your editor.

3. **It will fail at runtime** — If that line executes, you'll get:
   ```
   NameError: name 'ConstantInputWarning' is not defined
   ```

### Recommended Fix

Add the import at the top of your file:

```python
import warnings
from scipy.stats import ConstantInputWarning

warnings.filterwarnings('ignore', category=FutureWarning)
warnings.filterwarnings('ignore', category=ConstantInputWarning)
```
