You're right to push back — let me clarify.

Python *always* checks for the attribute normally first (in `__dict__`, class hierarchy, etc.) before falling back to `__getattr__`. It doesn't matter that we know `compute_mean` won't be there — Python still does the lookup attempt.

So the sequence is:

1. Python looks for `compute_mean` on the instance and its class hierarchy
2. It doesn't find it
3. *Then* it calls `__getattr__(self, "compute_mean")`

`__getattr__` is specifically the "fallback" hook — it only gets invoked when normal attribute resolution fails.
