`pytest.mark.parametrize` is a powerful feature in pytest that allows you to run a single test function with multiple sets of inputs and expected outputs. It enables parameterized testing, making it easy to test a function or feature with various inputs without writing separate test functions for each case.

---

### **Basic Syntax**
```python
@pytest.mark.parametrize(
    "param1, param2, expected",  # Parameter names as a comma-separated string
    [
        (value1, value2, expected1),  # Each tuple represents a set of inputs and the expected output
        (value3, value4, expected2),
        # Add more cases as needed
    ]
)
def test_function(param1, param2, expected):
    result = some_function(param1, param2)
    assert result == expected
```

---

### **Key Features**
1. **Multiple Test Cases in One Function**:
   - You can test multiple inputs with a single function by providing a list of parameter sets.

2. **Improves Readability**:
   - The test cases are clearly defined, and the test function remains clean and focused.

3. **Avoids Redundancy**:
   - No need to duplicate test functions for different inputs.

4. **Works with Fixtures**:
   - Parameters can be combined with fixtures to add even more flexibility.

---

### **Examples**

#### **1. Testing Simple Functionality**
```python
@pytest.mark.parametrize(
    "a, b, expected",
    [
        (1, 2, 3),  # Test case 1
        (-1, 1, 0),  # Test case 2
        (0, 0, 0),  # Test case 3
    ]
)
def test_add(a, b, expected):
    assert a + b == expected
```

#### **2. Testing Exceptions**
You can also test whether a specific input raises an expected exception:
```python
import pytest

@pytest.mark.parametrize(
    "input_val, exception",
    [
        (10, ValueError),  # Test case 1: expect ValueError
        ("string", TypeError),  # Test case 2: expect TypeError
    ]
)
def test_raises_exception(input_val, exception):
    with pytest.raises(exception):
        some_function(input_val)
```

#### **3. Parameterizing with IDs**
You can add descriptive labels to each test case for better test output:
```python
@pytest.mark.parametrize(
    "a, b, expected",
    [
        (1, 2, 3),
        (0, 0, 0),
        (-1, -1, -2),
    ],
    ids=["positive_numbers", "zeros", "negative_numbers"]
)
def test_add_with_ids(a, b, expected):
    assert a + b == expected
```

The output will show:
```plaintext
test_example.py::test_add_with_ids[positive_numbers] PASSED
test_example.py::test_add_with_ids[zeros] PASSED
test_example.py::test_add_with_ids[negative_numbers] PASSED
```

#### **4. Parameterizing a Fixture**
You can combine `@pytest.mark.parametrize` with fixtures for even more flexible testing:
```python
@pytest.fixture
def base_number():
    return 10

@pytest.mark.parametrize("multiplier, expected", [(1, 10), (2, 20), (3, 30)])
def test_multiplication(base_number, multiplier, expected):
    assert base_number * multiplier == expected
```

---

### **Best Practices**
1. **Use Descriptive IDs**:
   - Add labels using the `ids` parameter to make test output easier to understand.

2. **Keep Tests Independent**:
   - Ensure that each test case is isolated and doesn't depend on others.

3. **Combine with Fixtures for Complex Cases**:
   - Use fixtures to handle shared setup or teardown logic.

4. **Use Lists of Dictionaries for Clarity**:
   - For complex inputs, use dictionaries instead of tuples to make test cases more readable:
   ```python
   @pytest.mark.parametrize(
       "case",
       [
           {"input": 1, "multiplier": 2, "expected": 2},
           {"input": -1, "multiplier": 2, "expected": -2},
       ]
   )
   def test_function(case):
       assert case["input"] * case["multiplier"] == case["expected"]
   ```

---

### **Conclusion**
`pytest.mark.parametrize` is a highly versatile tool for writing clean and efficient tests in Python. It simplifies repetitive testing tasks, reduces code duplication, and makes test cases more manageable.
