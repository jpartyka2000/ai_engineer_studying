In Python, the recommended guideline for spacing in type hints is **no space before the colon (`:`)** and **one space after the colon** (if followed by a type hint). This is specified in **PEP 8**, Python's style guide, under the section about function annotations.

### Correct Spacing Example:
```python
def myfunc(some_param: int) -> int:
    ...
```

### Guidelines Summary:
1. **No space before the colon**:
   ```python
   some_param: int  # Correct
   some_param : int  # Incorrect
   ```
2. **One space after the colon if a type hint is provided**:
   ```python
   some_param: int  # Correct
   some_param:int  # Incorrect
   ```
3. **If no type hint is provided (just a colon), leave no space after the colon** (e.g., for dictionary literals):
   ```python
   my_dict = {"key": "value"}  # Correct
   ```

### Examples:
- **Correctly formatted function annotations**:
  ```python
  def calculate_area(length: float, width: float) -> float:
      return length * width
  ```

- **Incorrect formatting**:
  ```python
  def calculate_area(length :float, width :float)->float:
      return length * width
  ```

Following these guidelines ensures that your code adheres to Python's best practices and is clean and readable.
