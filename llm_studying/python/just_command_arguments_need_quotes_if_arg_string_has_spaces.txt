## It Depends on the Client Name

### **No Quotes Needed If:**
The client name is a **single word** with no spaces or special characters:
```bash
just dry-run acme "What is Python?"
just dry-run customer123 "What is Python?"
just dry-run my-client "What is Python?"
```

### **Quotes REQUIRED If:**
The client name has **spaces or special characters**:
```bash
just dry-run "Acme Corp" "What is Python?"
just dry-run "customer #123" "What is Python?"
just dry-run "my client & co" "What is Python?"
```

## Why This Works:

`just` passes the argument to the command, which becomes:
```bash
# Without quotes (single word):
just dry-run acme "question"
# Becomes: python -m cue_training.dry_run -c acme -q "question"

# With quotes (has spaces):
just dry-run "Acme Corp" "question"
# Becomes: python -m cue_training.dry_run -c Acme Corp -q "question"
#                                            ^^^^^ ^^^^
#                                            Problem! Two arguments
```

## Shell Special Characters Need Quotes:

```bash
# These need quotes:
just dry-run "client&co" "question"      # & is special
just dry-run "client|test" "question"    # | is special
just dry-run "client>file" "question"    # > is special
just dry-run "$CLIENT" "question"        # $ is special

# These are fine without quotes:
just dry-run client-123 "question"       # - is fine
just dry-run client_test "question"      # _ is fine
just dry-run client.prod "question"      # . is fine
```

## Best Practice:

**Always use quotes** for consistency and safety:
```bash
just dry-run "acme" "What is Python?"  # Works even if unnecessary
```

This prevents surprises if the client name changes later to include spaces.

**Bottom line**: Quotes are only **required** if the client name has spaces or shell special characters, but it's safer to always use them.
