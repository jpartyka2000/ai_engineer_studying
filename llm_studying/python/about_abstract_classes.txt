### **Abstract Classes in Python: A Primer**  

#### **What is an Abstract Class?**
An **abstract class** in Python is a class that **cannot be instantiated** and is meant to be **subclassed**. It typically defines **a common interface** (methods and attributes) that all subclasses must implement.  

Python implements abstract classes using the `ABC` module (`Abstract Base Classes`).

---

### **1️⃣ Defining an Abstract Class**
To create an abstract class, **inherit from `ABC`** and mark at least one method with `@abstractmethod`:  

```python
from abc import ABC, abstractmethod

class Animal(ABC):  # Abstract class
    @abstractmethod
    def make_sound(self):
        """Every subclass must implement this method"""
        pass
```
✔ **Cannot be instantiated:**  
```python
a = Animal()  # ❌ TypeError: Can't instantiate abstract class Animal
```
✔ **Must be subclassed and implemented:**
```python
class Dog(Animal):
    def make_sound(self):
        return "Woof!"

d = Dog()  # ✅ Works fine
print(d.make_sound())  # Woof!
```

---

### **2️⃣ Abstract Methods & Implementation Rules**
- An **abstract method** has **no implementation** in the base class.
- A subclass **must implement all abstract methods** to be instantiated.
- A subclass can **override non-abstract methods** from the base class.

```python
from abc import ABC, abstractmethod

class Vehicle(ABC):
    def __init__(self, name):
        self.name = name  # Regular instance attribute

    @abstractmethod
    def move(self):
        pass  # Must be implemented in subclasses

    def info(self):
        return f"This is a {self.name}"  # Regular method

class Car(Vehicle):
    def move(self):
        return "Driving on the road"

car = Car("Tesla")
print(car.move())  # Driving on the road
print(car.info())  # This is a Tesla
```
✔ **Abstract methods (`move`) must be implemented**  
✔ **Regular methods (`info`) can be inherited without modification**  

---

### **3️⃣ Partial Implementation: Subclassing Another Abstract Class**
A class can **inherit from an abstract class** but still be abstract itself if it doesn’t implement all abstract methods.

```python
class FlyingVehicle(Vehicle):  # Still an abstract class
    @abstractmethod
    def fly(self):
        pass

# Cannot instantiate
# fv = FlyingVehicle()  # ❌ TypeError

class Airplane(FlyingVehicle):
    def move(self):
        return "Taxiing on the runway"

    def fly(self):
        return "Flying in the sky"

plane = Airplane("Boeing 747")
print(plane.fly())  # Flying in the sky
```
✔ `FlyingVehicle` is still abstract because it doesn’t implement `move`.  
✔ `Airplane` is concrete because it implements both `move` and `fly`.

---

### **4️⃣ Abstract Properties**
You can define **abstract properties** using `@property` and `@abstractmethod` together:

```python
from abc import ABC, abstractmethod

class Shape(ABC):
    @property
    @abstractmethod
    def area(self):
        """Each shape must implement its own area calculation."""
        pass

class Circle(Shape):
    def __init__(self, radius):
        self.radius = radius

    @property
    def area(self):
        return 3.14 * self.radius ** 2

circle = Circle(5)
print(circle.area)  # 78.5
```
✔ `Circle` **must implement `area`** as a property.

---

### **5️⃣ Abstract Class with Default Implementations**
An abstract class **can provide some default implementations** for methods.

```python
class Bird(ABC):
    def eat(self):  # Regular method
        return "Pecking food"

    @abstractmethod
    def fly(self):
        pass

class Sparrow(Bird):
    def fly(self):
        return "Flapping wings"

sparrow = Sparrow()
print(sparrow.eat())  # Pecking food (inherited method)
print(sparrow.fly())  # Flapping wings
```
✔ **Regular methods (`eat`) are inherited as-is**  
✔ **Abstract methods (`fly`) must be implemented**  

---

### **6️⃣ When to Use Abstract Classes**
Use abstract classes when:
✅ You need to enforce a **common interface** across multiple subclasses.  
✅ You want to provide **some shared behavior** but force subclasses to implement key methods.  
✅ You are designing a **framework** where certain methods **must be overridden**.  

---

### **Key Takeaways**
| Feature | Abstract Class Behavior |
|---------|------------------------|
| Can be instantiated? | ❌ No |
| Can have regular methods? | ✅ Yes |
| Can have attributes? | ✅ Yes |
| Can provide default implementations? | ✅ Yes |
| Must be subclassed? | ✅ Yes |
| Must implement all `@abstractmethod`s in subclasses? | ✅ Yes |
