Great question — both `list[str]` and `List[str]` are used in type annotations, but they come from slightly different eras of Python typing.

---

## 1. `List[str]` (capital **L**)

* Comes from the **`typing`** module:

  ```python
  from typing import List
  ```
* Used in **Python 3.5–3.8** as the standard way to declare generic types in type hints.
* Example:

  ```python
  from typing import List

  def names() -> List[str]:
      return ["Alice", "Bob"]
  ```

---

## 2. `list[str]` (built-in, lowercase **l**)

* Introduced in **Python 3.9** (PEP 585).
* You can now use the built-in collection types (`list`, `dict`, `tuple`, etc.) directly with subscripts for type annotations.
* Example:

  ```python
  def names() -> list[str]:
      return ["Alice", "Bob"]
  ```

---

## 3. Key Differences

| Feature                   | `List[str]`              | `list[str]`                |
| ------------------------- | ------------------------ | -------------------------- |
| **Origin**                | `typing` module          | Built-in type              |
| **Python support**        | 3.5+                     | 3.9+                       |
| **Preferred in new code** | ❌ (backward-compat only) | ✅ (PEP 585 standard)       |
| **Runtime type**          | Just an alias for `list` | The actual built-in `list` |

---

## 4. Interoperability

They both mean the same thing to type checkers (mypy, Pyright, etc.), but **`list[str]` is now the modern, recommended style** if you’re on Python ≥3.9.

If you need to support older Python versions (like 3.7/3.8), you’ll still see `List[str]` in codebases.

---

✅ **In short:**

* Use **`list[str]`** in modern Python (3.9+).
* Use **`List[str]`** if you’re stuck supporting Python <3.9.
* Functionally, they mean the same thing: “a list of strings.”
