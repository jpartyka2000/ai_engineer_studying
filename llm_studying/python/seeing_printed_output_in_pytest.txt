To see output from `print` statements in your executed unit tests with `pytest`, use one of these methods:

---

### **1. Use the `-s` Option (Best Method)**
The `-s` flag tells `pytest` not to capture stdout/stderr, allowing `print` statements to appear in the terminal.

```bash
pytest -s
```

#### **Example**
```python
def test_example():
    print("This will be shown in the terminal.")
    assert True
```
✅ **Output will be displayed during test execution.**

---

### **2. Use `capsys` Fixture (If You Want to Capture Output)**
If you want to **capture and assert printed output** within your test, use `capsys`:

```python
def test_example(capsys):
    print("Hello, pytest!")
    captured = capsys.readouterr()
    assert "Hello, pytest!" in captured.out
```
✅ **Ensures `print` output is tested properly.**

---

### **3. Disable Output Capturing in `pytest.ini` (Permanent Setting)**
If you want to **always see print output**, you can disable output capturing in a `pytest.ini` configuration file:

```ini
[pytest]
addopts = -s
```
✅ **Now, running `pytest` will automatically show print outputs.**

---

### **4. Use Logging Instead of Print (Recommended for Large Projects)**
If you are debugging **large-scale tests**, consider using Python’s `logging` module instead of `print`. You can configure `pytest` to display logs:

```python
import logging

def test_logging():
    logging.basicConfig(level=logging.INFO)
    logger = logging.getLogger(__name__)
    logger.info("This is an info log message.")
```
Run with:
```bash
pytest --log-cli-level=INFO
```
✅ **Better for structured debugging and scalable projects.**

---

### **Summary**
| Method | Command | Best For |
|--------|---------|----------|
| `-s` flag | `pytest -s` | Quick debugging |
| `capsys` fixture | `capsys.readouterr()` | Testing printed output |
| `pytest.ini` | `addopts = -s` | Always showing output |
| Logging | `--log-cli-level=INFO` | Large-scale debugging |
