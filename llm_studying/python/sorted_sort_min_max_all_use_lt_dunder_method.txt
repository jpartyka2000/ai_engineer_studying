**Question:**
What method makes `sorted()` work on your custom objects?

```python
class Temperature:
    def __init__(self, celsius):
        self.celsius = celsius
    
    def __repr__(self):
        return f"Temperature({self.celsius})"
    
    def _______(self, other):
        return self.celsius < other.celsius

temps = [Temperature(30), Temperature(10), Temperature(20)]
print(sorted(temps))
# [Temperature(10), Temperature(20), Temperature(30)]
```

**Your Answer:** `__lt__`

**Correct Answer:** `__lt__`

**Result:** ✅ Correct

---

**Why `__lt__`?**

Python's `sorted()` and `list.sort()` only need `__lt__` (less than) to work. They use it to compare pairs of elements and determine order.

```python
# Python's sort essentially does comparisons like:
if a < b:  # Calls a.__lt__(b)
    # a comes before b
```

**Bonus:** This also works with `min()` and `max()`:

```python
temps = [Temperature(30), Temperature(10), Temperature(20)]

print(min(temps))  # Temperature(10)
print(max(temps))  # Temperature(30)
```
