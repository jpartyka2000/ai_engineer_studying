pytest --durations=N is a useful option in pytest that helps identify slow-running tests. It reports the N slowest test executions in your test suite, allowing you to pinpoint performance bottlenecks in your tests.


---

Usage

pytest --durations=10

This will display the 10 slowest tests in descending order of execution time.

Example Output

=========================== slowest 10 durations ============================
1.10s call     tests/test_model.py::test_large_dataset
0.95s call     tests/test_api.py::test_slow_api_response
0.85s call     tests/test_db.py::test_database_insert
...

This helps determine which tests take the longest and might need optimization.


---

Variants and Additional Options

1. Show all test durations (no limit)

pytest --durations=0

Lists all tests with their execution times.



2. Include setup and teardown times

pytest --durations=5 --durations-min=0.5

Shows the 5 slowest tests including setup and teardown times.

Only reports tests that take at least 0.5 seconds.



3. Combine with other options

pytest --durations=10 -v --maxfail=3

Runs in verbose mode (-v).

Stops execution after 3 failures (--maxfail=3).





---

How to Optimize Slow Tests

If pytest --durations reveals slow tests, consider:

Using Fixtures Efficiently: Ensure that setup/teardown isn’t redundant.

Mocking Expensive Operations: Replace database/API calls with mocks.

Running Tests in Parallel: Use pytest-xdist (pytest -n auto).

Profiling for Bottlenecks: Use cProfile or pytest-profiling.
