### Understanding `csv.DictWriter.writeheader()`

When you’re working with Python’s built-in `csv` module, the **`DictWriter`** class is handy for writing rows of data where each row is a dictionary. One common step is writing the **header row** (the column names). That’s exactly what `writeheader()` does.

---

#### 🔹 What it does

`writeheader()` writes a single row containing the field names you specified when you created the `DictWriter`. These field names become the column headers in your CSV file.

---

#### 🔹 Example

```python
import csv

headers = ["name", "age", "city"]

with open("people.csv", "w", newline="") as f:
    writer = csv.DictWriter(f, fieldnames=headers)

    writer.writeheader()  # writes: name,age,city

    writer.writerow({"name": "Alice", "age": 30, "city": "Toronto"})
    writer.writerow({"name": "Bob", "age": 25, "city": "Vancouver"})
```

**Resulting CSV file:**

```
name,age,city
Alice,30,Toronto
Bob,25,Vancouver
```

---

#### 🔹 When to call it

* Call `writeheader()` once, before writing any rows.
* If you’re appending to an existing file, you may want to skip it (otherwise you’ll repeat headers). You can check if the file exists or is empty first.

---

#### 🔹 Key points

* Uses the `fieldnames` you passed into `DictWriter`.
* Writes them in order, separated by the chosen delimiter (default is `,`).
* Does not automatically write headers — you must call it explicitly.

---

✅ **Takeaway:**
Use `csv.DictWriter.writeheader()` when you want the first row of your CSV file to contain the column names. It’s a one-time call that makes your CSV files easier to read and more useful when imported into spreadsheets or databases.
