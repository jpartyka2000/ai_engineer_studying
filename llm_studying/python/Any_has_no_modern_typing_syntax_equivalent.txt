There's no modern shorthand for `Any` — you still need to import it:

```python
from typing import Any

def analyze(data: Any) -> Any:
    ...
```

`Any` essentially opts out of type checking — it's compatible with every type. Use it sparingly, as it defeats the purpose of type hints. If you find yourself reaching for `Any`, consider whether `object` or a generic type might be more appropriate:

- `object` — accepts any type but restricts you to only using methods common to all objects
- `Any` — accepts any type and lets you do anything with it (no checking at all)
