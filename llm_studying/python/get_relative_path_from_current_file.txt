Yes, in Python, you can use `os.path.relpath()` to get the relative path from one file or directory to another. This function returns the relative path from the specified path to the target path.

### Usage:

```python
import os

# Example: Get relative path from current script to another file
current_script_path = os.path.dirname(os.path.abspath(__file__))  # Absolute path of the current script
target_file_path = "/path/to/target/file.txt"  # Absolute path to the target file

relative_path = os.path.relpath(target_file_path, current_script_path)
print("Relative path:", relative_path)
```

### Explanation:
- `os.path.abspath(__file__)` gives the absolute path of the current script.
- `os.path.dirname()` is used to get the directory of the current script.
- `os.path.relpath(target_file_path, current_script_path)` computes the relative path from the `current_script_path` to the `target_file_path`.

### Example:

Assume:
- The current script is located at `/home/user/scripts/test_script.py`.
- The target file is located at `/home/user/data/input.txt`.

The code will return a relative path like:

```
../data/input.txt
```

This tells you how to navigate from `test_script.py` to `input.txt` relative to the script's location.

### Notes:
- If the `target_file_path` is outside the current directory tree, `os.path.relpath()` will provide the necessary `..` (parent directory) navigation.

Yes, the `with open()` function in Python supports **relative paths**. You can use relative paths when opening files, just as you would with absolute paths. The relative path is resolved relative to the current working directory where the Python script is executed.

### Example:

```python
# Assuming 'data.txt' is in the same directory as your script
with open("data.txt", "r") as file:
    content = file.read()
    print(content)
```

### Relative Path Explanation:
- If `data.txt` is in the same directory as the script, `"data.txt"` is the relative path.
- If `data.txt` is in a subdirectory, you can specify a relative path like `"subfolder/data.txt"`.
- If the file is in a parent directory, you can use `..` to navigate up, e.g., `"../data.txt"`.

### Example with Subfolder and Parent Directory:

```python
# Opening a file in a subfolder
with open("subfolder/data.txt", "r") as file:
    content = file.read()
    print(content)

# Opening a file in the parent directory
with open("../data.txt", "r") as file:
    content = file.read()
    print(content)
```

### Important Notes:
1. **Current Working Directory**: The relative path is evaluated based on the **current working directory** (the directory where you run the script or the one set as the working directory in your IDE).
2. **Directory Changes**: If your script changes directories using `os.chdir()`, the relative paths will be interpreted relative to the new current working directory.
3. **File Not Found**: If the relative path is incorrect (e.g., the file doesn't exist in the specified location), Python will raise a `FileNotFoundError`. You can use `os.path.exists()` to check if the file exists before opening it.
