No, you do not need to explicitly import `None` for type hinting in Python. The `None` keyword is built into Python and is always available. 

### Example Usage
You can use `None` in type hints directly without any additional imports:
```python
from typing import Optional

def greet(name: Optional[str] = None) -> None:
    if name is None:
        print("Hello, world!")
    else:
        print(f"Hello, {name}!")
```

### Optional and `None`
When using `None` in type hints, it is common to combine it with `Optional` (from the `typing` module). `Optional[X]` is equivalent to `Union[X, None]`, meaning the value can be of type `X` or `None`.

For example:
```python
from typing import Optional

def process(value: Optional[int]) -> None:
    if value is None:
        print("No value provided")
    else:
        print(f"Processing value: {value}")
```

### Explicit Use of `None`
You can also use `None` directly in a `Union` if you prefer:
```python
from typing import Union

def process(value: Union[int, None]) -> None:
    if value is None:
        print("No value provided")
    else:
        print(f"Processing value: {value}")
```

However, `Optional[int]` is more concise and is the recommended approach for hinting a value that can be `None`.
