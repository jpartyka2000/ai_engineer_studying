In Python, the terms "pass by value" and "pass by reference" don't strictly apply in the same way they do in languages like C or Java. Instead, Python uses a mechanism called pass by object reference or call by sharing. Here's how it works:

1. Mutable vs Immutable Types:

Immutable objects (like integers, floats, strings, and tuples): When you pass these to a function, you cannot modify the original object. If you try to change the value, a new object is created. For example:

def modify(x):
    x = 10
a = 5
modify(a)
print(a)  # Output will be 5

In this case, x gets a new value (10), but it does not affect the original a because a is an immutable integer.

Mutable objects (like lists, dictionaries, and sets): When you pass these to a function, you can modify the original object. For example:

def modify_list(lst):
    lst.append(4)
my_list = [1, 2, 3]
modify_list(my_list)
print(my_list)  # Output will be [1, 2, 3, 4]

Here, my_list is modified because lists are mutable, and lst in the function points to the same object as my_list.



2. What Actually Happens:

When you pass an object to a function, Python passes a reference to that object (not the actual object itself). However, whether you can change the object depends on whether the object is mutable or immutable.

For immutable types, if you change the value, it creates a new object rather than modifying the original.

For mutable types, you can directly modify the original object using its reference.



3. Key Takeaway:

If you pass an immutable object, you cannot change its original value inside the function.

If you pass a mutable object, you can modify its contents inside the function.




Understanding this helps in designing functions that either modify the original objects (in-place) or return new objects, based on the use case.
