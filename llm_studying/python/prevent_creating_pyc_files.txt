To prevent Python from generating `.pyc` files (compiled bytecode files), you can use one of the following methods:

---

### **1. Use the `PYTHONDONTWRITEBYTECODE` Environment Variable**
Set the `PYTHONDONTWRITEBYTECODE` environment variable to `1`. This tells Python not to generate `.pyc` files during execution.

#### **On Linux/MacOS:**
```bash
export PYTHONDONTWRITEBYTECODE=1
python script.py
```

#### **On Windows (Command Prompt):**
```cmd
set PYTHONDONTWRITEBYTECODE=1
python script.py
```

#### **On Windows (PowerShell):**
```powershell
$env:PYTHONDONTWRITEBYTECODE = 1
python script.py
```

---

### **2. Use the `-B` Command-Line Option**
Run the Python interpreter with the `-B` option, which prevents the generation of `.pyc` files.

#### Example:
```bash
python -B script.py
```

---

### **3. Modify the `sys.dont_write_bytecode` Variable in Code**
You can disable the generation of `.pyc` files programmatically by setting the `sys.dont_write_bytecode` variable to `True`.

#### Example:
```python
import sys
sys.dont_write_bytecode = True
```

- Add this code at the top of your script to apply it globally for the script's runtime.
- Note: This method is limited to the script in which it is set.

---

### **Which Method Should You Use?**
- **Temporary Change**: Use the `-B` option or the `PYTHONDONTWRITEBYTECODE` environment variable.
- **Script-Specific Change**: Use the `sys.dont_write_bytecode` variable.
- For permanent or system-wide changes, you can set `PYTHONDONTWRITEBYTECODE` in your shell profile or system environment variables.
