The `indent` parameter in `json.dumps()` controls how “pretty‐printed” your JSON output is, by inserting newlines and indent spaces. Here’s what you need to know:

1. **Type and Meaning**

   * If you pass an integer $n$, each level of nesting in the resulting JSON will be indented by $n$ spaces.
   * If you pass `"\t"` (a string), each level will be indented by one tab character.
   * If you pass `None` (the default), the output is as compact as possible (no extra whitespace beyond the separators).

2. **Line Breaks**
   As soon as you set `indent` to a non-`None` value, `json.dumps()` breaks after every item and between braces/brackets. You get one element per line, making deeply nested structures far easier to read.

3. **Example**

   ```python
   import json

   data = {"name":"Alice", "scores":[10, 20, {"math":95,"eng":88}]}

   # Compact (default)
   print(json.dumps(data))
   # → {"name":"Alice","scores":[10,20,{"math":95,"eng":88}]}

   # Pretty, 2-space indent
   print(json.dumps(data, indent=2))
   # → {
   #       "name": "Alice",
   #       "scores": [
   #         10,
   #         20,
   #         {
   #           "math": 95,
   #           "eng": 88
   #         }
   #       ]
   #     }
   ```

4. **Combining with `sort_keys` and `separators`**

   * You can add `sort_keys=True` to alphabetically order your keys.
   * You can tweak the tuple `separators` to remove trailing spaces after commas/colons (e.g., `separators=(',', ': ')`) for more control over compactness vs. readability.

In short, `indent` is your “pretty‐print” switch: pick a small number (2 or 4) for human-friendly JSON, or stick with `None`/omit it for the tightest machine‐oriented output.
