Great question — Python exposes **many environment variables** that can affect how your script runs, how modules are imported, how errors are handled, and more.

Here’s a categorized overview of the most useful and commonly encountered ones 👇

---

## 🧩 **1. Core Interpreter Behavior**

| Variable                          | Description                                                                                                                 |
| --------------------------------- | --------------------------------------------------------------------------------------------------------------------------- |
| **`PYTHONPATH`**                  | A list of directories that are added to `sys.path` at startup. Used to control where Python looks for modules and packages. |
| **`PYTHONHOME`**                  | Overrides the location of the standard Python libraries. Useful when embedding Python or managing custom environments.      |
| **`PYTHONSTARTUP`**               | Path to a script executed every time you start an interactive Python shell.                                                 |
| **`PYTHONEXECUTABLE`**            | Path to the Python interpreter binary. Often used in virtual environments.                                                  |
| **`PYTHONCASEOK`** (Windows only) | Makes import statements case-insensitive. Rarely used.                                                                      |
| **`PYTHONUTF8`**                  | If set to `1`, forces UTF-8 mode (overrides locale settings). Great for consistent text encoding.                           |
| **`PYTHONIOENCODING`**            | Controls encoding used for stdin/stdout/stderr (e.g. `utf-8`). Useful in Docker or pipelines.                               |

---

## ⚙️ **2. Module Importing and Optimization**

| Variable                      | Description                                                                                                            |
| ----------------------------- | ---------------------------------------------------------------------------------------------------------------------- |
| **`PYTHONDONTWRITEBYTECODE`** | If set (to any value), prevents creation of `.pyc` files.                                                              |
| **`PYTHONOPTIMIZE`**          | If set (e.g. to `1` or `2`), runs the interpreter in optimized mode (`-O`), removing assert statements and docstrings. |
| **`PYTHONNOUSERSITE`**        | Skips adding user site directories (`site.USER_SITE`) to `sys.path`.                                                   |
| **`PYTHONPYCACHEPREFIX`**     | Changes the location where `.pyc` bytecode files are written.                                                          |

---

## 🧠 **3. Debugging, Tracing, and Performance**

| Variable                      | Description                                                                                                                     |
| ----------------------------- | ------------------------------------------------------------------------------------------------------------------------------- |
| **`PYTHONBREAKPOINT`**        | Defines the default debugger for `breakpoint()`. Example: `PYTHONBREAKPOINT=ipdb.set_trace`. Set to `0` to disable breakpoints. |
| **`PYTHONFAULTHANDLER`**      | If set to `1`, automatically enables the `faulthandler` module for crash tracebacks (like segfaults).                           |
| **`PYTHONTRACEMALLOC`**       | Enables memory allocation tracing (e.g. `PYTHONTRACEMALLOC=1` or `=10` for depth).                                              |
| **`PYTHONPROFILEIMPORTTIME`** | If set, prints import times for each module — very useful for startup profiling.                                                |
| **`PYTHONWARNINGS`**          | Controls how warnings are displayed (e.g. `ignore`, `error`, `default`, etc.). Equivalent to `-W` flag.                         |
| **`PYTHONHASHSEED`**          | Controls hash randomization for deterministic behavior (set to a fixed integer).                                                |

---

## 🧱 **4. Virtual Environment and Packaging**

| Variable                     | Description                                                                   |
| ---------------------------- | ----------------------------------------------------------------------------- |
| **`VIRTUAL_ENV`**            | Set by `venv` or `virtualenv` to point to the active environment’s directory. |
| **`PYTHONUSERBASE`**         | Changes the base directory for user site packages (default is `~/.local`).    |
| **`PIP_REQUIRE_VIRTUALENV`** | If set, disallows `pip` from running outside a virtual environment.           |

---

## 🌐 **5. Multi-threading and Miscellaneous**

| Variable                 | Description                                                                       |
| ------------------------ | --------------------------------------------------------------------------------- |
| **`PYTHONTHREADDEBUG`**  | Enables debug output for Python threads. Rarely used except in deep debugging.    |
| **`PYTHONMALLOC`**       | Chooses allocator: `malloc`, `pymalloc`, or `debug`. Useful for memory debugging. |
| **`PYTHONASYNCIODEBUG`** | Enables async debug mode (shows warnings for blocked loops, etc.).                |
| **`PYTHONINSPECT`**      | Forces interactive mode after running a script (same as `python -i script.py`).   |

---

## 💡 Example Usage

```bash
export PYTHONHASHSEED=0
export PYTHONWARNINGS=ignore
export PYTHONPATH=/my/custom/modules:$PYTHONPATH
export PYTHONFAULTHANDLER=1

python my_script.py
```

---

## 🧰 To List Them Programmatically

You can inspect Python-related environment variables inside a script:

```python
import os
for key, value in os.environ.items():
    if key.startswith("PYTHON"):
        print(f"{key}={value}")
```
