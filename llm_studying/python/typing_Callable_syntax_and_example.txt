**QUESTION:**
What's the correct type hint for a function that takes a callable which accepts two integers and returns a string?

A) `def process(func: Callable[[int, int], str]) -> None:`
B) `def process(func: Callable[int, int, str]) -> None:`
C) `def process(func: Function[[int, int], str]) -> None:`
D) `def process(func: (int, int) -> str) -> None:`

**YOUR ANSWER:** A

**CORRECT ANSWER:** A

**EXPLANATION:**
Option A is correct! The `Callable` type hint syntax is:
```python
Callable[[param_types], return_type]
```

**Key points:**
- Parameter types go in a **list** (double brackets): `[int, int]`
- Return type comes after, separated by comma: `str`
- Requires `from typing import Callable`

**Why other options are wrong:**
- **B:** Missing the list brackets around parameters - invalid syntax
- **C:** There's no `Function` type in the typing module (it's `Callable`)
- **D:** This arrow syntax isn't valid Python (though it looks like TypeScript/other languages)

**Example usage:**
```python
from typing import Callable

def process(func: Callable[[int, int], str]) -> None:
    result = func(5, 10)
    print(result)

# This function matches the signature:
def add_and_format(a: int, b: int) -> str:
    return f"Sum is {a + b}"
```
