Yes, **Black** has specific opinions on import formatting, but it does **not** enforce import ordering. Here’s how it handles imports:

### **1. Import Ordering**
- **Black does not reorder imports**—it leaves them in the order they were written.
- If you want to enforce ordering, you should use **isort** alongside Black.

### **2. Import Formatting**
- **One import per line**:  
  ```python
  import os
  import sys
  ```
  Instead of:
  ```python
  import os, sys
  ```

- **Multi-line imports**: Black enforces parentheses when a line is too long:  
  ```python
  from module import (
      function_one,
      function_two,
      function_three,
  )
  ```

- **Spaces around `as` in imports**:  
  ```python
  import numpy as np
  from pandas import DataFrame as DF
  ```

### **Using Black with isort**
Since Black doesn’t reorder imports, it’s common to use **isort** for sorting and Black for formatting. To make them compatible, run:

```sh
isort . && black .
```
Or configure **isort** to follow Black’s style:

```sh
isort . --profile black
```
