### 📦 Python Package Naming Conventions

When organizing your code into packages (directories of modules), following consistent naming conventions makes your projects easier to understand, import, and maintain. These conventions are drawn primarily from [PEP 8](https://peps.python.org/pep-0008/#package-and-module-names) and common practice in the Python community.

---

#### 1. All Lowercase

* **Package names** should be **all lowercase**.
* This avoids confusion on case-insensitive filesystems (macOS, Windows) and matches the wider ecosystem.

  ```text
  good:    mypackage/
           imageprocessing/
  bad:     MyPackage/
           ImageProcessing/
  ```

---

#### 2. Avoid Underscores Unless Needed

* Unlike modules (where `snake_case` is common), underscores in package names are **discouraged** if the name remains clear without them.
* Use `my_package/` only if `mypackage/` is hard to read or ambiguous.

  ```text
  preferred:  datatools/
  acceptable: data_tools/    # if it really clarifies meaning
  ```

---

#### 3. No Hyphens or Special Characters

* Hyphens (`-`) are **not valid Python identifiers**, so avoid them in directory names.
* Stick to letters and optionally digits.

  ```text
  good:    analytics3d/
  bad:     analytics-3d/
           analytics.3d/
  ```

---

#### 4. Keep It Short and Descriptive

* Aim for **conciseness**—long names are cumbersome in import statements.
* Yet ensure the name **conveys the package’s purpose**.

  ```text
  concise & clear:  auth/
                    config/
                    payments/
  overly long:     user_authentication_and_authorization/
  ```

---

#### 5. Top-Level vs. Namespace Packages

* If your project uses **namespace packages** (PEP 420), follow the same lowercase rules across all subpackages.
* Ensure each directory either contains an `__init__.py` (for regular packages) or is intentionally left without one (for implicit namespaces).

---

#### 6. Align Directory & Distribution Names

* The name you publish on PyPI (the distribution name) should generally match your top-level package name, using hyphens to separate words if necessary:

  ```text
  PyPI name:      my-package
  top-level pkg:  mypackage
  ```
* This alignment helps users find and import your package without confusion.

---

#### 7. Examples of Well-Named Packages

* **Requests** → `requests/`
* **NumPy** → `numpy/`
* **Flask** → `flask/`
* **scikit-learn** → `sklearn/` (PyPI name uses hyphen, package name is lowercase without underscore)

---

### ✅ Takeaway

* **Lowercase** and **concise** are your guiding lights.
* **Underscores** only when clarity demands.
* **Avoid** hyphens or special characters.
* Mirror your package’s directory name and your PyPI distribution name as closely as possible (modulo hyphens).

Following these simple rules keeps your Python packages clean, consistent, and immediately recognizable to other developers.
