In **pytest**, the definition of a "unit test" depends on the level of granularity you're considering:  

1. **A test function** counts as **one test** by default.  
   - Example:
     ```python
     def test_addition():
         assert 1 + 1 == 2
     ```
     - This is **one test** in pytest's count.  

2. **A parameterized test function** counts as **multiple tests**, one for each test case.  
   - Example:
     ```python
     import pytest

     @pytest.mark.parametrize("a, b, expected", [(1, 1, 2), (2, 2, 4), (3, 3, 6)])
     def test_addition(a, b, expected):
         assert a + b == expected
     ```
     - This counts as **three tests**, because pytest runs the function separately for each `(a, b, expected)` tuple.  

### **Checking the Test Count**
Run pytest with `-v` (verbose mode):
```sh
pytest -v
```
If the test function is parameterized, the output will show each test case as a separate test:
```
test_math.py::test_addition[1-1-2] PASSED
test_math.py::test_addition[2-2-4] PASSED
test_math.py::test_addition[3-3-6] PASSED
```
This confirms that **each test case in a parameterized function is counted as a separate test**.
