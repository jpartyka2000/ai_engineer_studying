The error you're encountering is because `distutils` was removed in Python 3.12, but PySpark (specifically older versions) still depends on it. This is a known compatibility issue between PySpark and Python 3.12.

Here are a few solutions to fix this:

## Solution 1: Install setuptools (Easiest)

The `distutils` module has been merged into `setuptools`. Installing setuptools should provide the necessary compatibility:

```bash
uv pip install setuptools
```

Then try running your command again:
```bash
uv run python -m cue_training.agent.analysis_tools
```

## Solution 2: Use Python 3.11 (Most Compatible)

Since the README specifies Python 3.11 or higher, but there's a compatibility issue with 3.12, you might want to use Python 3.11 specifically:

```bash
# Install Python 3.11 using UV
uv python install 3.11

# Create a new virtual environment with Python 3.11
uv venv --python 3.11

# Reinstall dependencies
uv sync

# Run your command
uv run python -m cue_training.agent.analysis_tools
```

## Solution 3: Update PySpark

If the project allows, you could try updating to a newer version of PySpark that's compatible with Python 3.12:

```bash
uv pip install --upgrade pyspark
```

However, this might cause compatibility issues with other parts of the project, so check the `pyproject.toml` or `requirements.txt` for version constraints first.

## Solution 4: Force distutils compatibility

As a temporary workaround, you can install the `distutils` package separately:

```bash
uv pip install setuptools
uv pip install distutils-stubs
```

## Recommended Approach

I'd recommend **Solution 1** first (installing setuptools), as it's the simplest and usually resolves the issue. If that doesn't work, **Solution 2** (using Python 3.11) is the safest option since it matches the project's documented requirements.

After applying one of these solutions, make sure your `.env` file is properly configured with your Databricks credentials before running the analysis tools.
