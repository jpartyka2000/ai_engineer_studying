If you have a list of class names and you want to instantiate each class into a new object, you can dynamically create instances using **`globals()`** (or **`locals()`** if the classes are locally defined) or by accessing the class definitions from a module using **`getattr()`**.

Here’s how you can do it:

### Example with `globals()`:
If the classes are globally defined in your script, you can use `globals()` to reference them.

```python
class ClassA:
    def __init__(self):
        self.name = "ClassA"

class ClassB:
    def __init__(self):
        self.name = "ClassB"

# List of class names as strings
class_names = ['ClassA', 'ClassB']

# Instantiate each class and store in a list
instances = [globals()[class_name]() for class_name in class_names]

# Check instances
for instance in instances:
    print(instance.name)
```

### Example using `getattr()`:
If the classes are defined in a module (or you prefer a safer method than `globals()`), you can use **`getattr()`** to retrieve the class from the module.

```python
import my_module  # Assuming the classes are defined in my_module

# List of class names as strings
class_names = ['ClassA', 'ClassB']

# Instantiate each class and store in a list
instances = [getattr(my_module, class_name)() for class_name in class_names]

# Check instances
for instance in instances:
    print(instance.name)
```

### Explanation:
- **`globals()`**: Returns a dictionary of all global variables, where the class names can be looked up.
- **`getattr()`**: Retrieves an attribute (in this case, the class) from a module by name.
- **`()`: Instantiates the class after retrieving it.

Both methods allow you to dynamically instantiate classes based on their names stored in a list.
