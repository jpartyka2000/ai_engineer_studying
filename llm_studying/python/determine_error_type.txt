To find out the type of error (exception) that was raised in Python, you can use the `type()` function along with the exception object, or use `traceback` to capture and display the full exception details. Here's how you can do it:

### Method 1: Using `type()` and `except` Block
You can catch the exception and print its type using `type()`:

```python
try:
    # Code that may raise an error
    result = 1 / 0  # Example of an error
except Exception as e:
    print(f"Error type: {type(e).__name__}")  # Prints the type of the error
    print(f"Error message: {e}")  # Prints the error message
```

**Output:**
```
Error type: ZeroDivisionError
Error message: division by zero
```

### Method 2: Using `traceback` to Get Detailed Information
The `traceback` module gives a more detailed view, including the error type, message, and where the error occurred:

```python
import traceback

try:
    # Code that may raise an error
    result = 1 / 0  # Example of an error
except Exception as e:
    print(f"Error type: {type(e).__name__}")
    print(f"Error message: {e}")
    traceback.print_exc()  # Prints the full traceback
```

**Output:**
```
Error type: ZeroDivisionError
Error message: division by zero
Traceback (most recent call last):
  File "script.py", line 5, in <module>
    result = 1 / 0
ZeroDivisionError: division by zero
```

### Method 3: Using `logging` to Log the Error Type and Message
If you're using logging, you can capture and log the error type and message:

```python
import logging

logging.basicConfig(level=logging.INFO)

try:
    # Code that may raise an error
    result = 1 / 0
except Exception as e:
    logging.error(f"Error type: {type(e).__name__}, Message: {e}")
```

**Output in log:**
```
ERROR:root:Error type: ZeroDivisionError, Message: division by zero
```

### Method 4: Directly Printing the Error Type and Message
If you don't want to use `traceback`, you can just print the error type and message:

```python
try:
    # Code that may raise an error
    result = 1 / 0
except Exception as e:
    print(f"Exception occurred: {e.__class__.__name__} - {e}")
```

**Output:**
```
Exception occurred: ZeroDivisionError - division by zero
```

### Summary:
- `type(e).__name__` gives you the name of the exception type.
- `e` will give you the message associated with the exception.
- `traceback.print_exc()` provides detailed information about where the exception occurred and the stack trace.

This will help you identify the exact error and gather more context about it.
