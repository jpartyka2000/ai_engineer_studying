No, installing a Python version with `pyenv` **does not replace** the system Python or any other versions already installed on your machine. Instead, `pyenv` keeps all installed Python versions isolated in `~/.pyenv/versions/`, and it manages which version is used via **shims** in your shell.

### **How `pyenv` Works:**
1. When you install Python with `pyenv install 3.11.6`, it places that version inside `~/.pyenv/versions/3.11.6/`.
2. Your system’s default Python (e.g., `/usr/bin/python` or `/usr/local/bin/python`) remains untouched.
3. `pyenv` works by adjusting your **`$PATH`** so that when you type `python`, it first looks in `~/.pyenv/shims/` instead of using the system Python.
4. If you don’t explicitly set a version with `pyenv global` or `pyenv local`, your system will continue using the default Python.

---

### **How to Use a Pyenv-installed Version**
To make sure you're using the newly installed Python:
- Set it globally for all shell sessions:
  ```sh
  pyenv global 3.11.6
  ```
- Or, set it only for a specific project directory:
  ```sh
  pyenv local 3.11.6
  ```
- Check which Python is active:
  ```sh
  python --version
  ```
  or
  ```sh
  pyenv which python
  ```
  If properly configured, it should show something like:
  ```
  /Users/yourname/.pyenv/versions/3.11.6/bin/python
  ```

---

### **How to Use the System Python Again**
If you ever need to switch back to the system Python, you can do:
```sh
pyenv global system
```
or
```sh
pyenv local system  # Only for the current directory

python --version  # Should now show the macOS default Python version
```
