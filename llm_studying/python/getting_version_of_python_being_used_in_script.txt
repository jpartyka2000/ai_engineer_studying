In a Databricks `.py` file, you can get the Python version using several methods:

## Method 1: Using `sys` module (most common)
```python
import sys
print(sys.version)  # Full version info including build details
print(sys.version_info)  # Structured version info as a tuple
```

## Method 2: Using platform module
```python
import platform
print(platform.python_version())  # Returns string like '3.9.5'
```

## Method 3: In a Databricks notebook cell
```python
# Just for the major.minor version
import sys
print(f"Python {sys.version_info.major}.{sys.version_info.minor}")
```

## Method 4: Using shell command in Databricks
```python
# Using dbutils to run shell command
dbutils.notebook.run("python --version")
# Or
import subprocess
result = subprocess.run(['python', '--version'], capture_output=True, text=True)
print(result.stdout)
```

## Method 5: Check in Spark context
```python
# Since Databricks uses PySpark
spark.conf.get("spark.databricks.python.version")
```

The most straightforward approach is using `sys.version` or `sys.version_info`. For example:

```python
import sys
print(f"Python version: {sys.version}")
# Output: Python version: 3.9.5 (default, May  4 2021, 03:36:27) [GCC 7.5.0]
```

Note that the Python version in Databricks depends on the runtime version you've selected for your cluster. Different Databricks Runtime versions come with different Python versions pre-installed.
