The error `"nothing to repeat at position 1"` occurs because the `*` character is interpreted as a regex quantifier that means "repeat the preceding element 0 or more times." If there's no preceding element, it causes an error.

In your regex pattern:

```python
r'^*text (you)? provided*'
```

The `*` after `^` and `provided` is causing the issue because `^` is an anchor (not a character) and doesn't represent something that can be repeated. To fix this, you need to remove `*` when it's not being used with a preceding character, or replace it with the correct regex syntax.
