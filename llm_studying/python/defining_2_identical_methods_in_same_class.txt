When you define two methods with the *same* name in a single class body, Python will bind that name to whichever function object was assigned most recently. In other words, the *second* (later) definition simply overwrites the first in the class’s namespace, and only that one “survives.”

```python
class Foo:
    def bar(self):
        print("first version")

    def bar(self):
        print("second version")

f = Foo()
f.bar()   # prints "second version"
```

Here, `Foo.__dict__['bar']` is bound to the second `bar`, so calling `f.bar()` uses the second definition. The first is discarded.
