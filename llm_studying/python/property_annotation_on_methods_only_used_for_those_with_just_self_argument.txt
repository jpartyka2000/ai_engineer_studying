Yes, `@property` is essentially only used with `(self)` — that's the whole point.

It makes a method behave like an attribute:

```python
class Circle:
    def __init__(self, radius):
        self.radius = radius
    
    @property
    def area(self):
        return 3.14159 * self.radius ** 2

c = Circle(5)
c.area      # Access like an attribute, not c.area()
```

**If you need arguments, just use a regular method:**

```python
class Circle:
    def area_scaled(self, factor):  # Needs argument, so regular method
        return 3.14159 * self.radius ** 2 * factor

c.area_scaled(2)  # Called with parentheses
```

**The property trio (all take only `self` or `self` + assigned value):**

```python
class Thing:
    @property
    def value(self):          # getter: self only
        return self._value
    
    @value.setter
    def value(self, val):     # setter: self + assigned value
        self._value = val
    
    @value.deleter
    def value(self):          # deleter: self only
        del self._value

t = Thing()
t.value = 5    # setter
t.value        # getter
del t.value    # deleter
```
