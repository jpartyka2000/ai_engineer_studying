A **PKL (Pickled) dataset** refers to a dataset that has been serialized using Python's `pickle` module, typically saved with a `.pkl` file extension. This format allows Python objects, including **Pandas DataFrames**, NumPy arrays, and other data structures, to be saved and loaded efficiently.

### **🔹 Why Use PKL Files?**
- **Fast Read/Write**: Pickled data loads much faster than CSV or JSON.
- **Preserves Data Types**: Unlike CSV, which may lose type information, pickling retains Python-specific structures.
- **Easy Storage & Retrieval**: Useful for caching processed datasets or models.

### **🔹 How to Save a Dataset as a PKL File**
#### **Saving a Pandas DataFrame**
```python
import pandas as pd

# Create sample DataFrame
df = pd.DataFrame({"A": [1, 2, 3], "B": [4, 5, 6]})

# Save as a .pkl file
df.to_pickle("dataset.pkl")
```

---

### **🔹 How to Load a PKL Dataset**
```python
df_loaded = pd.read_pickle("dataset.pkl")
print(df_loaded)
```

---

### **🔹 When Should You Use PKL?**
✅ **When working within Python** (not easily readable by non-Python apps).  
✅ **For fast read/write performance** (better than CSV or JSON).  
✅ **When storing complex data structures** like dictionaries or lists of objects.
