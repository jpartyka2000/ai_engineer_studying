**`sort_keys` in `json.dumps()`**

The `sort_keys` parameter tells Python to order the keys of dictionaries alphabetically in the output JSON. By default (`sort_keys=False`), the keys appear in the same order they were inserted into the original Python dict. Setting `sort_keys=True` produces deterministic, lexicographically sorted key order, which can be helpful for:

* **Readability & Consistency**
  When you’re manually inspecting JSON, having keys in A-to-Z order makes it easier to locate fields and compare structures across files.
* **Diff-Friendly Outputs**
  If you’re checking JSON files into version control, sorted keys prevent irrelevant reorderings from showing up in diffs, keeping changes focused on real data edits.
* **Canonicalization**
  In contexts where you need a “canonical” string form of your JSON (for hashing, signing, or caching), sorting keys ensures that logically identical dicts always serialize to the exact same byte sequence.

**Example**

```python
import json

data = {"b": 2, "a": 1, "c": {"z": 26, "y": 25}}

print(json.dumps(data, sort_keys=False))
# → {"b": 2, "a": 1, "c": {"z": 26, "y": 25}}

print(json.dumps(data, sort_keys=True, indent=2))
# → {
#      "a": 1,
#      "b": 2,
#      "c": {
#        "y": 25,
#        "z": 26
#      }
#    }
```

Use `sort_keys=True` whenever deterministic ordering or cleaner diffs matter; skip it for maximum performance or when preserving insertion order is important.
