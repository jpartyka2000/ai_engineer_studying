When using **Poetry** and **pyenv** together, you typically manage your project's dependencies with **Poetry** and manage the Python version with **pyenv**. 

Here's the right way to install a new dependency while ensuring that both **Poetry** and **pyenv** are working seamlessly together:

### **1. Make Sure the Correct Python Version is Set with `pyenv`**
Before installing any dependencies, you should ensure you're using the correct Python version for your project. You can do this with `pyenv`.

- **Check the current Python version in the directory**:
  
  ```bash
  pyenv version
  ```

- **Set a specific Python version for your project** (if needed):

  ```bash
  pyenv local 3.10.6  # or any version you want
  ```

This sets the Python version for the project directory, ensuring that the correct version is used when you install dependencies with Poetry.

### **2. Install the Dependency with Poetry**
Now that the correct Python version is set (via `pyenv`), you can install the new dependency using **Poetry**.

#### **To install a regular dependency:**

```bash
poetry add <dependency-name>
```

For example:

```bash
poetry add requests
```

This will:
- Install the dependency (`requests` in this example).
- Add it to your `pyproject.toml` file under `[tool.poetry.dependencies]`.
- Resolve and lock the versions in the `poetry.lock` file.

#### **To install a development dependency:**

If you want to install a package as a **development dependency** (e.g., for testing or linting):

```bash
poetry add --dev <dependency-name>
```

For example:

```bash
poetry add --dev pytest
```

This will add `pytest` to the `[tool.poetry.dev-dependencies]` section in your `pyproject.toml` file.

### **3. Verify the Installation**
After installing the dependency, Poetry will automatically update the `pyproject.toml` and `poetry.lock` files. You can verify that the dependency has been added successfully by running:

```bash
poetry show
```

This will show a list of all installed dependencies.

### **4. Installing All Dependencies (When Setting Up a New Environment)**
If you're setting up the project in a new environment and want to install all dependencies (both regular and dev dependencies):

```bash
poetry install
```

This will install all dependencies listed in the `pyproject.toml` and `poetry.lock` files into the active Poetry-managed virtual environment.

---

### **Summary:**
1. **Set the Python version** for your project using `pyenv` (if not already done).
2. **Install dependencies** using Poetry with `poetry add <dependency-name>` (for regular dependencies) or `poetry add --dev <dependency-name>` (for dev dependencies).
3. **Verify installation** using `poetry show`.

By using both `pyenv` and `poetry`, you can ensure the correct Python version and dependencies are used within your project.
