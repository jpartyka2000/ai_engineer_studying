In Python, `set()` is technically a **constructor** (specifically, the constructor for the `set` class), though it's often casually referred to as a "built-in function."

Here's the distinction:

`set` is actually a **type** (a class). When you call `set()`, you're invoking the class to create a new instance—just like calling any other class constructor. You can verify this:

```python
>>> type(set)
<class 'type'>

>>> isinstance(set, type)
True
```

This is different from a true function like `len()`:

```python
>>> type(len)
<class 'builtin_function_or_method'>
```

The same applies to other built-in types like `list()`, `dict()`, `int()`, `str()`, and `tuple()`—they're all class constructors, not functions.

That said, Python's documentation often lists these under "built-in functions" for practical purposes, since from a usage standpoint they behave similarly (you call them with parentheses and get a value back). The distinction matters more when you're thinking about Python's object model or doing things like subclassing.
