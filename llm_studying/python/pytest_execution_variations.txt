Yes, you can configure **pytest** to run only a specific `.py` file within a directory containing multiple test files. Here’s how you can do it:

---

### **Option 1: Specify the File Name**
You can pass the path of the specific `.py` file you want to run when invoking pytest.

#### Example Command:
```bash
pytest test_file_name.py
```

#### Programmatically:
```python
import pytest

pytest.main(["test_file_name.py", "-v"])
```

---

### **Option 2: Use the `-k` Option for a Specific Test**
If you only want to run tests from a specific file, you can use `-k` to match specific test names or keywords.

#### Example Command:
```bash
pytest -k "test_specific" test_file_name.py
```

---

### **Option 3: Using Markers (Custom Tags for Tests)**
If you organize your tests using **markers**, you can run only the tests in the specified file that match the marker.

#### In `test_file_name.py`:
```python
import pytest

@pytest.mark.my_marker
def test_example():
    assert True
```

#### Example Command:
```bash
pytest test_file_name.py -m my_marker
```

---

### **Option 4: Using Test Discovery**
If you just want to run tests in a specific file without specifying each test explicitly, simply pass the file's name or path.

#### Example Directory Structure:
```
tests/
├── test_one.py
├── test_two.py
```

#### Run `test_one.py`:
```bash
pytest tests/test_one.py
```

#### Programmatically:
```python
import pytest

pytest.main(["tests/test_one.py", "-v"])
```

---

### **Important Notes:**
- If you want to run specific tests inside a file, you can specify the test function names using the syntax `file_name::test_name`.
  
  #### Example:
  ```bash
  pytest test_file_name.py::test_specific_function
  ```
  
- If your test file doesn’t follow the naming convention (`test_*.py` or `*_test.py`), pytest may not discover it automatically. You can still run it by explicitly specifying the file.
