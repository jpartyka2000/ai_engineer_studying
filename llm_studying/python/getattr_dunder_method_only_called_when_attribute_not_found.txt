## 📋 Question Review

**Level:** Medium | **Type:** Multiple Choice | **Topic:** Attribute Access

**Question:**
What is the output of the `DefaultDict` attribute access code?

```python
class DefaultDict:
    def __init__(self, default_value):
        self.default_value = default_value
        self.data = {}
    
    def __getattr__(self, name):
        return self.default_value

d = DefaultDict("N/A")
d.data["name"] = "Alice"
d.x = 100

print(d.data["name"])
print(d.x)
print(d.missing)
print(d.anything)
```

**Your Answer:** C
```
Alice
100
AttributeError
AttributeError
```

**Correct Answer:** A
```
Alice
100
N/A
N/A
```

**Result:** ❌ Incorrect

---

**Explanation:**

The key is understanding **when `__getattr__` is called**.

**`__getattr__` is ONLY called when the attribute is NOT found normally.**

| Attribute | Found normally? | Result |
|-----------|-----------------|--------|
| `d.data` | ✅ Yes (set in `__init__`) | Returns the dict `{}` |
| `d.x` | ✅ Yes (set with `d.x = 100`) | Returns `100` |
| `d.missing` | ❌ No | Calls `__getattr__` → Returns `"N/A"` |
| `d.anything` | ❌ No | Calls `__getattr__` → Returns `"N/A"` |

**Step by step:**

1. `d.data["name"]` → `data` exists, returns dict, gets `"Alice"`
2. `d.x` → `x` exists (we set it), returns `100`
3. `d.missing` → Not found → `__getattr__("missing")` → `"N/A"`
4. `d.anything` → Not found → `__getattr__("anything")` → `"N/A"`

**Why C is wrong:**

`__getattr__` doesn't raise `AttributeError` — it **returns** `self.default_value` for any missing attribute. That's the whole point of this class!

---

**`__getattr__` vs `__getattribute__`:**

| Method | When called |
|--------|-------------|
| `__getattr__` | Only when attribute is **not found** |
| `__getattribute__` | For **every** attribute access |

```python
# __getattr__ = fallback for missing attributes
# __getattribute__ = intercepts ALL attribute access
```
