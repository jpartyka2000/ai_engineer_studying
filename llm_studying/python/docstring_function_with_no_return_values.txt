When writing a docstring for a function that has no return values (i.e., it returns `None`), you should still include a `Returns` section in the docstring to explicitly indicate this. It helps to make the function's behavior clear for anyone reading the documentation.

### Example of a Function with No Return Values:
```python
def log_message(message):
    """
    Logs a message to the console.

    This function takes a message and logs it without returning any value.
    
    Returns:
        None
    """
    print(message)
```

### Key Points:
- **No Arguments**: If the function has no arguments, simply omit the `Args` section.
- **No Return Values**: In the `Returns` section, explicitly state `None`, as this is Python's default return value when no explicit return is provided.
  
This clarity helps maintain consistency in documenting functions, even if they don’t produce return values.
