The built‑in `list.remove(value)` only ever removes a **single** element (the first matching one) per call. You can’t pass it a list of values to remove in one go.  

If you want to drop multiple items from a Python list, you have two common options:

---

### 1) Call `.remove()` in a loop

```python
cols = ['a', 'b', 'c', 'd', 'e']
to_drop = ['b', 'd']

for x in to_drop:
    if x in cols:
        cols.remove(x)

print(cols)   # ['a', 'c', 'e']
```

This will iterate over your `to_drop` list, removing each one in turn.

---

### 2) Use a list comprehension (often more Pythonic)

```python
cols = ['a', 'b', 'c', 'd', 'e']
to_drop = {'b', 'd'}   # a set is slightly faster for membership tests

cols = [x for x in cols if x not in to_drop]

print(cols)   # ['a', 'c', 'e']
```

This builds a brand new list containing only the items you want to keep.

---

Either approach works—if you care about preserving order *and* in‑place mutation use the loop; if you don’t need to preserve the original list object and prefer compact code, go with the comprehension.
