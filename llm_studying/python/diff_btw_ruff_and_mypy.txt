**Ruff** and **mypy** solve different problems:

**Ruff** is a linter and formatter. It checks for style issues, unused imports, undefined names, code complexity, and formatting inconsistencies. It answers questions like "Is this code clean and well-structured?" Ruff is extremely fast (written in Rust) and replaces tools like Flake8, isort, and Black.

**mypy** is a static type checker. It analyzes your type hints to catch type-related bugs before runtime. It answers questions like "Am I passing a string where a function expects an integer?" It doesn't care about style — only whether your types are consistent and correct.

**Example:**

```python
def greet(name: str) -> str:
    return "Hello, " + name

greet(42)  # mypy catches this: expected str, got int
```

```python
import os  # Ruff catches this: unused import

def greet(name):
    x = 1  # Ruff catches this: unused variable
    return "Hello, " + name
```

**In practice**, many projects use both together — Ruff for style and general code quality, mypy for type safety. They're complementary, not competing tools.
