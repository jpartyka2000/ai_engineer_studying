Python’s built-in `dict` **repr** will always use single quotes—there isn’t a switch to make `print(my_dict)` emit double-quoted keys/values. Instead, to get a JSON-compatible string (which by spec uses double quotes), serialize it with the `json` module:

```python
import json

my_dict = {"foo": 1, "bar": [2, 3]}
json_str = json.dumps(my_dict)          # '{"foo": 1, "bar": [2, 3]}'
print(json_str)
```

You can even pretty-print:

```python
json_pretty = json.dumps(my_dict, indent=2, sort_keys=True)
print(json_pretty)
```

Under the hood `json.dumps` always uses double quotes for object keys and string values, so you get valid JSON rather than Python’s single-quoted repr.
