Yes — absolutely 👍

In Python, `isinstance()` can check against **multiple types at once** by passing a **tuple of types**.

---

## 📌 Example

```python
value = 42

print(isinstance(value, int))                     # ✅ True
print(isinstance(value, (int, float)))            # ✅ True (42 is an int)
print(isinstance(3.14, (int, float)))             # ✅ True (3.14 is a float)
print(isinstance("hello", (int, float, str)))     # ✅ True (string is included)
```

* If `value` matches **any one** of the types in the tuple, `isinstance()` returns `True`.
* If it matches none, you get `False`.

---

## ⚖️ Notes

* The second argument must be a **type or a tuple of types** — you can’t pass a list.

  ```python
  isinstance(42, [int, float])  # ❌ TypeError
  isinstance(42, (int, float))  # ✅ works
  ```
* You can nest tuples, but usually not needed.
* Works with custom classes too:

  ```python
  class A: pass
  class B: pass

  obj = A()
  isinstance(obj, (A, B))  # ✅ True
  ```

---

✅ **Answer:** Yes — just pass a tuple of types to `isinstance(value, (Type1, Type2, ...))`. It returns `True` if the value is an instance of *any* of those types.
