### **Understanding `__name__` in Python**

In Python, every **function and method** has a special built-in attribute called `__name__`, which stores the **name of the function or method as a string**.

---

## **1️⃣ How `__name__` Works**
When you define a function or method, Python assigns it a `__name__` attribute:

```python
def my_function():
    pass

print(my_function.__name__)  # Output: "my_function"
```
✅ **`my_function.__name__` returns `"my_function"`, the function's name as a string.**

---

## **2️⃣ Example with a Class Method**
If a function is defined inside a class (i.e., a method), it also has a `__name__`:

```python
class MyClass:
    def my_method(self):
        pass

print(MyClass.my_method.__name__)  # Output: "my_method"
```
✅ `__name__` still works the same way for methods.

---

## **3️⃣ Why Use `__name__`?**
### 🔹 **1. Helpful for Debugging & Logging**
You can use `__name__` to track which function is executing:

```python
def process_data():
    print(f"Executing {process_data.__name__} function")

process_data()
# Output: Executing process_data function
```

### 🔹 **2. Dynamic Function Registration (e.g., in Dictionaries)**
You can use `__name__` as a **key** in a dictionary for function lookups:

```python
def add(a, b):
    return a + b

def subtract(a, b):
    return a - b

operations = {
    add.__name__: add,
    subtract.__name__: subtract
}

print(operations["add"](5, 3))  # Output: 8
```

### 🔹 **3. Meta-Programming & Reflection**
Sometimes, `__name__` is used in **introspection**, where Python code examines itself:

```python
def greet():
    pass

print(f"This function is named: {greet.__name__}")  
# Output: This function is named: greet
```

---

## **4️⃣ Special Case: `__name__` in Anonymous (`lambda`) Functions**
Lambdas **do not have meaningful names**—their `__name__` attribute is always `"lambda"`:

```python
my_lambda = lambda x: x * 2
print(my_lambda.__name__)  # Output: "lambda"
```
⚠️ Unlike normal functions, `lambda` functions always return `"lambda"` as their name.

---

## **5️⃣ `__name__` in the `__main__` Scope**
In **modules**, `__name__` behaves differently. If a Python file is run directly, `__name__` will be `"__main__"`:

```python
print(__name__)  # Output: "__main__" (if run directly)
```

But if the same script is **imported** as a module, `__name__` becomes the module's name.

---

## **🔹 Summary**
- **`__name__`** is an attribute of **functions and methods** that stores their name as a string.
- Used for **debugging, function registration, and introspection**.
- **Lambdas always have `__name__ = "lambda"`**.
- **Modules also have `__name__`**, which can be `"__main__"` or the module name.
