PEP 8, the official style guide for Python, provides guidance on how to format docstrings. For function-level docstrings, the recommendations are described in **PEP 257**, a sister document to PEP 8.

### Key Points for Writing a Function Docstring

1. **Placement**: The docstring should appear **immediately after the function definition** and before any code in the function body.
   ```python
   def example_function(param1, param2):
       """
       A summary of what the function does.

       Detailed description (if needed) that explains the function's behavior, 
       parameters, return values, or exceptions raised.
       
       Args:
           param1 (type): Description of the first parameter.
           param2 (type): Description of the second parameter.
       
       Returns:
           return_type: Description of the return value.
       """
       # Function code
   ```

2. **Triple Quotes**: Use triple double-quotes (`"""`) for docstrings, even for single-line descriptions.

3. **Summary Line**:
   - The first line should be a **short summary** of the function's purpose.
   - It should start with a capital letter and end with a period.

4. **Blank Line After the Summary**:
   - If there are additional details, include a blank line between the summary and the detailed description.

5. **Detail Section (Optional)**:
   - After the summary, you can include more detailed information, like parameter descriptions, return values, or examples.

6. **Use Consistent Formatting**:
   - Parameter and return value descriptions should be consistent. Formats like **Google style**, **NumPy style**, or **reStructuredText (Sphinx)** are commonly used in Python projects.

---

### Example Following PEP 8/257:
#### Single-Line Docstring:
```python
def add_numbers(a, b):
    """Return the sum of two numbers."""
    return a + b
```

#### Multi-Line Docstring:
```python
def calculate_area(width, height):
    """
    Calculate the area of a rectangle.

    Args:
        width (float): The width of the rectangle.
        height (float): The height of the rectangle.

    Returns:
        float: The area of the rectangle.

    Raises:
        ValueError: If width or height is negative.
    """
    if width < 0 or height < 0:
        raise ValueError("Width and height must be non-negative.")
    return width * height
```
