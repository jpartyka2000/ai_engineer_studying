In Python, you don’t need to write every import on a separate line if you’re bringing in multiple names from the **same module**. Instead, you can put them all in a single `from … import …` statement, separated by commas.

For example:

```python
# Instead of writing:
from math import sqrt
from math import sin
from math import cos

# You can condense it into one line:
from math import sqrt, sin, cos
```

This works for any module. It can make your code shorter and easier to scan when the imported names are related.

---

### A Few Things to Keep in Mind

* **Readability first**: If you’re importing many names, a long comma-separated line might become hard to read. In that case, it’s often clearer to use multiple lines or wrap with parentheses:

  ```python
  from math import (
      sqrt, sin, cos, tan, log, exp
  )
  ```

* **Namespace collisions**: If you import a lot of names directly, you might accidentally shadow another function in your code. Using `import math` and then calling `math.sqrt()` avoids this problem but requires the extra prefix.

* **Consistency**: Pick a style that fits your project. PEP 8 (Python’s style guide) generally recommends keeping lines under 79 characters, so sometimes spreading imports out is better.

---

👉 In short: yes, you can specify all imports from a module on one line with commas, but balance it against readability and style conventions.
