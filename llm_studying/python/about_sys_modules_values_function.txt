`sys.modules` is a dictionary that Python maintains of all modules that have been imported during the program's execution.

- **Keys**: module names (strings like `"os"`, `"cue_eval.runner"`, `"json"`)
- **Values**: the actual module objects

```python
import sys
import json

print(sys.modules['json'])  # <module 'json' from '/usr/lib/python3.11/json/__init__.py'>
print(sys.modules['json'].__file__)  # /usr/lib/python3.11/json/__init__.py
```

So `sys.modules.values()` gives you all the module objects that have been imported. Each module object has a `__file__` attribute pointing to its source file (except for built-in modules like `sys` itself, which have `__file__` as `None`).

Quick example:

```python
import sys
import json
import os

for name, module in sys.modules.items():
    if hasattr(module, '__file__') and module.__file__:
        print(f"{name}: {module.__file__}")
```

Output would be something like:

```
json: /usr/lib/python3.11/json/__init__.py
os: /usr/lib/python3.11/os.py
...
```

By checking this at program exit via `atexit`, you capture every module that was imported at any point during execution.
