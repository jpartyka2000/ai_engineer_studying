When you’re importing **entire modules** (not specific names from a module), you can also put them on a single line, separated by commas. For example:

```python
import re, sys, time
```

This works perfectly fine — Python allows it.

---

### But here’s the catch

PEP 8 (the official style guide) *recommends against this*. Instead, it suggests one import per line:

```python
import re
import sys
import time
```

Why?

* **Readability**: It’s easier to scan in a diff, see which imports changed, and avoid missing an import in a long list.
* **Consistency**: If you mix in things like aliases (`import numpy as np`) or conditional imports, the single-line style gets messy quickly.

---

✅ **When it’s acceptable**: For quick scripts, throwaway code, or notebooks, `import re, sys, time` is convenient and totally fine.
❌ **When to avoid**: In production code or team projects, stick to one import per line for clarity.
