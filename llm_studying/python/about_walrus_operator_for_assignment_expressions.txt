The “walrus” operator (`:=`)—formally called an **assignment expression**—lets you both **assign** to a variable and **produce** that value in the same expression. It was introduced in Python 3.8 via [PEP 572](https://www.python.org/dev/peps/pep-0572/).

---

## Core Syntax

```python
target := expr
```

* **`expr`** is evaluated first.
* Its value is bound to **`target`**.
* Then the entire `target := expr` yields that value, so you can use it inline.

---

## Common Use–Cases

### 1. In `if` statements

Avoid repeating a costly call:

```python
# Without walrus
data = fetch_data()
if data:
    process(data)

# With walrus
if (data := fetch_data()):
    process(data)
```

### 2. In `while` loops

Read chunks until exhaustion:

```python
# Without walrus
chunk = file.read(1024)
while chunk:
    handle(chunk)
    chunk = file.read(1024)

# With walrus
while (chunk := file.read(1024)):
    handle(chunk)
```

### 3. In comprehensions (Python 3.8+)

Capture sub‐expressions:

```python
# Filter and transform at once
results = [y for x in items if (y := expensive_transform(x)) is not None]
```

---

## Rules & Gotchas

1. **No bare names on the left** in function signatures or lambda parameters.
2. **Not allowed** at the top‐level of a module (only inside expressions).
3. **Readability**: overusing it can make code hard to follow—use sparingly where it clearly removes duplication.
4. **Precedence**: it has lower precedence than most operators, so you often need parentheses around the whole `target := expr`.

---

## Benefits

* **DRY**: don’t repeat calls or computations.
* **Atomic**: bind and test in one place, reducing temporary lines.
* **Expressive**: keeps related logic tightly grouped.

---

### TL;DR

Use `:=` when you want to compute something once, store it, and immediately use it in a larger expression—especially in conditions or loops—without separate assignment statements.
