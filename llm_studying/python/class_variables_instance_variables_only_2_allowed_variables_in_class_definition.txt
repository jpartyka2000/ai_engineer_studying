Right. Python doesn't have "static variables" as a distinct concept like Java or C++. What you have here is a **class variable** (also called a class attribute), which is shared across the class and all its subclasses (unless a subclass shadows it with its own value).

In Python, variables defined at the class level fall into two categories:

1. **Class variables** — defined directly in the class body, outside any method:
   ```python
   class Foo:
       eda_config = None  # class variable
   ```

2. **Instance variables** — defined on `self`, typically in `__init__`:
   ```python
   class Foo:
       def __init__(self):
           self.eda_config = None  # instance variable
   ```

Since your classes can't be instantiated (due to the `__new__` override), instance variables aren't an option anyway. So class variables are the only way to store state, and that's exactly what you're doing with `eda_config`.

When you call `cls.eda_config = eda_config` inside `configure`, you're setting (or shadowing) that class variable on whichever class `cls` refers to at call time. So if you call `SummaryAnalytics.configure(my_config)`, it sets `eda_config` on `SummaryAnalytics` specifically, not on `EDABase`.
