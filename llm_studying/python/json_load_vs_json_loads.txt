## `json.loads()` vs `json.load()`

### **The Key Difference:**
- **`json.loads()`** - Load from a **string** (the 's' stands for string)
- **`json.load()`** - Load from a **file object**

### **Examples:**

### `json.loads()` - From String
```python
import json

# From a string
json_string = '{"name": "Alice", "age": 30}'
data = json.loads(json_string)  # Pass the string directly
print(data)  # {'name': 'Alice', 'age': 30}

# Common use case: API responses
response = requests.get(url)
data = json.loads(response.text)  # response.text is a string
```

### `json.load()` - From File
```python
import json

# From a file
with open('data.json', 'r') as f:
    data = json.load(f)  # Pass the file object
    print(data)

# NOT the filename - the file object!
# json.load('data.json')  # ❌ Wrong!
# json.load(f)            # ✅ Correct!
```

### **Memory Trick:**
- **`loads()`** = load string (s for string)
- **`load()`** = load file (no s)

### **Same Pattern for Writing:**
```python
# Write to string
json_string = json.dumps(data)  # Returns a string

# Write to file
with open('data.json', 'w') as f:
    json.dump(data, f)  # Writes to file object
```

**Common mistake**: Using the wrong one causes:
- `json.loads(file_object)` → TypeError
- `json.load(string)` → AttributeError
