### How the `@property` Decorator Works in Python

In Python, the **`@property` decorator** is more than just syntactic sugar — it actually transforms a function into a **property object**.

---

#### 1. Normal Method

```python
class Rectangle:
    def area(self):
        return self.width * self.height
```

Here, `area` is just a regular method. You’d call it with parentheses:

```python
r = Rectangle()
r.width, r.height = 3, 4
print(r.area())   # 12
```

---

#### 2. Adding `@property`

```python
class Rectangle:
    @property
    def area(self):
        return self.width * self.height
```

Now, `area` is no longer a plain method — Python calls `property()` on it during class creation. The result is a **property object**, which is stored on the class under the name `area`.

That property object knows how to act like an **attribute**:

```python
r = Rectangle()
r.width, r.height = 3, 4
print(r.area)     # 12  (no parentheses!)
```

---

#### 3. What’s Inside a Property Object?

A property object can hold up to three functions:

* **getter** (`fget`) → the function decorated with `@property`
* **setter** (`fset`) → added with `@<prop>.setter`
* **deleter** (`fdel`) → added with `@<prop>.deleter`

For example:

```python
class Person:
    def __init__(self, age):
        self._age = age

    @property
    def age(self):
        return self._age

    @age.setter
    def age(self, value):
        if value < 0:
            raise ValueError("Age cannot be negative")
        self._age = value
```

After the class is defined:

```python
print(Person.age)        # <property object at ...>
print(Person.age.fget)   # <function Person.age at ...>
print(Person.age.fset)   # <function Person.age at ...>
```

---

#### ✅ Key Takeaway

When you decorate a method with `@property`:

* Python replaces the method with a **property object**.
* Accessing the attribute calls the getter function transparently.
* You can later attach setters or deleters to the same property.

That’s why `@property` lets you write methods that **look and feel like attributes**, while still allowing encapsulation and validation.
