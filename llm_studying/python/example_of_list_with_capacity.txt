Here's a Python class that extends a list and stores GPT messages as dictionaries. The class enforces a limit on the number of stored messages and removes the oldest question/answer pair when the limit is exceeded:

```python
class GPTMessageHistory:
    def __init__(self, max_size: int):
        self.max_size = max_size
        self.messages = [{"role": "system", "content": "You are a helpful assistant."}]  # System prompt always stays

    def add_message(self, role: str, content: str):
        new_message = {"role": role, "content": content}
        self.messages.append(new_message)
        
        # Remove the oldest message if the list exceeds the limit (excluding the system prompt)
        if len(self.messages) > self.max_size:
            self.messages.pop(1)  # Remove the oldest user/assistant pair
    
    def get_history(self):
        return self.messages


# Example usage:
history = GPTMessageHistory(max_size=5)

# Simulating some interactions
history.add_message("user", "Tell me about Python.")
history.add_message("assistant", "Python is a versatile programming language.")
history.add_message("user", "What are its uses in data science?")
history.add_message("assistant", "Python is widely used for data analysis, machine learning, and more.")
history.add_message("user", "How do you use Pandas?")
history.add_message("assistant", "Pandas is used for data manipulation and analysis.")

# Print message history
for message in history.get_history():
    print(f"{message['role'].capitalize()}: {message['content']}")
```

### Key Points:
1. **System Prompt**: The system message stays at the beginning of the list and does not get removed.
2. **Maximum Capacity**: Once the list exceeds the defined capacity (`max_size`), the oldest non-system message (question/answer pair) is removed.
3. **Message Addition**: Messages are added with a role (either `user` or `assistant`) and the content (the question or answer).

This structure ensures that your conversation history is stored efficiently and always maintains the context, even as the number of messages grows.
