### The Three Methods of a Python Property Object

When you use the **`@property` decorator** in Python, what you actually create is a **property object**. A property object can hold up to **three different methods** that control how the attribute behaves:

---

#### 1. Getter (`fget`)

* Defined with `@property`.
* Called whenever the attribute is **read**.

```python
class Person:
    def __init__(self, name):
        self._name = name

    @property
    def name(self):   # fget
        return self._name

p = Person("Alice")
print(p.name)   # "Alice"
```

---

#### 2. Setter (`fset`)

* Defined with `@<property>.setter`.
* Called whenever the attribute is **assigned** a value.

```python
class Person:
    def __init__(self, name):
        self._name = name

    @property
    def name(self):   # fget
        return self._name

    @name.setter
    def name(self, value):   # fset
        if not value:
            raise ValueError("Name cannot be empty")
        self._name = value

p = Person("Alice")
p.name = "Bob"     # calls fset
print(p.name)      # "Bob"
```

---

#### 3. Deleter (`fdel`)

* Defined with `@<property>.deleter`.
* Called when the attribute is **deleted** with `del`.

```python
class Person:
    def __init__(self, name):
        self._name = name

    @property
    def name(self):   # fget
        return self._name

    @name.deleter
    def name(self):   # fdel
        print("Deleting name...")
        del self._name

p = Person("Alice")
del p.name   # calls fdel
```

---

### Summary

Every property object can have these three optional methods:

* **Getter (`fget`)** → runs on attribute read.
* **Setter (`fset`)** → runs on attribute assignment.
* **Deleter (`fdel`)** → runs on attribute deletion.

If you only use `@property`, you get just the getter. Adding `.setter` and `.deleter` turns it into a fully managed attribute.
