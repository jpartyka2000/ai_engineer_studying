| Your Approach | Generator Pipeline |

def pipeline(numbers):
    evens = (n for n in numbers if n % 2 == 0)
    squared = (n ** 2 for n in evens)
    big_ones = (n for n in squared if n > 10)
    yield from big_ones

For future reference, a "true" generator pipeline uses 
generator expressions () instead of list comprehensions []
to chain operations lazily. This is more memory-efficient
because:
- No intermediate lists are created
- Items are processed one at a time through the chain
- Essential for large or infinite data streams

KEY INSIGHT:
- List comprehension: [x for x in ...] → builds full list
- Generator expression: (x for x in ...) → lazy evaluation
