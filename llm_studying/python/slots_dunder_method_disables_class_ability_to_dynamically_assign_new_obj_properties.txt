### 🐍 Arbitrary Attributes in Normal Python Classes

One of Python’s most flexible features is that you can add attributes to objects dynamically — even if those attributes were never defined in the class.

---

#### 1. How It Works

By default, Python class instances store attributes in a special dictionary called `__dict__`.

* When you assign `obj.foo = 123`, Python adds a key-value pair `{"foo": 123}` to that dictionary.
* This means you can attach **new attributes on the fly**.

Example:

```python
class Animal:
    pass

a = Animal()
a.name = "Fido"       # dynamically add new attribute
a.age = 3             # add another one

print(a.name)  # Fido
print(a.age)   # 3
```

Even though `Animal` never declared `name` or `age`, the instance accepts them.

---

#### 2. The Role of `__slots__`

If a class defines `__slots__`, Python disables the per-instance `__dict__` by default.
This restricts which attributes can exist on an instance.

Example:

```python
class Animal:
    __slots__ = ("name",)

a = Animal()
a.name = "Fido"    # ✅ allowed
a.age = 3          # ❌ AttributeError: 'Animal' object has no attribute 'age'
```

So `__slots__` is how you lock down instances to only a known set of attributes.

---

#### 3. Why This Matters

* **Flexibility:** Great for quick prototyping and dynamic behavior.
* **Safety & Memory Savings:** Using `__slots__` enforces a fixed attribute set and reduces memory usage, especially with many instances.

---

### ✅ Takeaway

* **Normal Python classes** (without `__slots__`) let you assign arbitrary new attributes at runtime.
* **With `__slots__`**, instances are restricted, and attempts to add undeclared attributes will raise an error.

This dynamic nature is one reason Python is so powerful — but in large systems, using `__slots__` or dataclasses/Pydantic models can help enforce structure.
