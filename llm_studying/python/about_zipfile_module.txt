The **`zipfile`** library in Python is a **built-in standard library module** that lets you **create, read, write, append, and extract ZIP archives**. In other words, it gives you programmatic control over `.zip` files without needing any third-party package.

---

## Key Features

* **Reading ZIPs**: You can open existing `.zip` files, list their contents, and extract files.
* **Writing ZIPs**: You can compress files or raw data into a new `.zip` archive.
* **Modes**: Works with different modes:

  * `"r"` → read (default)
  * `"w"` → write (overwrites existing)
  * `"a"` → append (add to an existing archive)
* **Compression options**: By default uses `ZIP_STORED` (no compression), but you can use `ZIP_DEFLATED` (with zlib) or other methods like `ZIP_BZIP2` or `ZIP_LZMA` if available.

---

## Example: Reading a ZIP file

```python
import zipfile

# Open a zip file in read mode
with zipfile.ZipFile("archive.zip", "r") as zf:
    # List all files inside
    print(zf.namelist())
    
    # Extract a specific file
    zf.extract("data.csv", path="output_folder")
```

---

## Example: Creating a ZIP file

```python
import zipfile

# Create a new zip file and add files into it
with zipfile.ZipFile("my_archive.zip", "w", compression=zipfile.ZIP_DEFLATED) as zf:
    zf.write("report.txt")
    zf.write("image.png")
```

---

## Why it’s useful

* Packaging logs, reports, or datasets for sharing.
* Extracting downloaded ZIP archives.
* Automating backups in compressed form.
* Works across platforms (Windows, macOS, Linux).

---

⚡ So in short: `zipfile` is Python’s **go-to library for handling `.zip` archives**—built in, versatile, and no extra install needed.
