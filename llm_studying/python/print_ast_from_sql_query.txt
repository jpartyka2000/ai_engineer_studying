`sqlglot` provides a few ways to see a **human-readable** representation of a parsed AST. The simplest are:

1. **Just print the `Expression` object**:  
   ```python
   ast = sqlglot.parse_one("SELECT x FROM y")
   print(ast)
   ```
   This shows a one-line representation (e.g., `Select(this=..., from=...)`) describing the top-level node and its children.

2. **Use `ast.tree_string()`** for a multi-line, indented parse tree**:  
   ```python
   ast = sqlglot.parse_one("SELECT x FROM y")
   print(ast.tree_string())
   ```
   This outputs a more detailed, hierarchical representation, for example:
   ```
   Select
     expressions:
       Column
         this: x
     from:
       Identifier
         this: y
   ```

3. **Other options**:
   - `ast.dump()` shows internal node info (similar to `tree_string()` but may be slightly more verbose).  
   - `ast.sql()` returns the SQL string for that AST.  
   - `ast.to_json()` returns a JSON-like structure representing the AST.

Which method you pick depends on whether you want a **human-friendly** view (`tree_string()`, `print(ast)`) or a **machine-readable** structure (`to_json()` or `dump()`).
