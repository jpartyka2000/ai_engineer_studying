## The 3 Most Important Differences:

### 1. **Human Readability**
```yaml
# YAML - Clean and minimal
name: Alice
age: 30
hobbies:
  - reading
  - coding
address:
  city: Seattle
  zip: 98101
```

```json
// JSON - More symbols and quotes
{
  "name": "Alice",
  "age": 30,
  "hobbies": ["reading", "coding"],
  "address": {
    "city": "Seattle",
    "zip": 98101
  }
}
```

### 2. **Comments Support**
```yaml
# YAML supports comments
name: Alice  # User's name
age: 30      # Age in years
# TODO: Add validation here
```

```json
// JSON has NO comment support
{
  "name": "Alice",  // ❌ This breaks JSON!
  "age": 30
}
```

### 3. **Strictness vs Flexibility**
```yaml
# YAML - Multiple ways to write the same thing
# All of these are valid:
is_active: true
is_active: True  
is_active: yes
is_active: on

# Strings often don't need quotes
message: Hello world
```

```json
// JSON - One strict way
{
  "is_active": true,  // Only lowercase true/false
  "message": "Hello world"  // Strings MUST have quotes
}
```

**Bottom line**: YAML prioritizes **human-friendliness** (easier to read/write), while JSON prioritizes **machine-friendliness** (strict, unambiguous, universal support).
