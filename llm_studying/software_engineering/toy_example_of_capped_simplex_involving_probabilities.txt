Here’s what happened in the toy example:

base_probability	actual	projected_probability
0.9	1				0.85
0.2	0				0.15
-0.1	0				0.00
1.2	1				1.00

* **Base probabilities** started as `[0.9, 0.2, -0.1, 1.2]`.
* **Actual sum** (the target total) was `1 + 0 + 0 + 1 = 2`.
* After the “water‑filling and clipping”:

  * The first value dropped from 0.9 → **0.85**,
  * The second from 0.2 → **0.15**,
  * The third (negative) got floored to **0.00**,
  * The fourth (above 1) got capped at **1.00**.

All together, the projected probabilities sum to exactly **2.00**, matching the actuals total, while each is within \[0,1]. This ensures minimal overall disturbance from the original guesses.
