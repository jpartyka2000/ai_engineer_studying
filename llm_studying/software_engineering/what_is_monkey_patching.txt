**Monkey‑patching** is the practice of **dynamically** adding, modifying, or replacing attributes or methods on existing classes or instances at runtime—without changing the original source code. In Python, because everything is an object and you can assign to attributes on the fly, you can “patch” in new behavior on the fly.

---

### Why & when you might use it

* **Quick fixes** to third‑party libraries without forking or editing their code.
* **Testing**: swap in fake methods or stub implementations for isolation.
* **Ad hoc extensions**: add convenience methods to an instance just where you need them.

---

### A simple example

```python
class Greeter:
    def greet(self):
        return "Hello"

g = Greeter()
print(g.greet())  # → "Hello"

# Monkey‐patch a new method
def goodbye(self):
    return "Goodbye"

import types
g.goodbye = types.MethodType(goodbye, g)

print(g.goodbye())  # → "Goodbye"
```

Here, without touching `Greeter`’s definition, we’ve injected a new method onto the **instance** `g`.

---

### Caveats

* **Maintainability**: future readers may not expect behavior to be altered at runtime.
* **Fragility**: upstream library changes can break your patch.
* **Debuggability**: stack traces and IDE‐hints can be confusing when methods aren’t on the original class.

Use it sparingly—when you really need to extend or fix behavior on‑the‑fly and you can’t otherwise modify the original code.
