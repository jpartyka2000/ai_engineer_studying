Cache busting is a general technique where you change something trivial to trick a caching system into thinking the content is new, forcing it to rebuild or re-fetch instead of using the cached version.

The concept shows up everywhere in software. In web development, you might append a version number to a CSS file URL (`styles.css?v=2`) so browsers download the new version instead of serving the old cached one. In Docker, people add a dummy `ARG` or `ENV` line to force layers below it to rebuild. Your Modal `.env({"CACHE_BUST": "1"})` is the same idea — Modal sees the image definition changed, so it rebuilds everything from that point forward.

The underlying principle is always the same: caching systems decide whether to use the cache by checking if the inputs have changed. If you change an input — even a meaningless one — the cache treats it as a miss and does a fresh build. It's a simple workaround for when there's no official "ignore the cache" button.
