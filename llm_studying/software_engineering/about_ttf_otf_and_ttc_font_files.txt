**TTF (TrueType Font)** — Developed by Apple and Microsoft in the late 1980s. Contains a single font style (e.g., just "Roboto Regular" or just "Roboto Bold") in one file. Uses quadratic Bézier curves to define glyph shapes. Works essentially everywhere — every OS, browser, and application supports it. Still the most common format you'll encounter.

**OTF (OpenType Font)** — Developed by Microsoft and Adobe in the mid-1990s as an evolution of TrueType. Also contains a single font style per file, but supports more advanced features: ligatures, stylistic alternates, small caps, swashes, and more complex glyph substitution rules. Uses cubic Bézier curves (from PostScript), which can describe curves more efficiently with fewer control points — meaning slightly smaller files and smoother curves at extreme sizes. OTF is essentially "TTF plus extras." In practice, most users won't notice a difference unless they're doing serious typography work in apps that support OpenType features (InDesign, Illustrator, modern CSS).

**TTC (TrueType Collection)** — A container that bundles *multiple* font styles into a single file. For example, one `.ttc` file might contain Regular, Bold, Italic, and Bold Italic all together. The fonts inside a TTC can share common glyphs between styles, which saves disk space. macOS uses TTC files extensively for its system fonts. The tradeoff is that some older software and some web contexts don't handle TTC files well — if you need maximum compatibility, individual TTF/OTF files are safer.

In short: TTF is the universal workhorse, OTF adds typographic power features, and TTC is just a way to pack multiple TTF/OTF fonts into one file.
