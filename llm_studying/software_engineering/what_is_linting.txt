**Linting** is the process of automatically analyzing your code to catch errors, bugs, and style issues before you run it.

A **linter** scans your source code and flags things like:
- Syntax errors
- Undefined variables
- Unused imports
- Style inconsistencies (indentation, naming conventions)
- Potential bugs (e.g., unreachable code, type mismatches)

For Python, common linters include **Ruff** (fast, modern), **Flake8** (style + errors), **Pylint** (very thorough), and **mypy** (type checking specifically).

You typically run a linter from the command line or integrate it into your editor so issues appear as you type. Many projects also run linters in CI pipelines to enforce code quality before merging.

The name comes from the original "lint" tool for C, which picked out small "fluff" problems — like a lint roller picks lint off clothing.
