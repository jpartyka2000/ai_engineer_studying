In Python (and many other languages), a **functor** is simply an object that you can “call” like a function.  Under the hood, that means the class defines a `__call__` method, so when you do:

```python
result = my_obj(arg1, arg2)
```

Python invokes:

```python
result = my_obj.__call__(arg1, arg2)
```

---

## Why use functors instead of plain functions?

1. **Stateful behavior**

   * A functor can carry internal state across calls.
   * E.g. a counter or a parameter that influences every invocation.

   ```python
   class Counter:
       def __init__(self):
           self.count = 0
       def __call__(self, increment=1):
           self.count += increment
           return self.count

   c = Counter()
   print(c())    # 1
   print(c(3))   # 4
   ```

2. **Encapsulation**

   * You can bundle data (attributes) and behavior (`__call__`) cleanly in one object.
   * Avoids global variables or closures if you need more complex internal logic.

3. **Polymorphism**

   * Any functor can be passed where a function is expected (e.g. as a callback).
   * Yet you can swap in different functor classes with shared interface but different internals.

---

## Functors vs. Functions vs. Methods

* **Function**: defined with `def`, stateless unless you close over variables.
* **Method**: a function bound to an object instance (has access to `self`).
* **Functor**: an *instance* of a class that behaves like a function, via `__call__`.

---

## When you see functors in Python

* **Decorators** are often implemented as functor *classes* so they can accept parameters.
* **Sklearn estimators** (e.g. `RandomForestClassifier`) implement `fit`/`predict`, but many wrappers use `__call__` functors for pipelines.
* **Command patterns** or **callbacks** in GUI frameworks: bundle up event handling state in a functor.

---

### In one sentence

> A **functor** is just any object with a `__call__` method, letting you treat it like a function—so you get all the benefits of an object (state, encapsulation, subclassing) with the simplicity of function‐call syntax.
