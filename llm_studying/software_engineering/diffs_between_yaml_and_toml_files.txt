## 3 Major Differences Between YAML and TOML:

### 1. **Indentation vs Explicit Structure**
```yaml
# YAML - Indentation defines structure
database:
  host: localhost
  port: 5432
  credentials:
    user: admin
    password: secret
```

```toml
# TOML - Explicit sections with brackets
[database]
host = "localhost"
port = 5432

[database.credentials]
user = "admin"
password = "secret"
```

### 2. **Type Clarity**
```yaml
# YAML - Ambiguous types
port: 08080      # Is this octal? String? Number?
enabled: yes     # Boolean? String?
time: 12:30      # Time? Number ratio?
```

```toml
# TOML - Unambiguous types
port = 8080              # Clearly an integer
enabled = true           # Clearly a boolean  
time = 12:30:00         # Clearly a time
```

### 3. **Arrays/Lists Syntax**
```yaml
# YAML - Multiple array styles
colors: [red, green, blue]  # Inline
# OR
colors:
  - red
  - green
  - blue
```

```toml
# TOML - Clear array syntax
colors = ["red", "green", "blue"]  # Always quoted strings

# Or for complex arrays:
[[products]]
name = "Table"
price = 100

[[products]]
name = "Chair"
price = 50
```

**Bottom line**: YAML prioritizes **human readability** but can be ambiguous. TOML prioritizes **clarity and explicitness** - you always know exactly what type something is.
