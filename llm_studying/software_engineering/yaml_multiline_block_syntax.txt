The `|-` symbols in YAML are **block scalar indicators** that define how multi-line strings should be handled. Specifically:  

- `|` (pipe) means **literal block scalar**, which preserves line breaks.  
- `|-` (pipe with a dash) is the **same as `|` but removes the trailing newline at the end** of the block.  

### **How It Works in Your YAML:**
```yaml
prompts:
  - |-
    Hi there LLM,
    Please translate the following text to French:
    "{{name}}: {{text}}"
  - |-
    Translate the following text to German:
    "{{name}}: {{text}}"
```
1. Each `|-` starts a multi-line string.  
2. The `-` removes any trailing newline at the end of the block.  
3. The indentation of the block is determined by the indentation level of the first non-empty line.  

### **Example Behavior:**
If you parsed this YAML, the first element in the `prompts` list would be stored as:  
```text
Hi there LLM,
Please translate the following text to French:
"{{name}}: {{text}}"
```
There **won’t** be an extra blank line at the end due to `|-`.  

---

### **Alternative: `|` vs. `|-` vs. `>`, `>-`**
- **`|`** → Preserves all line breaks, **including the final newline**.  
- **`|-`** → Preserves all line breaks but **removes the final newline**.  
- **`>`** → **Folds** newlines into spaces (except for double line breaks).  
- **`>-`** → Same as `>`, but removes the final newline.
