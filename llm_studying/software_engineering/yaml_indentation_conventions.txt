## YAML Indentation Standard:

### **Most Common: 2 Spaces**
```yaml
# 2 spaces per level (most common)
database:
  host: localhost
  credentials:
    user: admin
    password: secret
```

### **The Rules:**
1. **Must use SPACES, never tabs** (tabs cause errors)
2. **Be consistent** - same number throughout the file
3. **Common choices**: 2 spaces (most popular) or 4 spaces

### **Examples in Popular Tools:**

**2 Spaces (Default/Preferred):**
- GitHub Actions
- Docker Compose
- Kubernetes
- Ansible
- Most style guides

```yaml
# GitHub Actions example
name: CI
on:
  push:
    branches:
      - main
```

**4 Spaces (Sometimes):**
- Some Python projects (matching Python's 4-space standard)
- Personal preference

### **VSCode Settings:**
```json
// .vscode/settings.json
{
  "[yaml]": {
    "editor.insertSpaces": true,
    "editor.tabSize": 2  // Set to 2 or 4
  }
}
```

### **To Check/Fix:**
- Look at existing indentation in the file
- Use a linter like `yamllint` to enforce consistency
- VSCode shows spaces as dots if you enable `View → Render Whitespace`

**Bottom line**: **2 spaces** is the de facto standard, but **consistency** matters more than the exact number. Never use tabs!
