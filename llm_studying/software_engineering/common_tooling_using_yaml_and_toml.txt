## Tools That REQUIRE Specific Formats:

### **YAML-Only Tools:**

**DevOps/Infrastructure:**
- **GitHub Actions** - `.github/workflows/*.yml`
- **GitLab CI** - `.gitlab-ci.yml`
- **Docker Compose** - `docker-compose.yml`
- **Kubernetes** - All manifests (deployment.yaml, service.yaml)
- **Ansible** - Playbooks must be YAML
- **CircleCI** - `.circleci/config.yml`

**Development Tools:**
- **pre-commit** - `.pre-commit-config.yaml`
- **MkDocs** - `mkdocs.yml` (documentation)
- **ESLint** - `.eslintrc.yml` (one option)
- **Swagger/OpenAPI** - API specs typically YAML

### **TOML-Only Tools:**

**Python Ecosystem:**
- **pip/setuptools** - `pyproject.toml` (Python standard)
- **Poetry** - `pyproject.toml` (required)
- **Black** - `pyproject.toml` (formatter config)
- **Ruff** - `pyproject.toml` (linter config)
- **pytest** - `pyproject.toml` (preferred)

**Other Languages:**
- **Rust/Cargo** - `Cargo.toml` (required)
- **Hugo** - `config.toml` (static site generator)
- **Netlify** - `netlify.toml`

### **Flexible (Accept Multiple):**

These tools accept YAML, TOML, JSON, or others:
- **VS Code** - `settings.json` (prefers JSON)
- **Prettier** - `.prettierrc` (yaml/json/toml/js)
- **Jest** - `jest.config.*` (js/json/yaml)
- **Next.js** - `next.config.js` (JavaScript preferred)

**The Pattern**: 
- **DevOps/Cloud tools** → Usually YAML
- **Python packaging** → TOML (since PEP 518)
- **JavaScript tools** → JSON or JavaScript files
- **Rust ecosystem** → TOML

This is why Python projects often have both - Python tooling uses TOML while CI/CD and containerization use YAML!
