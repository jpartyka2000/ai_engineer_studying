The regex part `(?!\()` is a **negative lookahead assertion**. Here's a detailed breakdown:

---

### **Components of `(?!\()`**:
1. **`(?! ...)`**:
   - This is the syntax for a **negative lookahead assertion**.
   - It ensures that the specified pattern (inside the parentheses) does **not** immediately follow the current position in the string.
   - It is a **non-consuming assertion**, meaning it checks the condition without advancing the regex engine’s position.

2. **`\(`**:
   - Matches a literal opening parenthesis `(`.
   - The backslash `\` escapes the parenthesis since it is a special character in regex.

---

### **Purpose of `(?!\()`**:
This assertion ensures that the next character after the current position is **not an opening parenthesis `(`**.

---

### **Behavior**:
- **If the next character is not `(`**, the match continues.
- **If the next character is `(`**, the match fails at that position.

---

### **Example**:
Using the regex `UNION\s+(?!\()`, let’s analyze how `(?!\()` works.

#### Input Without Parentheses:
```sql
SELECT id FROM table1
UNION
SELECT id FROM table2
```

- The regex engine matches the `UNION` keyword.
- **At "UNION" in the input:**
  - The next character is a space (` `).
  - `(?!\()` passes because the next character is **not** `(`.
- The match succeeds.

#### Input With Parentheses:
```sql
SELECT id FROM table1
UNION
(SELECT id FROM table2)
```

- The regex engine matches `UNION`.
- **At "UNION" in the input:**
  - The next character is `(`.
  - `(?!\()` fails because the next character **is** `(`.
- The match is rejected at this position.

---

### **Use Cases of Negative Lookahead**:
1. Ensure a specific character or sequence does **not** appear after a match.
2. Validate patterns where matches should **not** occur in specific contexts.
3. Avoid unintended matches based on the content that follows.

---

### **Common Examples**:
#### 1. Match "word" not followed by "s":
```python
import re
text = "word words"
pattern = r"word(?!s)"
re.findall(pattern, text)  # Output: ['word']
```

#### 2. Match digits not followed by a decimal point:
- Match integers only:
```python
pattern = r"\d+(?!\.)"
```

#### 3. Match "cat" not followed by "nap":
```python
pattern = r"cat(?!nap)"
```

---

### **Comparison: Lookbehind vs. Lookahead**:
- **Lookbehind (`(?<!...)`)** checks what comes **before** the current position.
- **Lookahead (`(?!...)`)** checks what comes **after** the current position.
