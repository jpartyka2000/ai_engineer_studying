Font licensing is one of those things that trips people up because having a font file on your machine doesn't mean you can ship it inside your product.

**.ttf vs .otf** — the format itself doesn't determine the license. TTF (TrueType) and OTF (OpenType) are just container formats. TTF is older and more universally compatible; OTF supports more advanced typographic features like ligatures, alternate glyphs, and more complex curves. For end users and most developers, the practical difference is minimal — both work everywhere that matters. The licensing terms are set by whoever created the font, not by the file format.

**The licensing landscape** breaks into a few categories. Fonts under the **SIL Open Font License (OFL)** — which covers most Google Fonts, Nerd Fonts, and many of the fonts you'd install via Homebrew — can be freely bundled and redistributed with software, even commercially. The main restriction is you can't sell the font *by itself* and you must include the license file. This is the most permissive and developer-friendly license.

Fonts under **Apache 2.0** (like Roboto) are similarly permissive for bundling and redistribution.

**Commercial/proprietary fonts** are where it gets tricky. Fonts from foundries like Monotype, Adobe (non-open ones), Hoefler, etc. are typically licensed per-seat for *desktop use* — meaning you can use them in Photoshop or Word documents, but you cannot embed them in a software application, mobile app, or web product without purchasing a separate **embedding or distribution license**, which often costs significantly more. Some licenses distinguish between embedding in a PDF (sometimes allowed) versus bundling in an app binary (almost never allowed under a basic desktop license).

**Web fonts** have their own licensing tier — even with Google Fonts being free, some foundries charge separately for self-hosted web use versus using a CDN they control.

**Practical advice for software distribution:** stick with OFL or Apache-licensed fonts if you want to bundle fonts without legal headaches. If you're using Homebrew-installed fonts, run `brew info font-<name>` to check the license before shipping. Always include the font's LICENSE or OFL.txt file alongside the font in your distribution. And if a client insists on using a proprietary font like Helvetica Neue or Proxima Nova in a product, make sure someone is paying for the correct license tier — "we have it installed on our Macs" does not cover redistribution.
