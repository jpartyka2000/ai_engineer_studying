## `\begin` and `\end` in LaTeX

`\begin` and `\end` are commands that create **environments** in LaTeX. Environments are sections of your document with special formatting or behavior.

**Basic syntax:**
```latex
\begin{environmentname}
  content goes here
\end{environmentname}
```

Every `\begin` **must** have a matching `\end` with the same environment name.

---

### **The Most Important Environment: `document`**

Every LaTeX file must have the `document` environment, which contains all the actual content:

```latex
\documentclass{article}

% preamble here

\begin{document}
  All your content goes here
\end{document}
```

Nothing after `\end{document}` will appear in the output.

---

### **Common Environments**

**Math environments:**
```latex
\begin{equation}
  E = mc^2
\end{equation}

\begin{align}
  x + y &= 5 \\
  2x - y &= 1
\end{align}
```

**Lists:**
```latex
\begin{itemize}
  \item First item
  \item Second item
\end{itemize}

\begin{enumerate}
  \item First
  \item Second
\end{enumerate}
```

**Text formatting:**
```latex
\begin{center}
  This text is centered
\end{center}

\begin{verbatim}
  Code or text displayed exactly as typed
\end{verbatim}
```

**Figures and tables:**
```latex
\begin{figure}
  \includegraphics{image.png}
  \caption{My image}
\end{figure}

\begin{table}
  \begin{tabular}{|c|c|}
    A & B \\
    C & D
  \end{tabular}
\end{table}
```

**Abstract (for articles):**
```latex
\begin{abstract}
  This paper discusses...
\end{abstract}
```

---

### **Environments Can Be Nested**

```latex
\begin{itemize}
  \item First item
  \item Second item
  \begin{itemize}
    \item Nested item
    \item Another nested item
  \end{itemize}
  \item Third item
\end{itemize}
```

---

### **Environments with Options**

Some environments accept optional parameters:

```latex
\begin{tabular}{|l|c|r|}  % left, center, right aligned columns
  A & B & C \\
  D & E & F
\end{tabular}
```

---

### **Key Concept:**

Think of `\begin{...}` and `\end{...}` as **opening and closing tags** that tell LaTeX "apply special formatting/behavior to everything in between." They're similar to HTML tags or braces in programming — they define a scope where certain rules apply.
