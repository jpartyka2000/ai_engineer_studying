Yes—LightGBM’s Python API lets you silence training logs in a few ways:

1. **Set `verbose` (or `verbosity`) to 0 or –1**

   ```python
   from lightgbm import LGBMClassifier

   model = LGBMClassifier(
       n_estimators=1000,
       learning_rate=0.05,
       verbose=-1,         # or verbosity=-1 in newer versions
       # … other params …
   )

   model.fit(
       X_train, y_train,
       verbose=False       # also accepted here to override any defaults
   )
   ```

2. **Use the callback API (`lgb.train` only)**
   If you’re using the low‑level `lgb.train` interface, turn off eval logging with:

   ```python
   import lightgbm as lgb

   lgb.train(
       params,
       train_set,
       num_boost_round=1000,
       valid_sets=[valid_set],
       verbose_eval=False,           # no per‑round printouts
       callbacks=[lgb.log_evaluation(period=0)]
   )
   ```

3. **Globally mute LightGBM logger**

   ```python
   import lightgbm as lgb
   lgb.set_config(verbosity='silent')
   # or
   lgb.set_config(verbosity=0)
   ```

Any of those will suppress the “\[LightGBM] …” console output during training.
