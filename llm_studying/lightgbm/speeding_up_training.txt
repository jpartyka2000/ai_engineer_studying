Here are a variety of levers you can pull to make LightGBM train faster beyond just adding more threads or cutting down on rounds:

---

## 1. Subsample your data every iteration

* **`bagging_fraction` + `bagging_freq`**

  ```python
  LGBMClassifier(
      # … 
      bagging_fraction=0.8,    # use 80% of data per tree
      bagging_freq=1,          # do it every iteration
      # …
  )
  ```

  This randomly down‑samples your rows each boosting round, cutting the work per iteration.

* **Gradient‑based One‑Side Sampling (“goss”)**

  ```python
  LGBMClassifier(
      boosting_type='goss',
      top_rate=0.2,    # keep 20% of highest‑grad samples
      other_rate=0.1,  # + 10% of the rest
      # …
  )
  ```

  GOSS aggressively samples examples by gradient magnitude, dramatically reducing data processed.

---

## 2. Subsample your features each iteration

* **`feature_fraction`**

  ```python
  LGBMClassifier(
      feature_fraction=0.6,  # only 60% of features per split
      # …
  )
  ```

  Fewer features → fewer candidate splits → faster histograms.

---

## 3. Coarsen your histograms

* **`max_bin`** (default 255)

  ```python
  LGBMClassifier(
      max_bin=64,
      # …
  )
  ```

  Smaller bins mean less work building the histogram for each feature.

* **`subsample_for_bin`**

  ```python
  LGBMClassifier(
      subsample_for_bin=20000,  # default 200k
      # …
  )
  ```

  Only sample this many rows to build bin boundaries.

---

## 4. Simplify your trees

* **`num_leaves`**

  ```python
  LGBMClassifier(
      num_leaves=15,  # default 31
      # …
  )
  ```

  Fewer leaves per tree = less splitting work.

* **`min_data_in_leaf`** (aka `min_child_samples`)

  ```python
  LGBMClassifier(
      min_data_in_leaf=50,  
      # …
  )
  ```

  Larger leaves → fewer splits evaluated.

* **`max_depth`**

  ```python
  LGBMClassifier(
      max_depth=6,
      # …
  )
  ```

  Hard cap on depth to avoid exploring huge trees.

---

## 5. Use native categorical support

If you have high‑cardinality categoricals, let LightGBM handle them directly:

```python
LGBMClassifier(
    categorical_feature=['degree_offering', 'state', …],
    # …
)
```

That avoids expensive one‑hot encoding or string‑to‑int pipelines.

---

## 6. Offload to GPU

If you have a CUDA‐capable card:

```python
LGBMClassifier(
    device='gpu',
    gpu_platform_id=0,
    gpu_device_id=0,
    # …
)
```

The `gpu_hist` algorithm can be 5–10× faster on large datasets.

---

## 7. Pre‑select or engineer fewer features

Removing low‑importance or highly correlated features up front reduces the dimensionality that LightGBM has to scan.

---

## 8. Reduce logging overhead

Set verbosity low or off to cut console I/O:

```python
LGBMClassifier(
    verbose=-1,
    # …
)
```

---

### Putting it all together

```python
from lightgbm import LGBMClassifier

clf = LGBMClassifier(
    boosting_type='goss',
    n_estimators=500,
    learning_rate=0.1,
    num_leaves=15,
    max_bin=64,
    bagging_fraction=0.8,
    bagging_freq=1,
    feature_fraction=0.6,
    subsample_for_bin=20000,
    min_data_in_leaf=50,
    device='gpu',
    verbose=-1,
    random_state=42
)

clf.fit(
    X_train, y_train,
    eval_set=[(X_val, y_val)],
    early_stopping_rounds=30,
    verbose=100
)
```

By combining **data sampling**, **feature sampling**, **coarser histograms**, **smaller trees**, and (optionally) **GPU acceleration**, you’ll usually see a 2×–5× speedup in training time with minimal impact on performance.
