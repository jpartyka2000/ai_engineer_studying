LightGBM does **not** include F1‑score as a built‑in evaluation metric, but you **can** still monitor (and early‑stop on) F1 by supplying a custom evaluation function via the **`feval`** parameter. Under the hood, LightGBM will:

1. Call your `feval` at each iteration with the raw predictions and true labels.  
2. Expect a return tuple `(metric_name, metric_value, is_higher_better)`.  
3. Use that metric for logging and (if you’ve set `early_stopping_rounds`) deciding when to stop.

Here’s how you could implement and plug in F1 using Scikit‑Learn’s `f1_score`:

```python
import numpy as np
import lightgbm as lgb
from sklearn.metrics import f1_score

# 1) Define the custom metric
def f1_metric(preds: np.ndarray, train_data: lgb.Dataset):
    # LightGBM passes raw scores, so convert to probabilities then to 0/1 labels
    labels = train_data.get_label()
    # you can choose a threshold here; 0.5 is typical
    preds_binary = (preds > 0.5).astype(int)
    f1 = f1_score(labels, preds_binary)
    # return: (metric_name, metric_value, is_higher_better)
    return 'f1', f1, True

# 2) Prepare your data
dtrain = lgb.Dataset(X_train, label=y_train)
dvalid = lgb.Dataset(X_val,   label=y_val, reference=dtrain)

# 3) Train with the custom metric

    'objective': 'binary',
    'learning_rate': 0.05,
    'num_leaves': 31,
    # note: do NOT put 'f1' into params['metric']
}
bst = lgb.train(
    params,
    dtrain,
    num_boost_round=1000,
    valid_sets=[dtrain, dvalid],
    valid_names=['train','valid'],
    feval=f1_metric,                # <-- our custom F1
    early_stopping_rounds=50,
    verbose_eval=50
)
```
