Sure! Here's an example of using the class_weight parameter with lightgbm.LGBMClassifier to give more weight to the minority class:


---

Example: Using class_weight in LightGBMClassifier

import lightgbm as lgb
from sklearn.datasets import make_classification
from sklearn.model_selection import train_test_split
from sklearn.metrics import classification_report

# Create imbalanced dataset
X, y = make_classification(
    n_samples=10000,
    n_features=20,
    n_classes=2,
    weights=[0.95, 0.05],  # 5% minority class
    flip_y=0,
    random_state=42
)

# Split the dataset
X_train, X_test, y_train, y_test = train_test_split(X, y, stratify=y, random_state=42)

# Use class_weight to upweight the minority class
model = lgb.LGBMClassifier(
    class_weight={0: 1.0, 1: 10.0},  # 10x more weight for minority class
    random_state=42
)

# Train model
model.fit(X_train, y_train)

# Predict and evaluate
y_pred = model.predict(X_test)
print(classification_report(y_test, y_pred))


---

What class_weight Does

The parameter {0: 1.0, 1: 10.0} tells LightGBM to penalize errors on class 1 (minority) 10 times more than errors on class 0. This makes the model focus more on correctly classifying the minority class.


---

Alternative: 'balanced'

You can also use:

class_weight='balanced'

This automatically sets weights inversely proportional to class frequencies, i.e.:

weight_class_i = total_samples / (n_classes * samples_in_class_i)
