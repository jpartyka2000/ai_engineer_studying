Thought for 20 seconds

The error you’re seeing—

> `train() got unexpected keyword argument 'early_stopping_rounds'`

—happens because support for the **`early_stopping_rounds`** argument in **`lgb.train()`**, **`lgb.cv()`** and the sklearn **`.fit()`** wrapper was **removed** in **LightGBM 4.0.0**.  In other words, the last major release that still accepted:

```python
lgb.train(..., early_stopping_rounds=50, ...)
```

was the **3.x** series (most recently **3.3.5**, released January 24, 2023) citeturn1search2.  Starting in **4.0.0** (July 13, 2023), you must instead use the new callback API:

```python
from lightgbm import early_stopping

bst = lgb.train(
    params,
    train_set,
    valid_sets=[train_set, valid_set],
    callbacks=[early_stopping(stopping_rounds=50)],
    # no more `early_stopping_rounds=` here
)
```

### What you can do

- **If you really want the old signature**  
  Pin your LightGBM version to the latest 3.x release:
  ```bash
  pip install lightgbm==3.3.5
  ```
  Then you can again pass `early_stopping_rounds` directly. citeturn0search2

- **Or upgrade your code to use the new callback**  
  ```python
  from lightgbm import early_stopping

  model = lgb.train(
      params,
      train_data,
      num_boost_round=1000,
      valid_sets=[train_data, valid_data],
      callbacks=[early_stopping(stopping_rounds=50)],
      verbose_eval=100
  )
  ```
  This is the supported approach in all **4.x** releases.
