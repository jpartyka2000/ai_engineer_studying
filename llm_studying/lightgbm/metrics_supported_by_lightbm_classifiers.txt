**Listing LightGBM classification metrics**

LightGBM supports several evaluation metrics for classification tasks. For binary classification, common metrics are:

- binary_logloss
- binary_error
- auc
- average_precision

For multiclass classification, it includes:

- multi_logloss
- multi_error
- auc_mu

Additionally, there are other metrics like cross_entropy, cross_entropy_lambda, and kullback_leibler. Although ranking metrics like ndcg and map are available, they fall under a separate category. F1 score isn't built-in, though it can be calculated.

### How to enable multiple metrics

You can pass a single metric or a list:

```python
params = {
    'objective': 'binary',
    'metric': ['binary_logloss', 'auc', 'binary_error']
}
```

LightGBM will compute all of them each round, and—if you’ve enabled early stopping—use the **first** metric in the list to decide when to stop (unless you override that behavior via callbacks).
