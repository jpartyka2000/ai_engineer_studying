Yes. Regardless of what you put in your `system_prompt`, the agent will automatically inject its own, machine‐readable tool descriptions into the prompt it actually sends to the LLM. Internally it builds something like:

```
[System Prompt YOU WROTE]

TOOLS:
• compute(expr: str) → {"value": float}  
  “Compute the value of an arithmetic expression exactly (within float).”  
• another_tool(…) → …

[Serialized State / Context]

[User Question]
```

So even if your `system_prompt` doesn’t mention the tools (or mentions them only in prose), the agent runtime will append the precise signatures and docstrings for every `@agent.tool`-decorated function before handing the combined prompt off to the model. That way the LLM always “knows” exactly what tools are available and how to call them.
