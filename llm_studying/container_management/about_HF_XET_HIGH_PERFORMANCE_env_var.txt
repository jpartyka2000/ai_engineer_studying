This enables high-performance mode for Hugging Face's Xet storage backend.

## What is Xet?

Xet is a newer storage system that Hugging Face has been rolling out for hosting model files. It's designed to be more efficient than their traditional Git LFS-based storage, particularly for large files like model weights.

## What the variable does

When set to `"1"`, it enables optimizations in the Xet client that improve download speeds. According to Hugging Face's documentation, this typically involves:

- More aggressive parallelization
- Optimized chunk sizes
- Better memory management during downloads

## Why both this and `HF_HUB_ENABLE_HF_TRANSFER`?

They target different storage backends:

| Variable | Backend | Purpose |
|----------|---------|---------|
| `HF_HUB_ENABLE_HF_TRANSFER` | Traditional LFS storage | Rust-based parallel downloader |
| `HF_XET_HIGH_PERFORMANCE` | Xet storage | High-performance mode for Xet client |

Since models on Hugging Face Hub might be stored on either backend (and this can change over time), setting both variables ensures you get the fastest possible downloads regardless of where the model files actually live.

## The pattern

This is a common defensive approach — enable all the performance optimizations available, so you're covered no matter which code path gets triggered during the model download.
