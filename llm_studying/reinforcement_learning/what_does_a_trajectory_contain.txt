Great question! A **trajectory** contains **both** - it's the complete sequence of states, actions, and transitions.

## What a Trajectory Actually Contains

A trajectory is the full execution sequence:

```
s₀ --[action a₀]--> s₁ --[action a₁]--> s₂ --[action a₂]--> s₃ --[action a₃]--> ...
```

### Written out fully:
```
Trajectory τ = (s₀, a₀, s₁, a₁, s₂, a₂, s₃, a₃, ..., s_T)
```

This includes:
- **States:** s₀, s₁, s₂, ...
- **Actions:** a₀, a₁, a₂, ... (the actions that were actually taken)
- **Transitions:** s₀ → s₁ (caused by taking a₀), s₁ → s₂ (caused by taking a₁), etc.

## You Can View It Multiple Ways

From a trajectory, you can extract:

### 1. State-Action Pairs
```
{(s₀, a₀), (s₁, a₁), (s₂, a₂), ...}
```
"At state s₀, action a₀ was taken"

### 2. State Transition Pairs  
```
{(s₀, s₁), (s₁, s₂), (s₂, s₃), ...}
```
"State s₀ led to state s₁"

### 3. Full Transitions (state, action, next_state)
```
{(s₀, a₀, s₁), (s₁, a₁, s₂), (s₂, a₂, s₃), ...}
```
"Taking action a₀ in state s₀ led to state s₁"

## What Goes Into the SMILE/DAgger Dataset?

From the trajectories, we extract:
- The **states** that were visited: {s₀, s₁, s₂, ...}
- We then **relabel** them with expert actions: {(s₀, a₀*), (s₁, a₁*), (s₂, a₂*), ...}

Where a* denotes the expert's recommended action, **not** the action that was actually taken in the trajectory.

## So To Answer Your Question:

**A trajectory contains:** States, actions taken, and the resulting state transitions (the complete execution history)

**The dataset contains:** States from the trajectory, paired with expert-recommended actions (discarding what was actually done and the transition structure)
