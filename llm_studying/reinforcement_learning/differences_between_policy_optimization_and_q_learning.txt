Here’s a high‑level comparison of the two main model‑free paradigms—**policy optimization** (policy‑gradient/actor‑critic) vs. **Q‑learning** (value‑based):

| Aspect                     | Policy Optimization                                                               | Q‑Learning (Value‑Based)                                            |                                   |
| -------------------------- | --------------------------------------------------------------------------------- | ------------------------------------------------------------------- | --------------------------------- |
| **What’s learned**         | An explicit parameterized policy  (\pi\_\theta(a                                  | s))                                                                 | An action‑value function $Q(s,a)$ |
| **Policy extraction**      | Direct: $\pi_\theta$ is your controller                                           | Indirect: $\pi(s)=\arg\max_a Q(s,a)$                                |                                   |
| **On‑ vs Off‑policy**      | Often **on‑policy** (REINFORCE, PPO), though actor‑critic can be off‑policy (SAC) | **Off‑policy** by design (learn $Q^*$ from any data)                |                                   |
| **Update style**           | Monte Carlo or TD **gradients** on $J(\theta)$                                    | **Bootstrapped** TD updates: $Q\leftarrow r+\gamma\max Q$           |                                   |
| **Bias vs variance**       | **Low bias**, **high variance** (mitigated by baselines, GAE)                     | **Lower variance**, **higher bias** (due to bootstrapping)          |                                   |
| **Exploration**            | Intrinsic via stochastic $\pi_\theta$                                             | Needs external strategy (ε‑greedy, Boltzmann)                       |                                   |
| **Action spaces**          | Easily handles **continuous** actions                                             | Originally for discrete; continuous needs modifications (e.g. DDPG) |                                   |
| **Convergence guarantees** | Often **noisy**; few strict proofs beyond tabular cases                           | Tabular Q‑learning has convergence proofs under standard conditions |                                   |
| **Sample efficiency**      | Generally **less efficient** (on‑policy data)                                     | **More efficient** reuse of off‑policy data                         |                                   |
| **Stability**              | Requires care (trust regions, clipping)                                           | Sensitive to function‑approximation error (overestimation)          |                                   |
| **Typical algorithms**     | REINFORCE, A2C/A3C, PPO, SAC                                                      | Q‑Learning, SARSA, DQN, Double DQN                                  |                                   |

---

### When to pick which

* **Policy optimization** if you need smooth stochastic policies (e.g. in continuous control), want direct opt‑throughput on $J(\theta)$, or need built‑in exploration via $\pi(a|s)$.
* **Q‑learning** if you can afford to learn a discrete-value table or function, you want to reuse past data aggressively, and you don’t mind deriving a policy indirectly via $\arg\max$.
