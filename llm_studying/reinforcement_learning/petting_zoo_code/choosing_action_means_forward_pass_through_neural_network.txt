Yes—whenever your policy is represented by a neural network, choosing an action means running a **forward pass** through that network to get either:

1. **Action probabilities** (in a stochastic policy):

   ```python
   logits = policy_net(obs)              # forward pass
   dist   = Categorical(logits=logits)  
   action = dist.sample().item()         # sample from πθ(a|obs)
   ```
2. **Q-values** (in a value-based method):

   ```python
   q_values = q_net(obs)                # forward pass
   action   = torch.argmax(q_values).item()  # pick best a
   ```
3. **Deterministic actions** (in a continuous control policy):

   ```python
   action = policy_net(obs)             # directly outputs the action
   ```

Without that forward evaluation you have no way to compute the policy’s output, so you can’t pick or sample an action. In pure tabular or rule-based methods you might index or look up an action without a forward pass, but in **deep RL** your network is the function that **maps observations → actions**, so every decision step entails a forward pass (inference) through your policy or Q-network.
