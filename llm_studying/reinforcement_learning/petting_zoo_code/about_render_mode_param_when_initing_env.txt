The `render_mode` argument tells the environment **how** you’d like it to produce visual output (if at all). In PettingZoo’s Atari wrappers (and many Gym-style APIs), common modes are:

* **`render_mode="human"`**
  Opens a window (or uses your display) to **show** each game frame in real time. You’ll actually see the tennis court and the ball as the episode runs.

* **`render_mode="rgb_array"`**
  Instead of popping up a window, calls to `env.render()` return the current frame as a NumPy array of shape `(H, W, C)` (e.g. 210×160×3). Useful if you want to grab frames for your own processing or video recording.

* **`render_mode=None`** (the default for many envs)
  Disables rendering entirely. No windows open, and `env.render()` may either do nothing or raise an error.

So by doing:

```python
env = tennis_v3.env(render_mode="human")
```

you’re telling the Atari Tennis environment to launch an on-screen display so you can watch the agents play live.
