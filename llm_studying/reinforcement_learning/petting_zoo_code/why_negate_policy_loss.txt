The negative sign is there because our optimizer (e.g. Adam) by default **minimizes** a loss, but we actually want to **maximize** the expected return

$$
J(\theta)
= \mathbb{E}\bigl[\sum_t G_t\log\pi_\theta(a_t\!\mid\!s_t)\bigr].
$$

If we set

```python
loss = - ∑_t G_t · log π(a_t|s_t),
```

then minimizing `loss` via gradient descent

```python
loss.backward()
opt.step()
```

is exactly the same as **gradient ascent** on $J(\theta)$.  In other words:

* **Without the minus**, we’d be descending the wrong hill.
* **With the minus**, $\nabla_\theta\bigl[-J(\theta)\bigr] = -\nabla_\theta J(\theta)$, so a descent step
  $\theta ← \theta - α\,\nabla_\theta[-J]$
  becomes
  $\theta ← \theta + α\,\nabla_\theta J$,
  which is precisely the policy-gradient update.
