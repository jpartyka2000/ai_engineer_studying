Each `torch.nn.Parameter` object has a `.grad` attribute that, after you call `loss.backward()`, holds the **computed derivative** of your loss with respect to that parameter. Concretely:

```python
# Suppose your model has a parameter w
w = some_layer.weight  # this is a nn.Parameter

# After computing loss and calling backward():
loss.backward()

# Now:
print(w.grad.shape)    # same shape as w
print(w.grad)          # a tensor of ∂loss/∂w
```

* **Type:** a `torch.Tensor` of the same shape as the parameter.
* **Content:** the element-wise partial derivatives $\frac{\partial \text{loss}}{\partial w_{ij}}$.
* **When to zero it:** `.zero_grad()` sets all these `.grad` tensors back to zero so that the next backward pass doesn’t accumulate on top of old gradients.
* **Used by optimizers:** when you call `optimizer.step()`, PyTorch looks at each parameter’s `.grad` to decide how to update that parameter.
