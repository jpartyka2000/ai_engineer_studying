A **policy** in RL defines how an agent picks actions given its input (state or observation). The key distinction is whether that mapping is **deterministic** or **stochastic**:

---

## 1. Deterministic Policy

* **Definition**

  $$
    \mu : \mathcal{S} \;\to\; \mathcal{A}
    \quad\text{so that}\quad
    a_t = \mu(s_t)
  $$

  For every state $s$, the policy returns one fixed action $a$.

* **Characteristics**

  * No randomness: the same state always yields the same action.
  * Simpler representation (e.g. a neural network with a single output for continuous actions).
  * Exploration must be injected externally (e.g. via ε‐greedy or added noise).

* **Example**

  * In a continuous‐control task, $\mu_\theta(s)$ might be a neural net outputting a real‐valued torque command.

---

## 2. Stochastic Policy

* **Definition**

  $$
    \pi : \mathcal{S} \;\to\; P(\mathcal{A})
    \quad\text{so that}\quad
    a_t \sim \pi(\,\cdot\mid s_t)
  $$

  For each state $s$, the policy defines a probability distribution over actions; the agent samples $a$ from that distribution.

* **Characteristics**

  * Intrinsic exploration: randomness in action selection.
  * Often parameterized as softmax over logits (discrete) or a Gaussian (continuous).
  * Policy‐gradient methods naturally optimize such policies via
    $\nabla_\theta \log \pi_\theta(a\mid s)$.

* **Example**

  * In a discrete gridworld, $\pi(a\mid s)$ might assign $\{0.1,0.2,0.7,0.0\}$ to the four cardinal moves.

---

## 3. When to Use Which

| Aspect          | Deterministic                          | Stochastic                                                       |
| --------------- | -------------------------------------- | ---------------------------------------------------------------- |
| **Exploration** | Needs external noise/ε-greedy          | Built‑in via sampling                                            |
| **Algorithms**  | DDPG, D4PG, TD3 (continuous)           | REINFORCE, A2C/A3C, PPO, SAC                                     |
| **Convergence** | Can converge faster in high‐dim spaces | More robust early on, handles multimodal action preferences      |
| **Suitability** | Good for smooth continuous actions     | Ideal when you need to represent uncertainty or mixed strategies |

---

### Bottom Line

* **Deterministic**: one “best” action per state; simpler but requires added exploration.
* **Stochastic**: maps to a distribution over actions; naturally explores and represents uncertainty, but slightly more complex to learn.
