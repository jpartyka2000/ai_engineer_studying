You can ask pytest to show you exactly which test items it *would* run—without actually executing them—by using the `--collect-only` flag. For example, from your project root (or the `/dbfs/...` copy on Databricks):

```bash
pytest --collect-only
```

By default it’ll print a tree of all discovered test files, classes, functions, and fixtures. If you just want the flat list of node IDs, add `-q` (quiet) or `-qq`:

```bash
pytest --collect-only -q
```

Or target a single file:

```bash
pytest tests/test_end_to_end.py:: --collect-only -q
```

That last command will list only the tests in that file (e.g. `tests/test_end_to_end.py::test_end_to_end_tasks`) so you can verify pytest is seeing the function you expect.
