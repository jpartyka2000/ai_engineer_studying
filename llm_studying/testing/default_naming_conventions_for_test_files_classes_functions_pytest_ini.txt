Yes, both are the defaults.

**Default pytest discovery patterns:**

| Setting | Default | Your Config |
|---------|---------|-------------|
| `python_files` | `test_*.py` and `*_test.py` | `test_*.py` (more restrictive) |
| `python_classes` | `Test*` | `Test*` (same) |
| `python_functions` | `test_*` | `test_*` (same) |

So you could simplify to:

```ini
[pytest]
testpaths = tests
python_files = test_*.py
addopts = -v --tb=short
markers =
    slow: marks tests as slow (deselect with '-m "not slow"')
    integration: marks tests as integration tests
filterwarnings =
    ignore::DeprecationWarning
    ignore::PendingDeprecationWarning
```

The only non-default you're actually using is `python_files` — you're excluding `*_test.py` style naming, which is fine if you want to enforce a consistent naming convention.
