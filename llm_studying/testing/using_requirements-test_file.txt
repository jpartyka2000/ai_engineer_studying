It isn't — pytest doesn't read requirements files at all.

`requirements-test.txt` is just a convention for humans and CI systems. You manually install from it before running tests:

```bash
pip install -r requirements-test.txt
pytest
```

**Typical CI workflow (e.g., GitHub Actions):**
```yaml
steps:
  - uses: actions/checkout@v4
  - name: Install dependencies
    run: |
      pip install -r requirements.txt
      pip install -r requirements-test.txt
  - name: Run tests
    run: pytest
```

**Common naming conventions:**

| File | Purpose |
|------|---------|
| `requirements.txt` | Production dependencies |
| `requirements-test.txt` | Test-only dependencies (pytest, etc.) |
| `requirements-dev.txt` | Dev tools (linters, formatters, etc.) |

Alternatively, with `pyproject.toml` you can define optional dependency groups:

```toml
[project.optional-dependencies]
test = ["pytest>=7.0", "pytest-cov>=4.0"]
```

Then install with:
```bash
pip install -e ".[test]"
```
