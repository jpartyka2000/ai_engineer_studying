`testpaths` tells pytest which directories to search for tests when you run `pytest` without specifying a path.

**With `testpaths = tests`:**
```bash
pytest          # Only looks in ./tests/
```

**Without `testpaths`:**
```bash
pytest          # Searches from current directory recursively
```

It's useful for:

1. **Speed** - Skips scanning irrelevant directories (src, docs, data, etc.)
2. **Avoiding false matches** - Won't accidentally pick up test-like files elsewhere
3. **Cleaner command** - Just type `pytest` instead of `pytest tests/`

Note: If you explicitly pass a path, it overrides `testpaths`:
```bash
pytest tests/test_unit.py    # Ignores testpaths, uses your argument
```
