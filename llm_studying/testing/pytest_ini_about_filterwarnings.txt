By default, `filterwarnings` is empty — pytest shows all warnings in a summary at the end of the test run.

**Default behavior:**
```
=================== warnings summary ====================
test_foo.py::test_example
  /path/to/file.py:10: DeprecationWarning: something is deprecated
    some_function()

=============== 5 passed, 2 warnings in 0.5s ============
```

**Your config hides those:**
```ini
filterwarnings =
    ignore::DeprecationWarning
    ignore::PendingDeprecationWarning
```

**Filter action options:**

| Action | Effect |
|--------|--------|
| `ignore` | Never show |
| `default` | Show once per location |
| `always` | Always show |
| `error` | Turn into exception |
| `module` | Show once per module |
| `once` | Show once total |

**More specific filtering:**
```ini
filterwarnings =
    # Ignore specific message
    ignore:some message:DeprecationWarning
    
    # Ignore from specific module
    ignore::DeprecationWarning:somepackage.*
    
    # Turn specific warning into error
    error::UserWarning
```

A common approach is to use `error` during CI to catch warnings early:
```ini
filterwarnings =
    error
    ignore::DeprecationWarning:thirdpartylib.*
```
