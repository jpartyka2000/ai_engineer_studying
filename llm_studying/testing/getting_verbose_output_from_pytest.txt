Absolutely—pytest’s `-v`/`-vv` flags will give you more verbosity, and you can also combine it with `-s` (to see `print()` output) or turn on live logging. For example, if you’ve copied your tests to DBFS under `/dbfs/tmp/code/cue_reinforcement_learning`, you could run:

```bash
%sh
cd /dbfs/tmp/code/cue_reinforcement_learning
pytest \
  tests/test_end_to_end.py::test_end_to_end_tasks \
  -vv -s \
  --disable-warnings --maxfail=1
```

* `-vv` makes pytest very talkative (showing each test name, setup/teardown, etc.).
* `-s` “disables capture” so you see any `print()` output or logging to stdout.

If you want pytest’s own logging (e.g. from `logging.info()` calls), you can also add:

```bash
pytest ... -o log_cli=true -o log_cli_level=INFO
```

which will stream log messages into your test output in real time.
