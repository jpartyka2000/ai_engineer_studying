Your `print` is at module level — it runs during *collection*, not during test *execution*. `-s` only affects test execution.

**Option 1: Move print inside a test:**
```python
class TestQuestion:
    def test_question_creation_full(self, sample_generated_question):
        print(str(Path(__file__).parent.parent))  # Will show with -s
        q = sample_generated_question
        ...
```

**Option 2: Show collection output:**
```bash
pytest tests/test_unit.py -s --collect-only
```

**Option 3: Use logging instead:**
```python
import logging
logging.warning(str(Path(__file__).parent.parent))  # Shows during collection
```

The module-level code runs when pytest imports the file to discover tests — that happens before `-s` capturing kicks in. If you need to debug import-time issues, logging or moving the print into a test is the way to go.
