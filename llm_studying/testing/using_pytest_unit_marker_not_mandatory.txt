No, markers are optional. You only need them if you want to selectively run/exclude tests.

**Without markers (your current setup):**
```bash
pytest tests/test_unit.py        # Run by file
pytest tests/test_integration.py # Run by file
```

**With markers:**
```python
@pytest.mark.unit
def test_something():
    pass

@pytest.mark.slow
def test_heavy_computation():
    pass
```

```bash
pytest -m unit          # Run only @pytest.mark.unit
pytest -m "not slow"    # Skip @pytest.mark.slow
pytest -m "unit and not slow"  # Combine
```

**When markers are useful:**

| Scenario | Example |
|----------|---------|
| Skip slow tests during development | `pytest -m "not slow"` |
| Run only integration tests in CI | `pytest -m integration` |
| Skip tests requiring external services | `pytest -m "not external"` |

Your file-based separation (`test_unit.py`, `test_integration.py`) already achieves similar organization without markers. Markers add value when the distinction doesn't map cleanly to files.
