A `pytest.ini` (or equivalently `tox.ini` or a `[tool.pytest.ini_options]` section in `pyproject.toml`) is the canonical place to put your project-wide pytest configuration. It lives at the root of your repo and lets you customize defaults so you don’t have to sprinkle CLI flags everywhere.

Here are the most common settings you’ll find (or add) in a `pytest.ini`:

```ini
[pytest]
# ——————————————————————————————————————————————————————————————————————
# 1) Change collection behavior
# ——————————————————————————————————————————————————————————————————————
# Only look under these directories for tests
testpaths = tests

# Which file name patterns to treat as test files
python_files = test_*.py *_test.py

# Which classes and functions count as tests
python_classes = Test* 
python_functions = test_*

# ——————————————————————————————————————————————————————————————————————
# 2) Default CLI flags
# ——————————————————————————————————————————————————————————————————————
# -s to show print(), -q for quiet, --maxfail=1 to stop at first fail, etc
addopts = -s --maxfail=1

# ——————————————————————————————————————————————————————————————————————
# 3) Custom markers
# ——————————————————————————————————————————————————————————————————————
markers =
    slow: marks tests as slow (deselect with '-m "not slow"')
    integration: for integration tests

# ——————————————————————————————————————————————————————————————————————
# 4) Logging & output
# ——————————————————————————————————————————————————————————————————————
# control assertion rewriting, log capture
console_output_style = classic
log_cli = true
log_cli_level = INFO

# ——————————————————————————————————————————————————————————————————————
# 5) Coverage plugin defaults
# ——————————————————————————————————————————————————————————————————————
# if you use pytest-cov
cov = tests
cov_report = term-missing
```

### How to use it

1. **Create** a file named `pytest.ini` at your project root.
2. **Edit** it to include only the sections you need.
3. **Run** `pytest` normally—pytest will auto-detect that file and apply your settings.

---

#### A couple of tips

* **`addopts`** is your “magic pass-through” for any CLI flags you always want on (e.g. `-s`, `-q`, coverage flags).
* **`markers`** block declares any custom `@pytest.mark.*` you plan to use, which suppresses errors about unknown markers.
* **`testpaths`** plus `python_files`/`functions`/`classes` give you full control over exactly which files pytest will scan.

With a well-tuned `pytest.ini`, you can simplify your CI scripts and your team’s everyday workflow: just `pytest` (or `poetry run pytest`, etc.) and all your preferred defaults will already be in place.
