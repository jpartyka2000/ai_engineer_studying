Almost — but with one subtlety 👌

In **pytest**, by default:

* **Test function names must start with `test_`.**
* Similarly, **test class names must start with `Test`.**

This is how pytest’s test discovery works out of the box.

---

## ✅ Example (works)

```python
def test_addition():
    assert 1 + 1 == 2

class TestMath:
    def test_subtraction(self):
        assert 2 - 1 == 1
```

---

## ❌ Example (does NOT run by default)

```python
def addition_test():
    assert 1 + 1 == 2  # Pytest will NOT discover this automatically
```

Even though the function *ends* with "test", pytest won’t run it unless you explicitly tell it to (e.g. by naming it in the CLI or configuring discovery rules).

---

## ⚙️ Customization

* You *can* change discovery rules by editing `pytest.ini` or `pyproject.toml`:

```ini
# pytest.ini
[pytest]
python_files = test_*.py *_test.py
python_functions = test_* *_test
python_classes = Test* *Tests
```

With this config, pytest would pick up both `test_addition()` **and** `addition_test()`.

---

## ✅ Answer

By default:

* Test **functions** must **start with `test_`**.
* Test **classes** must **start with `Test`**.
* Ending with `test` only works if you configure pytest’s discovery rules.
