`addopts` specifies command-line options that are automatically added every time you run pytest.

**Your config:**
```ini
addopts = -v --tb=short
```

This means running `pytest` is equivalent to running `pytest -v --tb=short`.

| Option | Effect |
|--------|--------|
| `-v` | Verbose output (shows each test name) |
| `--tb=short` | Shorter traceback format on failures |

**Common addopts examples:**

```ini
# Show coverage
addopts = --cov=mypackage --cov-report=html

# Stop on first failure
addopts = -x

# Run in parallel (requires pytest-xdist)
addopts = -n auto

# Combine several
addopts = -v --tb=short -x --strict-markers
```

You can still override or extend these on the command line:

```bash
pytest -q              # Overrides -v with quiet mode
pytest --tb=long       # Overrides --tb=short
pytest -k "unit"       # Adds filtering (combines with addopts)
```
