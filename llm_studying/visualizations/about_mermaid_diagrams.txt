# What is a Mermaid Diagram?

Mermaid is a text-based diagramming language that lets you create diagrams using simple markdown-like syntax. Instead of dragging and dropping shapes in a visual editor, you write code that describes the diagram, and a renderer converts it into a visual graphic.

## Why It Exists

Traditional diagramming tools (Visio, Lucidchart, draw.io) require manual positioning of shapes and connectors. This creates problems:

- Diagrams are hard to version control
- Updating diagrams is tedious
- Collaboration requires sharing binary files
- Diagrams drift out of sync with code/documentation

Mermaid solves this by treating diagrams as code.

## Basic Example

This text:

```mermaid
flowchart LR
    A[User] --> B[Frontend]
    B --> C[API]
    C --> D[Database]
```

Renders as a left-to-right flowchart showing: User → Frontend → API → Database

## Supported Diagram Types

| Type | Use Case |
|------|----------|
| `flowchart` | Process flows, architecture diagrams |
| `sequenceDiagram` | API calls, system interactions over time |
| `classDiagram` | OOP class structures, relationships |
| `erDiagram` | Database entity relationships |
| `gantt` | Project timelines |
| `stateDiagram` | State machines |
| `pie` | Pie charts |
| `gitgraph` | Git branch visualizations |

## Where It Renders

Mermaid diagrams render natively in:

- GitHub README files and issues
- GitLab markdown
- Notion
- Obsidian
- VS Code (with extensions)
- Confluence (with plugins)
- Many documentation platforms (Docusaurus, MkDocs, etc.)

## Simple Syntax Examples

**Sequence diagram:**
```mermaid
sequenceDiagram
    Client->>Server: HTTP Request
    Server->>Database: Query
    Database-->>Server: Results
    Server-->>Client: HTTP Response
```

**Class diagram:**
```mermaid
classDiagram
    Animal <|-- Dog
    Animal <|-- Cat
    Animal: +name string
    Animal: +speak()
```

## Key Benefits

1. **Version control friendly** — diffs show exactly what changed
2. **Lives with your code** — diagrams in the same repo as the code they document
3. **Easy to update** — change a line of text instead of rearranging shapes
4. **Consistent styling** — themes ensure visual consistency
5. **No special tools** — just a text editor

## Limitations

- Less precise control over positioning than visual tools
- Complex diagrams can become hard to read in text form
- Some advanced styling requires verbose configuration
- Not ideal for pixel-perfect designs or marketing materials

Mermaid is best suited for technical documentation where accuracy and maintainability matter more than visual polish.
