## Short Answer: Yes, it can pause Claude Code

If your Mac goes to sleep (or the screen saver triggers sleep mode), **Claude Code can get paused** and won't continue working. This is a known issue that affects users who leave Claude Code running while stepping away.

---

## The Problem

When Claude works for a long time, your Mac may go to sleep and pause the agent. If you leave the agent running while you go make a coffee or get a snack, it's very annoying to return to the computer only to find that it hasn't been doing anything in the meantime.

---

## The Solution (for Mac)

You can use Claude Code hooks with the `caffeinate` command to prevent sleep while Claude is working. The `UserPromptSubmit` hook was added in version 1.0.54, which allows you to run a script that prevents the Mac from sleeping only while Claude is running.

The solution involves creating two shell scripts:
1. **prevent-sleep.sh** — Runs `caffeinate` when you submit a prompt
2. **allow-sleep.sh** — Kills `caffeinate` when Claude stops

---

## Quick Fix (Manual)

If you don't want to set up hooks, you can manually run this in a separate terminal before starting a long Claude Code session:

```bash
caffeinate -i -s
```

This keeps your Mac awake until you stop it with `Ctrl+C`.
