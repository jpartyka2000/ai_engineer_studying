Here's how to get started in Claude Code:

### 1. Set Up Your Project Directory

First, create your project folder and add the files we created:

```bash
mkdir genre-sim
cd genre-sim
mkdir -p docs
# Copy CLAUDE.md to ./CLAUDE.md
# Copy spec.md to ./docs/spec.md
```

### 2. Start Claude Code

```bash
claude
```

### 3. Initial Prompt

Once Claude Code is running, type something like:

```
Read docs/spec.md and implement the music genre evolution simulator. Start with the project structure and core models (TraitVector, Song, Album, Artist, Genre). Set up the pyproject.toml with all dependencies.
```

Or if you want to go step-by-step:

```
Read docs/spec.md and create the initial project structure with pyproject.toml. Don't implement anything yet, just set up the scaffolding.
```

### 4. Iterative Prompts

After the initial setup, you can guide it through phases:

```
Implement the TraitVector and Song models from the spec.
```

```
Implement the ArtistAgent and ListenerAgent Mesa agents.
```

```
Implement the DuckDB database schema and MusicEvolutionModel.
```

```
Implement the AlbumArtGenerator with all 10 art styles.
```

```
Implement the CLI using Typer with all the flags from the spec.
```

```
Implement the static site generator with AllMusic-style pages.
```

```
Implement the HistoricalImportanceCalculator for post-simulation analysis.
```

### 5. Running the Simulation

Once implemented:

```
Run a test simulation from 1960-1980 with 5 bands per year and generate the site.
```

---

### Quick Start (All-in-One)

If you want Claude Code to just build everything:

```
Read docs/spec.md and implement the complete music genre evolution simulator. Create all files, models, CLI, and site generator. Then run a test simulation from 1960 to 1970 with default settings and generate the website.
```

This is ambitious for one prompt though—the step-by-step approach usually produces better results.
