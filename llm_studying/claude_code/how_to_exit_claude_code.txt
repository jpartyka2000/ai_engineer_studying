There are several ways to exit Claude Code, depending on what you want to do:

## Graceful Exit (Recommended)

**Using the slash command:**
```bash
/exit
```
or
```bash
/quit
```

These commands gracefully exit the session and save your conversation state so you can resume later.

**Using keyboard shortcuts:**
- **Ctrl+D** - Standard Unix/Linux way to exit (sends EOF signal)
- **Ctrl+C** - Also exits, but less graceful (immediately terminates)

## Important: Don't Use Ctrl+Z!

**Ctrl+Z** is often mentioned but it **suspends** Claude Code (doesn't exit it). This puts the process in the background, which can be confusing. If you accidentally use Ctrl+Z:

```bash
fg  # Brings Claude Code back to the foreground
```

Or kill the suspended process:
```bash
jobs  # See background jobs
kill %1  # Kill the suspended Claude Code process
```

## Stopping Claude Mid-Response

If Claude is generating a response and you want to **stop it** (but not exit):
- Press **Escape** (ESC) - This stops Claude's current response without exiting the session

## Resuming Previous Sessions

One of the best features - you can resume your previous conversation:

```bash
claude --resume
```

This shows you a list of recent conversations to pick up where you left off, which helps with the fact that conversations aren't automatically persistent like in the web interface.

## Quick Reference

| Action | Command |
|--------|---------|
| Exit gracefully | `/exit` or `/quit` |
| Exit with keyboard | Ctrl+D |
| Stop current response | Escape |
| Suspend (not exit) | Ctrl+Z |
| Resume suspended | `fg` |
| Resume previous session | `claude --resume` |

## Pro Tip

Always use `/exit` or Ctrl+D rather than Ctrl+C when possible - this ensures your session is properly saved and you can resume it later with `claude --resume`. This is especially important if you're in the middle of something complex!
