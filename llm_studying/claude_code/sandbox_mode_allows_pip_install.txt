Good news — Claude Code's built-in sandbox **does allow `pip install`**.

### Allowlisted Domains

The sandbox permits network access to package registries:

| Domain | Purpose |
|--------|---------|
| `pypi.org` | Python Package Index |
| `files.pythonhosted.org` | Package downloads |
| `pythonhosted.org` | Package downloads |
| `github.com` | Git dependencies (if needed) |
| `npmjs.org` / `registry.npmjs.org` | Node packages (if needed) |

So this will work fine inside the sandbox:

```bash
pip install mesa networkx pydantic typer pandas jinja2 duckdb
```

### What *Won't* Work

The sandbox blocks domains not on the allowlist, so:

- No HTTP requests to arbitrary APIs at runtime ✗
- No web scraping ✗
- No downloading files from random URLs ✗

But that's exactly what you want for a standalone project anyway.
