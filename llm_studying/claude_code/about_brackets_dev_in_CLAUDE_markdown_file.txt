Great question! The `[dev]` in `pip install -e ".[dev]"` refers to **optional dependency groups** (also called "extras") defined in `pyproject.toml`.

### How it works

In your `pyproject.toml`, you'd define something like:

```toml
[project]
name = "my_project"
dependencies = [
    "pydantic>=2.0",    # Required for everyone
]

[project.optional-dependencies]
dev = [
    "pytest>=7.0",
    "pytest-cov",
    "black",
    "ruff",
    "mypy",
]
```

### What the commands mean

| Command | What it installs |
|---------|------------------|
| `pip install .` | Only the core runtime dependencies |
| `pip install ".[dev]"` | Core dependencies **plus** the `dev` extras |
| `pip install -e ".[dev]"` | Same as above, but in **editable mode** (changes to source reflect immediately without reinstalling) |

### Why use extras?

- **End users** only need the core dependencies to use your library
- **Contributors/developers** need additional tools (testing, linting, formatting) that shouldn't be forced on regular users

You can define multiple groups too, like `[test]`, `[docs]`, `[lint]`, or even combine them: `pip install -e ".[dev,docs]"`.

The quotes around `".[dev]"` are needed in some shells (like zsh) because brackets have special meaning.
