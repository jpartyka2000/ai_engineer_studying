Great question! There are several ways to view your current statusline configuration:

## 1. See It In Action (Easiest)

The statusline is **always visible at the bottom of your Claude Code terminal** when you're in a session. Just look at the bottom of your terminal window - if it's configured, you'll see information displayed there.

**Start Claude Code to see it:**
```bash
claude
```

Look at the very bottom of the terminal for the status line display.

## 2. Check Your Configuration

View what statusline command is configured:

```bash
cat ~/.claude/settings.json
```

Look for the `statusLine` section:
```json
{
  "statusLine": {
    "type": "command",
    "command": "~/.claude/statusline.sh",
    "padding": 0
  }
}
```

This tells you which script is being used.

## 3. View the Statusline Script

If you have a statusline script configured, view it:

```bash
cat ~/.claude/statusline.sh
```

Or if it's in your project directory:
```bash
cat .claude/statusline.sh
```

## 4. Test Your Statusline Manually

You can test what your statusline would display without starting Claude Code:

```bash
echo '{"model":{"display_name":"Claude Sonnet 4.5"},"workspace":{"current_dir":"/your/project"},"context_window":{"total_input_tokens":50000,"context_window_size":200000}}' | ~/.claude/statusline.sh
```

This sends mock JSON data to your statusline script and shows you what it outputs.

## 5. Check If Statusline Is Even Configured

```bash
# Check if settings.json exists and has statusLine config
if grep -q "statusLine" ~/.claude/settings.json 2>/dev/null; then
  echo "✅ Statusline is configured"
  grep -A 3 "statusLine" ~/.claude/settings.json
else
  echo "❌ No statusline configured"
fi
```

## 6. Find All Statusline-Related Files

```bash
# Search for statusline scripts
find ~/.claude -name "*statusline*" 2>/dev/null
find . -name "*statusline*" -maxdepth 3 2>/dev/null
```

## If You Don't See a Statusline

If you don't see anything at the bottom of your Claude Code session, it means:

1. **No statusline is configured** - You need to run `/statusline` to create one
2. **Script isn't executable** - Run `chmod +x ~/.claude/statusline.sh`
3. **Script has errors** - Test it manually (see step 4 above)
4. **Wrong path** - Check that the path in settings.json matches where the script is

## Quick Status Check

Here's a one-liner to check everything:

```bash
echo "=== Settings ===" && \
grep -A 3 "statusLine" ~/.claude/settings.json 2>/dev/null && \
echo -e "\n=== Script Location ===" && \
ls -lh ~/.claude/statusline.sh 2>/dev/null || echo "No script found" && \
echo -e "\n=== Script Preview ===" && \
head -10 ~/.claude/statusline.sh 2>/dev/null || echo "No script to preview"
```

## Reconfigure Statusline

If you want to change or create a new statusline, just run:

```bash
/statusline
```

Or with specific instructions:
```bash
/statusline show the model name, context percentage, and git branch
```

This will regenerate your statusline configuration!
