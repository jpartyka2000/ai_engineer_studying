That's a great question! Unfortunately, there isn't currently a way to customize voice-to-text transcription rules in Claude's voice mode—like having "dot" automatically convert to `.` when you're dictating code.

The voice transcription happens before your message reaches me, so I receive whatever the speech-to-text system produces.

A few workarounds you might try:

**In voice mode:** You could say the code more literally, like "F period lit parentheses quote someval quote" and see how well it transcribes, though results can be inconsistent.

**Hybrid approach:** For code-heavy conversations, you might type the code snippets and use voice for the explanatory parts of your message.

**Post-correction:** You could dictate naturally with "dot" and then ask me to interpret your intent—I can usually figure out that "F dot lit" means `F.lit()`.

If this is something you'd find valuable, I'd encourage you to share that feedback with Anthropic—custom voice transcription rules for technical use cases like coding would be a useful feature for many people.
