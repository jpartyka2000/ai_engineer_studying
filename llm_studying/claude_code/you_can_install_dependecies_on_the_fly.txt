Yes, Claude Code can install dependencies for you. It has terminal access and can run shell commands including `pip install`.

### How It Works

When Claude Code sees that a dependency is needed (either from CLAUDE.md, pyproject.toml, or when it encounters an import error), it can run:

```bash
pip install duckdb
```

Or if you have a `pyproject.toml` set up:

```bash
pip install -e ".[dev]"
```

### Permission Model

By default, Claude Code will ask for confirmation before running commands. You'll see something like:

```
Claude wants to run: pip install duckdb
Allow? [y/n/always]
```

You can configure auto-accept for certain commands if you trust them.

### Best Practice: Virtual Environment

To keep your system Python clean, specify in your CLAUDE.md that Claude Code should use a virtual environment:

```markdown
## Environment Setup

Before installing dependencies, ensure a virtual environment is active:

```bash
python -m venv .venv
source .venv/bin/activate  # or .venv\Scripts\activate on Windows
pip install -e ".[dev]"
```

If the virtual environment doesn't exist, create it first.
```

This way, Claude Code will set up an isolated environment rather than installing packages globally.

### What Claude Code *Can't* Do

- Install system-level packages requiring `sudo` (unless you explicitly allow it)
- Install things that require interactive prompts it can't handle
- Access networks if you've sandboxed it without network access
