You have several good options here:

### Option 1: Put More in CLAUDE.md

CLAUDE.md isn't limited to technical layout — you can include functional requirements, expected behaviors, and acceptance criteria. Claude Code reads this file automatically at the start of each session.

```markdown
## Functional Requirements

### Feature: Data Processor
- Must accept CSV or JSON input from local filesystem
- Must validate all records against schema before processing
- Invalid records should be collected and reported, not silently dropped
- Output must be sorted by [field] in ascending order

### Acceptance Criteria
- [ ] Process 10,000 records in under 5 seconds
- [ ] Memory usage stays under 500MB for typical workloads
```

### Option 2: Create a SPEC.md or REQUIREMENTS.md

Keep CLAUDE.md for technical standards and create a separate file for your detailed requirements:

```
project/
├── CLAUDE.md          # How to build (standards, structure)
├── SPEC.md            # What to build (requirements, behaviors)
└── ...
```

Then in CLAUDE.md, add:
```markdown
## Important Files
- Read `SPEC.md` before implementing any features
```

### Option 3: Save Your Prompt as a File

Save your detailed prompt in a file like `PROMPT.txt` or `task.md`, then your actual prompt becomes short:

```
Read task.md and implement what it describes.
```

### Option 4: Use Claude Projects (in this interface)

If you're using claude.ai, you can create a **Project** with custom instructions that persist across all conversations in that project. Your large prompt goes in the project instructions once, and every new chat inherits it.
