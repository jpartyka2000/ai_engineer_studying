Based on the search results, Claude Code does have the ability to accept input while working, but the behavior is somewhat inconsistent:

**The intended behavior:** A few versions back there was a new feature added where you could prompt while Claude Code was busy performing a step and it would steer it away to do something else instead (or do something in addition as requested).

**The current reality:** This no longer seems to be working, instead whatever I type (and it acknowledges) while it's busy doing something will soon get 'lost' in the conversation and when it's done, it hasn't taken those instructions into account. It won't even queue these anymore.

So while the feature exists in principle, there are reported bugs where mid-task prompts get lost rather than being incorporated.

**Your options:**

1. **Press Escape to interrupt** — Press Escape to interrupt Claude during any phase (thinking, tool calls, file edits), preserving context so you can redirect or expand instructions. Then provide your new instruction.

2. **Use background tasks** — When Claude Code runs a command in the background, it runs the command asynchronously and immediately returns a background task ID. Claude Code can respond to new prompts while the command continues executing in the background.

For reliable behavior, interrupting with Escape and then providing your updated instructions is the safest approach right now.
