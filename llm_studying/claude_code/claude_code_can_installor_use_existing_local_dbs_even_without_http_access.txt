Yes, with some caveats depending on the database type:

### Embedded/File-Based DBs (No Installation Needed)

These work great for standalone projects and require no network access:

| Database | Install | Notes |
|----------|---------|-------|
| **SQLite** | Included with Python | `import sqlite3` — nothing to install |
| **DuckDB** | `pip install duckdb` | Great for analytics, SQL-based, single file |
| **TinyDB** | `pip install tinydb` | Pure Python, JSON-based, simple |
| **LMDB** | `pip install lmdb` | Fast key-value store |

These are just files on disk — fully compatible with your "standalone, no network" requirement.

### Already-Installed DBs (PostgreSQL, MySQL, etc.)

This depends on your sandbox configuration:

- **Filesystem access needed** → for SQLite/DuckDB-style file DBs
- **Unix socket access needed** → for local PostgreSQL/MySQL via socket
- **Localhost network access needed** → for DBs listening on `127.0.0.1`

If your sandbox blocks localhost network connections, client-server databases won't work even if installed.

### Recommendation for Your Project

Since you want standalone with no external dependencies:

```python
# SQLite - zero setup, included in Python
import sqlite3
conn = sqlite3.connect("mydata.db")
```

or

```python
# DuckDB - if you need analytics/columnar performance
import duckdb
conn = duckdb.connect("mydata.duckdb")
```

Both produce a single file that can be shipped with your project or created at runtime.
