Great question! Based on recent Claude Code updates, **you generally do NOT need to restart** for most settings changes.

## Live Reload Feature (as of v1.0.90)

As of Claude Code version 1.0.90, settings changes take effect **immediately** without requiring a restart. Changes to your `settings.json` files are automatically detected and applied to your current session.

## Settings File Locations

Just to clarify where settings files should go:

**Project-specific settings:**
```
your-project/
└── .claude/
    ├── settings.json          # Shared with team (checked into git)
    └── settings.local.json    # Personal settings (gitignored)
```

**Global settings (apply to all projects):**
```
~/.claude/settings.json
```

So if you created `.claude/settings.json` in your project directory on your Desktop, that's correct for project-specific settings!

## Exceptions That May Still Require Restart

While most settings reload immediately, there are a few cases where you might still need to restart:

1. **Hooks configuration** - May require restart for security reasons
2. **MCP server changes** - Adding new MCP servers typically requires restart or using `/exit` and restarting
3. **Some environment variables** - Depending on how they're used

## How to Restart if Needed

If you do need to restart:
```bash
/exit   # Exit Claude Code gracefully
claude  # Start it again
```

## Test Your Changes

You can verify your settings are active with:
```bash
/status  # Shows current configuration
```

For most configuration changes (permissions, status line, model settings, etc.), you should see them take effect immediately in your current session without needing to restart!
