To create a custom slash command in Claude Code, create a Markdown file in one of two locations:

**Project-specific commands:**
```bash
.claude/commands/your-command.md
```

**Personal commands (available across all projects):**
```bash
~/.claude/commands/your-command.md
```

The filename becomes the command name. So `optimize.md` becomes `/optimize`.

---

**Simple example:**

```bash
mkdir -p .claude/commands
```

Create `.claude/commands/review.md`:
```markdown
Review this code for:
- Security vulnerabilities
- Performance issues
- Code style violations
```

Now you can run `/review` in Claude Code.

---

**With arguments:**

Create `.claude/commands/fix-issue.md`:
```markdown
---
description: Fix a GitHub issue
argument-hint: [issue-number] [priority]
---
Fix issue #$1 with priority $2.
```

Use: `/fix-issue 123 high`

- `$1`, `$2` = positional arguments
- `$ARGUMENTS` = all arguments as a string

---

**With frontmatter options:**

```markdown
---
description: Security vulnerability scan
allowed-tools: Read, Grep, Glob
model: claude-sonnet-4-5-20250929
---
Analyze the codebase for security vulnerabilities.
```

---

**Subdirectories for namespacing:**

```
.claude/commands/
├── review.md         → /review
├── deploy/
│   ├── staging.md    → /deploy:staging
│   └── prod.md       → /deploy:prod
```
