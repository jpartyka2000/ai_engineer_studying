Claude Code uses a **hierarchy of settings files**, so you have options depending on your goal:

## Settings File Locations (in order of precedence)

| Location | Scope | Use Case |
|----------|-------|----------|
| `.claude/settings.json` | Project (shared) | Checked into git, shared with team |
| `.claude/settings.local.json` | Project (personal) | Git-ignored, your personal project overrides |
| `~/.claude/settings.json` | Global (user) | Applies to all your projects |

## For Your Sandboxed Project

**Yes, put it in your project directory** — specifically at `.claude/settings.json` (note the dot prefix on the folder). This keeps your restrictive settings scoped to that project only.

```bash
# From your project folder:
mkdir -p .claude
nano .claude/settings.json   # or use your preferred editor
```

Then add your deny rules:

```json
{
  "permissions": {
    "deny": [
      "Bash(rm:*)",
      "Bash(sudo:*)",
      "Read(~/.ssh/**)",
      "Read(~/.aws/**)",
      "Read(**/.env)"
    ]
  }
}
```

## Alternatively: Global Settings

If you want these restrictions to apply to *all* your Claude Code projects (safer default), put them in:

```
~/.claude/settings.json
```

This way you don't have to remember to set it up for each new project.

## Quick Tip

You can also manage permissions interactively within Claude Code by running:

```
/permissions
```

This opens a UI to configure what Claude can and cannot do, and it will write the appropriate settings file for you.
