### Your Workflow

**First time / major features:**
```
implement the data processor described in docs/spec.md
```

**Iterating on issues:**
```
the validation step isn't catching empty strings - fix it per the spec
```

**Refining the spec itself:**
When Claude's output isn't what you want, update `docs/spec.md` with clarifications rather than re-explaining in chat. This way your requirements document gets better over time.

### Pro Tip: Be Specific in spec.md

Instead of:
> "validate the input data"

Write:
> "validate input data by checking: (1) required fields are present, (2) email fields match regex X, (3) dates are ISO 8601 format. Return a ValidationResult with .is_valid and .errors list"

The more concrete your spec, the less back-and-forth you'll need.
