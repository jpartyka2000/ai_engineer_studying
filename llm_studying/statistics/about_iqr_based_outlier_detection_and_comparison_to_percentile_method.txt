# Understanding IQR-Based Outlier Detection

## What is IQR?

**IQR (Interquartile Range)** is a measure of statistical dispersion - it tells you how spread out the middle 50% of your data is.

```
IQR = Q3 - Q1
```

Where:
- **Q1** (First Quartile / 25th percentile): 25% of data falls below this value
- **Q3** (Third Quartile / 75th percentile): 75% of data falls below this value

## The Outlier Detection Formula

To identify outliers, we calculate bounds using this formula:

```
Lower Bound = Q1 - (multiplier × IQR)
Upper Bound = Q3 + (multiplier × IQR)
```

Any value **below the lower bound** or **above the upper bound** is considered an outlier.

## Why This Works

The IQR method is **resistant to outliers** in the calculation itself because Q1 and Q3 are based on the middle of your data distribution, not the extremes.

**Example Dataset:** `[0, 0, 0, 0, 5, 10, 15, 20, 25, 30, 35, 40, 50, 60, 10000]`

```
Q1 = 5     (25th percentile)
Q3 = 40    (75th percentile)
IQR = 40 - 5 = 35
```

With multiplier = 1.5:
```
Lower Bound = 5 - (1.5 × 35) = 5 - 52.5 = -47.5
Upper Bound = 40 + (1.5 × 35) = 40 + 52.5 = 92.5
```

**Result:**
- The zeros (0) are **NOT outliers** (they're above -47.5) ✅
- The value 10000 **IS an outlier** (it's above 92.5) ✅

## Common Multiplier Values

| Multiplier | Use Case | How Strict |
|------------|----------|------------|
| **1.5** | Standard outlier detection (default) | Moderate - flags typical outliers |
| **2.0** | Relaxed detection | Less strict - only more extreme values |
| **3.0** | Extreme outliers only | Very conservative - only the most extreme values |

### Visual Comparison

With data ranging 0-100 plus some values at 10,000:

```
Multiplier = 1.5:  Upper bound ≈ 150   → 10,000 flagged as outlier
Multiplier = 2.0:  Upper bound ≈ 200   → 10,000 flagged as outlier  
Multiplier = 3.0:  Upper bound ≈ 300   → 10,000 flagged as outlier
```

## Why IQR is Better Than Percentile Methods

**Percentile method:** "Flag bottom 1% and top 1%"
- Problem: If you have 1000 zeros, some get flagged as outliers just because they're in the bottom 1%

**IQR method:** "Flag values that are statistically distant from the central bulk"
- Solution: Frequent values (like 1000 zeros) won't be flagged because they're part of the central distribution

## Key Insight

The IQR method identifies outliers based on **distance from the central tendency**, not **position in the sorted list**. This means:

✅ Frequent extreme values (like many 0s) → NOT outliers  
✅ Rare extreme values (like a few 10,000s) → outliers

Perfect for real-world data where extreme but common values shouldn't be treated as anomalies!
