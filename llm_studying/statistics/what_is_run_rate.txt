## Run Rate in Business/Analytics

**Run rate** is an **annualized projection** based on current performance - essentially "if things continue at this pace, here's what the full year looks like."

## Basic Formula:

```python
# Take a shorter period and extrapolate to a year
Run_Rate = Current_Period_Performance × (12/Number_of_Months)

# Examples:
Q1_revenue = $1M
Annual_run_rate = $1M × 4 = $4M

January_sales = $100K  
Annual_run_rate = $100K × 12 = $1.2M
```

## Common Examples:

### Revenue Run Rate
```python
# Company's current month revenue
march_revenue = $500,000
annual_run_rate = $500,000 × 12 = $6,000,000

# Or using quarterly data
q2_revenue = $1,500,000
annual_run_rate = $1,500,000 × 4 = $6,000,000
```

### For SaaS/Subscription Businesses (ARR)
```python
# Monthly Recurring Revenue (MRR) to Annual Run Rate (ARR)
current_MRR = $50,000
ARR = $50,000 × 12 = $600,000

# This is how startups often report growth:
"We're at a $600K run rate" (meaning $50K/month currently)
```

### Cost/Expense Run Rate
```python
# Cloud costs this week
weekly_aws_spend = $10,000
annual_run_rate = $10,000 × 52 = $520,000
# "At this rate, we'll spend $520K on AWS this year"
```

## Why Use Run Rate?

1. **Quick estimates** for planning/budgeting
2. **Standardized comparison** (everything in annual terms)
3. **Early-stage metrics** (when you don't have full year data)

## Important Limitations:

```python
# Run rate assumes current pace continues - often wrong!

# Seasonal business example:
december_sales = $1M  # Holiday peak
annual_run_rate = $12M  # Wrong! Other months much lower

# Growth company example:
month_1: $10K
month_2: $20K
month_3: $40K
run_rate_from_month_3 = $40K × 12 = $480K
# But they're doubling monthly - will be much higher!
```

## In Financial Reporting:

```python
# Common phrases you'll see:
"Q3 revenue of $25M represents a $100M run rate"
"We've reached a $1B run rate" (current quarter × 4 = $1B)
"Burn rate of $500K/month" (spending run rate = $6M/year)
```

**Bottom line**: Run rate is a simple way to **annualize** any metric by taking current performance and projecting it forward for a full year - useful for quick estimates but assumes steady state.
