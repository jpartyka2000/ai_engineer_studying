The **coefficient of variation (CV)** is a standardized measure of dispersion, defined as the ratio of the standard deviation to the mean:

$$CV = \frac{\sigma}{\mu}$$

It's often expressed as a percentage. Unlike standard deviation alone, CV is dimensionless, which makes it useful for comparing variability across features with different units or scales.

## How it helps in ML feature selection

**Low CV features may be uninformative.** If a feature has a very low CV (close to zero), it means nearly all values are clustered around the mean—there's little variation. A feature with almost no variation can't help a model distinguish between different outcomes. For example, if 99% of your data has the same value for a feature, that feature carries almost no predictive signal.

**Practical use: variance thresholding.** A common preprocessing step is to remove features below a certain CV threshold. This is a quick, unsupervised way to prune features that are unlikely to be useful before applying more sophisticated selection methods.

**Caveats to keep in mind:**

1. **High CV doesn't guarantee usefulness.** A feature can vary a lot but still be uncorrelated with your target variable. CV only measures spread, not predictive power.

2. **Sensitive to mean values near zero.** If the mean is close to zero, CV can explode or become meaningless. It's best suited for ratio-scale data with positive values.

3. **Not a substitute for proper feature importance.** Methods like mutual information, correlation with the target, or model-based importance (e.g., from tree ensembles) are more direct measures of a feature's predictive value.

So CV is best thought of as a quick filter for eliminating obviously useless features (those with near-zero variance), rather than a definitive measure of feature quality.
