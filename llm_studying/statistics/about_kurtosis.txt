## Kurtosis

**Kurtosis** measures the "tailedness" of a distribution—how much of the data lies in the extreme tails versus the center. It's the fourth standardized moment:

$$\text{Kurtosis} = \frac{E[(X - \mu)^4]}{\sigma^4}$$

You'll often see **excess kurtosis**, which subtracts 3 so that a normal distribution has a value of zero:

$$\text{Excess Kurtosis} = \frac{E[(X - \mu)^4]}{\sigma^4} - 3$$

## Interpreting values

| Type | Excess Kurtosis | Characteristics |
|------|-----------------|-----------------|
| **Mesokurtic** | ≈ 0 | Normal distribution; moderate tails |
| **Leptokurtic** | > 0 | Heavy tails, sharp peak; more outliers than normal |
| **Platykurtic** | < 0 | Light tails, flat peak; fewer outliers than normal |

A common misconception is that kurtosis measures "peakedness." It's really about the tails. A leptokurtic distribution has more probability mass in the extreme tails, meaning outliers are more likely.

## How kurtosis helps in feature evaluation

**1. Outlier detection signal**

High kurtosis indicates a feature has heavy tails with extreme values. This is useful to know because:
- It flags features that may need outlier treatment or robust scaling
- Models sensitive to outliers (linear regression, k-means, SVMs) may struggle with high-kurtosis features

**2. Distribution assumptions**

Many statistical methods assume normality. A feature with very high or very low kurtosis violates this assumption, which can affect:
- Confidence intervals and hypothesis tests during exploratory analysis
- Algorithms that implicitly assume Gaussian distributions

**3. Identifying unusual features**

Extremely high kurtosis might indicate data quality issues—perhaps the feature is mostly constant with a few erratic spikes (sensor errors, data entry mistakes, etc.). This warrants investigation before including the feature in a model.

**4. Feature transformation decisions**

Knowing a feature is leptokurtic can guide preprocessing choices. You might apply:
- Log or power transforms to compress tails
- Winsorization to cap extreme values
- Quantile transforms to force a more uniform or Gaussian shape

## Example

Consider two features with identical mean, variance, and even skewness:

- **Feature A:** Excess kurtosis = 0 (normal-like tails)
- **Feature B:** Excess kurtosis = 10 (very heavy tails)

Feature B will have far more extreme outliers. If you're using linear regression, those outliers could disproportionately influence the fitted coefficients. Knowing this upfront lets you decide whether to transform, clip, or use a more robust model.

## Caveat

Like CV and skewness, kurtosis describes the shape of a feature's distribution—not its relationship to the target. A feature with perfect kurtosis can still be useless for prediction if it's uncorrelated with what you're trying to predict. It's a diagnostic tool for understanding your data, not a direct measure of predictive power.
