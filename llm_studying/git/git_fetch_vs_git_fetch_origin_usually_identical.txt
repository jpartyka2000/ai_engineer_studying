Yes, in most cases they're the same.

`git fetch` without arguments fetches from the default remote, which is typically `origin`. If you only have one remote (which is the common case), there's no difference.

**When they might differ:**

If you have multiple remotes configured (e.g., `origin` and `upstream`), then:

- `git fetch` — fetches from the default remote (usually `origin`, but can be configured)
- `git fetch origin` — explicitly fetches from `origin`
- `git fetch upstream` — fetches from `upstream`

**To see your remotes:**

```bash
git remote -v
```

If you only see `origin` listed, then `git fetch` and `git fetch origin` are identical.
