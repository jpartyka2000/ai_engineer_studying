**Remote branches** and **remote-tracking branches** are two different things:

**Remote branches:**
- Live on the remote server (e.g., GitHub, GitLab)
- You can't see or interact with them directly
- Updated when someone pushes to the remote
- Example: the `main` branch on GitHub

**Remote-tracking branches:**
- Live in your local repository
- Named like `origin/main`, `origin/feature-branch`
- A local snapshot of where the remote branch was the last time you fetched
- Read-only — you can't commit to them directly
- Updated when you run `git fetch` or `git pull`

**How they relate:**

```
Remote (GitHub)          Your local repo
---------------          ---------------
main            -->      origin/main        (remote-tracking branch)
                         main               (local branch)
```

When you run:
- `git fetch` — updates your remote-tracking branches to match the remote
- `git pull` — fetches and merges the remote-tracking branch into your local branch
- `git push` — updates the remote branch and your remote-tracking branch

**Practical example:**

```bash
# Your remote-tracking branch
git log origin/main

# Your local branch
git log main

# See if they've diverged
git log main..origin/main --oneline   # commits on remote you don't have
git log origin/main..main --oneline   # commits you have that remote doesn't
```

**Key point:** Your remote-tracking branches can be out of date. If a teammate pushed new commits, you won't see them in `origin/main` until you run `git fetch`. This is why `git fetch --all` is important before checking on someone else's work.
