To view all files in the Git index (staging area):

**Basic list of staged files:**
```bash
git ls-files
```

**With more detail (mode, hash, stage number):**
```bash
git ls-files --stage
```

Output looks like:
```
100644 4c1a9e3b... 0    README.md
100644 7d2f8c1a... 0    main.py
100644 2b9e4a5f... 0    src/utils.py
```

**Only show modified files in the index:**
```bash
git ls-files --modified
```

**Only show untracked files:**
```bash
git ls-files --others
```

**Show deleted files:**
```bash
git ls-files --deleted
```

**Show cached (staged) files only:**
```bash
git ls-files --cached
```

**Combine flags for specific views:**
```bash
# Untracked, excluding ignored files
git ls-files --others --exclude-standard
```

**What the stage number means (in `--stage` output):**
- `0` — normal staged file
- `1`, `2`, `3` — conflict markers during a merge (base, ours, theirs)
