Yes, you need to resolve all conflicts before running `git rebase --continue`.

If you try to continue with unresolved conflicts, Git will tell you:

```
error: Committing is not possible because you have unresolved merge conflicts.
```

So open `cue_eval/runner.py`, resolve the conflict there, stage both files, then continue:

```bash
git add justfile cue_eval/runner.py
git rebase --continue
```
