**Interactive rebase is real, not a dry run.**

When you run `git rebase -i origin/main`, the interactive part just lets you see and optionally edit the list of commits *before* the rebase begins. Once you save and close the editor, Git proceeds with the actual rebase—your commits are really being replayed and your branch is really being rewritten.

The "interactive" aspect gives you the opportunity to:

- Reorder commits
- Squash commits together
- Edit commit messages
- Drop commits entirely
- Pause at specific commits to amend them

But it doesn't make the operation reversible or hypothetical. Once you close the editor, it's happening for real.

**If you want an actual dry run**, there's no built-in option, but you can simulate one by creating a temporary branch first:

```bash
git checkout -b test-rebase your-branch
git rebase -i origin/main
```

If something goes wrong or you don't like the result, you can just delete `test-rebase` and your original branch is untouched.

**Aborting mid-rebase**: If you're in the middle of a rebase and want to back out, you can always run:

```bash
git rebase --abort
```

This restores your branch to its state before the rebase started.
