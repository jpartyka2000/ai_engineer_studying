Yes, you can do this, though not directly in the GitHub web interface. Here's how:

**Locally (the standard approach):**

```bash
git fetch origin
git checkout your-branch
git rebase origin/main
```

During the rebase, Git will stop at each commit that causes a conflict, letting you resolve it before continuing with `git rebase --continue`. This gives you exactly what you want—seeing which specific commit introduced the conflict.

**Using GitHub's UI:**

GitHub's web interface only offers merge commits or squash merges, not interactive rebasing. You'll need to do the rebase locally and then force-push:

```bash
git push --force-with-lease origin your-branch
```

**Tip for reviewing conflicts:**

If you want to be extra careful, you can do a dry run first:

```bash
git rebase origin/main --dry-run  # doesn't exist, but you can simulate with:
git rebase origin/main --interactive
```

The interactive rebase (`-i`) shows you all the commits that will be replayed, and you can review the list before proceeding.
