"Blob" stands for "binary large object," but in Git it refers to **any file content**, not just large binary files. Every file you commit — whether it's a tiny `.txt`, a Python script, or a large image — gets stored as a blob.

A blob is just the raw content of a file, identified by its SHA-1 hash. Git doesn't store filenames or directory structure in blobs — that information lives in "tree" objects, which map names to blobs.

So if you have two files with identical content in different directories, Git stores only one blob and both trees point to it.

The confusion with "large binary" probably comes from the name and from the fact that Git handles large binary files (images, videos, compiled assets) inefficiently — they don't diff well and bloat the repo. That's why tools like **Git LFS** (Large File Storage) exist to handle actual large binaries separately.
