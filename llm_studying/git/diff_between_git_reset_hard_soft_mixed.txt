------------------------------------------------------------
QUESTION:
git reset has three modes: --soft, --mixed, and --hard.

If you run `git reset --soft HEAD~1`, what happens?

A) Last commit removed, changes go to staging area, 
   working directory unchanged
B) Last commit removed, changes go to working directory 
   (unstaged), staging area cleared
C) Last commit removed, all changes permanently deleted
D) Last commit message changed but files remain committed
------------------------------------------------------------
YOUR ANSWER: A
CORRECT ANSWER: A
STATUS: ✅ Correct
------------------------------------------------------------
EXPLANATION:
git reset --soft HEAD~1:
- Moves HEAD back one commit
- Changes from that commit go to STAGING AREA
- Working directory is UNTOUCHED
- Perfect for "uncommit but keep everything staged"

THE THREE RESET MODES VISUALIZED:
------------------------------------------------------------
Before reset (committed "Add feature"):

  [Working Dir]  [Staging Area]  [Repository]
       ✓              ✓           abc123 "Add feature"
     (clean)       (clean)            ↑ HEAD

After git reset --soft HEAD~1:

  [Working Dir]  [Staging Area]  [Repository]
       ✓              ✓           def456 (previous)
    (unchanged)   (has changes)       ↑ HEAD

After git reset --mixed HEAD~1 (default):

  [Working Dir]  [Staging Area]  [Repository]
       ✓              ✓           def456 (previous)
   (has changes)   (empty)            ↑ HEAD

After git reset --hard HEAD~1:

  [Working Dir]  [Staging Area]  [Repository]
       ✓              ✓           def456 (previous)
     (clean)       (clean)            ↑ HEAD
                                 ⚠️ Changes GONE!

QUICK COMPARISON:
------------------------------------------------------------
┌─────────────┬───────────────┬─────────────┬─────────────┐
│ Mode        │ Commit        │ Staging     │ Working Dir │
├─────────────┼───────────────┼─────────────┼─────────────┤
│ --soft      │ ✗ Removed     │ ✓ Preserved │ ✓ Preserved │
│ --mixed     │ ✗ Removed     │ ✗ Cleared   │ ✓ Preserved │
│ --hard      │ ✗ Removed     │ ✗ Cleared   │ ✗ DELETED   │
└─────────────┴───────────────┴─────────────┴─────────────┘

WHEN TO USE EACH:
------------------------------------------------------------
--soft:  "I want to uncommit but keep changes staged"
         (great for re-doing a commit differently)

--mixed: "I want to uncommit and unstage, but keep files"
         (default mode — start fresh with changes)

--hard:  "I want to completely undo and discard everything"
         ⚠️ Destructive! Changes are lost!
