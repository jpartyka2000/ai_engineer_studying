No — there's only one local branch. Setting up tracking doesn't create a new branch.

---

**Clarifying the terminology:**

When you "set a local branch to track a remote," you're just adding configuration to your existing local branch. The branch itself doesn't change or duplicate.

Before tracking:
```
refs/heads/feature-x  →  (your local branch, no upstream configured)
```

After setting tracking:
```
refs/heads/feature-x  →  (same local branch, now with upstream configured)
```

---

**What actually gets created:**

When you push and set up tracking, you end up with two related but separate refs:

| Ref | Type | Purpose |
|-----|------|---------|
| `refs/heads/feature-x` | Local branch | Where you commit your work |
| `refs/remotes/origin/feature-x` | Remote-tracking branch | Local cache of the remote's state |

These are different things:
- Your **local branch** is where you make commits
- The **remote-tracking branch** is a read-only snapshot that updates when you `git fetch`

---

**The "tracking" part is just configuration:**

Setting up tracking simply adds two config entries to your existing local branch:

```ini
[branch "feature-x"]
    remote = origin
    merge = refs/heads/feature-x
```

This tells Git which remote branch to pull from and push to by default — it doesn't create any new branches.
