**Git config has three scope levels:**

| Scope | Flag | Applies to | Config file location |
|-------|------|------------|---------------------|
| System | `--system` | All users on the machine | `/etc/gitconfig` |
| Global | `--global` | All repos for your user | `~/.gitconfig` or `~/.config/git/config` |
| Local | `--local` | Current repo only | `.git/config` in the repo |

---

**Precedence (most specific wins):**

```
local > global > system
```

A local setting overrides global, which overrides system.

---

**Examples:**

```bash
# Set your name globally (all your repos)
git config --global user.name "Alice"

# Override it for just one repo
git config --local user.name "Alice (Work)"

# Set something system-wide (requires admin/root)
sudo git config --system core.autocrlf input
```

---

**See where a value comes from:**

```bash
git config --show-origin --get user.name
```

Output might look like:

```
file:/home/alice/.gitconfig    Alice
```

---

**List settings at each level:**

```bash
git config --system --list
git config --global --list
git config --local --list
```

---

**Typical usage:**

- **System**: Rarely used; for organization-wide defaults on shared machines
- **Global**: Your personal defaults (name, email, editor, aliases)
- **Local**: Project-specific overrides (different email for work vs personal repos)
