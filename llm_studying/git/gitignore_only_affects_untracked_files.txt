This is a classic gotcha! 

**.gitignore only affects UNTRACKED files.** These files were already being tracked by Git before you added the rule.

---

## The Fix

```bash
# Remove from Git's tracking (keeps files locally)
git rm -r --cached **/__pycache__/

# Commit the removal
git commit -m "Stop tracking __pycache__ files"
```

Now `.gitignore` will prevent them from being tracked again.

---

## What Happened

| Timeline | What Git Does |
|----------|---------------|
| Files committed before .gitignore existed | Git tracks them |
| You add `*.pyc` to .gitignore | Git ignores NEW .pyc files |
| Already-tracked .pyc files | Still tracked! |
| After `git rm --cached` | Git stops tracking, .gitignore takes over |

---

## Better .gitignore for Python

While you're at it, add the whole `__pycache__` directory:

```
# Python
__pycache__/
*.pyc
*.pyo
*.pyd
.Python
```
