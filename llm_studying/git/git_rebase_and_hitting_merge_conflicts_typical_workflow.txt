Yes, absolutely. That's the normal workflow.

When Git hits a conflict during a rebase, it pauses and lets you resolve it. Then you continue:

```bash
# 1. Git stops at a conflict - resolve it in your editor or IDE

# 2. Stage the resolved files
git add <resolved-files>

# 3. Continue the rebase
git rebase --continue
```

Git will then move on to replaying the next commit. If another conflict occurs, the process repeats—resolve, stage, continue—until all your commits have been replayed.

**Other options at a conflict:**

- `git rebase --skip` — Skip the current commit entirely (use with caution, you'll lose that commit's changes)
- `git rebase --abort` — Cancel the whole rebase and restore your branch to its original state
