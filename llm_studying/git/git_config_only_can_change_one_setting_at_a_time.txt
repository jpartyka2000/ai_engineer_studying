Category: Configuration & Setup
Type: Coding / Command
------------------------------------------------------------
QUESTION:
You've just installed Git on a new computer. Before making 
any commits, you need to configure your identity.

Write the two commands to set your name and email globally.
(Use "Jane Doe" and "jane@example.com")
------------------------------------------------------------
YOUR ANSWER: 
git config --global author.email "jane@example.com" author.name "Jane Doe"

CORRECT ANSWER: 
git config --global user.name "Jane Doe"
git config --global user.email "jane@example.com"

STATUS: ❌ Incorrect
------------------------------------------------------------
EXPLANATION:
Two issues with your command:

1. WRONG PROPERTY NAME
   You used: author.name / author.email
   Correct:  user.name / user.email
   
   Git uses "user" for configuration. "author" appears in 
   commit metadata but isn't a config setting.

2. ONE SETTING PER COMMAND
   git config sets one key-value pair at a time. You cannot 
   combine multiple settings in a single command.

Correct syntax:
   git config --global user.name "Jane Doe"
   git config --global user.email "jane@example.com"

Configuration levels:
   --global    All repos for current user (~/.gitconfig)
   --local     Current repo only (.git/config) [default]
   --system    All users on the machine (/etc/gitconfig)

Useful config commands:
   git config --list              # View all settings
   git config user.name           # View specific setting
   git config --global --edit     # Open config in editor

Other common global settings:
   git config --global init.defaultBranch main
   git config --global core.editor "code --wait"
   git config --global pull.rebase true
