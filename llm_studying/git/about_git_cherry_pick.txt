QUESTION:
You're working on the `develop` branch. A critical bug fix 
was made in a single commit (b7c8d9e) on the `hotfix` 
branch. You need to apply ONLY that specific commit to your 
current `develop` branch without merging the entire 
`hotfix` branch.

Write the command to accomplish this.
------------------------------------------------------------
YOUR ANSWER: git cherry-pick b7c8d9e
CORRECT ANSWER: git cherry-pick b7c8d9e
STATUS: ✅ Correct
------------------------------------------------------------
EXPLANATION:
git cherry-pick takes a specific commit from anywhere in 
your repository and applies it to your current branch as a 
new commit.

This is perfect for situations like:
- Applying hotfixes across multiple branches
- Pulling in a specific feature without a full merge
- Rescuing commits from abandoned branches

Useful variations:
- git cherry-pick -n b7c8d9e
  Apply changes but don't auto-commit (lets you modify)

- git cherry-pick A..B
  Cherry-pick a range of commits (exclusive of A)

- git cherry-pick -x b7c8d9e
  Appends "cherry picked from commit..." to message 
  (good for traceability)

⚠️  Watch out: Cherry-picking creates a NEW commit with a 
different SHA. This can cause confusion if the original 
commit is later merged in — Git sees them as different 
commits even though the content is identical.
