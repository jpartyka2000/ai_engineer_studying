GIT TUTOR — QUESTION RECORD
============================================================
Skill Level: Easy
Category: Viewing History & Blame
Type: Coding / Command
------------------------------------------------------------
QUESTION:
You're reviewing `auth.js` and find a suspicious line of 
code. You want to find out WHO wrote that specific line 
and in WHICH commit it was introduced.

Write the command to see the author and commit information 
for each line in auth.js.
------------------------------------------------------------
YOUR ANSWER: git log ?
CORRECT ANSWER: git blame auth.js
STATUS: ❌ Incorrect
------------------------------------------------------------
EXPLANATION:
git blame shows line-by-line annotation of a file:
- Who last modified each line
- Which commit introduced it
- When it was changed

git log shows commit history for the whole repo or file, 
but doesn't break it down by individual lines.

SAMPLE OUTPUT OF git blame auth.js:
------------------------------------------------------------
^a1b2c3d (Alice   2024-06-01 10:00:00 +0000  1) const express = require('express');
f4e5d6c7 (Bob     2024-07-15 14:30:00 +0000  2) const jwt = require('jsonwebtoken');
f4e5d6c7 (Bob     2024-07-15 14:30:00 +0000  3) 
9h8i7j6k (Charlie 2024-09-20 09:15:00 +0000  4) function validateToken(token) {
9h8i7j6k (Charlie 2024-09-20 09:15:00 +0000  5)     return jwt.verify(token, SECRET);
...

Now you know Charlie wrote that validateToken function!

USEFUL git blame VARIATIONS:
------------------------------------------------------------
git blame auth.js
  Basic blame for entire file

git blame -L 10,20 auth.js
  Blame only lines 10-20

git blame -L 15,+5 auth.js
  Blame 5 lines starting at line 15

git blame -w auth.js
  Ignore whitespace changes

git blame -C auth.js
  Detect lines moved/copied from other files

git blame --since="2024-01-01" auth.js
  Only show changes after a date

git blame <commit> -- auth.js
  Blame as of a specific commit

WHY "BLAME"?
------------------------------------------------------------
The name sounds accusatory, but it's really for 
understanding code history. Some tools call it "annotate" 
instead (git annotate is an alias).

Related: git log -p auth.js shows full commit diffs for 
the file, which is complementary to blame.
