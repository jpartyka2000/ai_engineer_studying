## Previewing Commits Before Pushing

Before pushing, you can see exactly which commits will be sent to the remote:

```bash
git log origin/<branch>..HEAD --oneline
```

**Example:**
```bash
git log origin/main..HEAD --oneline
```

Output:
```
e4f5g6h Resolve merge conflict
a1b2c3d Add legacy agent wrapper
```

These 2 commits exist locally but not on `origin/main`, so they'll be pushed.

### Variations

**See full commit details:**
```bash
git log origin/main..HEAD
```

**See which files changed:**
```bash
git log origin/main..HEAD --stat
```

**See the actual diff:**
```bash
git diff origin/main..HEAD
```

**Count commits:**
```bash
git rev-list origin/main..HEAD --count
```

### Why this works

Git tracks your remote branch as `origin/<branch>`. The range `origin/branch..HEAD` asks: "What commits are in my HEAD that aren't in the remote's version?"

```
origin/main: A -- B -- C
                        \
               HEAD:     D -- E
```

`origin/main..HEAD` returns D and E — exactly what `git push` will send.
