**Aborting a rebase with `git rebase --abort`**

If you're in the middle of a rebase and want to stop, run:

```bash
git rebase --abort
```

**What it does:**

- Stops the rebase process immediately
- Restores your branch to exactly the state it was in before you started the rebase
- Discards any conflict resolutions or commit changes you made during the rebase
- Returns your working directory to a clean state

**When you might use it:**

- The conflicts are more complex than expected and you want to try a different approach
- You realize you're rebasing onto the wrong branch
- You just want to start over

**It's completely safe.** Git keeps a reference to your original branch state when a rebase begins, so aborting always brings you back to where you started. No work from before the rebase is lost.

**Note:** This only works while a rebase is in progress. Once a rebase completes successfully, you'd need to use `git reflog` to recover the previous state if you wanted to undo it.
