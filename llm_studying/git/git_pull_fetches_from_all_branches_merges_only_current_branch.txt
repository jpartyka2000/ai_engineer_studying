No, `git pull` does **not** merge commits from all branches - it only affects your **current branch**.

Here's what actually happens:

**When you run `git pull`:**
1. **Fetch phase**: Downloads updates for all remote branches (updates all your `origin/*` references)
2. **Merge phase**: Only merges the remote tracking branch that corresponds to your *current* local branch

**Example:**
- If you're on your local `main` branch and run `git pull`
- It fetches everything from the remote
- But only merges `origin/main` into your local `main`
- Your local `develop`, `feature-branch`, etc. remain unchanged

**To update other local branches:**
You'd need to:
```bash
git checkout develop
git pull

git checkout feature-branch
git pull
```

**Or fetch everything without merging:**
```bash
git fetch --all
```
This updates all your remote-tracking branches (`origin/main`, `origin/develop`, etc.) but doesn't merge anything into your local branches.

**Key point:** `git pull` is really just shorthand for:
```bash
git fetch
git merge origin/<current-branch>
```

So it's a convenient command for updating the branch you're currently working on, but it's not a "sync everything" command.
