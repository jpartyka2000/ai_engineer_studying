QUESTION:
You want to mark a specific commit as version v2.0.0 for 
release purposes. Git offers lightweight and annotated tags.

Which correctly describes the difference AND shows the 
command to create an annotated tag?

A) Lightweight stores metadata; annotated are just pointers.
   Command: git tag v2.0.0

B) Annotated stores metadata; lightweight are just pointers.
   Command: git tag -a v2.0.0 -m "Release version 2.0.0"

C) Annotated stores metadata; lightweight are just pointers.
   Command: git tag -l v2.0.0

D) No difference; they are interchangeable.
   Command: git tag v2.0.0
------------------------------------------------------------
YOUR ANSWER: B
CORRECT ANSWER: B
STATUS: ✅ Correct
------------------------------------------------------------
EXPLANATION:
ANNOTATED TAGS (recommended for releases):
- Full Git objects stored in the database
- Contain: tagger name, email, date, message
- Can be signed with GPG for verification
- Created with: git tag -a v2.0.0 -m "Message"

LIGHTWEIGHT TAGS:
- Just a pointer to a commit (like a bookmark)
- No metadata, no message
- Created with: git tag v2.0.0

Why the others are wrong:
- A) Definition is reversed
- C) -l flag LISTS tags, doesn't create them
- D) They are definitely different

COMMON TAG COMMANDS:
------------------------------------------------------------
git tag                          # List all tags
git tag -l "v1.*"                # List tags matching pattern
git tag -a v2.0.0 -m "Message"   # Create annotated tag
git tag v2.0.0                   # Create lightweight tag
git tag -a v1.5.0 abc123 -m "M"  # Tag a specific commit
git show v2.0.0                  # Show tag details
git push origin v2.0.0           # Push single tag to remote
git push origin --tags           # Push all tags to remote
git tag -d v2.0.0                # Delete local tag
git push origin --delete v2.0.0  # Delete remote tag

BEST PRACTICE:
------------------------------------------------------------
Use annotated tags for releases and milestones (they have 
full context). Use lightweight tags for temporary or 
personal bookmarks.
