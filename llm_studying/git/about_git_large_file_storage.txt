Git LFS (Large File Storage) is an extension that handles large files by storing them outside your main Git repository.

**The problem it solves:** Git stores the full history of every file. With text files, this is efficient because Git can compress and diff them. But binary files (images, videos, audio, datasets, compiled binaries) don't diff well — every version gets stored in full. A repo with many large binary files quickly becomes bloated, making clones and fetches painfully slow.

**How it works:** Instead of storing the actual file content in your repo, Git LFS stores a small pointer file (just a text reference with a hash). The real file content lives on a separate LFS server. When you clone or checkout, LFS fetches the actual files on demand.

**Typical usage:**
```bash
# Install and initialize
git lfs install

# Track certain file types
git lfs track "*.psd"
git lfs track "*.mp4"
git lfs track "datasets/*.parquet"

# This creates/updates .gitattributes
git add .gitattributes
git commit -m "Track large files with LFS"
```

**Common use cases:** Design assets, video/audio files, ML model weights, large datasets, game assets, compiled binaries.

**Tradeoff:** You need LFS support on your remote (GitHub, GitLab, Bitbucket all support it), and there are usually storage/bandwidth limits on free tiers.
