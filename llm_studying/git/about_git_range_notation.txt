## Git Range Notation

Git uses `..` and `...` to specify ranges of commits.

### Double dot (`..`)

`A..B` means: **commits reachable from B that are not reachable from A**

Think of it as "what does B have that A doesn't?"

```
      main
        ↓
A -- B -- C
           \
            D -- E ← feature
```

| Command | Result |
|---------|--------|
| `git log main..feature` | Shows D, E |
| `git log feature..main` | Shows nothing (main has nothing feature doesn't have) |

**Common uses:**
```bash
# What commits will I push?
git log origin/main..HEAD

# What commits will I pull?
git log HEAD..origin/main

# What's on this branch that's not on main?
git log main..feature
```

### Triple dot (`...`)

`A...B` means: **commits reachable from either A or B, but not both** (symmetric difference)

Using the same diagram:

| Command | Result |
|---------|--------|
| `git log main...feature` | Shows C, D, E |

**Common use:**
```bash
# What's different between these two branches?
git log main...feature
```

### With `git diff`

Note: `git diff` interprets these differently than `git log`:

| Command | Meaning |
|---------|---------|
| `git diff A..B` | Same as `git diff A B` (diff between A and B) |
| `git diff A...B` | Diff between common ancestor of A/B and B |
