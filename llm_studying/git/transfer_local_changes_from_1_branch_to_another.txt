It depends on whether you've committed the changes or not.

**If changes are uncommitted (still in working directory):**

Just stash and switch:

```bash
git stash
git checkout branch2
git stash pop
```

Or even simpler, if branch2 already exists:

```bash
git checkout branch2
```

Git will carry over uncommitted changes automatically if there are no conflicts.

**If you've already committed on branch1:**

Use cherry-pick to copy the commits to branch2:

```bash
git checkout branch2
git cherry-pick <commit-hash>  # for a single commit
```

For multiple commits, you can cherry-pick a range:

```bash
git cherry-pick <oldest-hash>^..<newest-hash>
```

Then reset branch1 to remove those commits from it:

```bash
git checkout branch1
git reset --hard HEAD~N  # where N is the number of commits to remove
```

**If you want to move an entire branch's work:**

You can rebase branch1 onto branch2's base, or merge—but cherry-pick gives you the most control for selectively moving commits.
