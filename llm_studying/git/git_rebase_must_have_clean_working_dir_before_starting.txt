**You must have a clean working directory to start a rebase**

If you have uncommitted changes (staged or unstaged), Git won't let you start a rebase:

```
error: cannot rebase: You have unstaged changes.
error: Please commit or stash them.
```

**Your options before rebasing:**

1. **Stash your changes:**
   ```bash
   git stash
   git rebase -i origin/main
   # after rebase completes
   git stash pop
   ```

2. **Commit your changes:**
   ```bash
   git add .
   git commit -m "WIP: work in progress"
   git rebase -i origin/main
   ```
   You can always amend or squash this commit later.

3. **Discard your changes** (if you don't need them):
   ```bash
   git checkout -- .
   git rebase -i origin/main
   ```

**Why this requirement exists:**

Rebasing rewrites commits and moves your branch pointer around. If you had uncommitted changes in your working directory, Git wouldn't know what to do with them—they could conflict with the commits being replayed, or get lost entirely. Requiring a clean state keeps things predictable and safe.
