**Refs (references) are pointers to commits.**

At their core, refs are just files that contain a commit SHA. They give human-readable names to specific commits so you don't have to remember hashes like `a1b2c3d4e5f6...`.

---

**Where refs live:**

Refs are stored in your `.git/refs` directory:

```
.git/refs/
├── heads/          # Local branches
│   ├── main
│   └── feature-x
├── remotes/        # Remote-tracking branches
│   └── origin/
│       ├── main
│       └── feature-x
└── tags/           # Tags
    └── v1.0.0
```

---

**What's inside a ref file:**

Each ref file simply contains a commit hash:

```bash
cat .git/refs/heads/main
# 3f2a1b9c8d7e6f5a4b3c2d1e0f9a8b7c6d5e4f3a
```

That's it — just a 40-character SHA pointing to a commit.

---

**Types of refs:**

| Ref type | Path | Example |
|----------|------|---------|
| Local branch | `refs/heads/*` | `refs/heads/main` |
| Remote-tracking branch | `refs/remotes/*` | `refs/remotes/origin/main` |
| Tag | `refs/tags/*` | `refs/tags/v1.0.0` |

---

**The `refs/heads/` vs shorthand:**

When you write `main`, Git expands it to `refs/heads/main`. These are equivalent:

```bash
git checkout main
git checkout refs/heads/main
```

Similarly, `origin/main` expands to `refs/remotes/origin/main`.

---

**So about `branch.*.merge`:**

When you see:

```ini
merge = refs/heads/feature-x
```

This is the full ref path to the branch on the remote. It's not a local path — it tells Git which ref to ask for when communicating with the remote. The remote repository has its own `refs/heads/feature-x`, and this setting says "that's the one I'm tracking."

---

**List all refs:**

```bash
git show-ref
```

Example output:

```
3f2a1b9c... refs/heads/main
8d7e6f5a... refs/heads/feature-x
3f2a1b9c... refs/remotes/origin/main
a1b2c3d4... refs/tags/v1.0.0
```

---

**Special ref — HEAD:**

`HEAD` is a special ref that points to your current branch (or directly to a commit if detached):

```bash
cat .git/HEAD
# ref: refs/heads/main
```

This tells Git which branch you're currently on.
