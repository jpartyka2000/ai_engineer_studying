**`git push --force-with-lease`**

This command force pushes your branch to the remote, but with a safety check.

**Why force push is needed after a rebase:**

Rebasing rewrites your commits (they get new hashes), so your local branch and the remote branch have diverged. A normal `git push` will be rejected because Git sees them as incompatible histories. You need to force push to overwrite the remote with your rebased version.

**Why `--force-with-lease` instead of `--force`:**

- `--force` overwrites the remote branch unconditionally
- `--force-with-lease` first checks that the remote branch is where you expect it to be

If someone else pushed commits to your branch after you last fetched, `--force-with-lease` will fail and warn you, preventing you from accidentally overwriting their work. With plain `--force`, those commits would be lost.

**When to use it:**

- After rebasing a branch
- After amending commits that were already pushed
- Any time you need to rewrite history on a branch you've already pushed

**When NOT to use it:**

- On shared branches like `main` or `develop` — rewriting public history causes problems for everyone else
- It's generally safe on feature branches that only you are working on
