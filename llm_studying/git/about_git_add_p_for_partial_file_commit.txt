QUESTION:
You modified app.js with multiple changes, but you only 
want to stage SOME of the changes (not the entire file).

Which command lets you interactively choose which parts 
(hunks) of a file to stage?

A) git add app.js --partial
B) git add -p app.js
C) git stage --interactive app.js
D) git commit --patch app.js
------------------------------------------------------------
YOUR ANSWER: A
CORRECT ANSWER: B
STATUS: ❌ Incorrect
------------------------------------------------------------
EXPLANATION:
git add -p (or --patch) lets you interactively select 
which changes to stage, one "hunk" at a time.

WHY THE OTHERS ARE WRONG:
------------------------------------------------------------
A) git add app.js --partial
   Not a valid flag — doesn't exist

C) git stage --interactive app.js
   git stage is an alias for git add, but --interactive 
   works differently (menu-based, not hunk-by-hunk)

D) git commit --patch app.js
   Not valid — committing doesn't work this way

HOW git add -p WORKS:
------------------------------------------------------------
$ git add -p app.js

Git shows each "hunk" (chunk of changes) and asks:

  @@ -10,6 +10,8 @@ function login() {
       validate();
  +    logAttempt();
  +    trackUser();
   }

  Stage this hunk [y,n,q,a,d,s,e,?]?

OPTIONS:
  y = Yes, stage this hunk
  n = No, skip this hunk
  q = Quit (don't stage remaining hunks)
  a = Stage this hunk and ALL remaining hunks
  d = Don't stage this hunk or any remaining
  s = Split this hunk into smaller pieces
  e = Manually edit this hunk
  ? = Help

WHY THIS IS USEFUL:
------------------------------------------------------------
- Keep commits focused on ONE logical change
- Separate bug fixes from feature work
- Avoid committing debug code by accident
- Create cleaner, more reviewable history

EXAMPLE WORKFLOW:
------------------------------------------------------------
You fixed a bug AND added a feature in the same file:

$ git add -p app.js
  (stage only the bug fix hunks, say 'n' to feature hunks)

$ git commit -m "Fix login validation bug"

$ git add -p app.js
  (now stage the feature hunks)

$ git commit -m "Add user tracking feature"

Result: Two clean, focused commits!
