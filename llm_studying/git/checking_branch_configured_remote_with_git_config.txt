**Check a branch's configured remote:**

```bash
git config branch.<branch-name>.remote
```

This returns the name of the remote that a branch is configured to track.

**Examples:**

```bash
git config branch.main.remote
# origin

git config branch.feature-x.remote
# origin

git config branch.experiment.remote
# (no output = no remote configured)
```

---

**Related setting — the remote branch it tracks:**

```bash
git config branch.<branch-name>.merge
```

This returns the ref on the remote that the branch tracks.

```bash
git config branch.main.merge
# refs/heads/main
```

Together, `branch.*.remote` and `branch.*.merge` define the full upstream relationship.

---

**Set an upstream manually:**

```bash
git config branch.feature-x.remote origin
git config branch.feature-x.merge refs/heads/feature-x
```

Or more simply:

```bash
git branch --set-upstream-to=origin/feature-x feature-x
```

---

**Remove an upstream:**

```bash
git branch --unset-upstream feature-x
```

Or manually:

```bash
git config --unset branch.feature-x.remote
git config --unset branch.feature-x.merge
```

---

**List all branch tracking configurations:**

```bash
git config --get-regexp 'branch\..*\.(remote|merge)'
```

Example output:

```
branch.main.remote origin
branch.main.merge refs/heads/main
branch.feature-x.remote origin
branch.feature-x.merge refs/heads/feature-x
```

---

These settings are stored in your local `.git/config` file and control what happens when you run `git pull` or `git push` without arguments.
