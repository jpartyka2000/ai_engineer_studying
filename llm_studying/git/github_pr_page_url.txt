Yes, GitHub automatically generates a URL for each Pull Request (PR) when it is created. You can find the link in the following ways:

### 1. **From the PR Page**
   - When you open the PR, the URL in your browser's address bar is the direct link to that PR.
   - It will look like this:
     ```
     https://github.com/<owner>/<repo>/pull/<pr_number>
     ```
     For example:
     ```
     https://github.com/example-user/example-repo/pull/42
     ```

### 2. **From the Repository Pull Requests Tab**
   - Navigate to the repository on GitHub.
   - Click on the **Pull requests** tab.
   - Find your PR in the list and click on it to open it. The URL in the address bar is the link to the PR.

### 3. **From the Command Line (GitHub CLI)**
   If you're using the [GitHub CLI](https://cli.github.com/):
   - Run the following command to view your open PRs:
     ```bash
     gh pr list
     ```
   - To get the URL for a specific PR, use:
     ```bash
     gh pr view <pr_number> --web
     ```
     This will open the PR in your default browser.

### 4. **From Email Notifications**
   - If notifications are enabled, GitHub will send an email when a PR is created. The email includes a direct link to the PR.

### 5. **From the Issue/PR Number Mention**
   If the PR is referenced elsewhere in the repository (e.g., in a commit or issue), you can click on the reference (e.g., `#42`) to go to the PR page.
