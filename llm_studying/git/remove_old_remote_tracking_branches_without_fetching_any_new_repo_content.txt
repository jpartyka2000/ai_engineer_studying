You can use `git remote prune` to remove stale remote-tracking branches without fetching new content:

```bash
git remote prune origin
```

This removes any remote-tracking branches under `origin/` that no longer exist on the remote, without downloading any new commits, branches, or tags.

**Useful options:**

```bash
# See what would be pruned without actually doing it
git remote prune origin --dry-run

# Prune all remotes at once
git remote prune --all
```

**Alternative approach:**

You can also delete specific remote-tracking branches manually:

```bash
git branch -dr origin/old-branch-name
```

---

**For future fetches**, if you want pruning to happen automatically whenever you fetch, you can set this config:

```bash
git config --global fetch.prune true
```

Then any `git fetch` or `git pull` will automatically prune stale branches (though this does involve fetching).
