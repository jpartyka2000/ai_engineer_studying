GIT TUTOR — QUESTION RECORD

Skill Level: Medium
Category: Remote Repositories
Type: Scenario / Coding
------------------------------------------------------------
QUESTION:
You have a local repo and want to connect it to a new 
GitHub repo at: https://github.com/yourusername/my-project.git

Part 1: Add this URL as a remote named `origin`.
Part 2: Push local `main` to remote for the first time 
        AND set it to track the remote branch.
------------------------------------------------------------
YOUR ANSWER: 
  Part 1: git remote add origin
  Part 2: git push -u main

CORRECT ANSWER: 
  Part 1: git remote add origin https://github.com/yourusername/my-project.git
  Part 2: git push -u origin main

STATUS: ⚠️ Partially Correct (Part 1 ❌ / Part 2 ❌)
------------------------------------------------------------
EXPLANATION:
PART 1 — Missing the URL ❌
The full syntax requires THREE parts:

  git remote add <name> <url>
                  │       │
                  │       └─ The GitHub URL
                  └─ The nickname (usually "origin")

Correct:
  git remote add origin https://github.com/yourusername/my-project.git

PART 2 — Missing the remote name ❌
The push command needs to specify WHERE to push:

  git push -u <remote> <branch>
              │         │
              │         └─ Which branch to push
              └─ Which remote (origin)

Correct:
  git push -u origin main

WHAT DOES -u DO?
------------------------------------------------------------
The -u (or --set-upstream) flag creates a TRACKING 
relationship between your local and remote branches.

After running "git push -u origin main":
- Local main → tracks → origin/main
- Future pushes: just type "git push" (no need for origin main)
- Future pulls: just type "git pull" (no need for origin main)

WITHOUT -u:
  $ git push              # Error: no upstream branch
  $ git push origin main  # Must specify every time

WITH -u (first time):
  $ git push -u origin main
  
AFTER -u (future pushes):
  $ git push              # Works! Git knows where to push

USEFUL REMOTE COMMANDS:
------------------------------------------------------------
git remote -v                    # List remotes with URLs
git remote show origin           # Details about origin
git remote rename origin upstream # Rename a remote
git remote remove origin         # Remove a remote
git remote set-url origin <url>  # Change remote URL
