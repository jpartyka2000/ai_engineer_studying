## 📚 Git Status -s — Overview

**Copy the block below for your records:**

```
============================================================
GIT STATUS -s (SHORT FORMAT) — OVERVIEW
============================================================

WHAT IT DOES
------------------------------------------------------------
The -s (or --short) flag gives you a compact, minimal view 
of your repository status — no explanatory text, just the 
essentials.

BASIC USAGE
------------------------------------------------------------
$ git status -s

SAMPLE OUTPUT:
------------------------------------------------------------
 M README.md
M  app.js
A  newfile.js
?? notes.txt
D  oldfile.js

Compare to regular git status which is much more verbose 
with hints and instructions.

UNDERSTANDING THE OUTPUT
------------------------------------------------------------
Each line has TWO columns, then the filename:

  XY filename
  ││
  │└─ Working directory status
  └── Staging area (index) status

COMMON CODES:
------------------------------------------------------------
??  Untracked (new file Git doesn't know about)
A   Added (staged for commit)
M   Modified
D   Deleted
R   Renamed
C   Copied
U   Unmerged (conflict)
 
(space) = No changes in that column

EXAMPLES EXPLAINED:
------------------------------------------------------------
 M file.txt    Modified in working dir, NOT staged
M  file.txt    Modified AND staged
MM file.txt    Staged, then modified again
A  file.txt    New file, staged
?? file.txt    Untracked (not added to Git yet)
D  file.txt    Deleted and staged for removal
 D file.txt    Deleted in working dir, not staged

USEFUL VARIATIONS
------------------------------------------------------------
git status -s          # Short format
git status -sb         # Short format + branch info
git status -s --ignored # Short format + show ignored (!!)

WITH BRANCH INFO:
------------------------------------------------------------
$ git status -sb

## main...origin/main [ahead 2]
 M README.md
A  newfile.js
?? notes.txt

The first line shows branch and tracking status.

WHEN TO USE
------------------------------------------------------------
• Quick check while working (less noise)
• Scripting and automation
• When you know Git well and don't need hints
• Faster to scan in terminal
============================================================
```
