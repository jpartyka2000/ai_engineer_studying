============================================================
GIT STATUS --IGNORED — OVERVIEW
============================================================

WHAT IT DOES
------------------------------------------------------------
By default, git status hides files that are ignored via 
.gitignore. The --ignored flag reveals them.

BASIC USAGE
------------------------------------------------------------
$ git status --ignored

SAMPLE OUTPUT:
------------------------------------------------------------
On branch main

Changes not staged for commit:
        modified:   app.js

Untracked files:
        newfile.txt

Ignored files:
  (use "git add -f <file>..." to force include)
        node_modules/
        .env
        dist/
        debug.log

WHY USE IT
------------------------------------------------------------
- Verify your .gitignore is working correctly
- Check if sensitive files (.env, secrets) are ignored
- Debug why a file isn't appearing in git status
- Audit what's being excluded from the repo

USEFUL VARIATIONS
------------------------------------------------------------
git status --ignored           # Show ignored files
git status -s --ignored        # Short format (!! = ignored)
git status --ignored=matching  # Show ALL files, not just dirs

SHORT FORMAT OUTPUT:
------------------------------------------------------------
$ git status -s --ignored

 M app.js
?? newfile.txt
!! node_modules/
!! .env

!! = Ignored file/directory

RELATED COMMAND
------------------------------------------------------------
To check WHY a specific file is ignored:

$ git check-ignore -v filename.txt
.gitignore:5:*.txt    filename.txt

Shows which rule in which file is causing the ignore.
