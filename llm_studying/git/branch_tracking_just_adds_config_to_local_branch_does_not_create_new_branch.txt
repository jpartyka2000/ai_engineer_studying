**Branch tracking is just configuration — not a new branch.**

When you set a local branch to track a remote branch, Git simply adds two config entries to your existing branch:

```ini
[branch "feature-x"]
    remote = origin
    merge = refs/heads/feature-x
```

That's it. No new branch is created.

---

**View this configuration:**

```bash
git config --get-regexp 'branch\.feature-x\.'
```

Output:

```
branch.feature-x.remote origin
branch.feature-x.merge refs/heads/feature-x
```

---

**What this configuration enables:**

| Command | Without tracking | With tracking |
|---------|------------------|---------------|
| `git push` | Error: no upstream | Pushes to `origin/feature-x` |
| `git pull` | Error: no upstream | Pulls from `origin/feature-x` |
| `git status` | No ahead/behind info | Shows "Your branch is ahead by 2 commits" |

---

**Setting tracking is just modifying config:**

These two approaches are equivalent:

```bash
# Using the branch command
git branch --set-upstream-to=origin/feature-x feature-x

# Manually editing config
git config branch.feature-x.remote origin
git config branch.feature-x.merge refs/heads/feature-x
```

---

**Removing tracking is just removing config:**

```bash
# Using the branch command
git branch --unset-upstream feature-x

# Manually
git config --unset branch.feature-x.remote
git config --unset branch.feature-x.merge
```

---

**Where this config lives:**

These settings are stored in your local `.git/config` file, not on the remote. Tracking is a purely local concept that tells your Git client how to behave.
