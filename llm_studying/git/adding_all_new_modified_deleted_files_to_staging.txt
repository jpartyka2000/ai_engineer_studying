QUESTION:
Complete the following command to stage ALL modified and 
new files in your repository for the next commit:

git ___________
------------------------------------------------------------
YOUR ANSWER: add .
CORRECT ANSWER: add .  (also acceptable: add -A, add --all)
STATUS: ✅ Correct
------------------------------------------------------------
EXPLANATION:
"git add ." stages all changes (new, modified, and deleted 
files) in the current directory and all subdirectories.

Other valid alternatives:
- git add -A (or --all): Stages everything across the 
  entire working tree, regardless of your current directory.

- git add -u: Stages only modified and deleted files 
  (ignores new untracked files).

Quick comparison:
┌─────────────┬─────────┬──────────┬─────────┐
│ Command     │ New     │ Modified │ Deleted │
├─────────────┼─────────┼──────────┼─────────┤
│ git add .   │ ✓       │ ✓        │ ✓       │
│ git add -A  │ ✓       │ ✓        │ ✓       │
│ git add -u  │ ✗       │ ✓        │ ✓       │
└─────────────┴─────────┴──────────┴─────────┘

Note: In Git 2.x, "add ." and "add -A" behave the same 
when run from the repository root.
