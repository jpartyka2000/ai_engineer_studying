Yes — when you push commits, the tree objects those commits reference get pushed along with them.

A commit is really just a pointer to a root tree (plus metadata like author, message, parent commits). Git can't send a commit without also sending the tree it points to, and any blobs or subtrees that tree references. Otherwise the remote would have a commit pointing to nothing.

Git is smart about it though — it only sends objects the remote doesn't already have. If a tree or blob already exists on the remote (because it was part of a previous commit), Git skips it. That's why pushing a small change is fast even in a large repo.
