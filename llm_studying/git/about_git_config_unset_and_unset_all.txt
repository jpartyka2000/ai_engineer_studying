**Remove a Git config setting:**

```bash
git config --unset <key>
```

This removes a configuration entry entirely.

---

**Examples:**

```bash
# Remove a branch's remote tracking
git config --unset branch.feature-x.remote

# Remove a branch's merge ref
git config --unset branch.feature-x.merge

# Remove your configured email
git config --unset user.email
```

---

**Specify scope:**

```bash
# Remove from local config (.git/config)
git config --local --unset user.email

# Remove from global config (~/.gitconfig)
git config --global --unset user.email

# Remove from system config (/etc/gitconfig)
git config --system --unset user.email
```

---

**Removing multi-value entries:**

Some keys can have multiple values. Use `--unset-all` to remove all of them:

```bash
# Remove all values for a key
git config --unset-all remote.origin.fetch
```

Using `--unset` when multiple values exist will cause an error:

```
warning: key has multiple values
```

---

**Verify removal:**

```bash
git config --get branch.feature-x.remote
# (no output = successfully removed)
```

---

**Remove an entire section:**

```bash
git config --remove-section branch.feature-x
```

This removes the whole `[branch "feature-x"]` block, including all its keys.

---

**What happens in the config file:**

Before:

```ini
[branch "feature-x"]
    remote = origin
    merge = refs/heads/feature-x
```

After `git config --unset branch.feature-x.remote`:

```ini
[branch "feature-x"]
    merge = refs/heads/feature-x
```

After `git config --remove-section branch.feature-x`:

```ini
(section completely removed)
```
