`git show` displays information about Git objects (commits, tags, trees, blobs).

## Basic usage

```bash
# Show the last commit (default)
git show

# Show a specific commit
git show abc123
git show HEAD~3

# Show a tag
git show v1.0.0
```

## What it displays (for commits)

- Commit hash, author, date
- Commit message
- Full diff of all changes

## Useful flags

```bash
# Stats summary (files changed, insertions/deletions)
git show --stat

# Just the file names that changed
git show --name-only

# Names with status (A=added, M=modified, D=deleted)
git show --name-status

# No diff, just commit info
git show --no-patch
git show -s

# Specific file from a commit
git show HEAD:path/to/file.py

# Diff for specific file only
git show HEAD -- path/to/file.py
```

## Show file contents at a commit

```bash
# View file as it existed at that commit (no diff)
git show abc123:src/main.py
```

## Formatting options

```bash
# One-line summary
git show --oneline

# Custom format
git show --format="%h %s" -s
```

## vs. other commands

| Command | Purpose |
|---------|---------|
| `git show` | Display a single object in detail |
| `git log` | List multiple commits |
| `git diff` | Compare two points (working tree, commits, etc.) |
