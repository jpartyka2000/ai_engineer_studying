A "dirty working tree" means you have uncommitted changes in your working directory. Git considers the tree "clean" when your working files match the last commit exactly, and "dirty" when they don't.

**What makes it dirty:**
- Modified files that haven't been staged or committed
- Staged changes that haven't been committed
- Untracked files (sometimes, depending on the operation)

**Why it matters:** Certain Git operations refuse to run on a dirty tree because they could overwrite or lose your uncommitted work. Common ones that complain:
- `git checkout` (switching branches)
- `git pull` or `git rebase`
- `git stash pop`
- `git merge` (in some cases)

**How to check:**
```bash
git status
```

If it shows modified or staged files, your tree is dirty.

**How to fix it:**
- Commit your changes: `git add . && git commit -m "WIP"`
- Stash them temporarily: `git stash`
- Discard them: `git checkout -- .` (careful — this loses changes)
