Yes—you can, right in your GitHub Actions job, inspect exactly which Black got pulled into your runner. There are two easy ways:

1. **Look at the pip install logs**
   When your workflow does:

   ```yaml
   - name: Install dev requirements
     run: pip install -r requirements-dev.txt
   ```

   you should see lines like:

   ```
   Collecting black==23.7.0
     Downloading black-23.7.0-py3-none-any.whl (1.2 MB)
   ```

   which tells you the version.

2. **Add an explicit step to print it out**
   Right after you install your dependencies, just run:

   ```yaml
   - name: Show Black version
     run: |
       black --version
       pip show black
   ```

   That will print something like:

   ```
   black, version 23.7.0
   Name: black
   Version: 23.7.0
   Summary: The uncompromising code formatter.
   ```

   so you know exactly which Black binary your Actions runner is using.
