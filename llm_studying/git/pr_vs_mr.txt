In most modern Git-based workflows:

- **PR (Pull Request)** is the GitHub (and Bitbucket) terminology for proposing changes from one branch (or fork) into another.
- **MR (Merge Request)** is GitLab’s (and sometimes GitKraken’s) terminology for essentially the same concept: proposing changes and asking reviewers to merge them.

Although they perform nearly identical functions, the different names (“Merge Request” vs. “Pull Request”) reflect each platform’s chosen wording:

- **GitHub**: You “pull” changes into the target branch, hence a “pull request.”  
- **GitLab**: You “request” that your changes get merged, hence a “merge request.”

So “MR” and “PR” are platform-specific terms for a very similar review-and-merge flow.
