`git remote` manages connections to remote repositories and lets you inspect their state.

**Basic commands:**

```bash
# List remotes
git remote

# List remotes with URLs
git remote -v
```

**Example output:**

```
origin  git@github.com:yourname/repo.git (fetch)
origin  git@github.com:yourname/repo.git (push)
upstream  git@github.com:original/repo.git (fetch)
upstream  git@github.com:original/repo.git (push)
```

**Inspect a remote:**

```bash
git remote show origin
```

This displays useful information:
- The remote URL
- Which branches exist on the remote
- Which local branches track which remote branches
- Which remote-tracking branches are stale

**Example output:**

```
* remote origin
  Fetch URL: git@github.com:yourname/repo.git
  Push URL: git@github.com:yourname/repo.git
  HEAD branch: main
  Remote branches:
    main           tracked
    feature-login  tracked
    refs/remotes/origin/old-feature  stale (use 'git remote prune' to remove)
  Local branches configured for 'git pull':
    main merges with remote main
```

**Pruning with git remote:**

```bash
# See what would be pruned (dry run)
git remote prune origin --dry-run

# Prune stale remote-tracking branches
git remote prune origin
```

This is equivalent to `git fetch --prune` but without fetching new data.

**Other useful commands:**

```bash
# Add a new remote
git remote add upstream git@github.com:original/repo.git

# Rename a remote
git remote rename origin old-origin

# Remove a remote
git remote remove upstream

# Change a remote's URL
git remote set-url origin git@github.com:newname/repo.git
```
