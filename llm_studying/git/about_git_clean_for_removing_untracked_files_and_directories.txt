------------------------------------------------------------
QUESTION:
Your working directory is cluttered with untracked files — 
build artifacts, temp files, test outputs. You want to 
remove ALL untracked files AND directories.

Which command forcefully removes all untracked files AND 
untracked directories?

A) git clean -f
B) git clean -fd
C) git clean -n
D) git reset --hard
------------------------------------------------------------
YOUR ANSWER: B
CORRECT ANSWER: B
STATUS: ✅ Correct
------------------------------------------------------------
EXPLANATION:
git clean removes untracked files from your working 
directory. The flags control what gets deleted:

-f (force)     Required to actually delete anything
-d (directories) Also remove untracked directories
-fd            Combine both — delete files AND directories

WHY THE OTHERS ARE WRONG:
------------------------------------------------------------
A) git clean -f
   Removes untracked FILES only
   Leaves untracked DIRECTORIES untouched

C) git clean -n
   Dry run — shows what WOULD be deleted
   Doesn't actually delete anything
   (Great for previewing before cleaning!)

D) git reset --hard
   Resets tracked files to last commit
   Does NOT touch untracked files at all

GIT CLEAN FLAGS:
------------------------------------------------------------
-n, --dry-run   Preview what would be deleted (safe!)
-f, --force     Required to actually delete
-d              Include untracked directories
-x              Also delete ignored files (.gitignore)
-X              Delete ONLY ignored files
-i              Interactive mode (choose what to delete)

SAFE WORKFLOW:
------------------------------------------------------------
Always preview first!

$ git clean -fd -n        # See what will be deleted
Would remove build/
Would remove temp.txt
Would remove test_output/

$ git clean -fd           # Actually delete

⚠️ WARNING: git clean is PERMANENT! 
   Deleted files are NOT recoverable (not in Git history)
