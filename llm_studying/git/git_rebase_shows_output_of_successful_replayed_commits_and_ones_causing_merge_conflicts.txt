Yes to both.

Git replays your commits one at a time, and you'll see output for each one. It looks something like this:

```
Rebasing (1/5)
Rebasing (2/5)
Rebasing (3/5)
error: could not apply abc1234... Your commit message here
```

If a commit applies cleanly, Git moves on to the next one. If there's a conflict, it stops and shows you which commit caused the problem, including the commit hash and message. This is helpful because you know exactly which of your changes is conflicting with main.

After you resolve and continue, you'll see it pick up where it left off:

```
Rebasing (4/5)
Rebasing (5/5)
Successfully rebased and updated refs/heads/your-branch.
```

So yes, you get a clear picture of progress and can see precisely where things go wrong if they do.
