Just one pattern per line — that's it. Sections/comments are purely for human readability.

---

## Minimal .gitignore (Works Fine)

```
node_modules/
.env
*.log
__pycache__/
.DS_Store
```

## Same Thing With Sections (Optional)

```
# Dependencies
node_modules/

# Secrets
.env

# Logs
*.log

# Python
__pycache__/

# Mac
.DS_Store
```

**Both work identically.** Git ignores lines starting with `#`.

---

## The Rules

| Rule | Example |
|------|---------|
| One pattern per line | `*.log` |
| Comments start with `#` | `# This is a comment` |
| Blank lines are ignored | (use for spacing) |
| Directories end with `/` | `node_modules/` |

That's really all there is to it.
