No, rebase does not create any additional commits like a merge does.

**With merge:**

```
main:        A -- B -- C ----------- M  (merge commit created)
                        \           /
your-branch:             D -- E -- F
```

A merge creates an extra "merge commit" (`M`) that ties the two histories together.

**With rebase:**

```
main:        A -- B -- C
your-branch:              \-- D' -- E' -- F'
```

No extra commits—just your original commits replayed with new hashes. The history stays linear.

This is one reason people prefer rebase for feature branches: it keeps the history clean without merge commits cluttering it up.
