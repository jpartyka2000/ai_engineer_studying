No, they're different.

- **`git fetch`** вЂ” fetches from one remote (the default, usually `origin`)
- **`git fetch --all`** вЂ” fetches from all configured remotes

If you only have `origin` as your remote, they do the same thing. But if you have multiple remotes (e.g., `origin` and `upstream`), then:

```bash
git fetch          # only fetches from origin
git fetch --all    # fetches from origin AND upstream
```

Since you only have `origin`, there's no practical difference for you.
