**`git rebase --continue`**

Yes, `--continue` is only valid when a rebase is paused. This happens in a few situations:

1. **After a merge conflict** — Git stops and waits for you to resolve the conflict and stage the files before continuing.

2. **After an `edit` command in interactive rebase** — If you mark a commit with `edit` in the interactive todo list, Git pauses after applying that commit, letting you amend it or make additional changes. You run `--continue` when you're done.

3. **After a `break` command in interactive rebase** — You can insert `break` (or `b`) in the todo list to make Git pause at that point, useful for inspection. Continue when ready.

**If you run it when no rebase is paused:**

```bash
$ git rebase --continue
fatal: No rebase in progress?
```

Git will simply tell you there's nothing to continue.

**The typical flow:**

```bash
# Resolve conflict or finish editing
git add <files>
git rebase --continue
```

Note that you need to stage your changes before running `--continue`. If you have unstaged changes or unresolved conflicts, Git will remind you to address them first.
