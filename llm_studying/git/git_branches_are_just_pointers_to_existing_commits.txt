**Branches don't have their own hashes — they point to commits that have hashes.**

A branch is just a named pointer that stores a commit's SHA-1 hash. The branch itself has no hash.

---

**What's inside a branch:**

```bash
cat .git/refs/heads/main
# 3f2a1b9c8d7e6f5a4b3c2d1e0f9a8b7c6d5e4f3a
```

That's a commit hash, not a branch hash. The file `main` simply contains a reference to a commit.

---

**Get the commit a branch points to:**

```bash
git rev-parse main
# 3f2a1b9c8d7e6f5a4b3c2d1e0f9a8b7c6d5e4f3a
```

---

**What does have a SHA-1 hash in Git:**

| Object type | Has SHA-1 | Description |
|-------------|-----------|-------------|
| Commit | Yes | A snapshot with metadata (author, message, parent) |
| Tree | Yes | A directory listing |
| Blob | Yes | File contents |
| Tag (annotated) | Yes | A tag object with metadata |
| Branch | No | Just a pointer to a commit |
| Lightweight tag | No | Just a pointer to a commit |

---

**When you commit, the branch pointer moves:**

```bash
# Before commit
cat .git/refs/heads/main
# aaa111...

git commit -m "New commit"

# After commit
cat .git/refs/heads/main
# bbb222...  (now points to the new commit)
```

The branch is simply updated to point to the new commit's hash.

---

**This is why branches are "cheap" in Git:**

Creating a branch is just creating a 41-byte file (40-character hash + newline). There's no copying of content — it's just a new pointer to an existing commit.
