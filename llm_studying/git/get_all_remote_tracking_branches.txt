The `git branch -r` command lists all remote-tracking branches in your local repository.

**Basic usage:**

```bash
git branch -r
```

**Example output:**

```
origin/HEAD -> origin/main
origin/main
origin/feature/login
origin/feature/api-refactor
origin/bugfix/header-alignment
```

**Related options:**

```bash
# List local branches only
git branch

# List all branches (local + remote)
git branch -a

# List remote branches with last commit info
git branch -r -v

# List remote branches with last commit and date
git branch -r -v --format="%(refname:short) %(committerdate:relative) - %(subject)"
```

**Filtering branches:**

```bash
# Find branches by teammate's name or feature
git branch -r | grep feature
git branch -r | grep john
```

**Important notes:**

- These are *remote-tracking branches* — local copies of what the remote looked like the last time you fetched
- They can be stale if you haven't fetched recently
- Always run `git fetch --all` first to get the latest branch info:
  ```bash
  git fetch --all
  git branch -r
  ```

**Deleting stale remote branches:**

If a branch was deleted on the remote but still shows up locally:

```bash
# Prune deleted remote branches
git fetch --prune
```
