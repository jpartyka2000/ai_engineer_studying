Yes, `git restore` accepts wildcards, but you need to quote them so the shell doesn't expand them first:

```bash
# Restore all .txt files
git restore '*.txt'

# Restore all files in a directory
git restore 'src/*'

# Restore all Python files recursively
git restore '**/*.py'
```

Without quotes, your shell expands the glob before Git sees it, which can cause unexpected behavior or errors if no files match.
