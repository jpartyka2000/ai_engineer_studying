`git switch` is a command introduced in Git 2.23 (August 2019) specifically for switching branches. It was created to provide a clearer, more focused alternative to `git checkout`, which handled many different operations.

## `git switch <branch>`

Switches to an existing branch:

```bash
git switch main
git switch feature-login
```

If the branch doesn't exist, Git will error out (unlike `checkout`, which might do something unexpected).

## `git switch -c <new-branch>`

Creates a new branch and switches to it in one step:

```bash
git switch -c feature-new-dashboard
```

This is equivalent to:
```bash
git branch feature-new-dashboard
git switch feature-new-dashboard
```

The `-c` stands for "create."

## Other useful options

- **`git switch -`** — Switch back to the previous branch (like `cd -` in bash)
- **`git switch -c <branch> <start-point>`** — Create a new branch starting from a specific commit or branch
- **`git switch --detach <commit>`** — Detach HEAD at a specific commit

## Why use `switch` over `checkout`?

`git checkout` does many things: switch branches, restore files, detach HEAD, etc. This made it confusing. Git split the functionality:

| Task | Old command | New command |
|------|-------------|-------------|
| Switch branches | `git checkout <branch>` | `git switch <branch>` |
| Create & switch | `git checkout -b <branch>` | `git switch -c <branch>` |
| Restore files | `git checkout -- <file>` | `git restore <file>` |

The new commands are more explicit and harder to misuse.
