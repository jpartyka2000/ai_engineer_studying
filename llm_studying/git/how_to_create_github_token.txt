Your **GitHub token** is not something I can see — it’s a personal secret you generate in your GitHub account.

There are a few kinds of tokens:

1. **Personal Access Token (PAT)** – The most common. Looks like:

   ```
   ghp_xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx
   ```

   You create one at:

   * GitHub → Settings → Developer settings → Personal access tokens → **Tokens (classic)** or **Fine-grained tokens**

2. **GitHub App tokens** – If your org uses a GitHub App, you might authenticate via an installation token.

3. **GITHUB\_TOKEN** – An ephemeral token automatically provided inside GitHub Actions workflows (not valid outside Actions).

---

### For your Docker build

Since your container needs to `git clone https://github.com/PathAhead/cue_platform.git`, you’ll want a **PAT with at least `read:packages` and `repo` (or `contents:read`) scope**.

You’d then store it in your `api/github_credentials` file like:

```bash
GITHUB_USERNAME=your-github-username
GITHUB_TOKEN=ghp_XXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXX
```

---

⚠️ **Important:**

* Never commit this token to GitHub.
* Treat it like a password — keep it only in your local secret file or a secret manager.
* You can revoke/regenerate it anytime in GitHub settings.
