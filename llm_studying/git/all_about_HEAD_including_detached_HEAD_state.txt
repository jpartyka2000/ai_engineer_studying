============================================================
GIT CONCEPT — WHAT IS HEAD?
============================================================

DEFINITION
------------------------------------------------------------
HEAD is a pointer to your CURRENT location in the repository. 
It typically points to whatever branch you have checked out.

HEAD = "You are here" marker


HEAD MOVES WHEN YOU SWITCH BRANCHES
------------------------------------------------------------
$ git checkout main
  HEAD → main → a1b2c3d

$ git checkout feature
  HEAD → feature → f4e5d6c

$ git checkout develop
  HEAD → develop → 9h8i7j6

HEAD always follows you to whichever branch you switch to.


VISUALIZING HEAD
------------------------------------------------------------
Before checkout:
  main     → a1b2c3d  ← HEAD (you are here)
  feature  → f4e5d6c
  develop  → 9h8i7j6

After "git checkout feature":
  main     → a1b2c3d
  feature  → f4e5d6c  ← HEAD (you are here now)
  develop  → 9h8i7j6


SO IN THE MERGE CONFLICT QUESTION...
------------------------------------------------------------
When I said "HEAD points to main," that was because in that 
scenario you were on main when you ran the merge.

If you had been on develop and merged in feature-branch:
  $ git checkout develop
  $ git merge feature-branch

Then the conflict would look like:
  <<<<<<< HEAD
      [code from develop — because HEAD points to develop]
  =======
      [code from feature-branch]
  >>>>>>> feature-branch


DETACHED HEAD STATE
------------------------------------------------------------
HEAD usually points to a branch, but you can also checkout 
a specific commit directly:

  $ git checkout a1b2c3d

Now HEAD points directly to a commit, not a branch. This is 
called "detached HEAD" state. Any new commits you make won't 
belong to any branch (and can be lost if you switch away 
without creating a branch).

Normal:      HEAD → main → a1b2c3d
Detached:    HEAD → a1b2c3d (no branch!)


COMMANDS TO SEE WHERE HEAD POINTS
------------------------------------------------------------
git branch
  Shows all branches, current one marked with *

git log --oneline -1
  Shows the commit HEAD currently points to

git symbolic-ref HEAD
  Shows refs/heads/main (or whichever branch)

cat .git/HEAD
  Shows raw content: "ref: refs/heads/main"
