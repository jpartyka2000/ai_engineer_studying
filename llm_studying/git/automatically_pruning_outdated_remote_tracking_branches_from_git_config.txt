Sure! Here's a focused answer on that option:

---

**Auto-prune stale branches on every fetch:**

```bash
git config --global fetch.prune true
```

This configures Git to automatically remove remote-tracking branches that no longer exist on the remote whenever you run `git fetch` or `git pull`.

**Scope options:**

```bash
# Global (all repos for your user)
git config --global fetch.prune true

# Local (current repo only)
git config fetch.prune true
```

**Check current setting:**

```bash
git config --get fetch.prune
```

**Disable it:**

```bash
git config --global --unset fetch.prune
```

This is handy if you want to keep things tidy without remembering to run `git remote prune` manually, though note that it only takes effect when you actually fetch.
