============================================================
GIT COMMIT --AMEND — COMPREHENSIVE GUIDE
============================================================

WHAT IS --AMEND?
------------------------------------------------------------
The --amend flag modifies the most recent commit. It lets 
you:
- Fix typos in the commit message
- Add forgotten files
- Remove files that shouldn't have been committed
- Change the author information

Instead of creating a new "oops" commit, you REPLACE the 
previous commit with a corrected version.


HOW IT WORKS
------------------------------------------------------------
When you run git commit --amend:

1. Git takes your current staging area
2. Combines it with the previous commit's content
3. Creates a NEW commit replacing the old one
4. The old commit is discarded (but recoverable via reflog)

IMPORTANT: The commit SHA changes! It's technically a 
completely new commit, not an edit of the existing one.

Before:  abc1234 → "Add feature"
After:   xyz5678 → "Add feature" (new SHA, same or updated content)


COMMON USE CASES
------------------------------------------------------------

1. FIX A TYPO IN THE COMMIT MESSAGE
--------------------------------------
You wrote: git commit -m "Add logni form"
You meant: "Add login form"

Fix it:
  $ git commit --amend -m "Add login form"

Done! Message is corrected.


2. ADD A FORGOTTEN FILE
--------------------------------------
You committed but forgot to include validation.js:

  $ git add validation.js
  $ git commit --amend --no-edit

--no-edit keeps the original message unchanged.
