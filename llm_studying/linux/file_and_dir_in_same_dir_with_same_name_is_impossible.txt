Nope 🙂 — in Linux (and all UNIX-like filesystems, including macOS), you **cannot** have a file and a directory with the exact same name inside the same parent directory.

Here’s why:

* Each entry in a directory is just a mapping from a **name** → **inode** (which describes the type: file, directory, symlink, etc.).
* Since names must be unique within a directory, you can’t have both `github_credentials` as a file and `github_credentials` as a directory in the same folder.
* If you try to create a file with the same name as a directory (or vice-versa), you’ll get a “File exists” error.

### Example:

```bash
mkdir test
cd test
mkdir foo
touch foo
# touch: cannot create file 'foo': File exists
```

### What you *can* do:

* Have a **file** and a **directory** with the same name in *different* directories:

  ```
  /path/to/github_credentials  (file)
  /other/path/github_credentials (directory)
  ```
* Or add suffixes/prefixes if you need both in the same place, e.g. `github_credentials/` (directory) and `github_credentials.txt` (file).
