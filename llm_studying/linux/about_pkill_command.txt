`pkill` is not Mac-only — it's available on most Unix-like systems:

## Availability

| System | Available? |
|--------|------------|
| macOS | ✅ Yes |
| Linux | ✅ Yes |
| FreeBSD | ✅ Yes |
| Windows | ❌ No (unless using WSL, Cygwin, or Git Bash) |

---

## What It Does

`pkill` kills processes by **name** (rather than by PID like `kill`).

```bash
# Kill all processes named "python"
pkill python

# Kill all processes matching a pattern
pkill -f "python my_script.py"
```

---

## Common Options

| Option | Description |
|--------|-------------|
| `-f` | Match against full command line, not just process name |
| `-9` | Force kill (SIGKILL) |
| `-u username` | Only kill processes owned by a specific user |
| `-i` | Case-insensitive matching (macOS) |
| `-l` | List signal names |

---

## Examples

```bash
# Gracefully terminate all Python processes
pkill python

# Force kill all Python processes
pkill -9 python

# Kill a specific script (using -f to match full command)
pkill -f "python my_long_running_script.py"

# Kill all processes by a specific user
pkill -u john python
```

---

## Related Commands

| Command | Description |
|---------|-------------|
| `pgrep` | Find processes by name (returns PIDs) |
| `kill` | Kill by PID |
| `killall` | Kill by name (similar to pkill, slightly different behavior) |

```bash
# Find PIDs first, then kill
pgrep python    # Returns PIDs
kill <pid>      # Kill specific PID
```
