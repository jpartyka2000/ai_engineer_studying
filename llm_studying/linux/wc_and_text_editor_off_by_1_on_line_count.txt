This usually happens because of how **`wc -l` counts lines** versus how your text editor displays them.

* **`wc -l` behavior**:
  It counts the number of **newline characters (`\n`)** in the file.
  That means:

  * If the file has 999 newline characters, `wc -l` reports **999**.
  * If there’s a line at the end of the file that does not end with a newline, `wc` won’t count it.

* **Text editors’ behavior**:
  Most editors (like VS Code, vim, nano, etc.) still show that last line — even if it’s missing a newline at the end. So you see **1000 lines**, but `wc` only counts **999** because that last line doesn’t have a terminating `\n`.

👉 To check, you can run:

```bash
tail -n 1 -c 1 yourfile | od -c
```

If the output is empty or doesn’t show `\n`, then the last line isn’t newline-terminated.

To fix it (and make `wc -l` show 1000):

```bash
echo >> yourfile
```

That appends a final newline. After that, `wc -l` should match your editor.
