Alright — here are two simple ways to see if your file ends with a newline, without `od`:

---

### 1. Use `hexdump`

```bash
hexdump -C yourfile | tail -n 1
```

This shows the last line of the file in hex + ASCII.

* If you see `0a` at the very end, that’s a **newline** (`\n`).
* If not, the file is missing the final newline.

---

### 2. Use `tail` with `cat -A`

```bash
tail -c 1 yourfile | cat -A
```

* If the file ends with a newline, you’ll see `^J` printed (that’s how `cat -A` shows a newline).
* If nothing appears, the file does not end with a newline.

---

👉 Example:

```bash
$ printf "line1\nline2\n" > test1.txt
$ printf "line1\nline2"   > test2.txt

$ tail -c 1 test1.txt | cat -A
^J            # newline present

$ tail -c 1 test2.txt | cat -A
               # nothing printed, newline missing
```
