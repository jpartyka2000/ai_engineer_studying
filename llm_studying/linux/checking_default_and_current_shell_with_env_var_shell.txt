# Understanding `echo $SHELL` - Your Default Shell Identifier

The `echo $SHELL` command is one of the quickest ways to identify which shell program your system uses by default when you open a terminal.

## What it does:
```bash
echo $SHELL
```
This command prints the path to your default login shell - the shell that starts when you open a new terminal window or SSH into a system.

## Common outputs:

**On macOS:**
```bash
/bin/zsh     # macOS Catalina (10.15) and later
/bin/bash    # Older macOS versions
```

**On Linux:**
```bash
/bin/bash    # Most common
/usr/bin/zsh # If zsh is set as default
/bin/sh      # Basic POSIX shell
/bin/dash    # Debian/Ubuntu system shell
```

## Important distinction:
`$SHELL` shows your **default** shell, not necessarily your **current** shell. You might be running a different shell in your current session.

## To check your current shell:
```bash
echo $0
# or
ps -p $$
```

## Why this matters:
Different shells have different:
- Syntax for commands
- Configuration files (`.bashrc` vs `.zshrc`)
- Built-in features and functions
- History management commands

## Changing your default shell:
```bash
# List available shells
cat /etc/shells

# Change default shell
chsh -s /bin/bash   # Switch to bash
chsh -s /bin/zsh    # Switch to zsh
```

## Quick test to see which config file is active:
```bash
# Add a test echo to each file
echo 'echo "Using .zshrc"' >> ~/.zshrc
echo 'echo "Using .bashrc"' >> ~/.bashrc
# Open new terminal to see which message appears
```

Understanding your shell environment helps you use the right commands and configuration files for your system.
