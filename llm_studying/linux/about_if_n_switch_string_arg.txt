## What `[ -n "{{model}}" ]` Does:

This **tests if the model variable is NOT empty**.

## Breaking It Down:

```bash
if [ -n "{{model}}" ]; then
    # Do this if model is NOT empty
```

- **`[ ]`** = Test command (same as `test` command)
- **`-n`** = "**n**on-zero length" - checks if string is NOT empty
- **`{{model}}`** = just template variable replaced with actual value
- Returns **true** if string has content, **false** if empty

## Examples:

```bash
# If model="gpt-4"
if [ -n "gpt-4" ]; then  # TRUE - string is not empty
    MODEL_ARG="--model gpt-4"

# If model=""
if [ -n "" ]; then  # FALSE - string is empty
    # Skip this branch
```

## Common Test Operators:

```bash
[ -n "$var" ]  # True if var is NOT empty
[ -z "$var" ]  # True if var IS empty (zero length)
[ "$var" = "value" ]  # True if var equals "value"
[ -f "$file" ]  # True if file exists
[ -d "$dir" ]  # True if directory exists
```

## Real Examples:

```bash
# Check if variable has a value
if [ -n "$USER" ]; then
    echo "User is $USER"
fi

# Opposite - check if empty
if [ -z "$API_KEY" ]; then
    echo "Error: API_KEY not set!"
fi
```

**Bottom line**: `-n` means "**not empty**" - it's testing whether the model parameter was actually provided with a value or left blank.
