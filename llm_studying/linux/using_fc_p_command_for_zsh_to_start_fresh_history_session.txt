# Understanding `fc -p` in ZSH - History Stack Management

The `fc -p` command is a ZSH-specific feature for managing history stacks. It's not available in bash, which is why you might encounter errors if you're using the wrong shell.

## What `fc -p` does:
```bash
fc -p
```
This command pushes a new history list onto a stack, effectively starting a "fresh" history session without destroying your existing history.

## Basic usage:

**Start a new temporary history:**
```bash
fc -p
# Your commands here won't mix with main history
# When you exit or pop, you return to original history
```

**Specify a custom history file:**
```bash
fc -p ~/.temp_history
# Uses ~/.temp_history as the history file for this session
```

**Set custom history size:**
```bash
fc -p ~/.custom_history 100 100
# First number: HISTSIZE (lines in memory)
# Second number: SAVEHIST (lines saved to file)
```

## Common use cases:

**1. Temporary private session:**
```bash
fc -p /dev/null
# Nothing gets saved to disk
# Perfect for sensitive commands
```

**2. Project-specific history:**
```bash
cd ~/projects/myapp
fc -p .local_history
# Maintains separate history for this project
```

**3. Clear history without losing it:**
```bash
fc -p          # Push new empty history
# Do your work with clean history
fc -P          # Pop back to original history (note capital P)
```

## Related commands:

```bash
fc -P          # Pop history stack (return to previous)
fc -l          # List recent history
fc -li         # List with timestamps
fc -W          # Write current history to file
fc -R          # Read history from file
```

## Check if you're in ZSH:
```bash
if [ -n "$ZSH_VERSION" ]; then
    echo "You're running ZSH"
    fc -p  # This will work
else
    echo "Not ZSH, fc -p won't work"
fi
```

## Practical example - sensitive work:
```bash
# Start private session
fc -p /dev/null
# Do sensitive work
ssh user@secret-server
mysql -u root -p
# Exit without saving any of this
fc -P  # or just close terminal
```

**Note:** `fc -p` is ZSH-only. In bash, you'd need different approaches like `HISTFILE=/dev/null` or `set +o history` for similar functionality.
