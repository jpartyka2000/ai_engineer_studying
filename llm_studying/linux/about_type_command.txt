The `type` command in bash and zsh tells you how the shell interprets a command name. It answers the question: "When I type this command, what actually runs?"

**Basic usage:**
```bash
type <command_name>
```

**What it reveals:**

- **External program:** `type ls` might show `ls is /bin/ls`, meaning it runs the actual binary at that path.

- **Shell builtin:** `type cd` shows `cd is a shell builtin`, meaning it's built into the shell itself, not a separate program.

- **Function:** `type cat` showing `cat is a shell function` means someone defined a function that overrides the real command (this was your issue).

- **Alias:** `type ll` might show `ll is an alias for ls -l`, meaning it's a shortcut defined somewhere in your config.

**Useful variations:**

- `type -a cat` shows *all* definitions (helpful if something is both a function and an external command)
- `which cat` is similar but only finds external programs in your PATH
- `command -v cat` is another portable alternative

It's a great debugging tool when a command behaves unexpectedly — often something has shadowed the real command with an alias or function.
