QUESTION:
---------
Level: Easy | Area: Path Creation & Basics | Type: Multiple Choice

What is the result of joining paths with the `/` operator?

    from pathlib import Path
    base = Path("/home/user")
    result = base / "documents" / "file.txt"
    print(result)

A) "/home/user/documents/file.txt" (a string)
B) Path("/home/user/documents/file.txt") (a Path object)
C) TypeError — cannot use `/` with paths
D) ["home", "user", "documents", "file.txt"]

YOUR ANSWER:
------------
B) Path("/home/user/documents/file.txt") (a Path object)

CORRECT ANSWER:
---------------
B) Path("/home/user/documents/file.txt") (a Path object)

EXPLANATION:
------------
The `/` operator is overloaded for Path objects to provide intuitive path 
joining. It always returns a new Path object, not a string.

    base = Path("/home/user")
    result = base / "documents" / "file.txt"
    
    type(result)  →  <class 'pathlib.PosixPath'>  (or WindowsPath)
    str(result)   →  "/home/user/documents/file.txt"

ALTERNATIVE — joinpath() method:
    base.joinpath("documents", "file.txt")  →  same result
    base.joinpath("documents/file.txt")     →  also works

This is one of pathlib's most elegant features — it makes path construction 
readable and avoids error-prone string concatenation or os.path.join().
