QUESTION:
---------
Level: Hard | Area: Reading/Writing Files | Type: Coding

Write code to append text to an existing file using pathlib (don't overwrite!):

    from pathlib import Path
    p = Path("log.txt")

    # Append "New log entry\n" to the file
    ______

YOUR ANSWER:
------------
p.append_text()

CORRECT ANSWER:
---------------
with p.open("a") as f:
    f.write("New log entry\n")

EXPLANATION:
------------
Pathlib does NOT have an append_text() method! This is a known limitation.

    p.write_text("hello")   # Overwrites entire file
    p.append_text("more")   # Does NOT exist!

THE SOLUTION — Use p.open():
----------------------------
Path objects have an open() method that works like the built-in open():

    with p.open("a") as f:      # "a" = append mode
        f.write("New log entry\n")

ALTERNATIVELY:
--------------
    # Using built-in open() with Path object
    with open(p, "a") as f:
        f.write("New log entry\n")
    
    # One-liner (read + write, less efficient for large files)
    p.write_text(p.read_text() + "New log entry\n")

P.OPEN() MODE OPTIONS:
----------------------
    p.open("r")    →  read (default)
    p.open("w")    →  write (overwrite)
    p.open("a")    →  append
    p.open("rb")   →  read binary
    p.open("wb")   →  write binary

WHY NO APPEND_TEXT()?
---------------------
It's a design gap in pathlib. The convenience methods (read_text, 
write_text, read_bytes, write_bytes) cover the most common cases, 
but appending requires falling back to open().
