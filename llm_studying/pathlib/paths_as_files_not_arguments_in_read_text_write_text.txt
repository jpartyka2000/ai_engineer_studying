PATHLIB TUTOR - EVALUATION
═══════════════════════════════════════════════════════════════

Level: Hard
Type: Coding
Area: File Operations

───────────────────────────────────────────────────────────────
QUESTION:
───────────────────────────────────────────────────────────────
Write code that reads the contents of a text file at 
/home/user/data.txt and writes it to a new file at 
/home/user/backup.txt using only pathlib methods (no open()).

───────────────────────────────────────────────────────────────
YOUR ANSWER:      
text_str = p.read_text("/home/user/data.txt")
p.write_text(text_str, "/home/user/backup.txt")

CORRECT ANSWER:   
Path("/home/user/backup.txt").write_text(Path("/home/user/data.txt").read_text())

RESULT:           ✗ INCORRECT
───────────────────────────────────────────────────────────────
EXPLANATION:
───────────────────────────────────────────────────────────────
You identified the right methods (.read_text() and .write_text())
but used them incorrectly. These methods are called ON a Path 
object — the path isn't passed as an argument.

The key insight:
───────────────────────────────────────────────────────────────
In pathlib, the Path object IS the file. Methods operate on 
that path directly.

  ✗ p.read_text("/some/path")      # Wrong: path as argument
  ✓ Path("/some/path").read_text() # Right: method on Path object

Correct solutions:
───────────────────────────────────────────────────────────────
# Solution 1: Concise one-liner
Path("/home/user/backup.txt").write_text(
    Path("/home/user/data.txt").read_text()
)

# Solution 2: More readable with variables
source = Path("/home/user/data.txt")
dest = Path("/home/user/backup.txt")
content = source.read_text()
dest.write_text(content)

# Solution 3: Using a single base path
base = Path("/home/user")
content = (base / "data.txt").read_text()
(base / "backup.txt").write_text(content)

Method signatures:
───────────────────────────────────────────────────────────────
.read_text(encoding=None, errors=None)
    → Returns file contents as a string
    
.write_text(data, encoding=None, errors=None, newline=None)
    → Writes string to file, returns number of bytes written

Related methods for binary files:
───────────────────────────────────────────────────────────────
.read_bytes()        → Returns contents as bytes
.write_bytes(data)   → Writes bytes to file

Example:
    Path("image.png").write_bytes(Path("original.png").read_bytes())
