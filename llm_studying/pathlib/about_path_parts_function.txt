QUESTION:
---------
Level: Hard | Area: Path Properties | Type: Multiple Choice

What does the `parts` property return for this path?

    from pathlib import Path
    p = Path("/home/user/docs/file.txt")
    print(p.parts)

A) ["/", "home", "user", "docs", "file.txt"]
B) ("/", "home", "user", "docs", "file.txt")
C) ["home", "user", "docs", "file.txt"]
D) ("/home", "/home/user", "/home/user/docs", "/home/user/docs/file.txt")

YOUR ANSWER:
------------
B) ("/", "home", "user", "docs", "file.txt")

CORRECT ANSWER:
---------------
B) ("/", "home", "user", "docs", "file.txt")

EXPLANATION:
------------
The `parts` property returns a TUPLE (not a list!) containing all components 
of the path.

Key details:
- Returns a tuple, so () not []  — this rules out A and C
- For absolute Unix paths, "/" is included as the first element
- For Windows, it might be ("C:\\", "Users", "name", ...) 
- Option D describes what you'd get iterating through p.parents (plus p itself)

    p = Path("/home/user/docs/file.txt")
    
    p.parts   →  ("/", "home", "user", "docs", "file.txt")
    p.parent  →  Path("/home/user/docs")
    p.parents →  sequence: /home/user/docs, /home/user, /home, /

This is useful for analyzing path structure or rebuilding paths component 
by component.
