QUESTION:
---------
Level: Hard | Area: File Operations | Type: Coding

Write code to delete a file using pathlib:

    from pathlib import Path
    p = Path("temp_file.txt")

    ______

YOUR ANSWER:
------------
p.delete()

CORRECT ANSWER:
---------------
p.unlink()

EXPLANATION:
------------
Pathlib uses unlink() to delete files — the name comes from the Unix 
system call that removes a file's directory entry (its "link").

    from pathlib import Path
    p = Path("temp_file.txt")
    
    p.unlink()   # File is deleted

HANDLING MISSING FILES:
-----------------------
    p.unlink()                   # Raises FileNotFoundError if missing
    p.unlink(missing_ok=True)    # No error if file doesn't exist (Python 3.8+)

DELETING DIRECTORIES:
---------------------
unlink() only works on files! For directories, use:

    p.rmdir()           # Delete empty directory only
    
    # For non-empty directories, use shutil:
    import shutil
    shutil.rmtree(p)    # Delete directory and all contents

COMPLETE FILE LIFECYCLE EXAMPLE:
--------------------------------
    from pathlib import Path
    
    p = Path("example.txt")
    p.touch()                    # Create
    p.write_text("Hello!")       # Write
    content = p.read_text()      # Read
    p.unlink()                   # Delete

WHY "UNLINK" AND NOT "DELETE"?
------------------------------
The name reflects Unix terminology. A file can have multiple "links" 
(directory entries pointing to it). unlink() removes one link — when 
the last link is removed, the file is deleted.
