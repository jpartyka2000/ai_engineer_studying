QUESTION:
---------
Level: Medium | Area: Navigation Methods | Type: Coding

Given a path, write an expression to get the path relative to a base directory:

    from pathlib import Path

    file_path = Path("/home/user/projects/webapp/src/app.py")
    base = Path("/home/user/projects")

    relative = ______
    # Expected result: Path("webapp/src/app.py")

YOUR ANSWER:
------------
p.follow(file_path)

CORRECT ANSWER:
---------------
file_path.relative_to(base)

EXPLANATION:
------------
The relative_to() method returns a new Path representing the portion of 
the path that comes after the base path.

    file_path = Path("/home/user/projects/webapp/src/app.py")
    base = Path("/home/user/projects")
    
    file_path.relative_to(base)  →  Path("webapp/src/app.py")

HOW IT WORKS:
-------------
    /home/user/projects/webapp/src/app.py
    ^^^^^^^^^^^^^^^^^^^
    base gets removed, leaving: webapp/src/app.py

IMPORTANT RESTRICTIONS:
-----------------------
The path MUST actually start with the base, or you get an error:

    Path("/home/user/file.txt").relative_to("/other")
    # Raises: ValueError: '/home/user/file.txt' is not in the subpath of '/other'

PYTHON 3.12+ — walk_up PARAMETER:
---------------------------------
    # If paths don't share a base, use walk_up=True to add ".." components
    Path("/home/user/docs").relative_to("/home/user/projects", walk_up=True)
    # Returns: Path("../docs")

COMMON USE CASE:
----------------
    # Display relative paths in a project
    project_root = Path("/home/user/myproject")
    for py_file in project_root.rglob("*.py"):
        print(py_file.relative_to(project_root))
