### Two easy work‚ÄĎarounds

1. **Use the legacy‚ÄĮ`TreeExplainer` directly**

   ```python

   # bypass the new Explainer API
   explainer = shap.TreeExplainer(model)  
   shap_values = explainer.shap_values(X_clean)
   ```

   This avoids the buggy wrapper path altogether.

2. **Pass a raw prediction function into `shap.Explainer`**

   ```python

   # treat model.predict as a plain function
   explainer = shap.Explainer(model.predict, X_clean)
   shap_values = explainer(X_clean)
   ```

   By giving `shap.Explainer` a function rather than the model object, it falls back to a simpler code path that doesn‚Äôt hit the `.values` bug.
