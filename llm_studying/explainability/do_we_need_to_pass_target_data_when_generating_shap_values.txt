For most SHAP explainers, especially the ones built for tree models (like TreeExplainer), you only need to pass in the feature data (X) when calling `shap_values()`. The target values (y) aren’t required because the model’s predictions are computed solely from the input features. 

However, if you were using a different explainer—such as KernelExplainer—you might sometimes pass in a background dataset (which isn’t exactly the target labels, but a representative sample of the features) to help with sampling. But even then, the targets themselves are not typically passed as an argument to `shap_values()`. 

So in your case, passing only X is expected behavior and not problematic for a TreeExplainer.
