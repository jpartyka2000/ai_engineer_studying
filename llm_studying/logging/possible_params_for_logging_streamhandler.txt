The `StreamHandler` constructor actually looks like this:

```python
class StreamHandler(Handler):
    def __init__(self, stream=None):
        super().__init__()
        if stream is None:
            stream = sys.stderr
        self.stream = stream
```

* **Default**: if you don’t pass anything, it uses `sys.stderr`.
* **You can pass** *any* file-like object that implements at least `.write()` (and ideally `.flush()`), for example:

  * `sys.stdout`
  * an open file (`open("out.log", "a")`)
  * an in-memory buffer (`io.StringIO()`)
  * even a custom object you’ve written that captures writes somewhere else

So it’s *not* limited to only `stderr` or `stdout`—you’re free to direct your logs to whatever stream-style object makes sense.
