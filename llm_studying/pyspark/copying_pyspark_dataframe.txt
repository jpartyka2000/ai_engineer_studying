PySpark DataFrames are immutable, so you don’t really “copy” them in the same way you might with a mutable object. Instead, when you assign a DataFrame to a new variable, you're just creating another reference to the same underlying data. Any transformation you perform on a DataFrame creates a new DataFrame anyway. 

If you really want a separate object with the same data, you can do one of the following:

1. **Simple Assignment:**  
   ```python
   df_copy = df
   ```
   This creates another reference to the same DataFrame.

2. **Select All Columns:**  
   If you want to emphasize that you're creating a new DataFrame (even though the data is the same), you can do:
   ```python
   df_copy = df.select("*")
   ```
   This returns a new DataFrame object that contains the same data as `df`.

3. **Using alias():**  
   You can also use the `alias` method:
   ```python
   df_copy = df.alias("df_copy")
   ```
   This gives the new DataFrame an alias name, which can be useful in certain contexts (like self-joins).

Remember, because DataFrames are immutable, these methods won't create a deep copy of the data in memory. Instead, they just create new references in your DAG of transformations. This is usually sufficient for most use cases in PySpark.
