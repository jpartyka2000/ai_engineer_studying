When you use the standard `explode()` function, if a row’s "campaign_set" is empty (or null), that row will not appear in the resulting DataFrame because `explode()` produces one row per element in the array. If there are no elements to produce, no row is output for that original row.

If you want to retain those rows with empty or null arrays (but with a null value for the exploded column), you can use the `explode_outer()` function instead. For example:

```python
from pyspark.sql.functions import explode_outer

df_exploded = df.withColumn("campaign", explode_outer("campaign_set"))
```

This will include all original rows in the output, assigning a null value to "campaign" for rows where "campaign_set" is empty or null.
