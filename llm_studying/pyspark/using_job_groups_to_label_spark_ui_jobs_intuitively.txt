You can use the SparkContext's job group functionality to label a group of operations with a descriptive name. This lets you see that description in the Spark UI, which helps you identify jobs corresponding to specific operations—like joining two DataFrames. For example:

```python
# Get the SparkContext from your Spark session
sc = spark.sparkContext

# Set a job group with a descriptive label for the join
sc.setJobGroup("join_job", "Joining df1 and df2", interruptOnCancel=True)

# Perform your join and trigger an action
joined_df = df1.join(df2, on="join_key")
joined_df.count()  # or .show() to trigger the execution

# Optionally, clear the job group once done
sc.clearJobGroup()
```

### Explanation

- **`setJobGroup(groupId, description, interruptOnCancel)`**:  
  This method allows you to assign a group ID and a description to all subsequent operations. In the Spark UI, you'll see the description ("Joining df1 and df2") for that job group.

- **Triggering an Action**:  
  The join is only executed when an action (like `count()` or `show()`) is called, at which point the job group description appears in the Spark UI.

- **Clearing the Job Group**:  
  Calling `sc.clearJobGroup()` resets the job group settings.

This approach helps you identify which Spark job corresponds to the join operation when you inspect the Spark UI in Databricks.
