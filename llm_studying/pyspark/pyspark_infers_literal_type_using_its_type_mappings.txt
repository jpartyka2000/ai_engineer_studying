PySpark infers the data type from the Python type you pass to `F.lit()`.

**Default mappings:**

| Python Type | PySpark Type |
|-------------|--------------|
| `int` | `LongType` |
| `float` | `DoubleType` |
| `str` | `StringType` |
| `bool` | `BooleanType` |
| `None` | `NullType` |
| `datetime.date` | `DateType` |
| `datetime.datetime` | `TimestampType` |
| `bytes` | `BinaryType` |
| `decimal.Decimal` | `DecimalType` |
| `list` | `ArrayType` |
| `dict` | `MapType` |

**Examples:**

```python
from pyspark.sql import functions as F

df.select(
    F.lit(42),           # LongType (not IntegerType)
    F.lit(3.14),         # DoubleType (not FloatType)
    F.lit("hello"),      # StringType
    F.lit(True),         # BooleanType
    F.lit(None),         # NullType
)
```

**Notable gotcha:**

Python `int` becomes `LongType`, not `IntegerType`. If you specifically need `IntegerType`, you must cast:

```python
F.lit(42).cast("integer")
F.lit(42).cast(IntegerType())
```

Same for `float` → `DoubleType`. If you need `FloatType`:

```python
F.lit(3.14).cast("float")
```
