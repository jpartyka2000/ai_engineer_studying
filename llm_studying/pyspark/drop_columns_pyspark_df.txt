To perform the equivalent operation in **PySpark** for dropping specific columns and resetting the index, you can use the `drop()` method to remove columns, and since **PySpark DataFrames** do not have an index like Pandas, the `reset_index()` part is not necessary in PySpark.

Here's the equivalent **PySpark** code:

### PySpark Equivalent Code:

```python
# Drop the specified columns from the DataFrame
df_joined_company_dc = df_joined_company_dc.drop('type_id', '_fivetran_deleted', '_fivetran_synced', 'id')

# Show the result
df_joined_company_dc.show()
```

### Explanation:
- **`drop()`**: In PySpark, `drop()` is used to remove columns from the DataFrame. You can pass the column names directly as arguments, separated by commas.
- **No `reset_index()` needed**: PySpark DataFrames do not use indices like Pandas DataFrames, so you don't need to reset the index in PySpark.

This will drop the columns `['type_id', '_fivetran_deleted', '_fivetran_synced', 'id']` from the `df_joined_company_dc` DataFrame.
