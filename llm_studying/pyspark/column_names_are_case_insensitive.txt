In **Pandas**, column labels are just Python strings, so they are **case‑sensitive**.

```python
import pandas as pd
df = pd.DataFrame({'Age': [20,30], 'age': [5,10]})
# df['Age'] ≠ df['age']
```

Here, `df['Age']` and `df['age']` refer to entirely different columns. ([Codecademy Forums][1])

In **PySpark** (Spark SQL), by contrast, identifier resolution is **case‑insensitive by default**. That means

```python
spark.conf.get("spark.sql.caseSensitive")  # → "false"
df.select("Age")   # matches column named "age", "AGE", or "Age"
```

If you really need strict case‑sensitivity, you can turn it on via:

```python
spark.conf.set("spark.sql.caseSensitive", True)
```

Once enabled, Spark will distinguish `Age` vs. `age`. ([stackoverflow.com][2])
