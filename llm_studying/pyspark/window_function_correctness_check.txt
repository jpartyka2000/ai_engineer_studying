To verify that your PySpark window function with a window size of 3 is correctly partitioning and ordering the rows, you can use the `F.collect_list()` function to capture the values of `'year_quarter_numeric'` in each window and display them alongside the original values. ### Step-by-Step Approach:

### PySpark Code Example:

```python
from pyspark.sql import functions as F
from pyspark.sql.window import Window

# Define your window spec with 3 quarters
window_spec_stage_ravg_3_qtr = Window.partitionBy('opportunity_record_type').orderBy('year_quarter_numeric').rowsBetween(-2, 0)

# Use collect_list() to gather the 'year_quarter_numeric' values in each window
df_verification = df.withColumn(
    'year_quarters_in_window',

```

- **`Window.partitionBy('opportunity_record_type')`**: This ensures that the window function groups rows by `opportunity_record_type`.

- **`orderBy('year_quarter_numeric')`**: This orders the rows by `year_quarter_numeric` within each partition.

- **`rowsBetween(-2, 0)`**: This specifies a window that includes the current row and the previous two rows, which ensures a 3-row window.
