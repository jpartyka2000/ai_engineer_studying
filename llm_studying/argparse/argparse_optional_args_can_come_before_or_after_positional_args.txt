Yes — in **argparse**, optional arguments (the ones with a dash like `-v` or `--count`) can be placed either **before or after positional arguments**.

---

### Example

```python
import argparse

parser = argparse.ArgumentParser()
parser.add_argument("--verbose", action="store_true")
parser.add_argument("filename")
args = parser.parse_args()
print(args)
```

#### All of these work fine:

```bash
python script.py --verbose data.txt
python script.py data.txt --verbose
```

Output:

```
Namespace(verbose=True, filename='data.txt')
```

---

### Why this works

* **Optional arguments** are recognized by their `-` or `--` prefix, so their position doesn’t matter.
* **Positional arguments** are identified only by order, so they must appear in the sequence you defined in code.

---

✅ **Summary:** You can put optional arguments **before or after positional arguments** — argparse will parse them correctly. Only the **relative order of multiple positionals** matters.
