The Apple M3 Pro uses a **System-on-a-Chip (SoC)** design where the CPU, GPU, Neural Engine, and other components are all fabricated onto a single piece of silicon.

**Unified Memory Architecture (UMA)**

Traditional computers have separate memory pools — the CPU uses system RAM while the GPU has its own dedicated VRAM. Data has to be copied back and forth over the PCIe bus, which creates latency and bandwidth bottlenecks.

On Apple Silicon, there's a single pool of high-bandwidth memory that all components share directly. The CPU, GPU, and Neural Engine can all access the same data without copying it around. For ML inference, this is a big win — the model weights sit in memory once, and the GPU can access them immediately.

**The M3 Pro specifically**

The M3 Pro integrates either 14 or 18 GPU cores (depending on configuration) right alongside the CPU cores. These GPU cores support hardware-accelerated ray tracing and mesh shading, but for LLM work, what matters most is:

- Direct memory access without transfer overhead
- Memory bandwidth around 150 GB/s
- Metal framework support for GPU-accelerated inference

**Why this matters for local LLMs**

When you run Mistral 7B through Ollama, the Metal backend offloads matrix multiplications to those integrated GPU cores. Because there's no data transfer penalty, you get performance closer to what you'd expect from a discrete GPU setup — just in a laptop form factor with much better power efficiency.

It's a fundamentally different approach than the traditional CPU + discrete GPU split.
