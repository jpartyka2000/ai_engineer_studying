By default, Databricks AutoML logs the best model run (along with its parameters, metrics, and artifacts) to MLflow, but it does not automatically register that model into the MLflow Model Registry. 

If you want the best model to appear in the Model Registry, you need to manually register it using the MLflow UI or MLflow’s API. In some newer versions or specific workflows, there may be options to auto-register models, but the standard behavior is that AutoML tracks the run without promoting it to the registry automatically.
