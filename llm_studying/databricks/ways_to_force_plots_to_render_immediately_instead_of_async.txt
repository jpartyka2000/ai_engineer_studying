This is a common issue in Jupyter/Databricks notebooks! The problem occurs because matplotlib renders plots asynchronously, while print statements execute immediately. Here are several solutions:

## Solution 1: Use plt.show() (Recommended)
Add `plt.show()` after your plot code to force matplotlib to render immediately:

```python
import matplotlib.pyplot as plt

# Your plotting code
plt.plot([1, 2, 3], [4, 5, 6])
plt.title("My Plot")
plt.show()  # Force the plot to render now

# Now print
print("This will appear after the plot")
print([1, 2, 3, 4])
```

## Solution 2: Use display() for controlled output
In Databricks, you can use the `display()` function:

```python
import matplotlib.pyplot as plt

fig, ax = plt.subplots()
ax.plot([1, 2, 3], [4, 5, 6])
display(fig)  # Explicitly display the figure

print("This will appear after the plot")
```

## Solution 3: Close the figure after displaying
This ensures matplotlib is done with the figure:

```python
import matplotlib.pyplot as plt

plt.plot([1, 2, 3], [4, 5, 6])
plt.title("My Plot")
plt.show()
plt.close()  # Close the figure

print("This will appear after the plot")
```

## Solution 4: Separate cells
The simplest approach is to put plots and print statements in separate notebook cells, which guarantees execution order.

The `plt.show()` method (Solution 1) is typically the most reliable and is considered best practice for ensuring proper output ordering in notebooks.
