**No — creating a branch in Databricks Git folders creates a local branch first.**

The branch is only pushed to the remote when you click **Commit & Push**.

---

**The workflow:**

1. Open the Git dialog. Click Create Branch. Enter a name for the new branch and select the base branch.

2. Make your changes to notebooks and files

3. Add a required commit message for the changes, and click Commit & Push to push the changes to the remote Git repository.

---

**Evidence that branches are local until pushed:**

The local version of a branch can remain present in the associated Git folder for up to 30 days after the remote branch is deleted.

This confirms that Databricks maintains local branches separately from remote branches.

---

**The CI/CD workflow also confirms this:**

A user creates a local user-specific branch for the work the user will commit to it and push to the remote repository.

Use the Git folders UI to create a feature branch from the main branch. You can create and use multiple feature branches to do your work. Make your modifications to Databricks notebooks and other files in the repository. Commit and push your changes to the remote Git repository.

---

**So the answer is:** When you create a new empty branch in Databricks, it exists only locally in your Git folder. The branch appears on the remote repository only after you make changes and click "Commit & Push".
