To access the terminal functionality in Databricks notebooks when the web terminal isn't available due to shared access mode, you can use the **bottom terminal feature in a notebook**. Here's how you can do it:

### Steps to Use the Bottom Terminal in a Notebook:
1. **Open a Databricks Notebook**:
   Navigate to the Databricks workspace and open any notebook associated with your cluster.

2. **Switch to the Command Line Terminal**:
   - At the bottom of the notebook, you will see an area labeled "Command Line Terminal" or similar, depending on your Databricks environment version.
   - If this terminal isn’t immediately visible, click on the small terminal icon (usually near the notebook toolbar or footer).

3. **Enter Commands**:
   - In this bottom terminal, you can type and execute shell commands directly on the cluster.
   - This is equivalent to accessing the cluster through the web terminal but within the notebook interface.

4. **Example Use**:
   - If you want to list the files in the working directory, type:
     ```bash
     ls
     ```
   - If you want to navigate to a specific directory, use:
     ```bash
     cd /path/to/directory
     ```

5. **Limitations**:
   - The notebook terminal is scoped to the current notebook session and doesn't provide full terminal functionality like SSH or a standalone web terminal.

If you need more robust terminal access, consider creating a cluster with a **Single User Access Mode**, which allows you to use the web terminal on the cluster apps page.
