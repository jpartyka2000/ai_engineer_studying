That `from databricks import sql` failing almost always means the Databricks SQL Connector for Python isn’t installed in your driver/process. You have two quick ways to get it onto your cluster:

---

## 1. Install via the UI (cluster-wide)

1. Go to **Compute → \[your cluster] → Libraries**
2. Click **Install New → PyPI**
3. In “Package” type:

   ```
   databricks-sql-connector
   ```
4. (Optionally pin a version, e.g. `databricks-sql-connector==0.10.0`)
5. Click **Install** and then **Restart** your cluster.

After the restart your notebooks and MLflow PyFuncs will be able to do:

```python
from databricks import sql
```

---

## 2. Install in a notebook session (session-scope)

If you just want to test in a notebook, run at the top of your first cell:

```python
%pip install databricks-sql-connector
```

Then immediately restart that notebook’s Python session (you’ll get a prompt after install). Once it’s back up:

```python
from databricks import sql
```

---

### Bonus: MLflow serving

If you’re packaging this into an MLflow pyfunc, be sure to list `databricks-sql-connector` under `pip` in your `conda_env.yml` or `MLmodel` so that the scoring container has it too.
