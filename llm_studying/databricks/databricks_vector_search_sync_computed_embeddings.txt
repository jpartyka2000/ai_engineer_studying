Yes, you should enable the **"Sync Computed Embeddings"** option in your example if you want the embedding vectors for **Column A** to be computed once and stored in the vector search index for subsequent use without needing to recompute them.

### Why Enable Sync for Computed Embeddings
- When you **sync computed embeddings**, the embedding vectors are stored directly in the index.
- After this initial computation and sync, future queries will use these precomputed embeddings, eliminating the need for additional computation at query time.

### Workflow with "Sync Computed Embeddings" Enabled
1. **Initial Setup**: The embedding model computes embeddings for the data in **Column A** (e.g., a text description).
2. **Storage in the Index**: These embeddings are stored in the vector index.
3. **Subsequent Queries**: When you perform a vector search, the stored embeddings are used directly, ensuring fast lookup without recomputation.

### If You Disable Sync for Computed Embeddings
If you choose not to sync the computed embeddings, the system would need to recompute embeddings on-the-fly whenever a query is made. This is less efficient, especially for large datasets or frequent queries, as it incurs extra computational overhead.

### Recommendation for Your Use Case
- **Enable "Sync Computed Embeddings"** for **Column A** to compute the embeddings once and use them efficiently for all future searches.
- **Do not enable embedding sync** for **Column B**; include it only as a metadata column to store the corresponding return values.
