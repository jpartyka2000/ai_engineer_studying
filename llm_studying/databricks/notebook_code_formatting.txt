In a Databricks notebook, you can apply a code formatter to maintain consistent code style and improve readability. Here's how you can enable and use formatting in a Databricks notebook:

---

### 1. **Using Built-In SQL Formatter**
Databricks notebooks have a built-in SQL formatter for cells containing SQL code. 

#### Steps:
1. **Enable the SQL Formatter:**
   - At the top-right corner of a SQL cell, click on the "..." (more options) menu.
   - Select **"Format SQL"** from the dropdown menu. This will auto-format the SQL code in the cell.

2. **Keyboard Shortcut:**
   - Press `Ctrl` + `Shift` + `F` (Windows) or `Cmd` + `Shift` + `F` (Mac) to format the SQL cell.

---

### 2. **Using Databricks Utilities for Python or Scala**
For other languages like Python or Scala, Databricks doesn't provide a built-in formatter, but you can integrate third-party formatters like `Black` for Python.

#### Example: Using `Black` Formatter in Python
1. Install `Black` in your Databricks cluster:
   ```python
   %pip install black
   ```

2. Format a Python cell programmatically:
   ```python
   from black import format_str, FileMode

   code = """
   def example_function(x, y): 
   return x+y
   """
   formatted_code = format_str(code, mode=FileMode())
   print(formatted_code)
   ```

---

### 3. **Configuring Notebook Settings**
You can also enable formatting at the notebook level:
1. Click the **gear icon** in the top-right corner of the notebook.
2. Select **Notebook Settings**.
3. Toggle options for language-specific features (e.g., SQL autocomplete and formatting).

---

### 4. **Using External IDEs**
If you prefer advanced formatting, you can edit your code in an external IDE like PyCharm, VS Code, or IntelliJ, which support formatters such as `Black`, `Prettier`, or `scalafmt`. Once formatted, copy the code back into the Databricks notebook.
