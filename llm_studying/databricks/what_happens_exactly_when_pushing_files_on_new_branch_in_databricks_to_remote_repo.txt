When you commit and push a new branch for the first time, several things happen:

---

**1. Local commit is created**

Git packages your changes into a commit object containing:
- A tree object (snapshot of your files)
- Blob objects (the actual file contents)
- Metadata (author, timestamp, commit message)
- A pointer to the parent commit (the commit your branch was based on)

---

**2. Push creates the branch on the remote**

When you push, Git:
- Sends the commit object (and any trees/blobs the remote doesn't have) to the remote repository
- Creates the branch reference on the remote (e.g., `refs/heads/feature-x`)

---

**3. Local tracking is established**

After the push, your local branch is configured to track the new remote branch:
- A remote-tracking branch is created locally (`origin/feature-x`)
- Your local branch's upstream is set (`branch.feature-x.remote` and `branch.feature-x.merge`)

---

**In Databricks specifically:**

The "Commit & Push" button does all of this in one action:

```
git add <files>
git commit -m "your message"
git push -u origin feature-x
```

The `-u` flag sets up the tracking relationship automatically.

---

**After the first push:**

Your new branch now exists in three forms:

| Location | Ref | Example |
|----------|-----|---------|
| Remote repository | `refs/heads/feature-x` | Lives on GitHub/GitLab/etc. |
| Local remote-tracking | `refs/remotes/origin/feature-x` | Your local cache of remote state |
| Local branch | `refs/heads/feature-x` | Where you work |
