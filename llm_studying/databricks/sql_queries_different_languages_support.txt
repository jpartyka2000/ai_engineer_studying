Databricks does not natively support running SQL queries in different spoken languages directly within the editor. SQL itself is an English-based standard, and most database systems, including Databricks, follow this convention. The keywords (`SELECT`, `WHERE`, `INSERT`, etc.) and syntax in SQL are consistent regardless of the spoken language of the user.

---

### **Options for Multilingual Support in Databricks**
While Databricks doesn't support SQL in other spoken languages directly, there are ways to provide multilingual support indirectly:

1. **Localization of Comments and Documentation**:
   - You can include comments in notebooks in your preferred spoken language for better understanding by non-English speakers.
     ```sql
     -- Selecciona todas las filas de la tabla "usuarios"
     SELECT * FROM usuarios;
     ```

2. **Translation Tools**:
   - Use external tools to translate SQL queries or descriptions into your preferred language.
   - Example: Translate query logic or variable names using automated tools like Google Translate, but the SQL syntax remains in English.

3. **Customized Tools and Wrappers**:
   - You can build a wrapper around SQL to accept input in a specific language, translate it to standard SQL syntax, and execute it. However, this would require significant development effort.

4. **Multilingual Query Results**:
   - While the SQL itself is in English, you can structure the data in a way that supports multilingual text in results (e.g., storing names or descriptions in different languages in the database).

5. **Interface Language in Databricks**:
   - Databricks' user interface is primarily in English, but modern browsers or extensions can translate the interface text into other languages.

---

### **Conclusion**
While SQL itself is inherently tied to English as the standard language, with external tools, localized comments, or customized systems, you can create a multilingual-friendly environment in Databricks. However, the core query language will remain in English.
