In a Databricks notebook, you can format all your Python and SQL cells at once using the built-in formatter. Go to **Edit > Format Notebook** in the menu bar. This applies black formatting to every Python cell and SQL formatting to every SQL cell in one pass.

You can also format individual cells. Right-click on a Python cell and select **"Format Python"**, or use the keyboard shortcut **Cmd+Shift+F** (Ctrl+Shift+F on Windows). For SQL cells, the option says **"Format SQL"** in the cell's context dropdown menu.

**What black does to your code:**

Black is an opinionated formatter — it enforces a consistent style without giving you options. It will reformat things like:

```python
# Before
def add(a,        b):
    answer  =  a   +       b
    return    answer

# After
def add(a, b):
    answer = a + b
    return answer
```

It standardizes spacing, line breaks, trailing commas, quote styles, and line length. The goal is that all black-formatted code looks the same regardless of who wrote it.

**Requirements:**

On Databricks Runtime 11.3 LTS and above, black comes preinstalled — no setup needed. On older runtimes (10.4 LTS and below), you need to install it yourself by running `%pip install black==22.3.0 tokenize-rt==4.2.1` in a notebook cell.

**Configuration:**

Black enforces 4-space indentation and this is not configurable. However, if your notebook is in a Databricks Git folder, you can add a `pyproject.toml` file to the root of the repo with a `[tool.black]` section to configure other settings like line length. That configuration will apply whenever you format any notebook in that Git folder.

**Things to keep in mind:**

Formatting is not automatic — you have to trigger it manually each time. It won't format Python strings inside SQL UDFs, or SQL strings inside Python functions. And you need **CAN EDIT** permission on the notebook to use the formatter.
