When you run a standalone `.py` script, whether on your local machine, in a Databricks `%run my_script.py`, or as a submitted job, Python:

1. **Reads & compiles** the entire file up front (or at least reads each module as it’s imported).
2. **Executes** the compiled code in memory.

Any edits you make to the `.py` file **after** the interpreter has read and compiled that section of code will **not** be reflected in the current run. You’d have to stop and re-import or re-run the script to pick up your changes.

—In short, unlike a notebook where each cell is fetched live at execution time, a `.py` script is loaded into Python’s process at start, so on-disk edits mid-execution have no effect on the code that’s already been read into memory.
