Unity Catalog is required in Databricks when you’re using a Unity Catalog–enabled workspace to centrally govern your data and AI assets. In practice, this means that if you want to use the MLflow Model Registry, tables, or other data assets under the umbrella of centralized, fine‑grained governance, Unity Catalog must be enabled. For example:

- **MLflow Model Registry:**  
  In a Unity Catalog–enabled workspace, when you register a model with MLflow, Unity Catalog enforces that the model has a complete signature (input and output types) to ensure safe serving and reproducibility.  
- **Data Governance and Security:**  
  Unity Catalog provides centralized access control, auditing, and metadata management for Delta tables, files, and other assets. If your organization uses Unity Catalog to manage data governance, then all data and AI assets—including models—must comply with its requirements.

If you’re working in a legacy workspace that isn’t Unity Catalog–enabled, these restrictions wouldn’t apply. However, as Databricks moves toward a unified governance model, many new workspaces are built on Unity Catalog, making it a requirement for those environments.
