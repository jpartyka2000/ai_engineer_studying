If I am selecting a few columns from table 1 and all columns from table 2 and joining them together, can I use the * notation for table 2 only?

Yes, in Databricks (and SQL in general), you can use the `*` notation to select **all columns from one table** while specifying individual columns from another table. When performing a **join** between two tables, you can specify the columns from **table1** explicitly and use the `*` to select all columns from **table2**.

### Example:

Assume you have two tables:
- **table1** with columns `col1`, `col2`, `col3`
- **table2** with columns `colA`, `colB`, `colC`

If you want to select only `col1` and `col2` from `table1` and all columns from `table2`, you can write the following query:

```sql
SELECT t1.col1, t1.col2, t2.*
FROM table1 t1
JOIN table2 t2
ON t1.id = t2.id;
```

### Explanation:
- **`t1.col1, t1.col2`**: Selects specific columns from `table1` (aliased as `t1`).
- **`t2.*`**: Selects **all columns** from `table2` (aliased as `t2`).
- **`ON t1.id = t2.id`**: This is the join condition.

This way, you're only retrieving selected columns from `table1`, while grabbing all columns from `table2` using the `*` notation.

### Summary:
You can combine selecting specific columns from one table and all columns from another table using the `*` notation for the second table in a SQL query. This approach is valid and efficient for joins in Databricks or any SQL-based environment.
