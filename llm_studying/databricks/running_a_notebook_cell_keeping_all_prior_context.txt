Databricks notebooks run in a shared, persistent driver session when attached to a cluster. This means that any variables, functions, imports, or data you define in earlier cells remain available to subsequent cells as long as you’re running on the same session (i.e., the cluster hasn’t been restarted or detached).

Here are some tips to ensure your context is maintained:

- **Stay Attached to the Same Cluster:**  
  Make sure you’re continuously attached to a live cluster. Restarting or detaching the cluster will clear the session state.

- **Run Cells in Order:**  
  Execute cells sequentially so that any dependencies are established before later cells run.

- **Use `%run` to Include Other Notebooks:**  
  If you need to share code across multiple notebooks, using `%run` imports the other notebook’s context into your current session.

- **Avoid Overwriting Important Variables:**  
  Be mindful of redefining variables or functions that you need later in the notebook.

By following these practices, you ensure that all the context from previous executions remains available throughout your notebook session.
