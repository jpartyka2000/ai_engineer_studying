In Databricks AutoML the MLmodel artifact includes a model signature—essentially a description of the input features and output—that is automatically generated based on the data used during training. In most cases, this signature accurately reflects the predictors that were actually used by the model at training time.

That said, a few caveats are worth noting:

- **Feature Engineering & Transformations:** If your AutoML process involves preprocessing or feature engineering that changes the original predictor set (for example, creating new features or excluding some), the model signature will reflect the features the model ultimately sees. It’s important to ensure that the signature is interpreted in that context.

- **Manual Adjustments:** If you manually modify the signature (using MLflow’s signature APIs) after training, then the artifact might not match the actual predictors used. But if you rely on AutoML’s default behavior, it’s generally accurate.

- **Versioning:** The MLmodel artifact represents the state of the model (and its inputs) at the time of that specific run. If you run AutoML multiple times with different predictor subsets, each run’s artifact will reflect the predictors used for that particular model.

So, under normal circumstances, the MLmodel artifact generated by Databricks AutoML is a reliable representation of the predictors used by the model.
