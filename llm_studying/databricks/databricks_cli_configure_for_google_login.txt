No, you should **not enter your Google account password** when the `databricks configure` command asks for a password. Databricks does not store or use Google passwords for authentication. If you're using Google SSO to log in, here's what you need to do:

### Steps for Google SSO Users:
1. **Use a Personal Access Token (PAT):**
   - Databricks CLI does not directly support Google SSO for authentication.
   - Instead of entering your Google password, you need to generate a **personal access token (PAT)**:
     1. Go to your Databricks workspace.
     2. Click on your user icon (top-right corner) and select **User Settings**.
     3. Navigate to the **Access Tokens** tab.
     4. Click **Generate New Token**, set an expiration date, and save the token securely.
   - When `databricks configure` asks for a password, paste the PAT instead.

2. **Run the `databricks configure` Command:**
   - Execute `databricks configure` and provide the following:
     - Databricks Host: (e.g., `https://dbc-<unique-instance-id>.cloud.databricks.com`)
     - Username: Your email address (used for your Databricks login).
     - Password: Paste the PAT you generated instead of a Google password.

By using a PAT, you authenticate securely and avoid issues with SSO.

Databricks CLI configuration details are stored in a hidden file on your computer. Here's where you can find them:

### Location of Configuration File
- **Default Path:** `~/.databrickscfg`  
  - On Unix-based systems (Linux/macOS), this resolves to `/home/<your-username>/.databrickscfg`.
  - On Windows, it resolves to `C:\Users\<your-username>\.databrickscfg`.

### Contents of the File
The file contains the credentials and configuration for each Databricks workspace you have configured, structured like this:

```ini
[DEFAULT]
host = https://<your-databricks-host>.cloud.databricks.com
token = <your-personal-access-token>

[profile1]
host = https://<your-other-databricks-host>.cloud.databricks.com
token = <your-personal-access-token-for-profile1>
```

- The `DEFAULT` section is used when you don't specify a profile explicitly in CLI commands.
- Other profiles can be added with their own `[profile-name]` headers.

### How to View the File
1. **On Unix/Linux/macOS:**
   - Use a command like:
     ```bash
     cat ~/.databrickscfg
     ```
   - Or open it with a text editor:
     ```bash
     nano ~/.databrickscfg
     ```

2. **On Windows:**
   - Open a file explorer and navigate to `C:\Users\<your-username>\`.
   - Ensure that "Show Hidden Files" is enabled (since `.databrickscfg` is hidden by default).
   - Open `.databrickscfg` with a text editor like Notepad.

### Security Considerations
- The file contains sensitive information like your personal access token. Do not share it or commit it to version control.
- Use proper file permissions to restrict access to your user only. For example:
  ```bash
  chmod 600 ~/.databrickscfg
  ```
