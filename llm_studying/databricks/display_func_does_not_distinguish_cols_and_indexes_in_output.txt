No, the display() function in a Databricks notebook does not show index columns distinctly — it only displays the DataFrame as a regular table, with all columns appearing as regular columns, regardless of whether they were originally indexes in the underlying Pandas or Spark DataFrame.

What this means:

If you're using a Pandas DataFrame, any index (even a MultiIndex) will be reset to a column when passed to display() unless you explicitly reset the index yourself.

If you're using a Spark DataFrame, there is no concept of an index at all — all columns are treated equally.



---

How to tell which are indexes (in Pandas):

If you're using a Pandas DataFrame in a Databricks notebook and want to inspect the index explicitly, use:

df.index

To show both index and columns:

print(df.index)
print(df.columns)

To convert the index into a column for clarity before using display():

df_reset = df.reset_index()
display(df_reset)

This will make previously hidden index levels visible as regular columns.


---

Summary:

🔍 display() does not distinguish index vs. non-index columns.

✅ To make index columns visible in display(), use .reset_index().

🧱 Spark DataFrames don’t have index columns — everything is a regular column.
