To create a new table in Databricks in **Parquet format** (which is more space-efficient due to its columnar storage and compression), you can use either **SQL** or **PySpark**. Below are the steps for both methods:

### 1. **Using SQL in Databricks**:

You can run the following SQL command in a **Databricks notebook** or **SQL editor** to create a table in Parquet format.

```sql
CREATE TABLE schema_name.new_table_name
USING PARQUET
AS 
SELECT * FROM existing_table;
```

### Explanation:
- **`CREATE TABLE`**: This command creates a new table.
- **`USING PARQUET`**: Specifies that the table should be saved in Parquet format.
- **`AS SELECT * FROM existing_table`**: Populates the new table with data from an existing table (you can replace this with any valid SQL query).

### Example:
```sql
CREATE TABLE my_schema.my_new_table
USING PARQUET
AS
SELECT * FROM my_schema.my_existing_table;
```

### 2. **Using PySpark in Databricks**:

If you prefer using **PySpark** in Databricks, you can write a DataFrame to a table in Parquet format as follows:

```python
# Load the existing table into a Spark DataFrame
df = spark.table("schema_name.existing_table")

# Write the DataFrame to a new table in Parquet format
df.write.format("parquet").saveAsTable("schema_name.new_table_name")
```

### Explanation:
- **`spark.table("schema_name.existing_table")`**: Loads the existing table into a Spark DataFrame.
- **`df.write.format("parquet").saveAsTable("schema_name.new_table_name")`**: Saves the DataFrame as a new table in Parquet format.

### Example:
```python
# Load data from an existing table
df = spark.table("my_schema.my_existing_table")

# Write the DataFrame to a new table in Parquet format
df.write.format("parquet").saveAsTable("my_schema.my_new_table")
```

### Why Use Parquet?
- **Space Efficiency**: Parquet is a columnar storage format, which allows for more efficient compression and faster querying, particularly when working with large datasets and selective columns.
- **Performance**: It improves query performance by only reading the columns that are required, rather than entire rows.

### Summary:
You can create a new table in Parquet format either using SQL or PySpark in Databricks. SQL allows for a simple creation of Parquet tables, while PySpark provides flexibility when working with DataFrames. Parquet helps save space and improve query performance.
