`displayHTML` is a built-in function in Databricks notebooks that renders raw HTML directly in the cell output. You don't need to import anything — it's available automatically.

**Basic usage:**

```python
displayHTML("<h1>Hello World</h1>")
```

**Common use cases:**

Displaying images:
```python
displayHTML('<img src="/files/path/to/image.png" width="800">')
```

Embedding base64 images (useful when the `/files/` URL mapping doesn't work):
```python
import base64
with open("/dbfs/tmp/my_plot.png", "rb") as f:
    img_data = base64.b64encode(f.read()).decode("utf-8")
displayHTML(f'<img src="data:image/png;base64,{img_data}" width="800">')
```

Rendering styled text or tables:
```python
displayHTML("""
<div style="background-color: #f0f0f0; padding: 10px; border-radius: 5px;">
    <h3>Summary</h3>
    <p>Total rows: <strong>357,857</strong></p>
</div>
""")
```

**Key things to know:**

It only works in Databricks notebooks — not in `.py` files, not in pipelines without an interactive display. It renders in the cell output area just like a plot from matplotlib would. The HTML you pass in is sandboxed in an iframe, so it won't affect the rest of the notebook's UI. You can use CSS, JavaScript, and even embed iframes to external content, though external resources need to be accessible from the browser.

It's particularly useful for situations where the standard `display()` function isn't flexible enough — like showing images, custom formatted reports, or interactive HTML widgets.
