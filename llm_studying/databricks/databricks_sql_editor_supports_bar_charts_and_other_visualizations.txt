You don't need a notebook. The Databricks SQL editor supports bar charts, histograms, and other visualization types directly. After your query runs, you can click the downward pointing arrow in the visualization tab to add or edit a chart, and use the Visualization Editor to configure it.

So you can run your query in the SQL editor, then add a bar chart visualization right there with `lead_quality_bucket` on the X axis and `bucket_count` on the Y axis. No notebook required.
