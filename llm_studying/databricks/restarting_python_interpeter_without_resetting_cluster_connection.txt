Yes, in Databricks you can restart the Python interpreter without detaching/reattaching the cluster using:

```python
dbutils.library.restartPython()
```

This will:
- Clear all Python state (variables, imports, cached data)
- Release memory held by Python objects
- Reset the Python interpreter

A few things to keep in mind:

1. **All variables are lost** – Any data in memory will be cleared, so make sure you've saved anything important to a table, file, or widget before calling this.

2. **Re-run imports** – After restarting, you'll need to re-import all your libraries.

3. **Widgets persist** – Databricks widgets survive the restart, so you can use them to pass values across the restart if needed.

4. **Cluster stays attached** – Unlike detach/reattach, the cluster connection remains, so it's faster.

This is particularly useful when:
- You've installed a new library with `%pip install` and need it to take effect
- You want to clear memory without losing your cluster connection
- You're debugging import/dependency issues

If you just need to clear variables but don't need a full restart, you could also use:

```python
%reset -f
```

This clears user-defined variables but doesn't restart the interpreter itself (so cached imports and some state may persist).
