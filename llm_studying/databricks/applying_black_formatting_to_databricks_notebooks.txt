Actually, you can! Databricks supports Python code formatting using black directly within notebooks. On Databricks Runtime 11.3 LTS and above, black and tokenize-rt come preinstalled.

You can trigger it a few ways:

- **Keyboard shortcut:** Cmd+Shift+F (or Ctrl+Shift+F on Windows)
- **Menu:** Edit > Format Notebook (formats all cells)
- **Per cell:** Right-click a Python cell and select "Format Python"

You can also configure black's settings via a `pyproject.toml` file in the root of your Git folder if you're using Databricks Git folders. One thing to note: black enforces PEP 8 standards for 4-space indentation, and indentation is not configurable.
