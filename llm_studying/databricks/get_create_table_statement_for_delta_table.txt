To generate a `CREATE TABLE` SQL statement for a table in the Databricks SQL editor (without including the data), follow these steps:

### Option 1: Using Databricks UI (if you're working with an existing table)
1. **Access the Table**: In the Databricks workspace, go to the **Data** section and locate the table you want to replicate.
2. **View Table Details**: Click on the table name to open its details.
3. **Generate DDL**: Look for an option like "Generate SQL" or "Generate DDL" (depending on your Databricks version). This will provide the `CREATE TABLE` statement for the table's schema without data.

### Option 2: Using SQL Commands
You can use the `SHOW CREATE TABLE` command:
```sql
SHOW CREATE TABLE table_name;
```
This will output the SQL statement required to recreate the table structure. Note that the table name should include the database if it's not in the default database, e.g., `database_name.table_name`.

### Option 3: Using Python or Notebook Script
If you want to extract the `CREATE TABLE` statement programmatically within a notebook:
```python
# Example in Python using Spark SQL
result = spark.sql("SHOW CREATE TABLE database_name.table_name")
print(result.collect()[0][0])
```

This will display the `CREATE TABLE` DDL for the specified table.

### Important Notes:
- The `CREATE TABLE` statement will not include the data; it only defines the schema and properties of the table.
- If you need to include additional options (e.g., table location or partitioning), check the properties provided in the generated DDL and adapt them as needed.
