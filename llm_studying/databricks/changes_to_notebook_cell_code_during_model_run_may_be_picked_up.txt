When you hit **Run All** in Databricks, the notebook will submit each cell, in order, to the active Python driver one after another. **Any edits you make before a cell actually executes will be picked up** when execution reaches that cell; edits you make after it’s already been sent to the driver will have no effect on that run.

Concretely:

* **Cells already executed** (or currently running) will keep using the code they were sent with.
* **Cells not yet executed** will use whatever text is in them at the moment they get sent.

So if you change a cell’s contents while the notebook is mid-run **but before that cell has executed**, you will see your updated code run. If you tweak a cell that’s already run (or is currently running), that change only takes effect the *next* time you run it.

---

### Tips

* If you want a completely reproducible “snapshot” run, avoid editing once you’ve hit **Run All**—any change will alter the rest of the run.
* If you do need to tweak, you can safely edit cells below the one that’s *currently* executing; those edits will be honored when execution reaches them.
* To reset to a clean slate, stop the run, make your edits, then start **Run All** again.
