Databricks is simply warning you that the libraries you added to your job/cluster don’t have fixed versions (“pins”) specified, so every time your job runs it will pull whatever the latest version happens to be from PyPI.

**Why it matters**

* **Reproducibility**: If you don’t pin, next week you might get pandas 1.5.7 instead of 1.5.6, and some API could have changed, breaking your code.
* **Stability**: Pinning locks you into a known-good set of dependencies, so you won’t suddenly have test-or-prod jobs start failing because someone pushed a backwards-incompatible release.

**How to “pin”**
Instead of writing in the UI or an init script:

```
pandas, numpy, scikit-learn
```

you should list each library **and** its exact version, for example:

```
pandas==2.3.0
numpy==1.26.4
scikit-learn==1.3.2
```

(or whatever versions you’ve tested against). Databricks will then install exactly those versions every time, and the “not pinned” warning will go away.
