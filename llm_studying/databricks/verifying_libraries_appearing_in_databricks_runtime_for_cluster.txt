You have a few options to see which libraries are included with your Databricks runtime:

1. **Databricks Documentation:**  
   Each Databricks Runtime version has release notes and documentation that list the pre-installed libraries and their versions. You can find these details on the [Databricks Runtime release notes page](https://docs.databricks.com/release-notes/index.html).

2. **Using a Notebook Cell:**  
   You can run a shell command in a notebook cell to list all installed Python packages. For example:
   ```python
   %sh pip freeze
   ```
   or
   ```python
   %sh pip list
   ```
   This command will display the packages and their versions that are available on your cluster.

3. **Cluster UI:**  
   In the Databricks UI, under the **Libraries** tab for your cluster, you can see both the pre-installed libraries and any additional libraries you have attached.

These methods will help you verify which libraries are part of your current Databricks runtime environment.
