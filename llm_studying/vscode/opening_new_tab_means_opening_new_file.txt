In **VS Code**, tabs are tied directly to *open editors* (files, notebooks, terminals, etc.). There isn’t a concept of an “empty tab” like in a web browser — each tab corresponds to some resource.

---

### Ways to Open a New Tab

1. **Open a new file (unsaved/untitled)**

   * Menu: **File → New File**
   * Shortcut:

     * **macOS**: `⌘N`
     * **Windows/Linux**: `Ctrl+N`
   * This gives you an untitled, unsaved file in a new tab.

2. **Open an existing file**

   * Explorer: double-click a file → opens in a new tab.
   * Single-click opens in **preview mode** (italic tab title); double-click or edit to make it permanent.

3. **Split editor (multiple groups)**

   * Shortcut:

     * **macOS**: `⌘\`
     * **Windows/Linux**: `Ctrl+\`
   * Creates side-by-side tabs in separate groups.
   * You can drag tabs between groups.

4. **Integrated terminal in a tab** (newer versions)

   * Terminals can open in the editor area as tabs.
   * Use the dropdown next to **+** in the terminal panel → *Move Terminal into Editor Area*.
   * Or command palette → **Terminal: Create Terminal in Editor Area**.

---

### Important Note

* VS Code does **not** support creating a “blank browser-style tab” without a file, terminal, or notebook attached.
* If you just want a new space, the way to do that is **File → New File** (untitled tab).
