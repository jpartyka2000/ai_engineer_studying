Yes, exactly. VSCode (via Pylance/Pyright) dims unused imports to indicate they're not referenced anywhere in that file.

For example:

```python
import json           # Normal color → used somewhere below
import logging        # Dimmed/greyed → not used in this file
import os             # Normal color → used somewhere below
from typing import Optional  # Dimmed → Optional never appears in the code
```

## What to do about it

**Option 1: Remove it** — If you don't need it, delete the import to keep things clean.

**Option 2: It's intentional** — Sometimes you import something for side effects or re-export. In that case you can:

```python
# Explicitly mark as intentionally unused
from some_module import something  # noqa: F401
```

or

```python
# For re-exporting from a package's __init__.py
from .models import Question as Question  # The "as Question" signals intentional re-export
```

## Common scenario during refactoring

When splitting your code into modules, you'll likely see several dimmed imports in your original file as you move code out. That's a helpful visual cue for what's no longer needed in that file.
