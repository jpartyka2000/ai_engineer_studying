Instead of manually pasting your `def grade(sample, item): …` code into a big string, you can load it straight from a Python file at runtime using:

1. **`importlib.import_module`**
   Dynamically imports your grader module by name (e.g. `"grader"` if it’s `grader.py` on your `PYTHONPATH`).

   ```python
   import importlib
   grader_module = importlib.import_module("grader")
   ```

2. **`inspect.getsource`**
   Pulls the full source text of that module (or a specific function) as a string, preserving indentation and comments.

   ```python
   import inspect
   source_code = inspect.getsource(grader_module)
   ```

3. **Build your grader payload**
   Use that `source_code` string in your JSON:

   ```python
   grader = {
       "type":      "python",
       "source":    source_code,
       "image_tag": "2025-05-08"   # or whichever runtime you need
   }
   ```

---

### Why this helps

* **Single source of truth**: You keep your grading logic in `grader.py` where it’s easy to edit, lint, test, and version-control.
* **DRY**: No duplication between your local code and the JSON you send to the Eval API.
* **Clarity**: You can import helper functions, classes, or data structures in `grader.py` as needed, and `inspect.getsource` will pull in the whole module text.

Just make sure that `grader.py` (or whatever module you import) lives on your Python path when you run this snippet so that `importlib` can find it.
