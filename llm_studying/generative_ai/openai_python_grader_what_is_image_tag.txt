The `"image_tag"` field tells the Eval service **which container image** (runtime environment) to spin up in order to execute your Python grading code. Concretely:

* It’s the **tag** (think Docker tag) of the pre­built evaluator image that already has the right Python version and dependencies installed.
* By specifying `"image_tag": "2025-05-08"`, you’re opting into the grader runtime snapshot that was built and frozen on May 8, 2025.
* If you don’t set it, the service will default to its current “latest” image tag, which may change over time and could break reproducibility.

In short, **`image_tag` pins the exact execution environment** for your `def grade(sample, item): …` function, ensuring consistency and isolating you from upstream runtime changes.
