For your tool calling fine-tuning scenario, I'd recommend **200-500 examples** as a starting point, but let me break down the reasoning:

## Key Factors Affecting Your Needs

**Complexity of Your Task:**
- 7 different functions (moderate complexity)
- 2-4 arguments per function with optional parameters
- Sequential tool calling (predicting next in chain)
- Need to understand context from prior tool calls

**Coverage Requirements:**
- Each of the 7 functions should appear multiple times
- Different argument combinations (required vs optional)
- Various sequence positions (1st tool, 2nd tool, etc.)
- Different question types and contexts

## Recommended Distribution

**Minimum viable: 200 examples**
- ~30 examples per function (7 × 30 = 210)
- Cover most common argument patterns
- Basic sequence variations

**Better performance: 300-500 examples**
- ~40-70 examples per function
- Multiple argument combinations per function
- Edge cases and error scenarios
- Complex multi-step sequences

**High performance: 500-1000 examples**
- Comprehensive coverage of all scenarios
- Robust handling of ambiguous cases
- Strong generalization across question types

## Quality Over Quantity Considerations

**Focus on:**
- **Diverse question types** that require different tool sequences
- **Argument variation** - don't just use the same parameters repeatedly  
- **Sequential context** - examples where the next tool depends heavily on prior results
- **Edge cases** - optional parameters, error conditions, ambiguous requests

**Example distribution:**
```
Function A (most common): 60 examples
Function B (common): 50 examples  
Function C-E (moderate): 35 examples each
Function F-G (specialized): 25 examples each
Total: ~330 examples
```

## Validation Strategy

Start with 300 examples and evaluate:
- Hold out 20% for validation
- Monitor for overfitting vs underfitting
- Test on questions requiring tool sequences not seen in training
- Scale up if performance plateaus

Given your current 10-example baseline showed the mechanics work, I'd aim for **300-400 examples** as your sweet spot for balancing performance and effort.
