The model itself remains weight-static—that is, it doesn’t update its parameters or “learn” from seeing more examples, even if you feed them one after the other. However, it *does* have short-term “memory” in the form of its context window: anything you include in the prompt history (system/developer/user messages, plus any previous examples you concatenate) can influence how it responds to the next query.

So:

* **No weight updates:** Each API call is an independent inference pass—there’s no gradient step or parameter tweak happening under the hood.
* **Contextual conditioning:** If you batch multiple examples into the same prompt, or if you keep a running conversation history in your messages array, the model will base its next answer on everything it “sees” in that prompt.

If you want the model to genuinely *learn* across sessions (so that tomorrow it “knows” what you showed it today), you’d need to either:

1. **Fine-tune** it on your example set (creating a new model version), or
2. **Implement a retrieval/augmented-generation pipeline** that stores your examples externally and prepends relevant ones to each new prompt.

But out of the box, the base LLM doesn’t self-modify its knowledge just by answering questions.
