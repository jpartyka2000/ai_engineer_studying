When you send messages with `role: "assistant"` in OpenAI’s Chat API (or when using Harmony), each message also carries a **channel** that determines its purpose and visibility:

1. **`analysis`**

   * **Purpose:** Internal chain-of-thought, reasoning steps, and tool-invocation rationale.
   * **Visibility:** *Not* shown to end-users. Used to expose intermediate reasoning for debugging, auditing, or tool orchestration.
   * **Example:**

     ```json
     {
       "role": "assistant",
       "channel": "analysis",
       "content": "I need to call get_weather() because the user asked for tomorrow’s forecast."
     }
     ```

2. **`commentary`**

   * **Purpose:** Non-final communications about function calls or step preludes; for instance, announcing that a function is about to be invoked or summarizing tool output.
   * **Visibility:** Typically *not* shown directly to end-users, but can be logged or displayed in developer UIs.
   * **Example:**

     ```json
     {
       "role": "assistant",
       "channel": "commentary",
       "content": "Calling get_weather(location='Paris', unit='celsius') now."
     }
     ```

3. **`final`**

   * **Purpose:** The actual user-facing response—what the assistant “says” to the end-user.
   * **Visibility:** *Always* shown to the end-user or consumer of the API.
   * **Example:**

     ```json
     {
       "role": "assistant",
       "channel": "final",
       "content": "The forecast for Paris tomorrow is 18 °C with light rain."
     }
     ```

---

By separating these channels, you can:

* **Expose** or **hide** detailed reasoning.
* **Orchestrate** multi-step tool calls without confusing the user.
* **Deliver** a clean, concise answer in the `final` channel.
