Yes, this is typical for the OpenAI Agents SDK streaming API. Here's what you're seeing:

## Event Flow Breakdown

**Many `RawResponsesStreamEvent`s** — These include:
- `ResponseTextDeltaEvent` — token-by-token text streaming
- `ResponseFunctionCallArgumentsDeltaEvent` — streaming function call args
- `ResponseOutputItemDoneEvent` — a complete item (message, function_call, etc.)
- `ResponseCompletedEvent` — response finished

**One `RunItemStreamEvent`** — The SDK's higher-level abstraction that fires when a logical "item" is complete (a message, a tool call, etc.)

**Final output** — The `message_output_item` at the end is the agent's final response.
