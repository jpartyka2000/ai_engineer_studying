In the OpenAI Agents SDK (and the underlying Responses API), a **content part** is a single unit within a message's content array. Messages can contain multiple content parts of different types.

## Common Content Part Types

| Type | Description |
|------|-------------|
| `text` / `output_text` | Plain text content |
| `image` | Image data (base64 or URL) |
| `audio` | Audio content |
| `refusal` | Model's refusal to respond |
| `file` | File attachment |

## Example Structure

A single assistant message might look like:

```json
{
  "role": "assistant",
  "content": [
    { "type": "output_text", "text": "Here's the image you requested:" },
    { "type": "image", "image_url": "..." }
  ]
}
```

## Why It Matters

- **Multimodal responses** — Allows mixing text, images, and other media in one message
- **Streaming** — Each part can complete independently, hence `ResponseContentPartDoneEvent` signals when one part (e.g., the text portion) is finished, even if other parts are still generating
