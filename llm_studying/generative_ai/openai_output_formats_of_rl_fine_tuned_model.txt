An RL-fine-tuned model simply inherits the same I/O interfaces as our other hosted LLMs, so you can call it via:

1. **Text completions**

   * Use the **Completions** endpoint and you’ll get back a top-level `choices[i].text` string containing the model’s reply. ([OpenAI Platform][1])

2. **Chat completions**

   * Use the **Chat Completions** endpoint and you’ll receive a `choices[i].message` object with

     ```json
     { "role": "assistant", "content": "…your reply…" }
     ```

   ([OpenAI Platform][2])

3. **Function-calling outputs**

   * If your fine-tuned model supports function calls, the `choices[i].message` will include a

     ```json
     "function_call": {
       "name":   "my_tool",
       "arguments": "{…json…}"
     }
     ```
   * You then invoke that tool locally and append its result as another message in your conversation. ([OpenAI Platform][2])

In all cases, an RL-FT model behaves just like any other OpenAI model under the hood—only its weights were adjusted via your reinforcement-learning loop.
