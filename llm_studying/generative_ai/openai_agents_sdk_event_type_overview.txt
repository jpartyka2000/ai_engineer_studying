Based on the code, here's a brief overview of the OpenAI Agents SDK event types being handled:

## Wrapper Events

- **`RawResponsesStreamEvent`** — A top-level wrapper that contains a `data` field with the actual event payload.
- **`RunItemStreamEvent`** — Another streaming event type that contains an `item` field directly (alternative event structure).

## Streaming/Delta Events (skipped in this code)

These are granular streaming events, typically used for real-time UI updates:

- **`ResponseTextDeltaEvent`** — Individual text chunks as they're generated
- **`ResponseTextDoneEvent`** — Signals text generation is complete
- **`ResponseContentPartDoneEvent`** — Signals a content part finished
- **`ResponseCompletedEvent`** — Signals the entire response is done

## Output Item Events (processed)

**`ResponseOutputItemDoneEvent`** — Contains completed items with different `type` values:

| Item Type | Meaning |
|-----------|---------|
| `message` | Assistant's text response (contains `content` list with `output_text` entries) |
| `function_call` | Tool invocation (contains `name` and `arguments`) |
| `function_call_output` | Tool result (contains `output` and `call_id`) |

## TL;DR

The SDK streams events at different granularities. This code ignores the fine-grained deltas and only captures **completed items** (final messages, tool calls, and tool results) to build a trajectory for evaluation/scoring purposes.
