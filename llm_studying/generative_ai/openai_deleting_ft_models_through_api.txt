While OpenAI's documentation historically included a delete endpoint for models, **this functionality is now restricted or removed for most users**. Here's what you can try:

## Attempting deletion via API:

```python
from openai import OpenAI

client = OpenAI(
    api_key="your-api-key",
    project="jeff_rl"
)

# Try to delete a fine-tuned model
try:
    # Older client versions
    response = client.models.delete("ft:gpt-3.5-turbo:personal::abc123")
    print(f"Deleted: {response}")
except Exception as e:
    print(f"Error: {e}")
```

Or with curl:
```bash
curl -X DELETE https://api.openai.com/v1/models/ft:gpt-3.5-turbo:personal::abc123 \
  -H "Authorization: Bearer YOUR_API_KEY"
```

## Common responses:

Most users now get one of these errors:
- `"You have insufficient permissions for this operation"`
- `"Deleting fine-tuned models is not allowed"`
- 403 Forbidden error

## Why it likely won't work:

OpenAI appears to have disabled self-service deletion of fine-tuned models around 2023-2024. This might be for:
- Audit trail purposes
- Preventing accidental deletions
- Compliance reasons
- Platform stability

## Your only reliable option:

**Contact OpenAI Support**:
1. Go to help.openai.com
2. Submit a ticket requesting model deletion
3. Include:
   - The exact model IDs (e.g., `ft:gpt-3.5-turbo:personal::abc123`)
   - Reason for deletion
   - Your organization ID

They typically process these requests within 2-5 business days.

It's frustrating that this basic management feature was removed, especially when you're trying to troubleshoot why your new models aren't appearing in the Playground.
