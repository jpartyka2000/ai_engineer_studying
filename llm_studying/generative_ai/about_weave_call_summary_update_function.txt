## `call.summary.update()` in Weave

This **adds or updates summary statistics** for the current function call that will be displayed in the Weave dashboard.

```python
@weave.op()
def process_data(data: list) -> dict:
    call = weave.get_current_call()
    
    # Add summary metrics
    call.summary.update({
        "item_count": len(data),
        "processing_time": 1.23,
        "accuracy": 0.95,
        "model_used": "gpt-4"
    })
    
    result = do_processing(data)
    return result
```

## What It Does:

The summary appears in Weave's UI as **key metrics** for this function call - like a dashboard showing important numbers at a glance.

```python
@weave.op()
def evaluate_model(test_data):
    call = weave.get_current_call()
    
    predictions = model.predict(test_data)
    accuracy = calculate_accuracy(predictions)
    
    # These metrics show up prominently in Weave dashboard
    call.summary.update({
        "accuracy": accuracy,
        "false_positives": 12,
        "false_negatives": 3,
        "total_samples": len(test_data)
    })
    
    return predictions
```

## Common Pattern:

```python
@weave.op()
def llm_call(prompt: str) -> str:
    call = weave.get_current_call()
    
    response = openai.complete(prompt)
    
    # Track important metrics
    call.summary.update({
        "tokens": count_tokens(response),
        "cost": calculate_cost(response),
        "latency_ms": response.latency
    })
    
    return response.text
```

**Purpose**: Makes important metrics easily visible in Weave's UI without digging through full logs - perfect for tracking model performance, costs, or any KPIs you care about.
