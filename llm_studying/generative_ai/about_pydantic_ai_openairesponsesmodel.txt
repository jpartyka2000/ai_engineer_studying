## `OpenAIResponsesModel` Overview

`OpenAIResponsesModel` is PydanticAI's **response model class** that structures and validates the responses you get back from OpenAI API calls.

## What It Does:

It wraps OpenAI's raw API responses into a **standardized format** with:
- Validated response data
- Token usage tracking  
- Message history
- Error handling

## Basic Usage:

```python
from pydantic_ai.models.openai import OpenAIResponsesModel, OpenAIResponsesModelSettings

# Configure response handling
settings = OpenAIResponsesModelSettings(
    max_tokens=1000,
    temperature=0.7,
    stream=False
)

# The model handles response parsing and validation
response_model = OpenAIResponsesModel(settings=settings)

# Works with PydanticAI agents to ensure responses 
# match your expected schema
agent = Agent(
    'openai:gpt-4',
    response_model=MyExpectedOutput  # Your Pydantic model
)
```

## Key Features:

1. **Type Safety** - Ensures OpenAI responses match your Pydantic models
2. **Token Tracking** - Built-in usage and cost monitoring
3. **Error Handling** - Structured handling of API errors
4. **Streaming Support** - Can handle streamed responses if configured

**Bottom line**: It's the bridge between OpenAI's API responses and PydanticAI's type-safe world - ensuring you get validated, structured data instead of raw JSON from OpenAI.
