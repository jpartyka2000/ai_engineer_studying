Here’s a summary of what I found: which LLMs / providers are known to support MCP (Model Context Protocol), and which open-source / smaller ones are starting to or can be used with it. A lot of this is based on public docs & experiments; in many cases “support” means in the agent / tool-wrapping layer rather than the core model itself.


---

Who definitely supports MCP

These are models / providers that are reported to have integrated MCP (or are being built with MCP compatibility) in their tooling / agent frameworks.

Model / Provider Notes on MCP support

Anthropic / Claude Because Anthropic is creator of MCP, their Claude models/environments are naturally built to support it. E.g. Claude Desktop apps have “local MCP server support”.
OpenAI OpenAI has officially adopted MCP across several of their tools / APIs: the Agents SDK, the Responses API, etc.



---

Who can or are reported to work with MCP

These are open models, or others, which have shown support or compatibility in experimental or user-facing / community tools.

Model / Provider Status / Details

Open LLMs / Open source (via “mcp-cli” etc.) Several projects show you can connect local/open LLMs to MCP servers using wrappers / CLI tools. E.g. mcp-cli supports local LLMs via Ollama and OpenAI.
Gemini (Google) There are blogposts / tutorials showing how to use MCP Servers with Google Gemini.
“Any open LLMs” generally Multiple examples in tutorials show how to build simple agents using MCP Servers + open LLMs + OpenAI / Gemini etc. So “support” is informal but demonstrably possible.



---

Who probably does not or where support is unclear

Many open source LLMs (e.g. Llama family, Vicuna etc.) do not have built-in MCP support (in their core) but can be used with MCP via client wrappers / agents. The model itself usually doesn’t need special code for MCP; the MCP Client or the agent framework handles that.

If you have an LLM in a closed environment with restrictive tooling, or one without flexible API / local deployment, you may hit limitations.



---

Summary / Key Takeaways

Core models typically don’t have “MCP inside them” as a unique feature; “supporting MCP” usually means there is an MCP Client / agent wrapper that works with that model (or the provider’s stack includes MCP Compatible endpoints).

Anthropic and OpenAI are the big names with native / first-class MCP integration.

Open source / smaller models are starting to be used via MCP through agent frameworks (“you can hook up your local model to an MCP server”).
