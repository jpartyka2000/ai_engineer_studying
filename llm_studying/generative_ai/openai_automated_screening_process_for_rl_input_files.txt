Before any file can be used to start a reinforcement-learning fine-tuning (RFT) job, it must pass through OpenAI’s **automated screening** pipeline. Here’s how it works:

1. **When screening runs**

   * Screening does **not** happen at upload time.
   * It kicks off only once you begin a fine-tuning job with your training (or test) file.

2. **What it checks**

   * **Content compliance**: Filters out policy-violating or disallowed content.
   * **Data format**: Ensures each example follows the required JSON schema for RFT (correct fields, types).
   * **Quality heuristics**: Looks for malformed records, extremely short or nonsensical entries, and other red flags.

3. **Why it matters**

   * **Safety**: Prevents unsafe or non-compliant data from influencing model behavior.
   * **Stability**: Guarantees that the fine-tuner sees only well-formed examples, reducing training errors.

4. **Impact on workflow**

   * Large files take longer to screen—expect a delay from “job started” until screening completes.
   * Once a file has successfully passed screening, you can reuse it in future RFT jobs **without** additional delay.

5. **Size limits**

   * Training files: up to **50 000** examples.
   * Test files: up to **1 000** examples (also screened before use).

**Best practice:**
Start with a small set (dozens to a few hundred) to iterate quickly. After screening succeeds, you can scale up to a larger, high-quality dataset knowing that it will be immediately available for new fine-tuning runs.
