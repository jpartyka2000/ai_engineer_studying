Yes, exactly. The `RunContextWrapper` class has an attribute literally named `context` that holds the instance of whatever type you parameterize it with.

So when you write `RunContextWrapper[CueReactRunContext]`, you're saying "a `RunContextWrapper` whose `.context` attribute is of type `CueReactRunContext`."

Conceptually, the class probably looks something like this:

```python
class RunContextWrapper[T]:
    context: T
    
    def __init__(self, context: T):
        self.context = context
```

So the slightly awkward `context.context` is just:

```python
context         # The RunContextWrapper instance (the parameter name)
    .context    # The attribute that holds your CueReactRunContext
```

A clearer parameter name might have been `wrapper` or `ctx_wrapper`:

```python
async def call_charting_agent(
    wrapper: RunContextWrapper[CueReactRunContext], 
    description_of_request: str
):
    # Now it reads more clearly:
    wrapper.context.openai_api_key
```

But the framework likely dictates the parameter name `context`, or it's just a convention the codebase uses.
