It depends on which provider you're using:

**Together AI** — accepts a **URL** via the `image_url` parameter:

```python
client.images.generate(
    model="black-forest-labs/FLUX.1-kontext-pro",
    prompt="Change the outfit to a leather jacket...",
    image_url="https://your-bucket.com/album-cover.png",
    width=1024,
    height=768,
)
```

Or via curl:
```bash
curl -X POST "https://api.together.xyz/v1/images/generations" \
  -H "Authorization: Bearer $TOGETHER_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "black-forest-labs/FLUX.1-kontext-pro",
    "prompt": "Change the outfit to a leather jacket...",
    "image_url": "https://your-bucket.com/album-cover.png",
    "width": 1024,
    "height": 768
  }'
```

**fal.ai** — accepts **both URL and base64 data URIs**: You can pass your own URL or a Base64 data URI. The API will handle the file decoding for you. Keep in mind that for large files, base64 although convenient can impact request performance. They also offer a file upload utility that returns a hosted URL.

**CometAPI** — You can pass publicly-hosted URLs or inline Base64 data URIs — Base64 is convenient for one-off scripts but may slow down very large files.

So if you're sticking with Together AI, you'll need your input image hosted at a publicly accessible URL (e.g., S3 bucket, Cloudflare R2, etc.). If you need base64 support, fal.ai is an option.
