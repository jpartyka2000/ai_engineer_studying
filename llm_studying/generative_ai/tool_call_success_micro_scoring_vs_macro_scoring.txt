## `tool_success_rate_micro` Explanation

This calculates the **overall tool success rate across ALL tool calls** in the entire evaluation dataset - a "micro-average" approach.

## What It Means:

```python
# Micro-average: Total successes / Total attempts (across everything)

# Example with 3 evaluation runs:
Run 1: 2/3 successful tool calls
Run 2: 5/5 successful tool calls  
Run 3: 1/4 successful tool calls

# Micro-average calculation:
Total successes: 2 + 5 + 1 = 8
Total attempts: 3 + 5 + 4 = 12
tool_success_rate_micro = 8/12 = 0.667 (66.7%)
```

## Micro vs Macro Average:

```python
# MICRO-average (what this code does):
# Treats all tool calls equally
tool_success_rate_micro = total_successes / total_attempts
# = (2+5+1) / (3+5+4) = 8/12 = 0.667

# MACRO-average (alternative approach):
# Averages the per-run success rates
run_rates = [2/3, 5/5, 1/4]  # [0.667, 1.0, 0.25]
tool_success_rate_macro = mean(run_rates)
# = (0.667 + 1.0 + 0.25) / 3 = 0.639
```

## Why Micro-Average?

**Micro-averaging is better when:**
- Some runs have many more tool calls than others
- You want every tool call to count equally
- You care about overall system performance

```python
# Example showing why micro matters:
Run A: 100/100 successful (large task)
Run B: 0/1 successful (tiny task)

Micro: 100/101 = 99.0% ✓ (reflects reality)
Macro: (100% + 0%) / 2 = 50% ✗ (misleading)
```

**Bottom line**: `tool_success_rate_micro` gives you the true overall success rate by counting every single tool call equally, regardless of which evaluation run it came from - it's the "global" success rate.
