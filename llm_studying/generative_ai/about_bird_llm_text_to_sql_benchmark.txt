BIRD-SQL (BIg Bench for LaRge-scale Database Grounded Text-to-SQL) is a major benchmark for evaluating how well LLMs can convert natural language questions into SQL queries.

## Overview

BIRD-SQL is the first cross-domain large-scale benchmark specifically designed to bridge the gap between academic research and real-world applications in text-to-SQL parsing. While earlier benchmarks like Spider and WikiSQL focused mainly on database schema, BIRD-SQL draws attention to database values, introducing new challenges such as external knowledge, dirty data, and SQL efficiency in vast databases.

## Dataset Scale

The dataset includes 12,751 text-to-SQL pairs, 95 databases encompassing 37 professional domains, and a total size of 33.4 GB.

## Evaluation Metrics

The benchmark uses several metrics:
- EX (Execution accuracy) for evaluating whether generated SQL produces correct results
- R-VES (Valid Efficiency Score) for measuring SQL efficiency

## Recent Extensions

The BIRD project has expanded considerably:

- **BIRD-Interact**: A comprehensive interactive evaluation containing conversational (c-Interact) and agentic (a-Interact) interaction modes. Top results show o3-mini achieving 24.4% success rate on c-Interact, while Claude-3.7-Sonnet reaches 17.78% on a-Interact.
- **LiveSQLBench**: The first contamination-free text-to-SQL benchmark covering the full SQL spectrum
- **bird-critic**: A challenging reasoning task for SQL debugging where top-performing models are reasoning-based

## Current State-of-the-Art

Databricks achieved a new state-of-the-art test accuracy of 73.5% on the single-model track using reinforcement learning (RLVR), training only on the BIRD training set. The previous best was 71.8%, achieved by augmenting training data and using GPT-4o.
