## `OpenAIProvider` Overview

`OpenAIProvider` is the **adapter class** that connects PydanticAI to OpenAI's API - it handles the actual communication with OpenAI services.

## What It Does:

It's the "driver" that:
- **Manages API connections** to OpenAI
- **Handles authentication** (API keys)
- **Routes requests** to the right OpenAI models
- **Transforms** between PydanticAI's format and OpenAI's API format

## Basic Usage:

```python
from pydantic_ai.models.openai import OpenAIProvider

# Create provider with your API key
provider = OpenAIProvider(
    api_key="sk-...",  # Your OpenAI API key
    organization="org-...",  # Optional: org ID
    base_url="https://api.openai.com/v1"  # Optional: custom endpoint
)

# Use it with an agent
agent = Agent(
    provider=provider,
    model="gpt-4",
    # ... other settings
)
```

## Key Features:

1. **API Management** - Handles retries, rate limits, timeouts
2. **Model Selection** - Routes to GPT-4, GPT-3.5, etc.
3. **Format Translation** - Converts PydanticAI calls to OpenAI API format
4. **Error Handling** - Wraps OpenAI errors in PydanticAI exceptions

## Common Pattern:

```python
# Provider can be reused across multiple agents
provider = OpenAIProvider(api_key=os.getenv("OPENAI_API_KEY"))

chat_agent = Agent(provider=provider, model="gpt-4")
code_agent = Agent(provider=provider, model="gpt-4-turbo")
cheap_agent = Agent(provider=provider, model="gpt-3.5-turbo")
```

**Bottom line**: `OpenAIProvider` is the middleman that makes OpenAI's API work seamlessly with PydanticAI's agent system - you configure it once with credentials and settings, then use it across your agents.
