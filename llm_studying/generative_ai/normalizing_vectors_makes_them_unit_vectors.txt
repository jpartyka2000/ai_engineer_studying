Yes—when you **L2-normalize** a vector, you scale it to have Euclidean length 1, i.e., it becomes a **unit vector**.

* For a nonzero vector $x$, L2 normalization is $\hat{x} = \dfrac{x}{\|x\|_2}$.
  Then $\|\hat{x}\|_2 = 1$, and the direction is unchanged.
* The **zero vector** can’t be normalized (division by zero).

In libraries like SentenceTransformers, `normalize_embeddings=True` uses **L2 normalization**, so each embedding returned is a unit vector. That’s why cosine similarity becomes just the dot product and Euclidean distance aligns with angle (spherical geometry).

Quick demo:

```python
import numpy as np

x = np.array([3., 4., 0.])
x_norm = x / np.linalg.norm(x)   # L2 normalize
print(np.linalg.norm(x_norm))    # 1.0
```

⚠️ Note: there are other normalizations. For example, **L1 normalization** makes $\sum_i |x_i| = 1$ (not a unit vector in the Euclidean sense). So “normalized” means “unit” only when you specify **L2** (or “unit with respect to that norm”).
