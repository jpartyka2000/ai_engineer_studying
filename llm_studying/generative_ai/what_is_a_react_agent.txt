A **ReAct agent** is a style of LLM-driven agent architecture that interleaves **Reasoning** (“chain-of-thought”) with **Action** (tool calls or environment interactions) in a single loop. It was introduced in the paper “ReAct: Synergizing Reasoning and Acting” and works like this:

1. **Reasoning step**
   The model produces an internal thought—often framed as a natural-language “chain of thought”—about what it needs to do next.

   ```
   Thought: I need to look up today’s weather for Tokyo before answering.
   ```

2. **Action step**
   The model emits a structured action, e.g. calling a weather API or database query:

   ```
   Action: get_current_weather(location="Tokyo")
   ```

3. **Observation step**
   The tool returns results (the weather), which the agent ingests:

   ```
   Observation: {"temp": 20, "condition": "sunny"}
   ```

4. **Repeat**
   The model uses that observation in the next reasoning step, eventually producing a final answer:

   ```
   Thought: Based on the API result, it’s 20°C and sunny.  
   Answer: Today in Tokyo it’s 20°C and sunny.
   ```

By alternating reasoning and acting, a ReAct agent can:

* Leverage **chain-of-thought** for transparency and complex deduction
* Invoke external **tools** or **APIs** to fetch up-to-date or external data
* Adapt its plan dynamically based on real-time observations

This synergy lets ReAct agents handle tasks that require both deep reasoning and interaction with external systems—making them more capable than pure “think-then-act” or “tool-only” designs.
