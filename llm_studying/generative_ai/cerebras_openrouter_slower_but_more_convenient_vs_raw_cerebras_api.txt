Using Cerebras via OpenRouter is actually **not** faster — it's slightly slower. Cerebras's own docs note that a marginal amount of additional latency appears when going through OpenRouter, since the request has to be proxied.

The advantage of OpenRouter isn't speed — it's convenience. OpenRouter provides a unified API interface that lets you switch between models and providers (GPT-4, Claude, Cerebras, etc.) by just changing the model string, without rewriting integration code. That's useful if you're comparing across providers or want a single billing relationship.

For raw speed, going direct through Cerebras's own API is the best option. The tradeoff is basically: direct API gives you maximum speed and potentially lower cost, while OpenRouter gives you flexibility to swap providers easily at the cost of a small latency overhead and possible markup. For your eval harness where you'd be making many repeated scorer calls, going direct would save you both time and money.
