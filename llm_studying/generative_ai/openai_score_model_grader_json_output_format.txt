Here’s the **Score Model** grader’s expected response format (including chain‐of‐thought steps), exactly as shown in the docs:

```json
{
  "result": <float>,
  "steps": [
    {
      "description": "<string>",
      "conclusion":  "<string>"
    },
    // …additional reasoning steps…
  ]
}
```

* **`result`**
  A floating-point score (e.g. a reward or similarity value) returned by the model and (if configured) clipped to your grader’s `range`.

* **`steps`**
  An array of **`ReasoningStep`** objects, where each step has:

  * **`description`**: the model’s chain-of-thought or rationale for this step.
  * **`conclusion`**: the key inference or decision drawn at that step.

This format lets the model not only output a numeric grade but also articulate the reasoning behind that grade in a structured way.
