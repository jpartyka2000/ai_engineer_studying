## What `@weave.op()` Does:

It **automatically logs** the inputs, outputs, and execution time of any function call to W&B.

```python
@weave.op()
def predict(self, query: str) -> str:
    response = call_llm(query)
    return response

# When you call this:
result = predict("What is Python?")

# Weave automatically logs:
# - Input: "What is Python?"
# - Output: "Python is a programming language..."
# - Execution time: 1.2s
# - Any errors if they occur
```

That's it. It's like adding automatic logging to your functions without writing any logging code. You can then view all these calls in the W&B dashboard to debug, compare, and analyze your model's behavior.

Think of it as **print debugging on steroids** - instead of adding print statements, you add `@weave.op()` and get a full history of what your functions did.
