That's actually a decent setup for this. Apple Silicon's unified memory and Metal GPU acceleration change the picture quite a bit compared to pure CPU inference.

**What should work well:**

**PickScore / CLIPScore** — These will fly on your machine. ~400M params, well within your memory. Expect sub-second inference easily via MPS (Metal Performance Shaders) backend in PyTorch.

**ImageReward** — ~1B params, should run comfortably. Maybe 1-3 seconds per image on MPS. Totally usable.

**VQAScore with smaller models** — `llava-v1.5-7b` or `clip-flant5-xl` (~3B params) should fit in your 18GB. Performance won't match the XXL model but will still beat ImageReward on compositional prompts. Expect maybe 5-15 seconds per image depending on the model. Worth trying.

**What won't work:**

**VQAScore with CLIP-FlanT5-XXL** — 11B params will exceed your 18GB once you account for the OS, activations, and overhead. Don't bother.

**My recommendation for your hardware:**

Start with **PickScore** for fast iteration, then try **VQAScore with `clip-flant5-xl`** if you need better compositional accuracy and can tolerate slower speeds. Make sure you're using the MPS backend in PyTorch — it makes a big difference on Apple Silicon:

```python
import torch
device = torch.device("mps" if torch.backends.mps.is_available() else "cpu")
```

Some models may need you to explicitly move them to the MPS device. If you hit compatibility issues (some ops aren't yet supported on MPS), falling back to CPU is still reasonable for the smaller models on your M3 Pro.
