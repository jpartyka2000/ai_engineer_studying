## The Hidden Step: Enable Fine-tuned Models in Project Settings

After creating a fine-tuned model, you need to:

1. **Go to your OpenAI Project Settings**
   - Navigate to platform.openai.com
   - Select your project (e.g., "jeff_rl")
   - Go to Settings → Limits (or Project Limits)

2. **Enable the fine-tuned model**
   - Look for a section about "Models" or "Allowed Models"
   - Find your fine-tuned model in the list (e.g., `ft:gpt-4o-mini:personal:jeff-test:abc123`)
   - Toggle it ON/Check the box to allow it

3. **Save the changes**
   - The model should now appear in the Playground dropdown
   - It should also be accessible via API calls with that project's API key

## Why This Happens

OpenAI's project system treats fine-tuned models as separate models that need explicit permission, even if you created them within that project. This is probably for:
- Security/access control in team environments
- Preventing accidental usage of expensive fine-tuned models
- Allowing granular control over which projects can use which models

## Complete Fine-tuning Workflow

So the full process is:
1. Create fine-tuning job (via API or UI)
2. Wait for completion
3. **Go to Project Settings → Limits**
4. **Enable the new fine-tuned model**
5. Model now appears in Playground and works via API

This explains why:
- Your models were created successfully but not visible
- You got 403 errors when calling them via API with the project key
- Your older models showed up (they were probably enabled before)
- The models worked without specifying a project (bypassed project restrictions)
