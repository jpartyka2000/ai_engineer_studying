Yes. `asyncio.run()` expects an **awaitable**—specifically, a **coroutine object**—not a plain function. In practice that means you invoke an `async def` function (which returns a coroutine) and pass *that* into `asyncio.run()`. For example:

```python
async def main():
    await asyncio.sleep(1)
    print("Done")

# Correct: main() is a coroutine object
asyncio.run(main())
```

If you pass the function itself without calling it:

```python
# ❌ Wrong: passing the function object, not a coroutine
asyncio.run(main)
```

you’ll get:

```
TypeError: a coroutine was expected, got function
```

Under the hood, `asyncio.run()` will:

1. Create a new event loop
2. Run the provided coroutine to completion
3. Close the loop

So you must call your `async def` to get a coroutine before handing it to `asyncio.run()`.
