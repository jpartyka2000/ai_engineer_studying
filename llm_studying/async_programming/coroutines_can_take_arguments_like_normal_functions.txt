Yes—any coroutine you pass to `asyncio.run()` can take arguments, just like a regular function. You simply call the `async def` with its parameters when you hand it to `asyncio.run()`. For example:

```python
import asyncio

async def greet(name: str, delay: float):
    await asyncio.sleep(delay)
    print(f"Hello, {name}!")

# Call with arguments directly
asyncio.run(greet("Alice", 1.0))
```

Here, `greet("Alice", 1.0)` returns a coroutine object (with `name="Alice"` and `delay=1.0` bound), and `asyncio.run()` runs it to completion. You cannot pass arguments to `asyncio.run()` itself—it only takes one positional argument, the coroutine—but you embed your arguments in the coroutine call.
