In standard Python code, **yes**: the `await` keyword can only appear inside an `async def`–defined coroutine. If you write:

```python
# ❌ SyntaxError in a normal .py file:
result = await some_coroutine()
```

you’ll get:

```
SyntaxError: 'await' outside async function
```

Instead, you'd have to do something like:

```python
import asyncio

async def main():
    result = await some_coroutine()
    print(result)

asyncio.run(main())
```

---

## The Jupyter Exception

Jupyter’s IPython kernel is special—it pre-starts an asyncio event loop and transparently wraps your cell body in a coroutine when it sees a top-level `await`. That lets you write in a notebook cell:

```python
result = await some_coroutine()
print(result)
```

**without** defining an `async def` or calling `asyncio.run()`. But that’s a notebook-specific convenience—outside of IPython, you still need an `async def`.
