**Top-Level `await` in Jupyter Notebooks**

Jupyter’s IPython kernel runs its own asyncio event loop under the hood, which lets you use `await` directly in a cell without any boilerplate. Instead of having to write:

```python
import asyncio

async def main():
    result = await some_async_call()
    print(result)

asyncio.run(main())
```

you can simply do:

```python
result = await some_async_call()
print(result)
```

Behind the scenes, IPython patches itself (via `nest_asyncio` and custom loop management) so that:

* **An event loop is already running** when your notebook starts.
* **Top-level `await`** is enabled, meaning the cell runner wraps your code in a coroutine and schedules it automatically.
* **No `async def` wrapper** or explicit `asyncio.run()` is needed—your code behaves as if you wrote it inside an async function.

This makes it seamless to integrate asynchronous libraries (like `aiohttp`, `asyncpg`, or `asyncio` itself) into exploratory analyses and data-fetching workflows without extra ceremony.
