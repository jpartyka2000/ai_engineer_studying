---

### 📌 Parameters of `model_dump_json()` in Pydantic v2

The method signature is:

```python
model_dump_json(
    *,
    indent: int | None = None,
    include: IncEx | None = None,
    exclude: IncEx | None = None,
    context: dict[str, Any] | None = None,
    by_alias: bool = False,
    exclude_unset: bool = False,
    exclude_defaults: bool = False,
    exclude_none: bool = False,
    round_trip: bool = False,
    warnings: bool = True,
) -> str
```

---

### 🔑 Parameter Details

* **`indent`** (`int | None`)

  * Number of spaces for pretty-printing JSON.
  * `None` (default) → compact JSON.
  * Example: `indent=2` → human-readable JSON.

* **`include`** (`set | dict | None`)

  * Specify fields to include.
  * Can be a set of field names or a nested dict for sub-models.

* **`exclude`** (`set | dict | None`)

  * Specify fields to exclude (opposite of `include`).
  * Same structure as `include`.

* **`context`** (`dict | None`)

  * Extra context for custom serialization (used in validators/serializers).

* **`by_alias`** (`bool`, default `False`)

  * If `True`, uses field aliases instead of actual field names.

* **`exclude_unset`** (`bool`, default `False`)

  * Exclude fields that weren’t explicitly set by the user.

* **`exclude_defaults`** (`bool`, default `False`)

  * Exclude fields that are equal to their default values.

* **`exclude_none`** (`bool`, default `False`)

  * Exclude fields whose value is `None`.

* **`round_trip`** (`bool`, default `False`)

  * Keep JSON compatible with `model_validate_json()` round-tripping (e.g., don’t simplify enums or datetime formatting).

* **`warnings`** (`bool`, default `True`)

  * If `True`, warnings may be issued for non-serializable objects.

---

### ✅ Example

```python
from pydantic import BaseModel

class User(BaseModel):
    id: int
    name: str | None = None

u = User(id=1)

print(u.model_dump_json(indent=2, exclude_none=True))
```

Output:

```json
{
  "id": 1
}
```

---

👉 **In summary:**
`model_dump_json()` gives you fine-grained control over how your Pydantic model is serialized into JSON, including formatting (`indent`), field selection (`include`/`exclude`), alias handling, and rules for omitting defaults/unset/None values.
