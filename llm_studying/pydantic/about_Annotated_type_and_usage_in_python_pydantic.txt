## `Annotated` in Python & Pydantic

`Annotated` is a Python typing feature (from `typing` module) that lets you attach **metadata** to type hints. Pydantic leverages this to add validation, transformation, and configuration to fields.

### Basic Concept

```python
from typing import Annotated

# Syntax: Annotated[BaseType, metadata1, metadata2, ...]
MyType = Annotated[str, "some metadata", 42]
```

Plain Python ignores the metadata—it's just `str` to the type checker. But Pydantic *reads* the metadata and acts on it.

### How Pydantic Uses Annotated

```python
from typing import Annotated
from pydantic import BaseModel, Field

class User(BaseModel):
    # These two are equivalent:
    age: int = Field(ge=0, le=120)
    age: Annotated[int, Field(ge=0, le=120)]
```

The `Annotated` style is more powerful because you can create **reusable types**:

```python
from typing import Annotated
from pydantic import Field

# Define once
PositiveInt = Annotated[int, Field(gt=0)]
Percentage = Annotated[float, Field(ge=0, le=100)]
ShortStr = Annotated[str, Field(max_length=50)]

# Reuse everywhere
class Product(BaseModel):
    quantity: PositiveInt
    discount: Percentage
    name: ShortStr

class Order(BaseModel):
    item_count: PositiveInt
    tax_rate: Percentage
```

### Validators with Annotated

```python
from typing import Annotated
from pydantic import BaseModel, BeforeValidator, AfterValidator, PlainValidator

def strip_whitespace(v: str) -> str:
    return v.strip()

def lowercase(v: str) -> str:
    return v.lower()

CleanStr = Annotated[str, BeforeValidator(strip_whitespace), AfterValidator(lowercase)]

class Form(BaseModel):
    username: CleanStr

form = Form(username='  ALICE  ')
print(form.username)  # 'alice'
```

### Validation Order

```python
Annotated[str, BeforeValidator(fn1), BeforeValidator(fn2), AfterValidator(fn3)]
```

Execution order:
1. `BeforeValidator(fn1)` — runs on raw input
2. `BeforeValidator(fn2)` — runs on fn1's output
3. Pydantic's internal validation (coercion to `str`)
4. `AfterValidator(fn3)` — runs on validated value

### Stacking Multiple Metadata

```python
from typing import Annotated
from pydantic import Field, AfterValidator

def must_be_title_case(v: str) -> str:
    if v != v.title():
        raise ValueError('must be title case')
    return v

ProperName = Annotated[
    str,
    Field(min_length=1, max_length=50),
    AfterValidator(must_be_title_case)
]
```

### Composing Annotated Types

```python
from typing import Annotated
from pydantic import Field, AfterValidator

# Base type with length constraint
ShortStr = Annotated[str, Field(max_length=20)]

# Extended with additional validation
Username = Annotated[ShortStr, AfterValidator(lambda v: v.lower())]
```

from typing import Annotated
from pydantic import BaseModel, BeforeValidator, AfterValidator

def add_prefix(v):
    return f"PREFIX_{v}"

def add_suffix(v):
    return f"{v}_SUFFIX"

CustomStr = Annotated[str, BeforeValidator(add_prefix), AfterValidator(add_suffix)]

class Item(BaseModel):
    label: CustomStr

item = Item(label='hello')
