## 📌 `BaseModel.model_dump_json()` in Pydantic v2

### 1. What It Does

* It’s a **method on Pydantic models** (v2).
* Serializes the model into a **JSON string**.
* It’s basically a shortcut for:

  ```python
  json.dumps(model.model_dump())
  ```

  …but with Pydantic’s own optimized serialization under the hood.

---

### 2. Example

```python
from pydantic import BaseModel

class User(BaseModel):
    id: int
    name: str

u = User(id=123, name="Alice")

print(u.model_dump_json())
# '{"id":123,"name":"Alice"}'
```

Notice:

* The output is a **string** (valid JSON).
* Field names are preserved.
* Types are properly serialized (dates, enums, UUIDs, etc.).

---

### 3. Common Parameters

`model_dump_json()` accepts many of the same options as `model_dump()`, for fine-grained control:

* **`exclude` / `include`** → choose which fields to serialize.

  ```python
  u.model_dump_json(exclude={"id"})
  # '{"name":"Alice"}'
  ```

* **`by_alias=True`** → use alias names instead of field names.

* **`indent=2`** → pretty-print JSON.

* **`exclude_none=True`** → drop fields with value `None`.

---

### 4. Difference vs `model_dump()`

* `model_dump()` → returns a **Python dict**.
* `model_dump_json()` → returns a **JSON string**.

Both can take similar keyword arguments (`exclude`, `include`, etc.).

---

### 5. When to Use

* Use `model_dump()` when you want to manipulate the data in Python.
* Use `model_dump_json()` when you need to **send/store the data** as JSON (e.g., APIs, logging, caching).

---

## ✅ Summary

`BaseModel.model_dump_json()` is Pydantic v2’s built-in way to serialize a model directly into a JSON string, with options to include/exclude fields, handle aliases, indent formatting, and more. It’s the JSON-producing sibling of `model_dump()`.
