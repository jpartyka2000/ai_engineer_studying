In **Pydantic AI**, `FunctionToolset` is a concept (and class) used to group together and register **local Python functions** as “tools” that an AI agent can call at runtime. It’s part of the tooling framework that makes agents more powerful and interactive. ([Pydantic AI][1])

---

## 🛠 What `FunctionToolset` Does

Here are the key features and behaviors of `FunctionToolset`:

* It’s a **toolset** (collection of tools) whose tools are **functions**. ([Pydantic AI][2])
* You can register functions in several ways:

  1. Using a decorator (`@toolset.tool`) to annotate a function and add it. ([Pydantic AI][1])
  2. Passing functions (or `Tool` instances) into the `tools` argument when constructing a `FunctionToolset`. ([Pydantic AI][1])
  3. Dynamically adding tools with `add_function()` or `add_tool()` methods during runtime. ([Pydantic AI][1])
* Tools registered this way can optionally receive a `ctx: RunContext` argument, so they know about the current agent run’s context. ([Pydantic AI][1])
* Once you have a `FunctionToolset`, you supply it when creating or running an `Agent` so that the agent “knows about” those functions as callable capabilities. ([Pydantic AI][1])
* Under the hood, all the function tools from all given toolsets get gathered into a single combined toolset that an agent uses. ([Pydantic AI][3])

---

## 🔍 Why Use `FunctionToolset`?

* **Modularity**: You can define groups of related functions (tools) and swap them in or out.
* **Reusability**: Same toolset can be used across different agents.
* **Dynamism**: You can add tools at runtime (i.e. after the agent has started), enabling flexible behavior.
* **Control**: You get to decide exactly which functions are exposed as tools (instead of having all functions globally available).
