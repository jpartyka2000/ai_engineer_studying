**Incorrect, but good try!**

The answer is **D) Raises `ValidationError` for both `rating` and `comment`**

Both fields fail validation:

1. **`rating=6`** fails because `Field(ge=1, le=5)` means the value must be ≥1 AND ≤5. Since 6 > 5, it fails.

2. **`comment='Great!'`** fails because `Field(min_length=10)` requires at least 10 characters. `'Great!'` is only 6 characters.

The key insight: **Pydantic validates all fields and collects all errors**, rather than stopping at the first failure. This is helpful because users get a complete list of what needs fixing.

```python
# The ValidationError would show both issues:
# rating: Input should be less than or equal to 5
# comment: String should have at least 10 characters
```

Common `Field` constraints:
- Numbers: `gt`, `ge`, `lt`, `le`, `multiple_of`
- Strings: `min_length`, `max_length`, `pattern`
- Collections: `min_length`, `max_length`
