No, your Python script will **not** keep running when your Mac goes into sleep mode. Here's what happens:

**When your Mac sleeps:**
- The CPU is essentially paused/powered down
- All running processes, including your Python script, are suspended
- RAM is preserved but not actively processed
- Network connections are typically dropped

**What happens to your script:**
- It will pause exactly where it was when sleep mode activated
- When you wake your Mac, the script will resume from that exact point
- Any time-sensitive operations (like API calls with timeouts) may fail
- Network connections will likely need to be re-established

**To prevent this issue:**

1. **Prevent sleep while script runs:**
   ```bash
   caffeinate -i python your_script.py
   ```
   The `caffeinate` command prevents your Mac from sleeping while the script runs.

2. **Change Energy Saver settings:**
   - System Preferences → Battery → Power Adapter
   - Set "Prevent your Mac from automatically sleeping" when plugged in

3. **Use Terminal command:**
   ```bash
   pmset -g assertions  # Check what's preventing sleep
   sudo pmset -a sleep 0  # Disable sleep (remember to re-enable later)
   ```

4. **For long-running scripts:**
   - Consider using `screen` or `tmux` sessions
   - Run on a server or cloud instance instead
   - Add checkpointing/resume functionality to your script

The `caffeinate` command is usually the easiest solution for one-off script runs.
