`open` is a built-in macOS command that opens files, directories, and URLs using the default (or specified) application — essentially the command-line equivalent of double-clicking something in Finder.

**Basic usage:**

```bash
# Open a file with its default app
open report.pdf              # opens in Preview
open photo.png               # opens in Preview
open spreadsheet.xlsx        # opens in Excel/Numbers
open index.html              # opens in your default browser

# Open a directory in Finder
open .                       # current directory
open ~/Downloads             # Downloads folder

# Open a URL in your default browser
open https://google.com
```

**Specify which app to use:**

```bash
open -a "Google Chrome" index.html
open -a "Visual Studio Code" myproject/
open -a Safari https://example.com
open -a Preview image.jpg
```

**Other useful flags:**

```bash
# -R  Reveal in Finder (highlight the file, don't open it)
open -R ~/Downloads/report.pdf

# -e  Open in TextEdit
open -e notes.txt

# -t  Open in your default text editor
open -t config.yaml

# -n  Open a new instance of the app (even if it's already running)
open -n -a "Google Chrome"

# -g  Open in the background (don't bring the app to front)
open -g report.pdf

# Open multiple files at once
open file1.pdf file2.pdf file3.pdf
```

**Handy tricks:**

```bash
# Open Finder to the current terminal directory
open .

# Quickly open a project in VS Code
open -a "Visual Studio Code" .

# Open the output of a script
python my_script.py > output.txt && open output.txt

# Open System Settings to a specific pane
open "x-apple.systempreferences:com.apple.preference.network"
```

It's one of those small macOS commands that bridges the gap between terminal and GUI workflows really well.
