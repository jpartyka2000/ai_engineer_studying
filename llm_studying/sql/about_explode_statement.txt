If your Databricks instance does not support `UNNEST`, you can achieve similar functionality using **`EXPLODE`**, which is supported in Databricks.

### Example Using `EXPLODE`

Suppose you have the following table in Databricks:

| id  | values          |
|------|----------------|
| 1    | [10, 20, 30]   |
| 2    | [40, 50]       |
| 3    | [60]           |

You can use the `EXPLODE` function to achieve the same result:

```sql
SELECT id, value
FROM example_table
LATERAL VIEW EXPLODE(values) AS value;
```

### Result:

| id  | value |
|------|-------|
| 1    | 10    |
| 1    | 20    |
| 1    | 30    |
| 2    | 40    |
| 2    | 50    |
| 3    | 60    |

### Explanation:
- `EXPLODE(values)` transforms the array in the `values` column into multiple rows, one for each element in the array.
- `LATERAL VIEW` is used to apply the `EXPLODE` function and project the resulting values into a new column (`value`).
