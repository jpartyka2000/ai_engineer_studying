You typically use column position (`1`, `2`, etc.) in `GROUP BY` and `ORDER BY` clauses when referencing computed expressions or aliased columns. Some examples:

**GROUP BY with expression:**
```sql
SELECT 
    YEAR(created_at) AS year,
    COUNT(*)
FROM orders
GROUP BY 1
```

**ORDER BY with expression:**
```sql
SELECT 
    first_name || ' ' || last_name AS full_name,
    salary
FROM employees
ORDER BY 1
```

**Multiple positions:**
```sql
SELECT 
    department,
    CASE WHEN salary > 100000 THEN 'high' ELSE 'low' END AS salary_band,
    COUNT(*)
FROM employees
GROUP BY 1, 2
ORDER BY 3 DESC
```

**Why it's useful:** It avoids repeating long expressions. Instead of writing the entire CASE statement twice (once in SELECT, once in GROUP BY), you just say `GROUP BY 1`.

**Caveat:** Some SQL dialects (like PostgreSQL) let you reference aliases directly in `GROUP BY`, so you could write `GROUP BY lead_quality_bucket`. But others (like MySQL in strict mode, or older SQL Server) require position or repeating the expression. Position numbers are portable across dialects.
