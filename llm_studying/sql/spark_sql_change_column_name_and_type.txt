In Databricks Delta Lake (and similar environments that support Delta Lake), yes—that’s the standard and supported way to change a column’s data type using Spark SQL. The syntax:

```sql
ALTER TABLE pathahead.tealhq_ecue.pre_chart_teal_prediction_data_20250407 
CHANGE COLUMN CLV_365 CLV_365 DOUBLE;
```

works as follows:

- The first `CLV_365` identifies the existing column to be modified.
- The second `CLV_365` tells Delta Lake that the column name should remain the same.
- `DOUBLE` is the new data type for the column.

This syntax (using `CHANGE COLUMN`) is specific to Delta Lake on Databricks. In standard Hive or vanilla Spark SQL (without Delta Lake), the syntax for altering a column might differ or be more limited (often requiring a table rewrite), but for Delta Lake this is the conventional method.
