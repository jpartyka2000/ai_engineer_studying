To calculate the number of days between today and the end of the current quarter (March 31, 2025, in this case), you can use SQL's date functions. Here's how you might write a query:

```sql
SELECT DATEDIFF(DATE('2025-03-31'), CURRENT_DATE) AS days_until_end_of_quarter;
```

### Explanation:
1. **`DATE('2025-03-31')`**: This is the end date of the current quarter for your specific scenario.
2. **`CURRENT_DATE`**: Retrieves the current date.
3. **`DATEDIFF(end_date, start_date)`**: Returns the number of days between two dates.

### Generalized Query for Dynamic Calculation:
If you want to dynamically calculate the end of the current quarter based on the current date (e.g., for future queries):

```sql
SELECT DATEDIFF(
    CASE 
        WHEN MONTH(CURRENT_DATE) IN (1, 2, 3) THEN DATE(CONCAT(YEAR(CURRENT_DATE), '-03-31'))
        WHEN MONTH(CURRENT_DATE) IN (4, 5, 6) THEN DATE(CONCAT(YEAR(CURRENT_DATE), '-06-30'))
        WHEN MONTH(CURRENT_DATE) IN (7, 8, 9) THEN DATE(CONCAT(YEAR(CURRENT_DATE), '-09-30'))
        WHEN MONTH(CURRENT_DATE) IN (10, 11, 12) THEN DATE(CONCAT(YEAR(CURRENT_DATE), '-12-31'))
    END,
    CURRENT_DATE
) AS days_until_end_of_quarter;
```

### How It Works:
1. **`MONTH(CURRENT_DATE)`**: Determines the current month.
2. **`CASE` Statement**: Dynamically identifies the end date of the current quarter based on the month.
3. **`CONCAT` and `YEAR(CURRENT_DATE)`**: Construct the end-of-quarter date for the current year dynamically.

This approach ensures the query works regardless of the current date and aligns with standard calendar quarters.
