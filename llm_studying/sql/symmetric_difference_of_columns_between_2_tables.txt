If you want the **symmetric difference** of columns between two tables (or a table and a view), you’re asking for:

> **Columns that exist in one table but not the other.**

That’s the *opposite* of an intersection — it’s a union of both “differences.”

---

## 🧠 Conceptually

Given two sets of column names:

```
A = columns in table1  
B = columns in table2
```

Then the **symmetric difference** is:

```
(A − B) ∪ (B − A)
```

---

## ✅ SQL Version (portable across most engines)

```sql
(
  SELECT column_name
  FROM information_schema.columns
  WHERE table_schema = 'sales' AND table_name = 'table1'
  EXCEPT
  SELECT column_name
  FROM information_schema.columns
  WHERE table_schema = 'sales' AND table_name = 'table2'
)
UNION
(
  SELECT column_name
  FROM information_schema.columns
  WHERE table_schema = 'sales' AND table_name = 'table2'
  EXCEPT
  SELECT column_name
  FROM information_schema.columns
  WHERE table_schema = 'sales' AND table_name = 'table1'
);
```

### 🧩 What this does

* The first part (`EXCEPT`) gets columns in `table1` that aren’t in `table2`.
* The second part gets the reverse.
* The `UNION` joins them into one list (removing duplicates automatically).

This works whether one of the objects is a **view** or a **table** — both appear in `INFORMATION_SCHEMA.COLUMNS`.

## 🐍 PySpark Version (if you prefer programmatic logic)

```python
cols1 = set([c.name for c in spark.table("sales.table1").schema])
cols2 = set([c.name for c in spark.table("sales.table2").schema])

sym_diff = cols1 ^ cols2   # symmetric difference operator in Python
print(sorted(sym_diff))
```

This returns all columns that exist in exactly one of the two tables/views.

## 🧾 Optional: Include Data Types

If you want to see *which table* each column belongs to, add a label:

```sql
SELECT column_name, 'table1' AS source
FROM information_schema.columns
WHERE table_schema = 'sales' AND table_name = 'table1'
EXCEPT
SELECT column_name, 'table1' AS source
FROM information_schema.columns
WHERE table_schema = 'sales' AND table_name = 'table2'
UNION ALL
SELECT column_name, 'table2' AS source
FROM information_schema.columns
WHERE table_schema = 'sales' AND table_name = 'table2'
EXCEPT
SELECT column_name, 'table2' AS source
FROM information_schema.columns
WHERE table_schema = 'sales' AND table_name = 'table1';
```

This gives you both the **column name** and **which table it’s unique to**.
