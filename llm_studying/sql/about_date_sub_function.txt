The SQL `DATE_SUB()` function is used to subtract a specified time interval (such as days, months, or years) from a given date, returning the resulting date. It is commonly used for date arithmetic, such as calculating past dates relative to the current date.

### Syntax

In SQL systems that support the `DATE_SUB()` function (e.g., MySQL):

```sql
DATE_SUB(date, INTERVAL expr unit)
```

- **`date`**: The starting date.
- **`expr`**: The numerical value of the interval to subtract.
- **`unit`**: The unit of time, such as `DAY`, `MONTH`, `YEAR`, etc.

### Example

```sql
SELECT DATE_SUB('2024-01-10', INTERVAL 7 DAY) AS new_date;
```

Result:
```
new_date
2024-01-03
```

This subtracts 7 days from `2024-01-10`, resulting in `2024-01-03`.

---

### Databricks SQL

Databricks SQL has a simplified version of `DATE_SUB()`, which only supports subtracting days:

```sql
DATE_SUB(date, days)
```

- **`date`**: The starting date.
- **`days`**: The number of days to subtract.

### Example in Databricks

```sql
SELECT DATE_SUB(CURRENT_DATE, 10) AS new_date;
```

This subtracts 10 days from the current date.

---

### Key Differences Between Implementations
1. **MySQL/Other SQL Systems**:
   - Uses `INTERVAL` with various units like `DAY`, `MONTH`, `YEAR`.
2. **Databricks SQL**:
   - Simplified syntax; only supports subtracting days directly with an integer.
