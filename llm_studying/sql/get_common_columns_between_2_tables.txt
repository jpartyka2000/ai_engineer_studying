Great question — you can get the **common (shared) column names** between two tables by comparing their **metadata**, i.e. their column definitions in the system catalog, rather than the data itself.

The exact query depends slightly on your SQL dialect (PostgreSQL, Databricks, MySQL, etc.), but here’s how to do it in general terms 👇

---

## 🧠 Generic SQL (ANSI-style) version

Most SQL databases expose a view called `INFORMATION_SCHEMA.COLUMNS`.
You can query it like this:

```sql
SELECT column_name
FROM information_schema.columns
WHERE table_name = 'table1'
INTERSECT
SELECT column_name
FROM information_schema.columns
WHERE table_name = 'table2';
```

### ✅ Explanation

* `information_schema.columns` lists all columns for all tables.
* `INTERSECT` returns only the **column names that appear in both** table definitions.

---

## 💡 Example (PostgreSQL, Databricks, Snowflake, BigQuery, etc.)

```sql
SELECT column_name
FROM information_schema.columns
WHERE table_schema = 'my_schema' AND table_name = 'customers'
INTERSECT
SELECT column_name
FROM information_schema.columns
WHERE table_schema = 'my_schema' AND table_name = 'orders';
```

This will return all column names that both `customers` and `orders` share.

---

## ⚙️ Databricks (Spark SQL) version

In Databricks, you can use the same `information_schema.columns` table introduced in **Unity Catalog**:

```sql
SELECT column_name
FROM my_catalog.information_schema.columns
WHERE table_schema = 'sales' AND table_name = 'table_a'
INTERSECT
SELECT column_name
FROM my_catalog.information_schema.columns
WHERE table_schema = 'sales' AND table_name = 'table_b';
```

Or, if you’re in a workspace without Unity Catalog, you can use `DESCRIBE` and then do it in Python instead (see below).

---

## 🐍 PySpark alternative (if you’re in Databricks or Spark)

You can get the common columns programmatically:

```python
cols1 = set([c.name for c in spark.table("db.table1").schema])
cols2 = set([c.name for c in spark.table("db.table2").schema])

common_cols = cols1 & cols2
print(sorted(common_cols))
```

This is often faster and works even if you don’t have `information_schema` access.

---

## 🔍 Summary

| Environment                  | Method                                   | Example                |
| ---------------------------- | ---------------------------------------- | ---------------------- |
| ANSI / PostgreSQL / BigQuery | `INFORMATION_SCHEMA.COLUMNS + INTERSECT` | ✅ Easiest portable way |
| Databricks UC                | `catalog.information_schema.columns`     | ✅ Recommended for UC   |
| Spark (no UC)                | Use Python / PySpark schema comparison   | ✅ Works anywhere       |
