The short answer is that **`MERGE INTO` is part of the ANSI/ISO SQL standard** (it was introduced in SQL:2003), and many commercial and open-source databases (Oracle, SQL Server, DB2, PostgreSQL 15+, etc.) implement it.

Spark’s own core SQL engine, however, did *not* support `MERGE` until you add an extension like Delta Lake.  So when you write:

```sql
MERGE INTO my_table
USING updates
  ON my_table.key = updates.key
WHEN MATCHED  THEN UPDATE …
WHEN NOT MATCHED THEN INSERT …
```

you’re using the ANSI-standard syntax—but in Spark you only get it if you’re running with **Delta Lake** (or another storage layer that implements it).  Plain Spark SQL (without Delta) does not provide a native `MERGE` command.
