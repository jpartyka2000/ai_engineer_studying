Yes, Delta Lake supports time travel, so you can query an older version of a table using Spark SQL. For example, if you want to query a specific version, you can use:

```sql
SELECT * FROM my_table VERSION AS OF 5;
```

Or, you can specify a timestamp:

```sql
SELECT * FROM my_table TIMESTAMP AS OF '2023-03-25 12:00:00';
```

This lets you work with historical versions of your table directly via Spark SQL.
