There’s no special “threshold” field in MLflow—just treat your decision threshold like any other piece of metadata about the run and log it as a **param** (or, if you prefer, a **tag**). For example:

```python
import mlflow

threshold = 0.37  # whatever you chose at training time

with mlflow.start_run():
    # … your .fit(), .predict_proba(), .log_model(), etc. …

    # Log the threshold so you (and downstream consumers) know exactly
    # what cutoff you used to turn probs into class labels.
    mlflow.log_param("prediction_threshold", threshold)
```

Later, in the MLflow UI under **Params**, you’ll see `prediction_threshold=0.37` alongside your other hyperparameters. If you’d rather keep it out of Params and in the “Tags” panel, you can do:

```python
mlflow.set_tag("prediction_threshold", threshold)
```

Either way, anyone inspecting the run (or programmatically retrieving it via the MLflow client) can see exactly which threshold was used.
