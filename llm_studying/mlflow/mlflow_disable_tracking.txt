There isn’t a built‐in “off switch” in MLflow, but you can effectively disable its tracking in your notebook by overriding its configuration. For example, if you don’t want MLflow to log any runs or data, you can set its tracking URI to a dummy location. One common approach is:

```python
import mlflow
mlflow.set_tracking_uri("file:///dev/null")
```

By doing this, all MLflow tracking calls will write to a location that essentially discards the data. Also, if you have code that calls autologging (e.g. `mlflow.autolog()`), simply avoid invoking that in your notebook.

Additionally, if a run is already active, you can call:

```python
mlflow.end_run()
```

to close it.

Using these techniques lets you “turn off” MLflow’s side effects without removing the library from your environment.
