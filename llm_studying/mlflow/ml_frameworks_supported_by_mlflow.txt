MLflow supports many popular model frameworks, either natively (as “flavors”) or via its generic Python model interface (pyfunc). Here's a categorized breakdown of what MLflow supports out of the box as of recent versions:


---

1. ML Libraries with Native MLflow Flavors

These have dedicated logging and loading APIs (like mlflow.sklearn.log_model()):

Framework MLflow Flavor Notes

scikit-learn sklearn One of the most fully supported flavors
XGBoost xgboost Includes booster and scikit-learn APIs
LightGBM lightgbm Supports classifier and regressor models
CatBoost catboost Native flavor support
TensorFlow tensorflow Includes support for Keras and TF SavedModel
PyTorch pytorch Logs models via TorchScript or traced modules
statsmodels statsmodels Good for classical statistical models
Prophet prophet For time series forecasting
Glucose glucose For modeling glucose responses in medical data (niche)



---

2. Python Function Interface (pyfunc)

Universal interface that wraps any Python model, including custom ones.

You implement your own predict() method (like you did earlier).

This is how MLflow supports arbitrary models like:

Ensembles

Rule-based systems

Multi-step pipelines

Custom deep learning models with non-standard inputs/outputs




---

3. Others via Plugin or ONNX/MLflow Extensions

Format Flavor Notes

ONNX onnx Interoperability format for many model types
H2O h2o For models built with H2O.ai
Spark MLlib spark Spark pipelines and transformers
Java MLeap mleap For scoring in Java environments
FastAI via pyfunc No native flavor yet, but easy to wrap
Keras (standalone) deprecated in favor of tensorflow



---

Summary

MLflow natively supports most major ML libraries.

If your model isn’t listed, you can still use mlflow.pyfunc to wrap and log it.

You can inspect a logged model’s flavors to see how it was saved.
