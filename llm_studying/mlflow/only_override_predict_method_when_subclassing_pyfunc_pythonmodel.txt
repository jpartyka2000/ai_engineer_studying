You only **must** override **`predict(self, context, model_input)`**—that’s the core entry point MLflow will call when serving or scoring your model.  Without it, your class won’t do anything useful.

As for **`load_context(self, context)`**, it’s purely optional:

* **Override it** if you need to grab artifacts (files, pickled objects, parameters) that you logged alongside your model via `mlflow.pyfunc.log_model(..., artifacts=…)`.
* **Leave it alone** if your simulator has no external dependencies beyond what’s already in your code (e.g. it doesn’t need to load extra lookup tables or serializers).

So in most simple cases you’ll end up with:

```python
class MCSimulator(mlflow.pyfunc.PythonModel):

    def predict(self, context, model_input: pd.DataFrame) -> pd.DataFrame:
        # your Monte Carlo logic here
        return run_my_simulation(model_input)
```

—and you can omit `load_context` entirely.
