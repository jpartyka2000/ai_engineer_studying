No, you don't always have to use runs:/ as the prefix in MLflow URIs. The prefix depends on where the model is stored and how you intend to load it. Here are the different cases:

1. runs:/ Prefix (When Loading from an Experiment Run)

Use runs:/ when retrieving a model from a specific run within an MLflow experiment.

model_uri = f"runs:/{run.info.run_id}/random_forest_model"
loaded_model = mlflow.sklearn.load_model(model_uri)

Use case: Loading the model directly from an experiment run.

Stored location: Within a specific MLflow run (not the model registry).



---

2. models:/ Prefix (When Loading from the MLflow Model Registry)

Use models:/ when retrieving a registered model from the MLflow Model Registry.

model_name = "RandomForest_Classifier"
model_version = "1"
model_uri = f"models:/{model_name}/{model_version}"
loaded_model = mlflow.pyfunc.load_model(model_uri)

Use case: Loading a model that has been registered in the MLflow Model Registry.

Stored location: Centralized Model Registry in MLflow.


You can also load the latest version in a given stage (Staging, Production, etc.):

model_uri = f"models:/{model_name}/latest"
loaded_model = mlflow.pyfunc.load_model(model_uri)


---

3. artifacts:/ Prefix (When Loading from an Artifact Path)

Use artifacts:/ to load a model from the MLflow Artifact Store (e.g., S3, DBFS, or local storage).

model_uri = "artifacts:/random_forest_model"
loaded_model = mlflow.pyfunc.load_model(model_uri)

Use case: When models are logged as artifacts but not registered in the Model Registry.

Stored location: Artifact store (dbfs:/mlflow/artifacts/... in Databricks, S3, GCS, etc.).



---

4. databricks:/ Prefix (When Using Databricks MLflow Tracking)

If you're using Databricks-hosted MLflow, you can specify the Databricks workspace:

model_uri = f"databricks://models/{model_name}/{model_version}"
loaded_model = mlflow.pyfunc.load_model(model_uri)

Use case: When retrieving models from Databricks-managed MLflow.

Stored location: Databricks Model Registry.



---

Which One Should You Use?

✅ Use runs:/ if you’re pulling the model from a specific MLflow experiment run.

✅ Use models:/ if you’ve registered the model in MLflow's Model Registry.

✅ Use artifacts:/ if you’re loading a model from MLflow’s Artifact Store.

✅ Use databricks:/ if you’re working with Databricks-hosted MLflow.
