No—it doesn’t automatically go into “Staging” or “Production.”  Any brand‐new version you register will come up in the **None** stage by default.  You’ll see it listed in the Model Registry with a little “None” badge.

If you want it to be in Staging or Production, you need to move it there explicitly, either via:

```python
from mlflow.tracking import MlflowClient

client = MlflowClient()
client.transition_model_version_stage(
    name="MyModel",
    version=5,
    stage="Staging",
    archive_existing_versions=False
)
```

—or by clicking the “Transition to Staging/Production” buttons in the MLflow UI.
