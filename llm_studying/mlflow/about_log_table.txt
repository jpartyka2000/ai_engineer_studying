In MLflow 2.11.4, `mlflow.log_table` is a lightweight helper for writing tabular data (typically a Pandas DataFrame) into your run’s artifacts as a Parquet file, and making it natively viewable in the UI. Its simplified signature is:

```python
mlflow.log_table(
    table,           # pandas.DataFrame (or path to file on disk)
    artifact_file    # e.g. "tables/my_table.parquet" (including any sub‑folder)
)
```

**Key points:**

* **`table`** can be either a `pandas.DataFrame` (most common) or a string path pointing to an existing local table file.
* **`artifact_file`** is the *relative* path (within the run’s artifacts) where the Parquet will be written. If you want it in a sub‑directory, just include that in the name (e.g., `"tables/win_rates.parquet"`).
* MLflow will serialize the DataFrame to Parquet under the hood, log it as an artifact, and expose it in the “Artifacts → Tables” section of the run UI so you can preview it in‑browser.
* There is **no** separate `artifact_path` parameter in 2.11.4—instead, you control folders via the `artifact_file` path.
* On the Python side, you can later retrieve it with `mlflow.artifacts.download_artifacts()` or use `mlflow.load_table()` to read it back into a DataFrame.

This makes it super‐easy to bundle any intermediate or result tables alongside your model and metrics in MLflow without manual file I/O.
