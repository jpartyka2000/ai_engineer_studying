To disconnect a Databricks notebook from **MLflow** without deleting the notebook, you can stop automatic MLflow tracking and remove any related configurations without deleting the notebook itself. There are a few steps you can take to ensure MLflow tracking is disabled while keeping the notebook intact.

### Steps to Disconnect MLflow:

1. **Turn Off Autologging**:
   If you have enabled **MLflow autologging**, you can disable it using the following command:
   
   ```python
   import mlflow
   mlflow.autolog(disable=True)
   ```

   This will stop MLflow from automatically tracking any model training, metrics, or parameters without affecting your existing notebook content.

2. **Stop Active MLflow Run (if any)**:
   If there's an active MLflow run that is still open, you can end the run by calling `mlflow.end_run()`.

   ```python
   mlflow.end_run()
   ```

   This will close the current MLflow tracking run, effectively disconnecting your notebook from tracking new runs.

3. **Remove MLflow Tracking URIs**:
   If you had manually set an **MLflow tracking URI** in the notebook, you can remove or reset it using:

   ```python
   mlflow.set_tracking_uri(None)
   ```

   This will reset the tracking URI to the default behavior (local logging), which disconnects the notebook from any external tracking servers.

4. **Check for Manual Logging**:
   If you are logging parameters, metrics, or models manually (via `mlflow.log_param()`, `mlflow.log_metric()`, etc.), ensure that you either remove or comment out those lines to prevent further MLflow logging.

5. **Clear MLflow Configuration**:
   Optionally, you can clear the MLflow experiment settings associated with your notebook. If an experiment is set, you can reset it using:

   ```python
   mlflow.set_experiment(None)
   ```

### Example:

Here’s a quick example of how you can disable MLflow tracking in your notebook:

```python
import mlflow

# Turn off MLflow autologging
mlflow.autolog(disable=True)

# End any active MLflow run
mlflow.end_run()

# Reset the tracking URI to disconnect from any external tracking server
mlflow.set_tracking_uri(None)

# Optionally, reset the experiment to None
mlflow.set_experiment(None)

# If you have manual logging, comment out or remove mlflow.log_param, mlflow.log_metric, etc.
```

### Summary:
- Use `mlflow.autolog(disable=True)` to turn off automatic logging.
- Use `mlflow.end_run()` to stop active MLflow runs.
- Reset the tracking URI with `mlflow.set_tracking_uri(None)`.
- Remove or comment out any manual logging lines (e.g., `mlflow.log_param()`).

This will effectively disconnect your notebook from MLflow tracking without deleting or modifying the notebook itself.
