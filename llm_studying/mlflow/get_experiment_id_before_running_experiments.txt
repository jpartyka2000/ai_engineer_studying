Even if you haven't run any experiments yet, MLflow still tracks experiments. You can retrieve an experiment's ID by its name. For example, you can use:

```python
import mlflow

experiment_name = "/Workspace/Shared/DiabetesClassification"  # or your experiment name
experiment = mlflow.get_experiment_by_name(experiment_name)

if experiment is None:
    # Optionally, create the experiment if it doesn't exist:
    experiment_id = mlflow.create_experiment(experiment_name)
else:
    experiment_id = experiment.experiment_id

print("Experiment ID:", experiment_id)
```

### Explanation

- **mlflow.get_experiment_by_name:**  
  This function checks whether an experiment with the given name exists. It returns an Experiment object (which includes the experiment ID) if found, or `None` if it doesn't exist.

- **mlflow.create_experiment:**  
  If the experiment doesn't exist, you can create it, which returns the new experiment ID.

- **No Runs Required:**  
  You don't need to have run any experiments yet to retrieve or create an experiment ID. The experiment record exists independently of the runs.

This approach ensures that you always have an experiment ID available, whether or not you have executed any runs yet.
