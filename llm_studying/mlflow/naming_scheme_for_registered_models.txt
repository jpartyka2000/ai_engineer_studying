In Unity Catalog–enabled workspaces on Databricks, model names must follow a fully qualified three-level naming convention:  

```
catalog_name.schema_name.model_name
```

For example, if your catalog is named `prod_catalog`, your schema is `ml_models`, and your model is `diabetes_model`, you would register it as:  

```
prod_catalog.ml_models.diabetes_model
```

If you try to register a model without including all three levels (catalog, schema, and model name), MLflow will throw the error you saw. To resolve this, simply provide a fully qualified model name that meets this format when you register or log your model in MLflow.
