You don’t have to rely on MLflow’s automated inference (which can sometimes mis-type columns or choke on complex nested structures). Instead, you can **manually define** your model’s input and output schemas using MLflow’s `ModelSignature`, `Schema`, and `ColSpec` classes.

1. **Input schema**: list each expected column name and its type (`double`, `integer`, `string`, etc.) so MLflow knows exactly what it will receive.
2. **Output schema**: do the same for every column in your returned DataFrame—even nested objects can be marked as JSON strings.

By passing that signature into `mlflow.pyfunc.log_model(signature=…)`, you guarantee correct schema enforcement at serve-time, clear documentation in the Model Registry, and compliance with Unity Catalog’s requirements—no ambiguous inference, no surprises when you deploy.
