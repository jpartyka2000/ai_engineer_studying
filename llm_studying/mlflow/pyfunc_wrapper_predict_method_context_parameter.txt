Yes—the `predict()` signature **must** include that first `context` argument, even if you don’t use it.  MLflow’s pyfunc runner will always call:

```python
model.predict(context, model_input)
```

so you need:

```python
def predict(self, context, model_input: pd.DataFrame) -> pd.DataFrame:
    # … your code …
```

If you drop `context` from your method signature, MLflow will raise a “wrong number of arguments” error at serve-time.  If you really don’t use it, you can name it `_` to signal that:

```python
def predict(self, _context, model_input: pd.DataFrame) -> pd.DataFrame:
    # … ignore _context …
    return sim.run_simulation()
```
