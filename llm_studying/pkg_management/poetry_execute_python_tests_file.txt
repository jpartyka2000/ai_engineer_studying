The error **"Command not found: run_tests.py"** occurs because Poetry is trying to treat `run_tests.py` as a command, but it doesn't know how to execute it directly.

In Poetry, you need to explicitly specify how to run Python scripts (i.e., you need to invoke Python to run the script). Here’s how you can run your `run_tests.py` file properly using Poetry:

### **1. Use Poetry to Run the Script via Python**
Since `run_tests.py` is a Python file, you can run it like this:

```bash
poetry run python run_tests.py
```

This will use Poetry’s virtual environment to run the Python script `run_tests.py`.

### **2. Alternatively, Add a Script to `pyproject.toml` (Optional)**
If you often run `run_tests.py` and want to make it a named command, you can add a custom script in the `[tool.poetry.scripts]` section of your `pyproject.toml`.

For example:

```toml
[tool.poetry.scripts]
run-tests = 'run_tests:main'
```

This assumes that `run_tests.py` has a function named `main()` that you want to run. With this setup, you could run:

```bash
poetry run run-tests
```

### **3. Verify Poetry Environment**
If the issue persists, ensure that Poetry is correctly setting up and using the virtual environment by checking the active environment:

```bash
poetry env info
```

This will confirm that Poetry is using the correct virtual environment.

### **Summary**
To run a Python script with Poetry, you should use:

