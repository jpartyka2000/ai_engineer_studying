By default, `uv` looks for a `pyproject.toml` in your **current working directory** and works its way up to find the project root.

However, if you're using a **workspace** setup (monorepo), the behavior is different:

- The root `pyproject.toml` can define `[tool.uv.workspace]` with member packages
- A single `uv.lock` at the workspace root covers all member packages
- Running `uv lock --check` from anywhere in the workspace validates against the root lock file

You can check which project `uv` is detecting by running:

```bash
uv pip show <your-package-name>
```

Or to see where `uv` thinks the project root is, you can try:

```bash
uv tree
```

This will show the dependency tree and implicitly tell you which `pyproject.toml` it's reading from.

If you want to explicitly target a specific project, you can use:

```bash
uv lock --check --directory /path/to/project
```
