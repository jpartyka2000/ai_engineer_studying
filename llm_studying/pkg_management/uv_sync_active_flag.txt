The `--active` flag tells `uv run` to use whatever virtual environment is currently activated in your shell, rather than having uv manage one for you.

Without `--active`, uv will:
1. Look for a `.venv` in your project
2. Create one if it doesn't exist
3. Use that environment regardless of what you have activated

With `--active`, uv respects your shell's current environment — so if you've run `source some-other-venv/bin/activate`, that's what gets used.

This is useful when:
- You're working with an existing venv you manage yourself
- You have multiple environments and want explicit control over which one runs
- You're integrating uv into a workflow that already handles environment activation
