That error usually means your `uv.lock` file has gotten into an inconsistent state - specifically, `sqlglot` appears multiple times without proper source information to disambiguate them.

Here are a few things to try:

**1. Regenerate the lock file (safest):**
```bash
rm uv.lock
uv lock
```

**2. If that doesn't work, try with a clean cache:**
```bash
rm uv.lock
uv cache clean
uv lock
```

**3. Check for conflicting dependencies:**
Look in your `pyproject.toml` to see if `sqlglot` is specified multiple times or coming from multiple sources (like both PyPI and a Git repo). If so, consolidate them.

**4. Update uv itself:**
```bash
pip install --upgrade uv
```
Sometimes this is a bug in older versions of uv.

The regeneration approach (option 1) usually fixes this since it forces uv to rebuild the lock file from scratch with consistent source information.
