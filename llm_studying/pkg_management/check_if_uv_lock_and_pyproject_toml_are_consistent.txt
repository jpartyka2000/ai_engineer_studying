Yes! With `uv`, you can use the `--check` flag to verify consistency without making any changes:

```bash
uv lock --check
```

This will exit with a non-zero status code if the `uv.lock` file is inconsistent with your `pyproject.toml` (meaning dependencies have been added, removed, or changed). If everything is in sync, it exits silently with code 0.

This is particularly useful in CI pipelines to ensure developers have committed an up-to-date lock file.

If you find they're out of sync, just run:

```bash
uv lock
```

to regenerate the lock file, or:

```bash
uv sync
```

to both update the lock file and install the dependencies.
