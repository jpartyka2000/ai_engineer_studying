You likely have multiple installations of `modal` — one in your conda `(base)` environment and one managed by `uv`. Check where each one lives:

```bash
which modal
```

This will show you which `modal` binary your terminal is actually using. It's probably the conda one at something like `/Users/jpartyka/anaconda3/bin/modal`.

To fix it, either:

1. **Upgrade in conda directly:**
   ```bash
   pip install modal==1.3.0
   ```
   (without `uv`, since conda's `pip` is what's on your PATH)

2. **Or deactivate conda** so your shell picks up the `uv`-managed version:
   ```bash
   conda deactivate
   modal --version
   ```

Option 1 is probably the simplest path forward.
