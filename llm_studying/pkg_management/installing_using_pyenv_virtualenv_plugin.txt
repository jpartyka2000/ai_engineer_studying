To set up a virtual environment with `pyenv`, you can use `pyenv`'s `pyenv virtualenv` plugin, which allows you to manage both Python versions and isolated environments.

Here are the steps to set it up:

---

### **1. Install `pyenv` and `pyenv virtualenv`**

First, ensure that you have `pyenv` and `pyenv virtualenv` installed. If you haven‚Äôt already installed `pyenv`, follow these steps:

#### **Install `pyenv` (if you haven't already):**

```bash
brew install pyenv
```

After installation, add `pyenv` to your shell configuration (`~/.bash_profile`, `~/.zshrc`, etc.):

```bash
# Add this to your shell profile (e.g., ~/.bash_profile or ~/.zshrc)
export PATH="$HOME/.pyenv/bin:$PATH"
eval "$(pyenv init --path)"
eval "$(pyenv init -)"
eval "$(pyenv virtualenv-init -)"
```

#### **Install `pyenv virtualenv` plugin (if you haven‚Äôt already):**

If you installed `pyenv` with Homebrew, you likely already have `pyenv virtualenv`. If not, install it:

```bash
brew install pyenv-virtualenv
```

Then, add the following line to your shell configuration file (if not already added):

```bash
eval "$(pyenv virtualenv-init -)"
```

Reload your shell configuration file:

```bash
source ~/.bash_profile   # or source ~/.zshrc depending on your shell
```

---

### **2. Install the Python Version You Want to Use**
You need to install a Python version that you will use for the virtual environment. For example:

```bash
pyenv install 3.10.6
```

This will install Python 3.10.6. You can install any other version you need.

---

### **3. Create a Virtual Environment**
Once you have the desired Python version installed, you can create a virtual environment using `pyenv virtualenv`.

```bash
pyenv virtualenv 3.10.6 myenv
```

In this example:
- `3.10.6` is the Python version you want to base the environment on.
- `myenv` is the name of your virtual environment.

---

### **4. Activate the Virtual Environment**
To use your virtual environment, run:

```bash
pyenv activate myenv
```

This will activate the `myenv` virtual environment. Your shell prompt will change to indicate that you‚Äôre working in the virtual environment.

---

### **5. Deactivate the Virtual Environment**
To deactivate the virtual environment and return to the global Python environment:

```bash
pyenv deactivate
```

---

### **6. Set Local Python Version for a Project (Optional)**
If you want to set the virtual environment to be used automatically in a specific directory (for example, for a project), run the following command inside the project directory:

```bash
pyenv local myenv
```

This creates a `.python-version` file in the directory, which tells `pyenv` to automatically use the `myenv` virtual environment when you're in that directory.

---

### **7. Check Your Python Environment**
To verify the active environment:

```bash
python --version
```

Or check which Python binary is being used:

```bash
which python
```

This should point to your `pyenv` virtual environment, like:

```bash
~/.pyenv/versions/myenv/bin/python
```

---

### **Summary**
- **Install `pyenv` and `pyenv virtualenv`.**
- **Install a Python version** using `pyenv install`.
- **Create a virtual environment** using `pyenv virtualenv`.
- **Activate the virtual environment** with `pyenv activate`.
- Optionally, use `pyenv local` to set a virtual environment for a specific directory.
