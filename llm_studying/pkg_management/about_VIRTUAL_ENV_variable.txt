`$VIRTUAL_ENV` is set by the `activate` script when you activate a virtual environment. It's not set by the `venv` module itself when creating the environment — it only gets set when you source the activate script:

```bash
source .venv/bin/activate
```

At that point, the activate script:
1. Sets `VIRTUAL_ENV` to the full path of the venv
2. Modifies `PATH` to put the venv's `bin/` directory first
3. Modifies your prompt to show the venv name

When you run `deactivate`, it unsets `VIRTUAL_ENV` and restores your original `PATH` and prompt.

So if you create a venv but never activate it (e.g., you just call `.venv/bin/python` directly), `$VIRTUAL_ENV` won't be set.
