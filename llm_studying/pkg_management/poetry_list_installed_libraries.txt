To list the libraries (or dependencies) that have been installed in your Poetry-managed virtual environment, you can use the following commands:

### **1. List Installed Libraries with Poetry**
You can use the `poetry show` command to list all the installed libraries in the current virtual environment:

```bash
poetry show
```

This will display a list of all installed packages, including their versions and a brief description.

### **2. List Only Direct Dependencies**
If you want to see only the **direct dependencies** (i.e., those listed in your `pyproject.toml`), you can use:

```bash
poetry show --no-dev
```

This will exclude development dependencies (i.e., dependencies listed in the `[tool.poetry.dev-dependencies]` section of `pyproject.toml`).

### **3. List Development Dependencies**
If you want to see only the **development dependencies**, run:

```bash
poetry show --dev
```

This will list only the libraries that are installed under the `dev-dependencies` section of your `pyproject.toml`.

### **4. List All Installed Libraries (Including Dependencies)**
If you want to see **all installed libraries**, including both direct and transitive dependencies (those required by the libraries you installed), you can just use:

```bash
poetry show --tree
```

This will show a hierarchical tree of the installed packages, so you can see which packages depend on others.

---

### **Using `pip` within Poetry's Virtual Environment**
You can also use `pip` to list the installed libraries directly within Poetry's virtual environment. Run:

```bash
poetry run pip list
```

This will display all installed libraries in the virtual environment, including their versions.

---

### **Summary**
- **`poetry show`**: List all installed libraries.
- **`poetry show --no-dev`**: List only direct dependencies (excluding dev dependencies).
- **`poetry show --dev`**: List only dev dependencies.
- **`poetry show --tree`**: Display installed libraries in a tree structure.
- **`poetry run pip list`**: List all installed libraries via `pip`.
