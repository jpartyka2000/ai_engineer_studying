`uv tree` displays your project's dependency tree in a visual format. It's great for understanding your dependency graph.

**Basic usage:**

```bash
uv tree
```

This shows all your dependencies and their transitive dependencies in a tree structure.

**Useful flags:**

| Flag | Description |
|------|-------------|
| `--depth <N>` | Limit how deep the tree goes |
| `--invert` | Invert the tree to show why a package is included |
| `--package <name>` | Show tree for a specific package |
| `--outdated` | Highlight packages with newer versions available |

**Examples:**

```bash
# See only direct dependencies
uv tree --depth 1

# Find out WHY a package is in your tree (reverse lookup)
uv tree --invert --package requests

# Check for outdated packages
uv tree --outdated
```

The `--invert` option is particularly handy when you're wondering "why do I have this dependency?" — it shows you the chain of packages that pulled it in.

For the full list of options, run:

```bash
uv tree --help
```
