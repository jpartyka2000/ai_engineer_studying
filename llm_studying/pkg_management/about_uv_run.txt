Yes, `uv` can run Python scripts using the `uv run` command. Here's what it does:

**Basic functionality:** `uv run` executes a Python script or command within an environment that `uv` manages. It automatically handles dependency resolution and virtual environment creation.

**What happens when you run `uv run script.py`:**

1. **Dependency detection** — uv looks for dependencies declared in a `pyproject.toml`, `requirements.txt`, or inline script metadata (PEP 723 style comments at the top of the script).

2. **Environment creation** — It creates or reuses a virtual environment with those dependencies installed, all cached for speed.

3. **Execution** — It runs your script in that environment.

**Inline script dependencies (PEP 723):**
You can declare dependencies directly in a script:
```python
# /// script
# dependencies = ["requests", "rich"]
# ///

import requests
from rich import print
# ...
```

Then just run `uv run script.py` — uv installs `requests` and `rich` automatically.

**Other useful options:**
- `uv run --with package_name script.py` — temporarily add a package for this run
- `uv run python` — launch a Python REPL in the managed environment
- `uv run pytest` — run tools like pytest with your project's dependencies

The main appeal is speed (uv is written in Rust and is extremely fast) and convenience (no manual `pip install` or venv activation needed).
