When you install `pre-commit` via **Homebrew**, yes — Homebrew will also install a Python runtime if it’s needed as a dependency. Right now that would likely be **Python 3.13**, since that’s the latest release line.

But here are some clarifications:

* ✅ **It won’t overwrite your system Python** (the one that ships with macOS). macOS still has its own Python (historically 2.7, and in newer macOS, none at all by default). Homebrew keeps its Python separate in `/usr/local/Cellar/python@3.13/...` (on Intel Macs) or `/opt/homebrew/Cellar/...` (on Apple Silicon).
* ✅ You’ll get a `python3` binary on your PATH that points to the Homebrew Python. This becomes the *default* `python3` if `/usr/local/bin` or `/opt/homebrew/bin` is before `/usr/bin` in your PATH.
* ✅ Other Python versions (like 3.11 or 3.12) that you’ve installed manually (e.g., via `pyenv` or `conda`) won’t be deleted. They’ll still exist, but you’ll need to manage your PATH to decide which one is "first".

So basically:

* Homebrew will **add** Python 3.13 to your system, not replace macOS’s internal one.
* Your shell may start using it by default depending on PATH order.

---

👉 If you’re worried about version conflicts:

* You can install `pre-commit` with **pipx** instead. That way, it uses whichever Python you prefer, without pulling in Homebrew’s Python.
* Or you can keep Homebrew’s Python and manage multiple versions with `pyenv`.
