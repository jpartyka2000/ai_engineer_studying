Exactly right.

**`pip install .`** (or `pip install some-package`) copies files into `site-packages`. The source could be:
- Your local directory (when you run `pip install .`)
- PyPI (when you run `pip install requests`)
- A git repository (when you run `pip install git+https://github.com/...`)

Regardless of the source, the result is the same: files get copied into `site-packages`, and that copy is what Python uses.

**`pip install -e .`** doesn't copy anything. It just tells Python "when you need this package, look over here." Your source directory *is* the installed package.

One small clarification: it's not necessarily the "most updated version" — it's just whatever is in that directory at the moment you import it. If you delete half the code, that's what you get. It's a live link, for better or worse.
