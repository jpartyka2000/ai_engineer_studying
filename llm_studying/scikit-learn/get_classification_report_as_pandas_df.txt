Yes—under the hood `classification_report` can spit back a dict of numbers instead of a pretty‐printed string, and you can just wrap that in `pd.DataFrame`. For example:

```python
from sklearn.metrics import classification_report
import pandas as pd

# y_true and y_pred are your true/predicted labels
report_dict = classification_report(
    y_true, 
    y_pred, 
    output_dict=True,   # ← return a dict instead of a string
    target_names=["Class 0","Class 1"],
    digits=3
)

# convert to DataFrame and transpose so that each class/“accuracy” row becomes a DataFrame row
df_report = pd.DataFrame(report_dict).T

print(df_report)
```

That will give you a DataFrame that looks something like:

|               | precision | recall | f1‐score | support |
|--------------:|----------:|-------:|---------:|--------:|
|      Class 0  |      0.812|   0.850|     0.830|   2073 |
|      Class 1  |      0.745|   0.689|     0.716|    759 |
|      accuracy |      0.796|    —   |       —  |      — |
|     macro avg |      0.778|   0.770|     0.773|   2832 |
|  weighted avg |      0.792|   0.796|     0.793|   2832 |

And then you can sort it, round it, add or drop rows as you like.
