Not always – it depends on the solver you pick:

* **For `solver='sag'` or `'saga'`**
  Scikit-learn treats **one “iteration”** as **one full pass (epoch) over your data**. So in that case

  ```python
  max_iter=1000
  ```

  really means “up to 1000 epochs.”

* **For `solver='lbfgs'` or `'newton-cg'`**
  An **iteration** is a single quasi-Newton step (you’ll never do a full data sweep each time). These solvers build and invert (or approximate) the Hessian, so one iteration is one line-search + Hessian update step — not an epoch.

* **For `solver='liblinear'`**
  Iterations correspond to passes in the **coordinate-descent** inner loop over features, not data-set epochs.

You can always inspect exactly how many “iterations” the solver ran via `model.n_iter_` (or per class for multiclass), but **only for SAG/SAGA will that number directly equal the number of epochs**.
