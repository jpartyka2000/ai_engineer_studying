Yes, you can retrieve the model's type directly from the estimator object using Python’s built-in functions. Here are a couple of ways to do that:

1. **Using the `type()` function:**  
   ```python
   from sklearn.ensemble import ExtraTreesRegressor
   
   model = ExtraTreesRegressor(random_state=42)
   model_type = type(model)
   print(model_type)
   # Output: <class 'sklearn.ensemble._forest.ExtraTreesRegressor'>
   ```

2. **Using the `__class__` attribute:**  
   ```python
   model_type = model.__class__
   print(model_type)
   # Output: <class 'sklearn.ensemble._forest.ExtraTreesRegressor'>
   ```
   
Both methods will give you the model type object. If you need just the name of the class, you can do:
   
```python
print(model.__class__.__name__)
# Output: ExtraTreesRegressor
```

These approaches allow you to introspect the estimator and obtain its type information directly from the object.
