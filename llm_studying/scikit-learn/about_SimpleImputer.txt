**SimpleImputer** is a scikit-learn class for handling missing values in datasets. It's part of the `sklearn.impute` module and replaces missing values with a specified statistic computed from the data.

## Basic Usage

```python
from sklearn.impute import SimpleImputer
import numpy as np

# Create imputer
imputer = SimpleImputer(strategy='mean')

# Fit and transform data
X = [[1, 2], [np.nan, 3], [7, 6]]
X_imputed = imputer.fit_transform(X)
```

## Key Parameters

| Parameter | Description |
|-----------|-------------|
| `strategy` | How to impute: `'mean'`, `'median'`, `'most_frequent'`, or `'constant'` |
| `missing_values` | The placeholder for missing values (default: `np.nan`) |
| `fill_value` | Value to use when `strategy='constant'` |
| `keep_empty_features` | Whether to keep features that are all missing |

## Strategies Explained

- **mean** — replaces with column mean (numeric only)
- **median** — replaces with column median (numeric only)
- **most_frequent** — replaces with the mode (works with strings/categorical)
- **constant** — replaces with a fixed value you specify via `fill_value`

## Example with a Pipeline

```python
from sklearn.pipeline import Pipeline
from sklearn.linear_model import LogisticRegression

pipe = Pipeline([
    ('imputer', SimpleImputer(strategy='median')),
    ('classifier', LogisticRegression())
])

pipe.fit(X_train, y_train)
```

## Useful Attributes

After fitting, you can access:
- `imputer.statistics_` — the computed fill values for each feature
- `imputer.n_features_in_` — number of features seen during fit

SimpleImputer is often the first step in a preprocessing pipeline since most scikit-learn estimators can't handle `NaN` values directly.
