---

What is BayesSearchCV?

BayesSearchCV is a drop-in replacement for GridSearchCV or RandomizedSearchCV, but instead of exhaustively or randomly searching hyperparameters, it uses Bayesian Optimization to intelligently explore the space. It models the performance function and chooses hyperparameters that are more likely to improve it, based on past evaluations.


---

Key Concepts:

1. Surrogate Model (usually a Gaussian Process):

Learns a model of the objective function (e.g., validation score vs. hyperparameters).

It’s cheap to evaluate compared to training a full model.



2. Acquisition Function:

Decides where to sample next (explore vs. exploit trade-off).

Common functions: Expected Improvement (EI), Upper Confidence Bound (UCB), etc.



3. Sequential Search:

Hyperparameters are selected one at a time based on previous results.

Tends to converge faster to good values than grid/random search.





---

How to Use BayesSearchCV

Here’s a basic example:

from skopt import BayesSearchCV
from sklearn.ensemble import RandomForestClassifier
from sklearn.datasets import load_iris
from sklearn.model_selection import train_test_split

# Load data
X, y = load_iris(return_X_y=True)
X_train, X_test, y_train, y_test = train_test_split(X, y, test_size=0.2)

# Define search space
search_space = {
    'n_estimators': (10, 100),
    'max_depth': (1, 20),
    'min_samples_split': (2, 10)
}

# Set up Bayesian Optimization search
opt = BayesSearchCV(
    estimator=RandomForestClassifier(),
    search_spaces=search_space,
    n_iter=32,                   # Number of parameter settings to try
    cv=3,                        # Cross-validation folds
    n_jobs=-1,                   # Parallelism
    random_state=42
)

# Fit the model
opt.fit(X_train, y_train)

# Best results
print("Best score:", opt.best_score_)
print("Best parameters:", opt.best_params_)


---

Features of BayesSearchCV:

Supports categorical, integer, and real-valued parameters.

Can specify different spaces per estimator if needed.

Faster convergence vs. RandomizedSearchCV when evaluating expensive models.



---

Parameter Types:

Real: for continuous values (from skopt.space import Real)

Integer: for discrete values (from skopt.space import Integer)

Categorical: for fixed categories (from skopt.space import Categorical)


You can define the search space more explicitly:

from skopt.space import Real, Integer, Categorical

search_space = {
    'max_depth': Integer(1, 50),
    'max_features': Real(0.1, 1.0),
    'criterion': Categorical(['gini', 'entropy'])
}


---

When to Use BayesSearchCV:

Model training is expensive (deep nets, large datasets).

You have a small number of hyperparameters (1–10).

You want to balance exploration and exploitation intelligently.
