`LabelEncoder` in `scikit-learn` is used to convert categorical labels (strings or other non-numeric labels) into numerical values. It assigns **a unique integer to each category**, making it useful for encoding target variables in classification tasks.

### **How `LabelEncoder` Works**
- It **maps each unique category to an integer** starting from `0`.
- The transformation is **consistent** (i.e., the same category will always get the same number).
- It’s typically used for **encoding target labels** in classification tasks.

---

### **Example Usage**
```python
from sklearn.preprocessing import LabelEncoder

# Sample categorical labels
labels = ["positive", "negative", "positive", "negative", "positive"]

# Initialize and fit LabelEncoder
encoder = LabelEncoder()
encoded_labels = encoder.fit_transform(labels)

print(encoded_labels)  # Output: [1 0 1 0 1]
print(encoder.classes_)  # Output: ['negative' 'positive']
```
- **"negative" → 0**, **"positive" → 1**

---

### **Use Case in ML Models**
- Many ML algorithms (e.g., **Random Forest, Logistic Regression**) require **numerical labels** instead of strings.
- `LabelEncoder` helps when the **target variable** (classification labels) is categorical.

---

### **When NOT to Use `LabelEncoder`**
- **For encoding features**: If categorical **input features** have no ordinal relationship (e.g., "red", "blue", "green"), use **One-Hot Encoding (`OneHotEncoder`)** instead.
- **For large datasets**: If categories have many unique values, `LabelEncoder` may not be ideal.

---

### **Alternative for Multiple Columns**
For encoding multiple categorical columns in a **DataFrame**, use `OrdinalEncoder`:
```python
from sklearn.preprocessing import OrdinalEncoder
import pandas as pd

data = pd.DataFrame({"Color": ["Red", "Blue", "Green", "Blue"]})
encoder = OrdinalEncoder()
encoded_data = encoder.fit_transform(data)

print(encoded_data)  # Output: [[2.] [0.] [1.] [0.]]
```
