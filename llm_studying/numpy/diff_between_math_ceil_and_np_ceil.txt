Here’s the quick rundown:

### `math.ceil`

* **Scope:** Scalar numbers only (one value at a time).
* **Returns:** A Python `int`.
* **Types:** Works with `int`/`float` and any object that implements `__ceil__` (e.g., `fractions.Fraction`).
  `math.ceil(float('nan'))` → **ValueError**; `math.ceil(float('inf'))` → **OverflowError**.
* **Use when:** You need an integer count from a single number.

### `numpy.ceil` (`np.ceil`)

* **Scope:** **Vectorized**—works elementwise on arrays/lists; supports broadcasting.
* **Returns:** **Floating-point** array (e.g., `float64`) or NumPy scalar. Even if input is int, output is float.
* **Types:** Works on NumPy numeric dtypes (`float32/64`, etc.).
  `np.ceil(np.nan)` → `nan`; `np.ceil(np.inf)` → `inf` (no exception).
* **Use when:** You want to ceil many values at once (arrays/Series), then optionally cast:

  ```python
  y = np.ceil(x).astype(int)  # if you need integers and x is finite
  ```

### Behavior examples

```python
import math, numpy as np

math.ceil(1.2)        # 2          (int)
np.ceil(1.2)          # 2.0        (numpy float)

math.ceil(-1.2)       # -1
np.ceil(-1.2)         # -1.0

# Arrays:
np.ceil([1.2, 3.0, -1.2])   # array([2., 3., -1.])

# NaN / Inf:
# math.ceil(float('nan')) -> ValueError
# math.ceil(float('inf')) -> OverflowError
np.ceil(np.nan)       # nan
np.ceil(np.inf)       # inf
```

**Rule of thumb:**

* Use `math.ceil` for single numbers where you want a true `int`.
* Use `np.ceil` for arrays (it’s fast and vectorized); cast to int afterward if needed (and only if values are finite).
