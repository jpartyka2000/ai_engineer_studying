`np.full` is a convenience function in NumPy that creates a new array of a specified shape, where **every** element is initialized to the same value you provide. Its signature is roughly:

```python
np.full(shape, fill_value, dtype=None, order='C')
```

* **`shape`**: An integer or tuple of integers defining the dimensions of the output array.
* **`fill_value`**: The scalar value to place in every position of the array.
* **`dtype`** *(optional)*: The desired data-type for the array. If omitted, NumPy infers it from `fill_value`.
* **`order`** *(optional)*: Memory layout, either C-contiguous (`'C'`) or Fortran-contiguous (`'F'`).

---

### Example

```python
import numpy as np

# Create a 2×3 array where every entry is 7
arr = np.full((2, 3), 7)
# array([[7, 7, 7],
#        [7, 7, 7]])

# Create a length-5 float array of NaNs
nan_vec = np.full(5, np.nan, dtype=float)
# array([nan, nan, nan, nan, nan])
```

Under the hood, `np.full` is more efficient than doing something like `np.array([fill_value] * size)` because it allocates the array at once and writes the fill value directly in optimized C loops.
