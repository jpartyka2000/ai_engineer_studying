You only need to reach for `np.round()` any time you want to round **all** the entries of an array-like in one go—because the built-in `round()` is really just for scalars (or objects that implement `__round__`).

---

### Example 1: a plain Python list

```python
>>> round([1.234, 5.678], 2)
TypeError: type list doesn't define __round__  
>>> np.round([1.234, 5.678], 2)
array([1.23, 5.68])
```

### Example 2: a NumPy array

Although NumPy arrays do implement `__round__`, it’s clearer and more idiomatic to call

```python
arr = np.array([1.234, 5.678])
rounded = np.round(arr, 2)    # array([1.23, 5.68])
```

rather than

```python
rounded = round(arr, 2)       # works, but hides the fact you’re doing vectorized ops
```

### Example 3: a pandas Series

```python
>>> s = pd.Series([1.234, 5.678])
>>> round(s, 2)                # calls s.__round__(2) under the hood
0    1.23
1    5.68
dtype: float64

>>> np.round(s, 2)             # also works, returns an ndarray
array([1.23, 5.68])
```

But if you want to get back a **Series** rather than a raw `ndarray`, you’d prefer

```python
s.round(2)
```

---

**Bottom line:**

* Use **`round()`** for one-off scalar rounding.
* Use **`np.round()`** (or `.round()`) whenever you need a **vectorized** elementwise operation on lists, arrays, or entire DataFrame/Series columns.
