Yes. Typically, the duration for each customer in a survival analysis context is measured from a common starting point—usually the customer's first transaction (or subscription start date)—to an endpoint that depends on whether the customer is active or inactive:

- **For Inactive (Churned) Customers:**  
  The duration is calculated as the time from the first transaction until the date when the customer churned (often the date of the last transaction if that's used as a proxy for churn).

- **For Active Customers:**  
  Since these customers haven’t churned yet, their duration is calculated from the first transaction until the end of the observation period (or the date when the data was collected). This duration is “censored” because we only know they’ve survived until that date, but not how much longer they will remain active.

In both cases, the **oldest date** typically is the customer’s first transaction or subscription start date, which serves as the baseline for calculating duration. 

For example, if a user’s first transaction was on January 1st, and:
- They churned on June 1st, their duration would be 151 days.
- They’re still active as of today (say, July 1st), their duration would be 181 days (and they’d be considered right-censored in the survival analysis).

This framework allows you to incorporate both active and inactive customers into the Kaplan-Meier survival analysis.
