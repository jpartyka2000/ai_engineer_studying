It's not inherently undesirable from a modeling standpoint, but it can make individual-level explanations less straightforward. When you have multiple boolean features that are mutually exclusive (since a user can only subscribe to one plan at a time), an explainability tool like SHAP will show the contributions of each boolean. However, for any individual user, only one of these features will be "active" (True) while the others are False, which can make the explanation less intuitive if you try to describe it as a set of independent features.

If interpretability and the ability to easily communicate individual predictions are important to your stakeholders, you might consider the following options:

- **Combine the Booleans into a Single Categorical Feature:**  
  Instead of having five separate boolean columns, create one categorical feature that represents the current plan (e.g., "Plan A", "Plan B", etc.). This way, the model still benefits from one-hot encoding if needed, but the explanation can simply state the active plan type.

- **Post-Processing Explanations:**  
  If you keep the boolean features, you can always post-process the SHAP outputs for an individual prediction to combine the information into a single, human-friendly message. For example, if the SHAP analysis shows that the "Plan A" feature is the only one with a significant contribution, you can interpret that as "the user currently subscribes to Plan A."

- **Focus on Overall Model Behavior:**  
  For many applications, explaining the overall influence of each feature on the model's behavior (via aggregate plots) is more important than explaining each individual instance. But if individual explanations are needed, a single categorical feature can be more immediately interpretable.

In summary, while having multiple booleans isn't problematic for model performance, it may complicate individual-level explanations. If clear, straightforward explanations are critical, consider feature engineering that consolidates mutually exclusive attributes into a single, more interpretable variable.
