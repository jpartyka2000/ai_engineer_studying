`n_init` = how many **independent restarts** of k-means to run with different random centroid seeds.
K-means is non-convex, so different initial centroids can land in different local minima. With:

```python
km = KMeans(n_clusters=k, n_init=20, random_state=RANDOM_SEED)
```

scikit-learn will:

1. pick 20 different initializations (deterministically if you set `random_state`),
2. run k-means 20 times,
3. keep the **best** solution (lowest inertia / within-cluster sum of squares).

### Practical tips

* Larger `n_init` ⇒ more compute, **better chance of a good clustering**.
* With good initializers like **k-means++** (the default), you may not need a huge value, but 10–20 is common when you want stability.
* If you pass an explicit `init` array of centroids, `n_init` is effectively ignored (only that init is used).
* Defaults vary across scikit-learn versions, so it’s good practice to **set it explicitly** as you did.
