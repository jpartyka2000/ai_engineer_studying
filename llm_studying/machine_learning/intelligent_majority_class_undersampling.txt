Yes, when undersampling the majority class, you can employ techniques that ensure you select a diverse and representative subset of instances, helping to maintain model performance. This way, the selected instances provide the most informative and varied examples, reducing the risk of losing valuable information. Here are some effective methods to achieve this:

### 1. **Cluster-Based Undersampling**
- **How it Works:** Apply clustering algorithms (e.g., KMeans) to group the majority class into different clusters and then randomly sample from each cluster.
- **Why It's Effective:** This ensures that your sample contains diverse instances that represent the overall structure of the majority class.

**Example Code Using KMeans:**
```python
import pandas as pd
from sklearn.cluster import KMeans

# Assuming df is your DataFrame with features and 'label' column
majority_class = df[df['label'] == 0]
minority_class = df[df['label'] == 1]

# Apply KMeans clustering to the majority class
n_clusters = len(minority_class)  # Set the number of clusters to the size of the minority class
kmeans = KMeans(n_clusters=n_clusters, random_state=42)
majority_class['cluster'] = kmeans.fit_predict(majority_class.drop(columns=['label']))

# Sample one instance from each cluster to form the undersampled majority class
undersampled_majority = majority_class.groupby('cluster').apply(lambda x: x.sample(1, random_state=42))

# Combine with the minority class
undersampled_df = pd.concat([undersampled_majority, minority_class])

print(undersampled_df)
```

### 2. **NearMiss Undersampling**
- **How it Works:** NearMiss is a technique that selects majority class samples based on their proximity to minority class samples. It chooses majority samples that are closest to the minority samples, ensuring that these points are relevant to the classification boundary.
- **Why It's Effective:** It retains instances that are likely to be more informative for distinguishing between classes.

**Example Using imbalanced-learn:**
```python
from imblearn.under_sampling import NearMiss

X = df.drop(columns=['label'])
y = df['label']

# Apply NearMiss
nearmiss = NearMiss(version=1)
X_res, y_res = nearmiss.fit_resample(X, y)

# Create a DataFrame from the resampled data
undersampled_df = pd.concat([pd.DataFrame(X_res), pd.DataFrame(y_res, columns=['label'])], axis=1)
```

### 3. **Tomek Links**
- **How it Works:** Tomek Links are pairs of instances (one from each class) that are nearest neighbors but belong to different classes. By removing the majority class samples from these pairs, you eliminate overlapping and borderline examples.
- **Why It's Effective:** This method preserves diverse instances while removing those that don’t contribute to clear class separation.

**Example Using imbalanced-learn:**
```python
from imblearn.under_sampling import TomekLinks

tomek = TomekLinks()
X_res, y_res = tomek.fit_resample(X, y)

undersampled_df = pd.concat([pd.DataFrame(X_res), pd.DataFrame(y_res, columns=['label'])], axis=1)
```

### 4. **Undersampling with Synthetic Minority Oversampling (Hybrid Approach)**
- Combine undersampling with techniques like SMOTE (Synthetic Minority Over-sampling Technique). First, create synthetic samples for the minority class, then use one of the above methods to undersample the majority class. This helps in achieving a balanced and diverse dataset.

### 5. **Diversity Sampling Based on Feature Importance**
- Train a model on the full dataset first, then use feature importance to identify the most critical features. Select majority class samples that have the most variation in these important features.
  
### Summary
- **Cluster-Based** and **NearMiss** methods are generally the most effective at ensuring diversity while undersampling.
- These methods ensure the retained majority class instances are representative and provide good coverage of the feature space, reducing the risk of losing critical information for training the model.

By using these techniques, you can maintain a wide diversity of instances from the majority class, thereby minimizing the negative impact on model performance when applying undersampling.
