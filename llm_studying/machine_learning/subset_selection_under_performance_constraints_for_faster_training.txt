Yes, what you're describing is subset selection under performance constraints — and yes, it's possible, though not trivial. You want to:

> Find the smallest training subset such that the model trained on it has evaluation metrics that are within a defined distance (or tolerance) from the model trained on the full dataset.



This is related to the broader field of data summarization for machine learning, including topics like:

Core-set selection

Active learning (in reverse)

Model distillation via data pruning

Submodular optimization for data selection



---

✅ Feasibility

Yes, it's feasible, but it depends on:

The model you're using (linear vs deep)

The data complexity

The metric you're preserving (e.g., accuracy, AUC, F1)


For many real-world scenarios, you can retain 90–99% of the model’s performance with much less than 100% of the data, especially if there's redundancy or imbalance.


---

🔧 Practical Techniques

Here are some practical methods to solve this:

A. Greedy Subset Search

1. Start with a small sample (e.g. stratified 5%)


2. Train model and evaluate


3. Incrementally add most informative samples (e.g. using uncertainty or margin scores)


4. Stop when performance is within the threshold



This is similar to backward active learning.

B. Core-Set Selection

Use techniques like:

k-Center Greedy

Facility Location (submodular optimization)

Gradient Matching: Select subset such that its gradients approximate those of the full dataset


Libraries:

cords: Core-set based data subset selection

subset selection in skorch (if using PyTorch)


C. Model-aware Subset Pruning

Use influence functions or SHAP to determine which examples contribute least to model performance and prune them.

D. Optimization-Based Methods

You could even use Bayesian Optimization or Genetic Algorithms to search for the smallest subset under a constraint.

📦 Example: Using cords for subset selection

from cords.selectionstrategies import CRAIGStrategy
from cords.utils.data.data_utils import get_dataloader

# Suppose `train_dataset` and `val_dataset` are your torch datasets

strategy = CRAIGStrategy(train_dataset, val_dataset, model, loss_fn, device='cuda')
subset_indices = strategy.select(budget=0.1)  # select 10% most informative samples
subset_loader = DataLoader(Subset(train_dataset, subset_indices), batch_size=32)

# Train on the selected subset and evaluate


---

🧠 Key Considerations

Make sure your subset preserves class balance and feature diversity

Use stratified sampling as a baseline

This technique is model-dependent — a subset good for one model may not work for another
