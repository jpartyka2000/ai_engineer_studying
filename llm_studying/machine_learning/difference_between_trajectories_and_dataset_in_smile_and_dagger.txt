## The Process

### At Each Iteration:

**1. Execute N trajectories** (using current policy or mixture)
   - These trajectories are **fixed** once collected
   - They record which states were visited

**2. Create dataset from expert labels**
   - For each state visited in those N trajectories: ask expert "What action would you take?"
   - Build dataset: D(i) = {(state, expert_action) pairs}
   - This dataset is completely separate from the trajectories themselves

**3. Train new policy**
   - Use supervised learning on the aggregated dataset
   - D_total = D(1) ∪ D(2) ∪ ... ∪ D(i)
   - New policy π(i+1) learns to map states → expert actions

## Key Points

✓ The N trajectories are **never modified** - they're just used to determine which states to collect

✓ The expert labels create a **separate dataset** for training

✓ The dataset is a mapping: state → expert's recommended action

✓ Training is standard supervised learning on this dataset

You've understood it perfectly! The trajectories are like "waypoints" that tell us which states to get expert labels for, and those labels become our training data.
