The classic BG/NBD model is built specifically to work with the three summary statistics—frequency, recency, and the customer's age (T). In its standard formulation, it doesn't take in extra predictors directly. 

However, there are a few points to consider:

- **Extended Models:**  
  Researchers have explored extending the BG/NBD framework by incorporating covariates. These extensions typically involve modeling the underlying latent parameters (such as the transaction rate or dropout probability) as functions of additional predictors. However, these models are more complex and aren't as widely implemented or available in standard libraries like [lifetimes](https://lifetimes.readthedocs.io/).

- **Alternative Approaches:**  
  If you have rich customer data with many predictors, you might consider a two-stage approach:
  1. Use the BG/NBD model to capture the core transaction process.
  2. Use an additional regression or machine learning model to adjust the predictions based on the extra predictors.
  
- **Model Choice Considerations:**  
  While incorporating extra predictors can potentially improve performance, doing so within the BG/NBD framework requires significant modifications to its likelihood function and estimation process. This is why many practitioners opt for models specifically designed for high-dimensional predictor spaces (like ML regression methods) when additional covariates are critical.

**Overall:**  
The standard BG/NBD model only needs the three inputs. If you need to incorporate additional predictors, you would typically have to explore extended versions of the model or combine it with other modeling techniques.
