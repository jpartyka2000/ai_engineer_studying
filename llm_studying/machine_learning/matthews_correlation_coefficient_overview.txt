The Matthews Correlation Coefficient (MCC) is a performance metric used to evaluate how well a binary classifier is performing.
It’s particularly useful when your classes are imbalanced (for example, many more 0s than 1s).
It takes into account true positives (TP), true negatives (TN), false positives (FP), and false negatives (FN) — all four parts of a confusion matrix.


---

Here’s the basic formula:

\text{MCC} = \frac{ (TP \times TN) - (FP \times FN) }{ \sqrt{(TP+FP)(TP+FN)(TN+FP)(TN+FN)} }


---

Interpretation:

MCC = +1 → Perfect prediction

MCC = 0 → No better than random guessing

MCC = –1 → Total disagreement between prediction and true labels (worst possible)



---

Why is MCC useful?

Unlike accuracy, which can be misleading on imbalanced data (e.g., 95% accuracy could mean your model just always predicts the majority class),

MCC gives you a balanced view: it’s high only if the model is doing well on both classes.


In short:

> MCC is basically like a "correlation" between the predicted and actual labels — and correlation is always better than just percentages when the classes are lopsided.




---

Quick example:

Suppose:

TP = 70

TN = 900

FP = 20

FN = 10


Plugging into the formula:

\text{MCC} = \frac{(70 \times 900) - (20 \times 10)}{ \sqrt{(70+20)(70+10)(900+20)(900+10)} }

= \frac{63000 - 200}{ \sqrt{90 \times 80 \times 920 \times 910} }

= \frac{62800}{ \sqrt{6021120000} }

The denominator is huge, but the MCC would turn out close to ~0.81, showing strong performance.
