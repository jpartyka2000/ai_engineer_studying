Permutation feature importance is a model-agnostic technique that evaluates the importance of each feature by measuring the decrease in model performance when that feature's values are randomly shuffled. This approach allows you to understand the impact of each feature on the model's predictions, independent of any potential correlation.

Here's how you can implement permutation feature importance using Python:

Step 1: Train Your Model

Ensure that you have a trained machine learning model. For this example, let's assume you're using a scikit-learn-compatible model.

from sklearn.ensemble import RandomForestClassifier
from sklearn.model_selection import train_test_split
from sklearn.metrics import accuracy_score

# Assuming X and y are your features and target variables
X_train, X_test, y_train, y_test = train_test_split(X, y, test_size=0.3, random_state=42)

# Train a Random Forest model
model = RandomForestClassifier(random_state=42)
model.fit(X_train, y_train)

# Evaluate model performance on the test set
baseline_accuracy = accuracy_score(y_test, model.predict(X_test))
print(f"Baseline Accuracy: {baseline_accuracy}")

Step 2: Implement Permutation Feature Importance

We will shuffle each feature one by one and observe the impact on model performance. The more the accuracy drops, the more important that feature is.

Option 1: Implementing Manually

import numpy as np

def permutation_importance(model, X_test, y_test, metric_function=accuracy_score):
    baseline_score = metric_function(y_test, model.predict(X_test))
    feature_importance = {}

    for col in X_test.columns:
        # Shuffle the column values
        X_test_permuted = X_test.copy()
        X_test_permuted[col] = np.random.permutation(X_test_permuted[col])

        # Calculate the accuracy with the shuffled feature
        permuted_score = metric_function(y_test, model.predict(X_test_permuted))

        # The importance is the drop in accuracy caused by shuffling
        importance_score = baseline_score - permuted_score
        feature_importance[col] = importance_score

    return feature_importance

# Get permutation feature importance scores
importance_scores = permutation_importance(model, X_test, y_test)
sorted_importance = sorted(importance_scores.items(), key=lambda x: x[1], reverse=True)

print("Feature Importance Scores (Permutation Method):")
for feature, importance in sorted_importance:
    print(f"{feature}: {importance}")

Option 2: Using Scikit-Learn’s Built-in Function

Scikit-learn provides a built-in function for permutation feature importance, which can save you some coding effort:

from sklearn.inspection import permutation_importance

# Calculate permutation feature importance
results = permutation_importance(model, X_test, y_test, n_repeats=10, random_state=42, scoring='accuracy')

# Display the results
importance_means = results.importances_mean
importance_std = results.importances_std

# Create a sorted list of feature importances
sorted_idx = np.argsort(importance_means)[::-1]
print("Feature Importance Scores (Permutation Method):")
for idx in sorted_idx:
    print(f"{X_test.columns[idx]}: Mean Importance = {importance_means[idx]}, Std Dev = {importance_std[idx]}")

Step 3: Interpret the Results

A higher importance score indicates that the feature is more critical for the model's predictions.

If the model's accuracy drops significantly when a feature is shuffled, it suggests that the feature is highly important.

Conversely, if the accuracy does not change much, the feature might be less relevant.


Tips and Considerations

Permutation feature importance works best on independent test data, not training data, to avoid overfitting biases.

This method can be computationally intensive, especially with large datasets or complex models. Consider using fewer repetitions (n_repeats) for quicker but less precise estimates.

Permutation feature importance captures the relationship between each feature and the model, not necessarily the target variable, and is robust to multicollinearity compared to other importance methods.


This process will provide you with a clear understanding of how much each feature contributes to your model’s performance.

By "shuffling" a feature, we mean randomly rearranging the values of that feature (column) in the test dataset. This process disrupts the relationship between that specific feature and the target variable, allowing us to assess how much the model's performance is affected when that feature's information is essentially removed.

Detailed Explanation of Shuffling:

Original Feature Values: Suppose you have a feature "Age" with values [25, 35, 45, 30, 50] in your test dataset.

Shuffled Feature Values: After shuffling, this feature might look like [35, 50, 30, 45, 25].


This randomized version of the "Age" feature now has no meaningful connection to the original target variable. By evaluating the model's performance with this shuffled feature, you can determine how important "Age" was to the model's accuracy. If the performance drops significantly, "Age" is an important predictor; if not, it might be less relevant.

Why Shuffling Helps Assess Importance:

When you shuffle a feature, you maintain the distribution of its values but break its correlation with the target variable.

This disruption helps isolate the feature's contribution, as any drop in model performance can be attributed to the loss of information from that specific feature.


The Process in Practice:

For each feature:

1. Shuffle its values while keeping all other features unchanged.


2. Measure the model’s performance with this shuffled feature.


3. Compare the performance to the baseline (unshuffled) model performance.



The greater the decrease in performance, the more critical that feature is for your model's predictions.
