### **2. Performance Plateau (Validation-Based)**

Stop when performance stops improving:

```python
best_performance = -infinity
patience = 0
max_patience = 5

for i in range(max_iterations):
    # Train π(i+1), add to mixture
    current_performance = evaluate_on_validation_set()
    
    if current_performance > best_performance:
        best_performance = current_performance
        patience = 0
    else:
        patience += 1
    
    if patience >= max_patience:
        break  # Stop! No improvement for 5 iterations
```

### **3. Loss Threshold**

Stop when you match the expert well enough:

```python

    # Train π(i+1), add to mixture
    loss = measure_loss_vs_expert()
    
    if loss < threshold:
        break  # Good enough!
```

## **How to Decide Stopping Criterion**

**Choose based on your scenario:**

**Fixed iterations if:**
- You have computational budget constraints
- You know roughly how many iterations work from experience
- You want predictable runtime

**Validation-based if:**
- Performance is critical
- You can afford extra computation
- You have a good validation set

**Loss threshold if:**
- You have a specific accuracy target
- You know what "good enough" means
- You're deploying in safety-critical applications

**Computational budget if:**
- You're limited by time/money
- You're doing hyperparameter search over many runs
- You need to train many agents

## **Typical Values (From Literature)**

**Based on the experiments I found:**
- **Simple tasks:** 5-15 iterations
- **Medium tasks:** 15-30 iterations  
- **Complex tasks:** 20-50 iterations

**Example from search results:**
- Mario Bros: ~20 iterations
- Driving tasks: 15+ iterations

## **What Happens If You Don't Stop?**

**Keep adding policies forever?**

**Potential issues:**
1. **Overfitting to training distribution**
2. **Diminishing returns** - later policies add little
3. **Memory/computation costs** - storing many policies
4. **Diluted weights** - each policy gets tiny weight

**In practice:** Performance usually plateaus after some iterations

## **Rule of Thumb**

**Start with:**
- Max iterations: 20-30
- Patience: 3-5 iterations
- Min improvement: 1% of metric range

**Then tune based on:**
- Validation performance
- Computational budget  
- Task complexity

## **Summary**

**You stop when:**
1. **Fixed iterations reached** (most common for simplicity)
2. **Performance plateaus** (most common for best results)
3. **Loss threshold met** (if you have target accuracy)
4. **Budget exhausted** (practical constraint)

**The paper suggests validation-based stopping** - train multiple iterations and return the mixture that performs best on a held-out validation set.
