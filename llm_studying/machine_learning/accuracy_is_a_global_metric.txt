Accuracy is always computed over the **entire** test set:

\[
\text{Accuracy}
= \frac{\text{# correct predictions}}{\text{total # samples}}
= \frac{TP + TN}{TP + TN + FP + FN},
\]

so you get exactly **one** accuracy number—there is no “positive‐accuracy” vs. “negative‐accuracy” in the usual definition.

If you **did** try to compute accuracy on each class separately by restricting to only that class’s examples, you would in effect be computing the **recall** (sensitivity/specificity) for that class:

- On the subset where \(y=1\) (all the positives),  
  \[
    \text{accuracy}_\text{pos}
    = \frac{\text{# correct on positives}}{\text{# positives}}
    = \frac{TP}{TP + FN}
    = \text{Recall}_{+}.
  \]

- On the subset where \(y=0\) (all the negatives),  
  \[
    \text{accuracy}_\text{neg}
    = \frac{\text{# correct on negatives}}{\text{# negatives}}
    = \frac{TN}{TN + FP}
    = \text{Recall}_{-}\;(\text{specificity}).
  \]

Since positive recall and negative recall generally differ, you’ll get two different values if you do this “per‐class accuracy” trick—but understand that you’ve just re‐derived recall (for positives) and specificity (for negatives).

---

### Key takeaway

- **Accuracy** itself is a single global metric:  
  ```python
  from sklearn.metrics import accuracy_score
  acc = accuracy_score(y_true, y_pred)  # one number
  ```
- If you want class‐specific performance, use **recall** (sensitivity) for the positive class and **specificity** for the negative class (or precision, F1, etc.), rather than trying to split accuracy.
