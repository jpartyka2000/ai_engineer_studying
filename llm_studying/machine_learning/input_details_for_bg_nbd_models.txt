Yes, that's correct. The BG/NBD model requires calculating frequency, recency, and T for each customer. These values are aggregated from each customer's historical transaction data:

- **Frequency:** The number of repeat transactions (purchases after the first one).
- **Recency:** The time span between the customer's first and most recent transaction.
- **T (Age):** The total time period from the first purchase until the end of the observation period.

Each customer's behavior is summarized by these three metrics, which then serve as the inputs to the BG/NBD model for predicting future transactions.

-------------------------

Yes. Once you calculate the three key metrics—frequency, recency, and T (customer age)—for each customer, you can use a fitted BG/NBD model to generate a prediction of the expected number of future transactions for each customer (i.e., each row in your dataset). Typically, you'll:

1. **Fit the Model:**  
   Use the aggregated data (all customers' frequency, recency, and T) to estimate the model parameters.

2. **Generate Predictions:**  
   For each customer, use a function (like `conditional_expected_number_of_purchases_up_to_time` in the lifetimes library) that takes in the customer's frequency, recency, and T along with a specified future time horizon (e.g., 6 months, 12 months) to predict the expected future transactions.

This per-customer prediction effectively gives you the future transaction likelihood for each row in your dataset.
