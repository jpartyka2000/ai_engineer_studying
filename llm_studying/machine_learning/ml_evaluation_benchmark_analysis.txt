Analysis:

1. Is it too repetitive?

No, not really.
Each metric measures different things:

Accuracy: Overall proportion correct, but can be misleading in imbalanced datasets.

ROC AUC: How well you separate the classes (threshold-independent).

Log Loss: How confident your probabilistic predictions are — penalizes overconfident wrong guesses.

Matthews Correlation Coefficient (MCC): An excellent single summary that balances all 4 parts of the confusion matrix (TP, FP, TN, FN).


F1, Precision, Recall per class gives you insights that ROC AUC or MCC can't:

Recall measures how much you find positives (important for medical diagnoses, fraud detection, etc.)

Precision measures how much you trust your positives.

F1 balances them if you care about both.


Log loss per class is a little unusual (since log loss is usually computed globally), but it can give insight into whether your model is more confident in one class than the other.
(However, this might be harder to interpret unless you carefully explain it.)


---

2. Pros:

Very complete view: You cover raw classification correctness (Accuracy), ranking ability (ROC AUC), probability calibration (Log Loss), and balanced confusion matrix effects (MCC).

Class-specific fairness: Precision, Recall, and F1 per class detect class-specific problems that global metrics can miss.

Robust against imbalance: MCC and per-class Recall are very useful if you have class imbalance.

Practical insights: Different stakeholders (e.g., business vs. engineering) might prefer different metrics — you can give both kinds.



---

3. Cons:

Slight complexity: Might overwhelm a non-technical audience if not summarized properly ("too many metrics").

Small overlaps:

F1 is just the harmonic mean of Precision and Recall — so it is derived from the two. Some could argue it’s a bit duplicative.

Accuracy and MCC are both "global" metrics but MCC is much better when imbalance is present; Accuracy might be redundant if imbalance is large.


Per-class log loss could be overkill unless you really care about probability quality class-by-class (e.g., in high-risk domains like medicine or credit scoring).



---

My final objective opinion:

This is a very strong and almost ideal set of metrics — slightly heavy, but justifiable if you’re doing a rigorous evaluation, or reporting to a technical team or for publication.

If you want a slightly cleaner version, you could consider:

Drop either F1 or keep Precision/Recall (since F1 is derived from them).

Maybe omit Accuracy if your dataset is highly imbalanced (MCC + ROC AUC + Log Loss already cover the important aspects).

Be cautious about per-class log loss unless you can present or explain it clearly.
