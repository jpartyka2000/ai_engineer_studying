Yes‚Äîevery `Subgroup` result carries a `.subgroup_size` attribute (the total number of rows matching that rule), plus a `.cover` attribute (the number of those rows where the target is positive). Here‚Äôs a quick example of how you might print description, size, and positive‚Äìrate all in one go:

```python
import pysubgroup as ps

# ‚Ä¶ after you‚Äôve run your BeamSearch or DFS ‚Ä¶
results = ps.BeamSearch(
    task=ps.SubgroupDiscoveryTask(df, ps.BinaryTarget('prediction', value=1), ps.WRAccQF(), depth=3),
    beam_width=5,
    result_set_size=10
).run()

for sg in results:
    desc = sg.subgroup_description
    size = sg.subgroup_size         # total rows matching the rule
    positives = sg.cover            # number of those rows with prediction==1
    rate = positives / size if size else 0
    print(f"{desc!s}\n  ‚Ü≥ size = {size:,} rows, positives = {positives:,} ({rate:.1%} positive)\n")
```

---

### Alternative: deriving the count from the DataFrame

If you ever want to double‚Äëcheck against your original `df`, you can also do:

```python
for sg in results:
    mask = sg.subgroup.covers(df)      # boolean mask of rows matching the rule
    size = mask.sum()
    print(f"{sg.subgroup_description}: {size} rows")
```

The `.covers(df)` method returns a boolean array of the same length as `df`, so `mask.sum()` is the row count. Either approach will give you exactly how many rows fall under each discovered rule.
