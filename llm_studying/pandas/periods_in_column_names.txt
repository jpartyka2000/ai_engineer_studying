Yes. In Pandas, column names are just labels (strings), so you can include periods (`.`) anywhere. For example:

```python
import pandas as pd

df = pd.DataFrame({
    'first.name': [1, 2, 3],
    'last.name': [4, 5, 6]
})
```

Pandas will store those columns without complaint. The only caveat is how you access them:

* **Bracket notation** always works:

  ```python
  df['first.name']   # Series [1, 2, 3]
  ```
* **Attribute (dot) access** does *not* work when there’s a period in the name:

  ```python
  df.first.name   # tries to look for column "first" then attribute "name" → KeyError/AttributeError
  ```

If you need to use methods like `df.query()` or `df.eval()`, you can wrap the column name in backticks:

```python
df.query("`first.name` > 1")
```

In short:

* **Yes**, you can have `.` in column names.
* Just be sure to use `df['col.with.dot']` (or backticks in any SQL‐style string expressions).
