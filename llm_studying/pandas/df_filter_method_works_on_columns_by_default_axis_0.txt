Yes, by default `filter()` operates on **column names** (along `axis=0` which refers to the columns axis).

**Default behavior:**
```python
import pandas as pd

df = pd.DataFrame({'name': [1, 2], 'age': [3, 4], 'name_suffix': [5, 6]})

# Filters columns by default
df.filter(like='name')  # Returns columns 'name' and 'name_suffix'
df.filter(regex='^age')  # Returns column 'age'
df.filter(items=['name', 'age'])  # Returns columns 'name' and 'age'
```

**To filter on row index instead:**
```python
# Need to explicitly set axis=0 or axis='index'
df.filter(like='something', axis=0)  # Filters on row index
df.filter(like='something', axis='index')  # Same thing
```

**The axis parameter can be confusing:**
- `axis=1` or `axis='columns'` = filter on columns (DEFAULT)
- `axis=0` or `axis='index'` = filter on rows

This is somewhat counterintuitive since in many other pandas operations, `axis=0` refers to rows and `axis=1` refers to columns in terms of the direction of operation. But for `filter()`, think of it as "which axis's labels are you filtering on."

So yes, without specifying an axis parameter, `filter()` will filter based on column names, not row indices or data values.
