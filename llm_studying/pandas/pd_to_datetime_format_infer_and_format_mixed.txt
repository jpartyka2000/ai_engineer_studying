You're absolutely right—I misstated that.

Specifying `format="%Y-%m-%d"` is **not** a replacement for `infer_datetime_format`. It's the opposite: you're explicitly telling pandas the format rather than letting it figure it out.

The actual replacement for `infer_datetime_format=True` is:

```python
parsed = pd.to_datetime(dates, format='mixed')
```

Or, if you know all values share the same format but don't want to specify it:

```python
parsed = pd.to_datetime(dates, format='infer')
```

**Clarification of the two options:**

- **`format='infer'`**: This is the true successor to `infer_datetime_format=True`. Pandas examines the data and attempts to determine a single consistent format across all values.

- **`format='mixed'`**: This is more flexible—it allows different rows to have different formats. Pandas will try to parse each value individually, accommodating variation within the series.

So for the original code, the appropriate modern replacement would be:

```python
# Old (deprecated)
parsed = pd.to_datetime(sample, errors='coerce', infer_datetime_format=True)

# New equivalent
parsed = pd.to_datetime(sample, errors='coerce', format='infer')
```
