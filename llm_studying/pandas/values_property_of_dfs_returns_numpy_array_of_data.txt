The `.values` attribute on a `pandas.DataFrame` gives you the raw array of the data (i.e. all of the cell values), without any of the row or column labels.  Concretely:

* **Type**: In most cases it returns a two-dimensional `numpy.ndarray` of shape `(n_rows, n_columns)`.
* **Homogeneous dtypes**: If every column shares a numeric dtype (e.g. all `float64`), the array will have that dtype.
* **Mixed dtypes**: If your columns have different types (e.g. some strings, some ints, some floats), you’ll get back a single NumPy array of `dtype=object`, so that every element can hold any Python object.
* **Extension arrays**: Under the hood Pandas sometimes stores data in its own “ExtensionArray” types (e.g. for nullable integers or categoricals).  In those cases `.values` may return that ExtensionArray rather than a plain `ndarray`.  (If you want a guaranteed `ndarray`, use `.to_numpy()` instead.)

**Example**

```python
>>> df = pd.DataFrame({
...     "A": [1, 2, 3],
...     "B": [0.1, 0.2, 0.3],
... })
>>> df.values
array([[1. , 0.1],
       [2. , 0.2],
       [3. , 0.3]])
```

In short, use `.values` (or the more‐modern `.to_numpy()`) when you just want the bare 2D array of your DataFrame’s contents.
