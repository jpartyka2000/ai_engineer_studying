A **full outer join** in pandas is the operation that combines two DataFrames so that **all** rows from both are retained. Rows with matching key values are merged side-by-side, while non-matching rows show up with `NaN` for the other side’s columns.

```python
# basic syntax
df_full = dfA.merge(
    dfB,
    on=['key1', 'key2'],   # list of join columns
    how='outer'            # full outer join
)
```

**Key points:**

* **Keeps everything:** unlike “inner” (only matches), “left” (all A), or “right” (all B), the “outer” join preserves rows unique to either DataFrame.
* **Missing data:** wherever there’s no counterpart, pandas fills in `NaN` for the missing columns.
* **Column overlaps:** if both DataFrames have non-join columns with the same name, pandas appends suffixes (`_x`, `_y` by default), which you can customize via `suffixes=(…)`.
* **Diagnostics:** add `indicator=True` to produce a `_merge` column showing whether each row came from the left only, right only, or both—handy for sanity checks.

Full outer joins are especially useful when you need a **master list** that reflects every entity or event seen in either source, then reconcile or fill in gaps in downstream processing.
