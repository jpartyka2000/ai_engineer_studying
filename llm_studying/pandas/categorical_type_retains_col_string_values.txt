Yes—when you do:

```python
df['my_col'] = df['my_col'].astype('category')
```

Pandas will:

1. **Detect all the unique string values** in that column and store them as the “categories” list.
2. **Internally replace** each value in the column with a small integer code pointing to its category.
3. **Still present** the original strings when you look at `df['my_col']`; pandas “re-maps” the codes back to the strings for display.

---

### Example

```python
import pandas as pd

# a simple string column
df = pd.DataFrame({
    'color': ['red', 'blue', 'green', 'blue', 'red']
})
print(df['color'].dtype)
# → object

# convert to categorical
df['color'] = df['color'].astype('category')
print(df['color'].dtype)
# → category

print(df)
#     color
# 0     red
# 1    blue
# 2   green
# 3    blue
# 4     red

# inspect the underlying categories
print(df['color'].cat.categories)
# Index(['blue', 'green', 'red'], dtype='object')

# see their integer codes
print(df['color'].cat.codes)
# 0    2
# 1    0
# 2    1
# 3    0
# 4    2
# dtype: int8
```

* **What you see** (`df['color']`) still shows “red”, “blue”, etc.
* **Under the hood**, pandas just stores small integer codes and a categories list for memory and speed efficiency.

So converting to `category` retains all your original strings exactly as values.
