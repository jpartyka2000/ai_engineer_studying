You only need to pull out the integer‐column position when you’re using **`.iloc`**, because it strictly takes integer row and column positions. If you switch to **`.loc`** (label‐based indexing), you can refer to the column by name directly:

```python
# Make sure the column exists
X_probs['new_probability'] = np.nan

# If val_idx is an array/list of index LABELS (not positions):
X_probs.loc[val_idx, 'new_probability'] = proba_val
```

Or, if you really want to stay with `.iloc` but make it a bit cleaner, you can still use:

```python
col_pos = X_probs.columns.get_loc('new_probability')
X_probs.iloc[val_idx, col_pos] = proba_val
```

—but with `.loc` you don’t need that extra step. Just be sure that **`val_idx`** matches your DataFrame’s **index labels**, not its positional row numbers.
