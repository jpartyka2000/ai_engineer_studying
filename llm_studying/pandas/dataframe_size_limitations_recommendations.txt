The usability of a **Pandas DataFrame** depends on several factors, but **memory limits and processing speed** are the main constraints. Here’s a rough guideline:  

### **1. Memory Limitations (RAM)**
- **Small (≤ 100 MB, <1 million rows):** Runs smoothly on most machines.  
- **Medium (100 MB - 1 GB, ~10 million rows):** May start slowing down, especially if performing frequent operations like joins or sorting.  
- **Large (1 GB - 5 GB, ~50 million rows):** Requires optimization (e.g., using `dtype` reduction, indexing, or chunking).  
- **Very Large (>5 GB, 100+ million rows):** Becomes **unusable in-memory** for most users. Consider using **Dask, Polars, or a database**.  

### **2. Usability Factors**
- **Processing Speed:** Operations like `groupby()`, `merge()`, or `apply()` become slow as data grows.  
- **Memory Overhead:** Pandas loads data **fully into RAM**, unlike databases that fetch on demand.  
- **Garbage Collection:** Frequent object allocation (e.g., many string operations) can increase latency.  

### **3. How to Handle Large DataFrames?**
- **Downcast numeric types** (`float64` → `float32`, `int64` → `int32` where possible).  
- **Use categoricals** for low-cardinality string columns (`astype('category')`).  
- **Chunking** (`pd.read_csv(..., chunksize=500000)`) for large CSVs.  
- **Parallel processing** (`Dask`, `Modin`, `Polars`).  
- **Move to a database** (`SQL`, `DuckDB`, `SQLite`).  

If you're working with **~10 million+ rows**, you’ll likely need **optimizations** or an **alternative tool** like Dask.
