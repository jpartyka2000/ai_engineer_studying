Pandas respects several **environment variables** that influence its **execution behavior**, **I/O performance**, and **debugging**.

Here are three particularly useful ones that you can set with the `os` module before importing or using Pandas.

---

## 🧠 1. `PANDAS_BACKEND`

**Purpose:**
Controls which DataFrame backend Pandas uses (e.g., “default” vs. experimental engines like PyArrow).

**Usage:**

```python
import os
os.environ["PANDAS_BACKEND"] = "pyarrow"
import pandas as pd
```

**Effect:**
Pandas will use **PyArrow** as the backend for DataFrame operations (if available).
This can significantly speed up some operations and improve memory efficiency.

✅ *Introduced in Pandas 2.0+.*

---

## ⚙️ 2. `PYARROW_IGNORE_TIMEZONE`

**Purpose:**
Controls how Pandas (when using PyArrow) handles timezone-aware datetimes.

**Usage:**

```python
import os
os.environ["PYARROW_IGNORE_TIMEZONE"] = "1"
import pandas as pd
```

**Effect:**
When set to `"1"`, timezone information is **ignored** when converting Arrow tables or reading Parquet/Feather files, which can simplify interoperability but may lose tz info.

---

## ⚡ 3. `PANDAS_VENDORED_LIBRARIES`

**Purpose:**
Controls how Pandas uses or vendors certain optional dependencies.

**Usage:**

```python
import os
os.environ["PANDAS_VENDORED_LIBRARIES"] = "0"
import pandas as pd
```

**Effect:**
When `"0"`, Pandas **prefers external installations** of optional libraries like `dateutil` and `tzdata` instead of its internal vendored copies — useful for debugging, packaging, or dependency control.

---

## 🧩 Other Notable Examples

Here are a few more advanced ones you might find helpful:

| Variable              | Purpose                                                                        |
| --------------------- | ------------------------------------------------------------------------------ |
| `PANDAS_SLOW`         | Forces Pandas to use slower, pure-Python code paths (for testing).             |
| `PANDAS_CYTHON_DEBUG` | Enables verbose logging for Cython-accelerated operations.                     |
| `PYARROW_USE_PYTHON`  | Forces Arrow operations to run in pure Python mode (debugging).                |
| `NUMEXPR_MAX_THREADS` | Controls how many threads `numexpr` (used by Pandas) can use for computations. |
| `OMP_NUM_THREADS`     | Controls OpenMP thread count used by Pandas’ underlying numeric libraries.     |

---

## ✅ Example Setup

```python
import os

# Use PyArrow backend and ignore timezones
os.environ["PANDAS_BACKEND"] = "pyarrow"
os.environ["PYARROW_IGNORE_TIMEZONE"] = "1"
os.environ["NUMEXPR_MAX_THREADS"] = "4"

import pandas as pd
print(pd.__version__)
```
