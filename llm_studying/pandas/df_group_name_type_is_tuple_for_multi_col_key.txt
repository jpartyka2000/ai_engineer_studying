When you group by multiple columns, pandas sets `grp.name` to a **tuple** of the key values, in the same order you passed them to `.groupby()`.

* **Single‐column grouping**:

  ```python
  for grp in df.groupby('colA'):
      # grp.name is whatever type colA holds (e.g. an int, float, or string)
  ```

* **Multi‐column grouping**:

  ```python
  for grp in df.groupby(['colA','colB']):
      # grp.name is a tuple: (value_of_colA, value_of_colB)
  ```

So if you did

```python
df.groupby(['DEGREE_OFFERING','COHORT'])
```

then inside your apply function

```python
def f(grp):
    print(type(grp.name), grp.name)
    …
```

you’d see something like

```
<class 'tuple'> ('usc-dpt','January 2025')
```
