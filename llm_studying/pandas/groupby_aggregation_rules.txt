In Pandas, when you use .groupby() and then select a few other columns without explicitly specifying an aggregation function (like .sum(), .mean(), etc.), Pandas will not automatically aggregate those columns for you.

Here's what happens:

1. If you chain .groupby() with just column selection (e.g., df.groupby("key")[["col1", "col2"]]) — this returns a GroupBy object, not a final result. No aggregation happens yet.


2. If you call .agg() or a method like .mean(), then:

Numeric columns are aggregated using the specified function.

If you don’t specify one and instead call something like .mean(), then all numeric columns are aggregated using mean.

Non-numeric columns are ignored unless explicitly handled.





---

Example:

df = pd.DataFrame({
    "group": ["A", "A", "B", "B"],
    "val1": [10, 20, 30, 40],
    "val2": [100, 200, 300, 400],
    "text": ["x", "y", "z", "w"]
})

# This alone does nothing except define the group
grouped = df.groupby("group")[["val1", "val2"]]

# You must call an aggregator
result = grouped.mean()

Default behavior:

There is no automatic aggregation unless you:

call an aggregation method (.sum(), .mean(), etc.)

use .agg() explicitly


Summary:

> If you select columns after .groupby(), no aggregation is applied by default — you must specify it explicitly. If you do apply an aggregation method, only numeric columns are included unless otherwise defined.
