You’re right—`df.index.levels[1]` (or any `MultiIndex.levels`) gives you the *full set* of labels for that level in **sorted order**, not in the order they actually appear in your DataFrame.

If you need the regions in the order they first show up in your data, do this instead:

```python
# Pull out the Region values in their original sequence...
regions_with_dupes = df.index.get_level_values('Region')
# ...then dedupe while preserving first‑seen order
all_regions = regions_with_dupes.unique()
```

* **`levels[1]`** → sorted, global list of all possible Region labels in the index.
* **`get_level_values('Region')`** → the Region label for each row, in row order.
* **`.unique()`** on that preserves the order of first occurrence (insertion order) and drops duplicates.

That way you’ll get exactly the sequence in which each Region first appears in your DataFrame’s MultiIndex.
