Yes—you can convert a categorical string column to integer labels without implying an inherent order by using either `pd.Categorical(...).codes` or `pd.factorize()`. Both methods assign integer codes to each unique category in an arbitrary (usually alphabetical or appearance order) manner. For example:

```python
df['cat_int'] = pd.Categorical(df['cat_string']).codes
```

or

```python
df['cat_int'] = pd.factorize(df['cat_string'])[0]
```

While these integer codes are simply labels and do not have a true ordinal meaning, note that they are still numbers (with a natural ordering 0, 1, 2, …), so some machine learning algorithms might mistakenly treat them as having an ordinal relationship. If the algorithm you are using does not automatically treat such inputs as categorical, you may need to either tell the algorithm that the values are categorical or consider alternative encodings. 

Since you mentioned you have 50 unique values and you prefer not to use one-hot encoding, using one of the methods above (and ensuring the model treats them as categorical) is the common approach.
