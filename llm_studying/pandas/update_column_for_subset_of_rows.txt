In Pandas the simplest—and idiomatic—way to set or update a column for just a subset of rows is to use boolean indexing with `.loc`. That assignment always writes back into the original DataFrame, so you don’t have to “merge” the subset later.

---

### 1. Boolean mask + `.loc`

```python
# say we want to flag expensive items in a price column
mask = df['price'] > 100

# this sets df.loc[mask, 'expensive'] = True, in-place on df
df.loc[mask, 'expensive'] = True
```

* `mask` is a Series of True/False the same length as `df`.
* `.loc[mask, 'col'] = value` writes directly into `df`, not into a copy.

---

### 2. Using a transformed subset and then writing back

If you really need to pull out a subset, transform it, and then push changes into the original:

```python
# 1) extract subset
subset = df[df['group']=='A'].copy()

# 2) mutate the subset
subset['score'] = subset['score'] * 1.1

# 3) write those updates back into df
df.update(subset)  
```

`.update()` aligns on index & columns and only overwrites in `df` where `subset` has non-NA values.

---

### 3. Vectorized `np.where` inside `.loc`

For more complex logic you can combine masks and `np.where`:

```python
mask = df['score'].isna()
df.loc[:, 'score'] = np.where(mask, df['fallback'], df['score'])
```

---

### Key Takeaways

* **Prefer** `df.loc[condition, 'col'] = new_val` for in-place, subset-specific editing.
* **Avoid** assigning into `df[condition]['col'] = ...`—it may modify a copy, not your original.
* If you must work on a separate DataFrame, use `df.update(subset)` to merge your changes back.
