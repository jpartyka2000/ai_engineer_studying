Absolutely—you can use `.shift()` all on its own. It doesn’t need to be chained into a `.rolling()` call.

---

### What `.shift()` does

* **Signature**: `Series.shift(periods=1, freq=None, axis=0)`
* **Behavior**: Moves your data up or down by the number of `periods`.

  * `periods=1` (the default) replaces each value with the *previous* row’s value, and inserts a `NaN` at the top.
  * Negative periods look “forward” instead of backward.

You only need `.rolling()` if you want a moving-window aggregation.  If you just need a lag (or lead) column, `.shift()` by itself is the right tool.

---

### Example

```python
import pandas as pd

s = pd.Series([10, 20, 30, 40], index=pd.date_range("2025-01-01", periods=4))
print(s)
# 2025-01-01    10
# 2025-01-02    20
# 2025-01-03    30
# 2025-01-04    40
# dtype: int64

# Shift by 1 period backward (lag)
lag1 = s.shift(1)
print(lag1)
# 2025-01-01     NaN
# 2025-01-02    10.0
# 2025-01-03    20.0
# 2025-01-04    30.0
# dtype: float64

# Shift forward by 2 periods (lead)
lead2 = s.shift(-2)
print(lead2)
# 2025-01-01    30.0
# 2025-01-02    40.0
# 2025-01-03     NaN
# 2025-01-04     NaN
# dtype: float64
```

You can now use `lag1` or `lead2` however you like—no `.rolling()` required.
