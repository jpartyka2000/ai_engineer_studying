**Converting Columns to Categorical in Pandas with `.astype()`**

Pandas' categorical dtype can significantly reduce memory usage and improve performance when working with string columns that have repeated values.

## Basic Conversion

```python
# Single column
df['color'] = df['color'].astype('category')

# Multiple columns at once
df[['color', 'size', 'brand']] = df[['color', 'size', 'brand']].astype('category')

# All object columns
object_cols = df.select_dtypes(include='object').columns
df[object_cols] = df[object_cols].astype('category')
```

## Memory Benefits

```python
# Check memory usage before/after
print(df.memory_usage(deep=True))
df['product_type'] = df['product_type'].astype('category')
print(df.memory_usage(deep=True))
# Often 50-90% reduction for string columns!
```

## Working with Categories

```python
# Access category attributes
df['color'].cat.categories  # View unique categories
df['color'].cat.codes       # Get integer codes

# Add ordering
df['size'] = df['size'].astype('category')
df['size'].cat.reorder_categories(['S', 'M', 'L', 'XL'], ordered=True)

# Add new categories
df['color'] = df['color'].cat.add_categories(['purple'])
```

## When to Use Categorical

Best for:
- Low cardinality columns (< 50% unique values)
- Repeated string operations
- GroupBy operations
- Memory optimization

Not ideal for:
- High cardinality (many unique values)
- Columns you'll frequently modify
- Numeric data

Quick tip: `df.info(memory_usage='deep')` shows the real memory impact of converting to categorical!
