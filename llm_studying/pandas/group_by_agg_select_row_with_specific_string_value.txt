To group by `internal_id` and keep only the row with `'Closed Lost'` from the `opportunity_status` column, you can use the `.groupby()` method in combination with `.agg()` and `first()` to filter the rows. Here is how you can do it:

### Example Code:
```python
# Define a custom function to prioritize 'Closed Lost'
def keep_closed_lost(series):
    return series[series == 'Closed Lost'].iloc[0] if 'Closed Lost' in series.values else series.iloc[0]

# Group by 'internal_id' and keep 'Closed Lost' if it exists
df_grouped = df.groupby('internal_id', as_index=False).agg({
    'opportunity_status': keep_closed_lost
})

print(df_grouped)
```

### Explanation:
1. **Custom Function `keep_closed_lost(series)`:** 
   - This function checks if the value `'Closed Lost'` exists in the `opportunity_status` column within the group.
   - If `'Closed Lost'` is present, it returns that value.
   - If not, it defaults to the first value in the series.

2. **`.groupby('internal_id').agg()`:**
   - Groups the DataFrame by `internal_id`.
   - Uses the custom aggregation function to determine which value to keep in the `opportunity_status` column.
   - `as_index=False` ensures that `internal_id` remains a column in the resulting DataFrame instead of becoming an index.

This approach allows you to group the data by `internal_id` while prioritizing the `opportunity_status` value `'Closed Lost'` whenever it exists within the group.
