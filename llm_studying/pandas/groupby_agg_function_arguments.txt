Aggregation methods like `mean()` on pandas **DataFrame** (and many other aggregators) support an **`axis`** parameter plus several useful keyword arguments:

* **`axis`**

  * `axis=0` (default): compute the mean **down each column**, returning one value per column.
  * `axis=1`: compute the mean **across each row**, returning one value per row.

* **`skipna`** (`bool`, default `True`)
  Whether to ignore `NaN` values. If `False`, any `NaN` in the slice will make the result `NaN`.

* **`numeric_only`** (`bool` or `None`, default `None`)
  When set to `True`, only include numeric‚Äêdtype columns; when `False`, include all dtypes (and often error if non‚Äênumeric present); `None` tries numeric first then falls back.

* **`level`** (`int` or `str`, default `None`)
  For **MultiIndex** axes: compute the aggregation along a particular level of the index.

* **`ddof`** (for variance/standard‚Äêdeviation methods)
  The ‚Äúdelta degrees of freedom‚Äù divisor adjustment‚Äîe.g. `ddof=0` for population std, `ddof=1` for sample std.

* **`min_count`** (`int`, default `0`)
  Require at least this many non‚Äê`NaN` values for the result to be non‚Äê`NaN`; otherwise return `NaN`.

---

#### Example

```python
df.mean(axis=1,       # average across each row
        skipna=False, # any NaN ‚Üí result is NaN
        numeric_only=True)
```

Most other built-in aggregators (`sum()`, `std()`, `median()`, `max()`, `min()`) share the same signature, so once you know these options you can fine-tune virtually any pandas aggregation.
