Yes—you can pass a **list** of index‐level names (or positions) to `sort_index()` in order to sort by multiple levels in one call. For example, assuming your DataFrame has a two‐level row MultiIndex with levels named `"Region"` and `"Year"`:

```python
# Sort first by Region (A→Z), then by Year (ascending)
df_sorted = df.sort_index(level=['Region','Year'])
```

You can also control ascending/descending separately for each level:

```python
# Sort Region ascending, Year descending
df_sorted = df.sort_index(
    level=['Region','Year'],
    ascending=[True, False]
)
```

If you prefer to refer to levels by integer position instead of name, that works too:

```python
# Level 0 ascending, Level 1 descending
df_sorted = df.sort_index(level=[0,1], ascending=[True, False])
```

And of course you can sort columns (i.e. a column‐label MultiIndex) by setting `axis=1`:

```python
# Sort columns by the first and then second level of a column MultiIndex
df = df.sort_index(axis=1, level=[0,1])
```

---

### Key points

* **`level`** accepts a single name/int or a **list** of them.
* **`ascending`** can be a single bool or list matching the length of `level`.
* To sort by column‑index levels, add `axis=1`.
