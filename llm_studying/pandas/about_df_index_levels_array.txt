**Accessing Unique Labels of a MultiIndex Level with `df.index.levels[i]`**

When your DataFrame (or Series) has a **MultiIndex**, you might want to see just the distinct values that occur in one of its levels. Pandas provides the `.levels` attribute on a `MultiIndex` for exactly that.

---

## What is `df.index.levels`?

* `df.index` is your `MultiIndex` object.
* `df.index.levels` is a **list** of `Index` objects—one per level.
* `df.index.levels[i]` returns the **unique**, **ordered** labels for level `i` (0-based).

---

## Why use it?

* **Enumerate all categories** in that level (e.g. all stores, all dates, all regions).
* **Build loops** or **dropdowns** in UIs: you know exactly which keys exist.
* **Validate** that your data contains the expected set of labels.

---

## Example

```python
import pandas as pd

arrays = [
    ['East','East','West','West','East'],
    ['2025-01-01','2025-01-02','2025-01-01','2025-01-02','2025-01-01']
]
midx = pd.MultiIndex.from_arrays(arrays, names=['region','date'])
df = pd.DataFrame({'value': [1,2,3,4,5]}, index=midx)

# Display the distinct regions (level 0)
print(df.index.levels[0])
# Index(['East', 'West'], dtype='object', name='region')

# Display the distinct dates (level 1)
print(df.index.levels[1])
# Index(['2025-01-01', '2025-01-02'], dtype='object', name='date')
```

---

## Key Points

* **Length**: `len(df.index.levels[i])` is the number of unique labels in that level.
* **Type**: Each element is a pandas `Index`, not a Python `set`, so you retain ordering and dtype information.
* **Immutable**: You cannot modify `levels` in place; to change level labels you’d use `rename_axis`, `set_levels`, or rebuild the MultiIndex.

---

## Common Pitfall

* **Level vs. per-row values**:

  * Use `.levels[i]` when you want the **distinct** set of labels at level `i`.
  * Use `.get_level_values(i)` when you need the label for **every single row** in that level (length = number of rows).

---

**In summary**, `df.index.levels[i]` is your go-to accessor for the collection of unique keys at the i-th level of a MultiIndex, giving you quick insight into the structure of your hierarchical index.
