This often happens if the column already contains nulls (NaN). When you add a numeric value to NaN, the result is still NaN. If your 'probability' column is entirely null, then the operation won't produce any visible non-null values. You can fix this by filling the NaNs with 0 (or another appropriate value) before adding the random uniform values. For example:

```python
df_raw_stats_table_new['probability'] = df_raw_stats_table_new['probability'].fillna(0)
df_raw_stats_table_new['probability'] += np.random.uniform(0.05, 0.10, size=len(df_raw_stats_table_new))
```

This way, you'll start with 0 for any missing values, and then the addition will update the column with the generated values.
