The call to

```python
.fillna(method="bfill")
```

is simply taking each NaN in the newly-computed `"enrolled_last_100"` column and replacing it with the *next* non-NaN value *below* it in that same column.

– In practice the very first few rows of each cohort (or any spikes of NaNs you get because of your `shift(1)` or `min_periods`) will be back-filled by the *first* valid rolling mean that appears later in that cohort’s series.  If there’s no next valid value (e.g. at the very end), those NaNs remain.
