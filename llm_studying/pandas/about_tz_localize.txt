## 🕓 Understanding `tz_localize()` in Pandas — Assigning Time Zones to Datetime Columns

When dealing with timestamps in Pandas, you’ll often need to **assign a timezone** to naive datetime data (datetimes that have no timezone info).
That’s where `.dt.tz_localize()` comes in.

---

### 🔹 1. Naive vs. Timezone-Aware Datetimes

* **Naive datetime:** Has no timezone attached.
  Example: `2025-10-17 10:00:00`
* **Timezone-aware datetime:** Includes a timezone offset.
  Example: `2025-10-17 10:00:00-05:00`

Check if your column is timezone-aware:

```python
df['timestamp'].dtype
```

* `datetime64[ns]` → naive
* `datetime64[ns, UTC]` → timezone-aware

---

### 🔹 2. What `tz_localize()` Does

`tz_localize()` **adds** timezone information to a naive datetime column — it does **not** change the actual clock time.

Example:

```python
df['timestamp'] = pd.to_datetime(df['timestamp'])
df['timestamp'] = df['timestamp'].dt.tz_localize('UTC')
```

| Before              | After                     |
| ------------------- | ------------------------- |
| 2025-10-17 10:00:00 | 2025-10-17 10:00:00+00:00 |

The clock time stays the same; Pandas now *interprets* it as being in UTC.

---

### 🔹 3. How It Differs from `tz_convert()`

| Method           | Use Case                             | Works On                 | Effect                                                       |
| ---------------- | ------------------------------------ | ------------------------ | ------------------------------------------------------------ |
| `.tz_localize()` | Assign a timezone to naive datetimes | Naive datetimes          | Adds timezone info, no clock-time change                     |
| `.tz_convert()`  | Convert between timezones            | Timezone-aware datetimes | Adjusts clock time to represent same instant in another zone |

Example:

```python
# Step 1: Start with naive datetime
df['timestamp'] = pd.to_datetime(df['timestamp'])

# Step 2: Localize to UTC (assume times are UTC)
df['timestamp'] = df['timestamp'].dt.tz_localize('UTC')

# Step 3: Convert to Eastern time
df['timestamp'] = df['timestamp'].dt.tz_convert('America/New_York')
```

---

### 🔹 4. Handling Ambiguous or Nonexistent Times

During daylight saving transitions, some local times **don’t exist** or **occur twice**.
`tz_localize()` handles this via optional parameters:

```python
df['timestamp'].dt.tz_localize('America/New_York', ambiguous='infer', nonexistent='shift_forward')
```

* `ambiguous='infer'` — tries to guess DST transitions.
* `nonexistent='shift_forward'` — shifts times that “don’t exist” by one hour.

These are useful when working with real-world timestamps in local timezones.

---

### 🔹 5. Common Pattern: Parse → Localize → Convert → Drop

Here’s a common and safe workflow when importing timestamps:

```python
df['impression_dt'] = (
    pd.to_datetime(df['impression_ts'], errors='coerce')  # parse text to datetime
      .dt.tz_localize('UTC')                              # assume they're UTC
      .dt.tz_convert('America/Chicago')                   # convert to local time
      .dt.tz_convert(None)                                # drop timezone if needed
)
```

---

### ✅ Summary

| Goal                            | Method                                                   | Example                   |
| ------------------------------- | -------------------------------------------------------- | ------------------------- |
| Add timezone to naive datetimes | `.dt.tz_localize('UTC')`                                 | assume timestamps are UTC |
| Handle DST safely               | `.dt.tz_localize('America/New_York', ambiguous='infer')` | infer DST transitions     |
| Combine with conversion         | `.dt.tz_localize('UTC').dt.tz_convert('US/Pacific')`     | assign → convert          |

---

### 💡 Quick Rule of Thumb

> * Use **`tz_localize()`** to *add* a timezone.
> * Use **`tz_convert()`** to *change* a timezone.
> * Use **`.dt.tz_convert(None)`** to *remove* timezone info.
