They’re almost the same under the hood—both perform a SQL-style join—but differ in how you call them:

1. **`pd.merge(left, right, **kwargs)`**

   * **Function form**: you pass in both the left- and right-hand tables explicitly.
   * Example:

     ```python
     import pandas as pd

     df1 = pd.DataFrame({"key": [1,2], "A": ["a","b"]})
     df2 = pd.DataFrame({"key": [2,3], "B": ["x","y"]})

     merged = pd.merge(df1, df2, on="key", how="inner")
     ```
   * Useful if you’re merging more than two DataFrames in a single expression, or prefer the functional style.

2. **`df1.merge(df2, **kwargs)`**

   * **Method form**: you call `.merge()` on the “left” DataFrame and pass the “right” one in.
   * Exactly the same parameters under the covers—`on=`, `how=`, `left_on=`, `right_on=`, etc.
   * Example:

     ```python
     merged = df1.merge(df2, on="key", how="inner")
     ```
   * Can feel more natural when you think of “take df1, merge df2 into it.”

---

### A few small differences/tricks

* **Chaining**

  ```python
  # method form can be piped in a chain
  result = (
      df1
      .query("A == 'b'")
      .merge(df2, on="key", how="left")
      .assign(C=lambda d: d["A"] + d["B"])
  )
  ```

* **`pd.merge` is a standalone function**, so you could pass in any objects that implement the DataFrame merge protocol; the method form only works on DataFrame instances.

* **Readability**
  – Some teams prefer the functional `pd.merge(df1, df2)` for consistency.
  – Others like the method form because it reads left-to-right: “take `df1`, merge `df2`.”

---

> **Bottom line:**
> Use whichever style you find more readable—under the covers they both call the same pandas join machinery.
