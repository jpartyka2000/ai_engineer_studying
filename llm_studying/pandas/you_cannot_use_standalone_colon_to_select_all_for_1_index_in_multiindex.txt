Here’s the key point: when you only care about the **first** level of a MultiIndex, you don’t need (and in fact **can’t**) drop literal `:` tokens into a tuple—Python won’t parse `('A', :, :)`. Instead, Pandas will implicitly treat missing lower‐level selectors as “all” if you simply omit them:

```python
# Suppose df has a 3‐level row MultiIndex: Level0, Level1, Level2

# 👉 Only care about Level0 == 'A'
sub = df.loc['A']  
# exactly equivalent to:
# df.loc[('A', slice(None), slice(None))]
```

---

### Why you can’t write `('A', :, :)`

* In subscription syntax `df.loc[ key ]`, the commas and colons together build a tuple of slice objects.
* **Inside** a Python tuple literal you **cannot** just write `:`, because `:` only parses as a slice *within* a subscript, not as a standalone expression.

---

### If you really must be explicit

```python
# full explicit form—using slice(None) for the “:” at each lower level
df.loc[( 'A', slice(None), slice(None) ), :]
```

But in practice:

> **df.loc\['A']**
> → pulls out **all** Level 1 & Level 2 values under Level 0 = `'A'`.

No need for manual `slice(None)` (or any colons) unless you’re mixing in a column selector after the comma.
