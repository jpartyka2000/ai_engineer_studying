When you create a `pd.Series`, you can supply an **`index`** to label each element:

```python
import pandas as pd

# 1. Basic usage: data list + matching index labels
s = pd.Series([10, 20, 30], index=['a', 'b', 'c'])
# → index:  a   10
#            b   20
#            c   30
```

* **Length must match**: `len(data) == len(index)` or you’ll get a `ValueError`.
* If you **omit** `index`, you get the default `RangeIndex(0, 1, 2, …)`.

---

### Other common patterns

1. **From dict**, but preserve a specific order or add labels:

   ```python
   d = {'x': 1, 'y': 2}
   s = pd.Series(d, index=['y','x','z'])
   # → y   2.0
   #   x   1.0
   #   z   NaN    # label 'z' not in dict → fills with NaN
   ```

2. **Reindexing an existing Series**:

   ```python
   s0 = pd.Series([1,2,3])
   s1 = pd.Series(s0.values, index=['one','two','three'])
   # maps positionally, not by old labels
   ```

3. **Naming your index**:

   ```python
   s = pd.Series(
       [100, 200, 300],
       index=pd.Index(['Q1','Q2','Q3'], name='quarter'),
       name='sales'
   )
   ```

---

✓ **Key takeaway**: the `index=` parameter gives you explicit control over each element’s label and positioning—critical for alignment in merges, joins, and time‐series work.
