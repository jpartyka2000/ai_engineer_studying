Yes, that’s correct:

* **`reset_index()`** defaults to `drop=False`, meaning it will by default **keep** your current index as a column in the DataFrame when you reset back to a `RangeIndex`.
* **`set_index()`** defaults to `drop=True`, meaning it will by default **remove** the column(s) you’re promoting to the index from the DataFrame’s columns.

You can override both with their `drop` parameter:

```python
# reset_index: drop the old index entirely, don’t save it as a column
df = df.reset_index(drop=True)

# set_index: keep the column you’re indexing on as a normal column too
df = df.set_index('date', drop=False)
```

This symmetry keeps the most common workflows concise: you usually want to preserve your data when resetting an index, but drop the column when setting a new index.
