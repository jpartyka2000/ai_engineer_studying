In Pandas or JSON contexts, "record" and "split" refer to different serialization formats for dataframes — particularly when using .to_json() or .from_json() methods.

Here’s the key difference:


---

🔹 1. Record Format

Each row of the dataframe is converted to a dictionary (i.e., a "record").

Useful when you want each row to be a standalone JSON object.


df.to_json(orient='records')

Example:

[
  {"name":"Alice","age":25,"city":"NYC"},
  {"name":"Bob","age":30,"city":"SF"}
]

✅ Good for row-by-row processing
✅ Easy to read or insert into document-based databases (e.g., MongoDB)


---

🔹 2. Split Format

Dataframe is split into 3 components: index, columns, and data.

More compact, but less human-readable.


df.to_json(orient='split')

Example:

{
  "index":[0,1],
  "columns":["name","age","city"],
  "data":[["Alice",25,"NYC"],["Bob",30,"SF"]]
}

✅ Efficient for transmission and reconstruction
✅ Great for lossless round-tripping (especially in APIs)


---

🔄 Summary Table:

Format Structure Human-readable Suitable for

records List of dicts (row-wise) ✅ Yes Logs, APIs, MongoDB
split Dict with keys: index, columns, data 🚫 No Reconstructing exact DataFrames
