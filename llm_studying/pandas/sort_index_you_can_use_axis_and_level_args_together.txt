No—the `level=` argument in `sort_index()` only applies to the **index levels of the axis you’re sorting**. So:

* If your **row** index is a MultiIndex, you sort its levels with

  ```python
  df = df.sort_index(axis=0, level=[0,1])
  ```
* If your **column** index is a MultiIndex, you must switch to

  ```python
  df = df.sort_index(axis=1, level=[0,1])
  ```

Under the hood, `sort_index()` looks at the index **of the axis** you tell it (rows = 0, columns = 1). If you try to sort rows (`axis=0`) by levels that only exist on the columns, Pandas will raise because those levels aren’t in the row index.

---

### Quick recap

* **Row MultiIndex →**

  ```python
  df.sort_index(level=[…])              # same as axis=0
  ```
* **Column MultiIndex →**

  ```python
  df.sort_index(axis=1, level=[…])
  ```

If you ever need to reorder **both** axes, just chain the two calls.
