`DataFrame.where()` and `DataFrame.mask()` are two sides of the same coin: one keeps values **where** a condition is true and replaces the rest, the other **masks** (replaces) values **where** a condition is true.

---

### `where(cond, other)`

* **Keeps** the original value if `cond` is `True`.
* **Replaces** it with `other` (default `NaN`) if `cond` is `False`.

```python
df = pd.DataFrame({'A':[1,2,3,4]})
# keep only even values, turn odds to NaN
df_where = df.where(df.A % 2 == 0)
#    A
#0 NaN
#1  2
#2 NaN
#3  4
```

### `mask(cond, other)`

* **Replaces** the original value with `other` (default `NaN`) if `cond` is `True`.
* **Keeps** it if `cond` is `False`.

```python
# mask (hide) even values, show only odds
df_mask = df.mask(df.A % 2 == 0)
#    A
#0    1
#1  NaN
#2    3
#3  NaN
```

---

### Relationship

Under the hood,

```python
df.where(cond, other)  ==  df.mask(~cond, other)
```

and

```python
df.mask(cond, other)  ==  df.where(~cond, other)
```

---

### When to use which?

* Use **`.where`** when you have a “keep these rows” condition.
* Use **`.mask`** when you have a “hide/mask these rows” condition.

They’re functionally equivalent but offer clearer intent depending on whether you think of your logic as “I want to keep where this is true” (`where`) or “I want to blank out where this is true” (`mask`).
