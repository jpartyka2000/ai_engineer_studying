Yes — as of pandas 0.25.0 there’s a built-in `explode()` method on both `Series` and `DataFrame` that works much like PySpark’s, expanding list-like elements to separate rows.

```python
import pandas as pd

# example DataFrame with a list in each row
df = pd.DataFrame({
    'id': [1, 2, 3],
    'values': [[10, 20, 30], [40, 50], []]
})

# explode the 'values' column so each element gets its own row
df_exploded = df.explode('values')

print(df_exploded)
```

Output:

```
   id values
0   1     10
0   1     20
0   1     30
1   2     40
1   2     50
2   3    NaN
```

A few notes:

- Empty lists become a single row with `NaN`.  
- You can explode multiple columns at once by passing a list of column names to `DataFrame.explode([...])` (pandas 1.3.0+).  
- If you’re on pandas < 0.25.0, you can mimic explode with something like:  
  ```python
  (df.set_index('id')['values']
     .apply(pd.Series)       # expand each list to columns
     .stack()                # stack into a long Series
     .reset_index(level=1, drop=True)
     .rename('values')
     .reset_index())
  ```  

But if you’re on a modern pandas release, just use `df.explode('col_name')`.

-------------------

Yes—pandas’ `explode()` will work on any column whose entries are “list-like,” including lists of strings (or even NumPy arrays of strings). Under the hood pandas checks `pandas.api.types.is_list_like()` (which returns `True` for Python lists, tuples, NumPy arrays, etc., but **not** for scalar strings), and then it unpacks each element onto its own row.

So if you’ve got a column of Spark’s `ArrayType(StringType())` and you do:

```python
pdf = spark_df.toPandas()
```

that column in `pdf` will already be a column of Python lists of strings, and you can simply do:

```python
pdf_exploded = pdf.explode('your_array_col')
```

to get one row per string.

#### Example

```python
import pandas as pd

# simulate a column of lists of strings
df = pd.DataFrame({
    'id': [1, 2, 3],
    'tags': [['red','blue'], ['green'], []]
})

print(df)
#    id         tags
# 0   1  [red, blue]
# 1   2      [green]
# 2   3          []

# explode it
df2 = df.explode('tags')
print(df2)
#    id   tags
# 0   1    red
# 0   1   blue
# 1   2  green
# 2   3    NaN
```

A couple of gotchas:

1. **Empty lists** turn into a single row with `NaN`.  
2. If your column is actually a pandas **string** dtype (i.e. each entry is a single string), pandas will **not** treat it as list-like and won’t split it into characters. You’d first have to convert it to lists (e.g. `df['col'] = df['col'].str.split(',')`) before exploding.  
3. From pandas **1.3.0** onward you can even explode multiple columns at once by passing a list of column names:  
   ```python
   df.explode(['col1','col2'])
   ```
   but when exploding multiple columns, they must all be the same length in each row.

In short, as long as your “array[string]” column ends up as an object-dtype Series of Python lists (or tuples/arrays) of strings, `df.explode('that_col')` will do exactly what you want.
